# PRD: 조직 생성 UX 개선

> SDD 작성자가 작성하는 제품 요구사항 문서입니다.
> 사업 에이전트의 `docs/business/STATUS.md` 데이터 이상/오픈 이슈를 기반으로 작성합니다.

## 배경/문제 요약

- 참고: `docs/business/STATUS.md` > 데이터 이상, 오픈 이슈
- 문제:
  1. **organization_type 오선택**: 우면동 "초등부주일학교"가 MIDDLE_HIGH로 등록 → 운영자 수동 DB 수정. 타입 선택 UI가 직관적이지 않고, 이름과 타입 불일치를 감지하지 못함
  2. **동일 본당 내 중복 모임 생성**: 대치2동 동일 이름 2개, 양재2동 "양재2동" + "양재2동 중고등부" 2개. 기존 모임 존재 여부 확인 없이 새로 생성 가능
  3. **중복 본당 생성**: 같은 본당이 이름 변형으로 중복 생성 (예: "양재2동성당" vs "양재2동 성당"). 검색 없이 바로 "새로 만들기"로 진입 가능
- 현재 상태:
  - 조직 생성 시 이름(필수) + 타입(드롭다운, 기본값 MIDDLE_HIGH) 입력
  - 백엔드/프론트엔드 모두 중복 검증 없음
  - 타입 라벨이 "초등부 / 중고등부 / 청년" 3개뿐 — 선택 의미 불명확
  - 사제 네트워크로 하루 13명 가입 폭발 (03-13), 유입 증가 시 반복 발생 예상
- 목표 상태:
  - 타입 선택 시 의미와 영향을 명확히 전달하여 오선택 방지
  - 동일 본당 내 중복 모임 생성 전 기존 모임 안내로 중복 방지
  - 본당 생성 시 유사 이름 본당 경고로 중복 본당 방지

## 목표/성공 기준

- **목표**: 조직/본당 생성 시 타입 오선택과 중복 생성을 줄여 운영자 수동 DB 수정 건수를 0건으로 만든다
- **성공 지표**: 배포 후 타입 오선택 0건, 동일 본당 내 중복 모임 0건, 중복 본당 생성 0건
- **측정 기간**: 배포 후 4주

## 사용자/대상

- **주요 사용자**: 신규 가입 후 모임을 생성하는 교리교사 (ADMIN이 되는 첫 사용자)
- **사용 맥락**: 회원가입 → 교구/본당 선택 → 모임 합류 또는 생성 단계. 대부분 처음 사용하는 상황

## 범위

### 포함

- 조직 타입 선택 UX 개선 (선택지 설명 추가, 이름-타입 불일치 경고)
- 동일 본당 내 중복 모임 방지 (기존 모임 목록 안내, 동일/유사 이름 경고)
- 본당 생성 시 유사 이름 본당 경고 (기존 church.search API 활용)

### 제외

- 기존 잘못된 데이터 일괄 수정 (운영자 수동 처리)
- 조직 타입 변경 기능 (관리자 설정 페이지)
- 조직 삭제/병합 기능

## 사용자 시나리오

1. **타입 오선택 방지**: 교리교사가 "초등부주일학교"라는 이름으로 모임을 만들려고 한다. 타입 선택 드롭다운에서 각 옵션에 설명(졸업 연령 등)이 표시되어 올바른 타입을 선택할 수 있다. 이름에 "초등"이 포함되는데 MIDDLE_HIGH를 선택하면 불일치 경고가 표시된다.
2. **중복 모임 방지**: 교리교사가 이미 "중고등부"가 존재하는 본당에서 "중고등부"를 새로 만들려고 한다. 생성 다이얼로그 상단에 "먼저 목록에서 찾아보세요" 안내가 표시되고, 동일 이름 경고와 함께 합류를 권유한다.
3. **중복 본당 방지**: 교리교사가 "양재2동 성당"을 새로 만들려고 한다. 생성 다이얼로그 상단에 "먼저 검색에서 찾아보세요" 안내가 표시되고, 이름 입력 시 "양재2동성당"이 이미 존재한다는 경고가 나와 검색으로 돌아간다.

## 요구사항

### 필수 (Must)

- [ ] 조직 타입 기본값 제거 — 미선택 상태에서 시작하여 사용자가 반드시 능동적으로 선택하도록 강제
- [ ] 조직 타입 선택 시 각 옵션에 설명 텍스트 표시 (졸업 연령 기준 등)
- [ ] 동일 본당 내 동일 이름 모임 존재 시 생성 차단 (버튼 비활성화) + 합류 유도
- [ ] 이름에 타입 키워드(초등/중고등/청년)가 포함될 때 선택한 타입과 불일치하면 경고 표시
- [ ] 본당 생성 다이얼로그에서 동일 이름 본당 존재 시 생성 차단 (버튼 비활성화) + 검색 유도
- [ ] 백엔드에서도 동일 이름 모임/본당 생성 차단 (프론트엔드 우회 방지)

### 선택 (Should)

- [ ] 유사 이름(부분 일치) 모임이 있을 때 안내 표시
- [ ] 생성 확인 단계 추가 (이름 + 타입 최종 확인)

### 제외 (Out)

- 조직 타입 사후 변경 기능
- 기존 데이터 마이그레이션

## 제약/가정/리스크

- **제약**: organization.list API(모임 중복), church.search API(본당 중복) 모두 기존 API로 추가 개발 없이 활용 가능
- **가정**: 이름에 "초등" 키워드가 포함되면 ELEMENTARY 타입일 확률이 높다. 키워드 매칭으로 대부분의 오선택을 방지할 수 있다
- **리스크**: 동일 이름 차단이 정당한 중복 생성을 막을 수 있음 → 이름-타입 불일치는 비차단 경고로 유지

## 의존성

- **내부**: 기존 organization.list API, church.search API (모두 구현됨)
- **외부**: 없음

## 롤아웃/검증

- **출시 단계**: 전체 배포 (모든 사용자 대상)
- **이벤트/로그**: GA4 이벤트 — 경고 노출 횟수, 경고 후 타입 변경 횟수, 경고 무시 후 생성 횟수
- **검증 방법**: 배포 후 4주간 신규 모임의 타입 오선택/중복 건수 모니터링 (DB 쿼리)
