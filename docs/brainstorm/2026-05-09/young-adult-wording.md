# 브레인스토밍 결과 보고서

## 주제

청년-워딩 — 주일학교 포커스 제품에서 청년회(YOUNG_ADULT, 현재 5곳) 모임이 가입했을 때 "학년/학생" 라벨이 부자연스러운 문제. 워딩 분기 트리거(자동 vs 선택), 변경 범위, 청년 유입이 적은 진짜 원인이 워딩인지 채널인지 인과 검증.

## 사용자 발화 (입력)

> "주일학교에 포커스를 맞춘거다보니, 청년들이 상대적으로 덜 들어오는거같아. 그래서 좀 청년으로 들어오게 된 경우는 워딩을 좀 바꿔야 할 거같아."
> (R3 후) "난 워딩으로도 유입이 막아진다고 생각해. 청년 단체가 그래서 이탈한 사례가 많을텐데?"

## 사실 베이스

### YOUNG_ADULT 5곳 활성 상태 (STATUS 기준)

| 모임                 | 패턴                                   | 단계                          |
| -------------------- | -------------------------------------- | ----------------------------- |
| 사창동필립보         | **그룹 미생성**                        | 가입 후, 그룹 만들기 직전     |
| 다산성당             | 04-28~29 신규 도입 후 4~5일 침묵 (R24) | 가입 후, 그룹 1개 1학생 0출석 |
| 가톨릭상지대(예따사) | 학생 5명 정체, 변동 없음               | 가입 후, 사용 중              |
| 가톨릭상지대(본)     | 변동 없음                              | 가입 후                       |
| 목포해군             | 04-12 마지막 후 21일 침묵, 5/3 복귀    | 가입 후, 사용 중              |

**5곳 중 4곳이 가입 후 정체.** 외부 거절 사례(가입 전 이탈)는 사용자 도메인 관찰에서도 없음 → 진짜 문제는 **유입 막힘이 아니라 가입 후 사용 단계**. 사창동필립보의 "그룹 미생성"은 "학년 추가" 버튼 앞에서 멈춘 결정적 단서.

### 시스템 사실

- ORGANIZATION_TYPE: ELEMENTARY / MIDDLE_HIGH / YOUNG_ADULT 3종. **YOUNG_ADULT 단일 타입에 청년회·레지오·군종·성가대·교사단 다 매핑된 도메인 부채**
- 기존 도메인 용어 변경(2026-02-20 완료): 그룹→학년, 멤버→학생 (주일학교 가정)
- LandingPage.tsx FAQ_ITEMS 6번에 이미 "주일학교 운영에 최적화되어 있지만, 출석 관리가 필요한 모임이면 어디든 사용할 수 있어요" 문구 존재
- 1인 개발, 5월 P1 결제 인프라(PG 본계약 체인) 의존성

## 토론 요약

3라운드 토론 후 비판자가 v3-B(라벨 분기)를 BLOCK 처리했지만, R3 후속에서 **사용자 도메인 직관 + 데이터 정밀 해석**이 비판자 결론을 뒤집었다.

### 비판자 BLOCK 근거 vs 뒤집힌 이유

| 비판자 BLOCK 근거             | 뒤집힌 이유                                                                       |
| ----------------------------- | --------------------------------------------------------------------------------- |
| 3중 게이트 곱확률 10~20%      | 사용자 도메인 직관으로 N=2/2 게이트 제거 → MAO 50 + D유형 루프만 남으면 도달 가능 |
| medium 비용 (DB 컬럼 추가)    | **하드코딩 라벨 매핑 = small**. DB 변경 불필요                                    |
| ORGANIZATION_TYPE 도메인 부채 | 첫 단계는 "YOUNG_ADULT 전부 동일 라벨" 일괄 적용. 세분화는 DM 결과 후 결정        |
| 가설 협소 (워딩만 검사)       | 데이터 자체가 "가입 후 사용 단계" 정체를 보여줌. (b)(c)(d)는 평행 검증으로        |
| FAQ 6번 중복                  | "유입 막힘이 아닌 가입 후 정체"가 진짜 문제 → v3-C(랜딩) 우선순위 후순위          |

```diff
- 출석, 축일, 학생 현황까지.
```
