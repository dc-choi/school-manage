# 로드맵

## 로드맵 원칙(가설/근거/검증)
### 가설
- 무료 핵심 완성 → 유저 확장 → 유료 멤버십 → 외부 연동 순서가 도입 장벽과 리스크를 최소화한다

### 근거
- 초기에는 무료 핵심으로 진입 장벽을 낮추고, 확산 후 유료 전환을 검증해야 한다는 합의
- 외부 연동(알림/결제/전자서명)은 비용/계약 리스크가 커서 후순위가 합리적이다

## 1단계(무료 핵심 완성)
### 가설
- 목표: 유저 피드백 기반으로 무료 핵심 기능 완성
- 범위
  - 출석/학생/그룹 관리 고도화: 학년 전환 스케줄러 안정화, 일괄 졸업 처리, 데이터 일관성 검증
  - 통계 개선: 전체/성별/학년별/최근 출석
  - 출석부 UX 개선: 자동 저장/정렬/간편 체크, 의무축일/행사 구분
  - 온보딩 자동화: 엑셀 Import, 일괄 추가/삭제
  - 기본 리포트/의사결정: 주간/월간 출석 리포트, 사제 보고용 요약
  - 측정 인프라: 최소 이벤트 로깅 + GA4 연동(획득/전환 + 내부 이벤트 최소 세트)
  - 파일럿 도입: 서울대교구 3~5개 본당 파일럿(교리교사 주도), 사제 공유/피드백 수집

### 근거
- 반복 행정과 데이터 분산이 핵심 문제이며, 무료 핵심으로 주간 리듬을 먼저 안정화해야 도입 장벽이 낮다

## 2단계(유저 확장)
### 가설
- 목표: 더 많은 주일학교의 플랫폼 가입 유도
- 범위
  - 세일즈/온보딩 에셋: 템플릿, FAQ, 자가 온보딩 가이드, 랜딩/체험 경로 제공
  - 채널 확대: 교리교사 네트워크 추천, 청소년국 협력 채널 활용
  - 광고 수익: 전체 사용자 대상 광고 도입(부적절한 광고 필터링), 광고 형태는 행동 패턴 분석 후 결정
  - NFC 출석 유료 파일럿: 소수 본당에서 사용률/리텐션/지불 의향 검증

### 근거
- 1단계 안정화 이후 확산 채널을 검증해야 유료 전환율을 예측할 수 있다

## 3단계(유료 멤버십 + 유료 기능)
### 가설
- 목표: 유료 멤버십 도입 및 유료 기능 완성
- 범위
  - 행사 운영: 일정/준비물/역할 배정, 참가자 명단/비상연락망 관리
  - 축일 관리: 이번달/다음달 축일 관리
  - 조직/권한·데이터 연동: 본당 단위 다중 주일학교 관리, 교리교사 역할/권한 세분화, 초등부→중고등부 전환 이관 지원
  - NFC 출석 유료 애드온 정식 출시(별도 과금)

### 근거
- 확장 기능은 운영 복잡도를 높여 유료 가치 검증이 필요하다

## 4단계(외부 솔루션 연동)
### 가설
- 목표: 외부 솔루션 연동 유료 기능
- 범위
  - 공지/알림: 공지 발행/배포, 공지 리마인드, SMS/카톡 연동(비용 발생 시 애드온 고려)
  - 등록 관리: 연/학기 등록제, 등록 여부/보상 기준 관리(전자서명 연동)
  - 시상/보상 기준 자동 산출(등록제 연계)
  - 동의/전자서명·결제: 개인정보 제공 동의서/참가 동의서(전자서명 연동), 행사비/교재비 등 결제 모듈

### 근거
- 외부 연동은 비용/계약 리스크가 크며, PMF 확인 후 확장해야 한다

## 의존성 관리
### 가설
- 단계 착수 전에 의존성을 확보하면 일정 리스크를 낮출 수 있다
- 필수 의존성
  - 본당/주일학교 조직 모델 및 인증/권한 설계(3단계 착수 전)
  - 알림/결제/전자서명 연동을 위한 통신/PG/전자서명 솔루션 계약(4단계 착수 시)
  - 데이터 마이그레이션/백업 체계(학년 전환/행사 데이터 보존)
  - 등록제 운영 기준 정의(연/학기, 등록 여부, 보상 기준 다양성 반영)

### 근거
- 인증/권한과 데이터 보존이 불안정하면 유료 기능 품질이 하락한다
- 외부 연동은 계약/비용 확정 없이는 검증이 불가능하다
