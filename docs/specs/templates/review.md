# 검수 리뷰 템플릿

> 검수자가 각 단계(Feature/Task/Development/구현)를 검수할 때 사용하는 템플릿입니다.
> **PRD와 기능 설계 문서를 기준**으로 각 산출물이 요구사항을 충족하는지 검증합니다.
> 검수 유형에 맞는 템플릿을 복사하여 사용하세요.

---

## 검수 원칙

검수자는 다음 원칙에 따라 검수를 수행합니다:

1. **상위 문서 기준 검증**: 항상 PRD/기능 설계를 기준으로 요구사항 충족 여부 판단
2. **추적성 확보**: 요구사항 → Feature → Task → Development → 구현의 흐름이 일관된지 확인
3. **누락 방지**: Must Have 요구사항이 빠짐없이 반영되었는지 확인

---

## Feature 검수 리뷰

```markdown
## Feature 검수 결과

**문서**: [문서 경로]
**결과**: 승인 / 수정 필요

### 참조 문서
- PRD: [PRD 경로]
- 기능 설계: [기능 설계 경로]

### 요구사항 추적 체크리스트
- [ ] PRD의 목표/성공 기준이 Feature 개요에 반영되었는가?
- [ ] PRD의 Must Have 요구사항이 모두 "기능 요구사항"에 포함되었는가?
- [ ] 기능 설계의 사용자 흐름이 "사용자 스토리"에 반영되었는가?
- [ ] 기능 설계의 예외/엣지 케이스가 "엣지 케이스"에 포함되었는가?

### 문서 품질 체크리스트
- [ ] 사용자 스토리가 명확한가?
- [ ] 인수 조건이 검증 가능한가?
- [ ] 엣지 케이스가 정의되었는가?

### 수정 요청 (수정 필요 시)
| # | 항목 | 설명 | 우선순위 |
|---|------|------|---------|
| 1 | [섹션명] | [구체적 수정 내용] | High/Medium/Low |

### 코멘트
[추가 의견]
```

- [ ] 유스케이스에 구현 방향이 명시되었는가?

|---|------|------|---------|
| 1 | [UC-N / 업무 #N] | [구체적 수정 내용] | High/Medium/Low |

### 코멘트
[추가 의견]

|---|------|------|---------|
