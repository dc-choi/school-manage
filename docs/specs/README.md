## PRD & 기능 설계

> **작성자**: SDD 작성자

### PRD (제품 요구사항 문서)

> 23건 전체 Approved (구현 완료). 경로: `docs/specs/prd/`

### Functional Design (기능 설계)

> 16개 도메인 문서 전체 구현 완료. 경로: `docs/specs/functional-design/`
> **병합 규칙**: 도메인별 단일 문서 관리. 상세: `.claude/rules/specs.md`

---

> **SSoT**: 구현 완료된 기능의 진실 원천은 **기능 설계 문서 + 코드베이스**. Task/Development는 구현 완료 후 삭제.

### FUNCTIONAL (로드맵 3단계 — 베이직 플랜 결제 인프라, 2026-04-30 등록)

> 브레인스토밍 결과 반영: `docs/brainstorm/2026-04-30/pg-subscription.md`. PG 본계약 선행 의존성.

| 우선순위 | 기능명                                   | SDD 상태 | 비고                                                                                                                                                                                                        |
| -------- | ---------------------------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| P1       | 사이트 Footer + 약관 3종 (1차 배포)      | 미착수   | PG 가맹 심사 선행 조건. 사업자번호·상호·대표자·연락처·이용약관·개인정보처리방침·환불약관. 통신판매업 신고번호는 2차 갱신. R21 근거                                                                          |
| P1       | 베이직 플랜 정기결제 (토스페이먼츠 빌링) | 미착수   | 카드/계좌이체 정기결제. 빌링키 발급 + 매월 자동 청구 + 결제 실패 7일 유예 + 무료 플랜 다운그레이드. 카카오/네이버페이는 정기결제 불가 (일회성 결제 한정). 3단계 의존성                                      |
| P1       | 영수증/세금계산서 자동 발행              | 미착수   | MVP 필수 (2026-04-30 격상). 토스페이먼츠 자동 발행 활용 — 현금영수증(개인) + 전자세금계산서(본당법인) + 카드 매출전표. 결제 완료 시 PDF 다운로드 + 이메일 + 마이페이지 영수증함. 가톨릭 본당 회계 정산 필수 |
