# 비즈니스 모델

## 비전

- 가톨릭 교회 내 모든 모임을 위한 그룹 운영 플랫폼 (주일학교 중고등부 PMF 검증 중)

## 가치 제안

- 한 줄 정의(가설): **매주 일요일, 이거 하나 열면 다 되는 가톨릭 모임 운영 도구** — 출석 확인·축일·멤버 현황이 한곳에
- 시장 근거: 가톨릭 교리교사/모임 운영자가 매주 여는 주간 운영 도구가 시장에 없음 (상세: `docs/business/1_market/competitive-analysis.md`)
- 핵심 관찰: 출석 "기록"보다 "빠른 조회"가 핵심 가치, 축일 검색 등 가톨릭 정보 도구로도 활용됨 (장위동 4년 사용)
- 현재 검증 대상: 교리교사(중고등부 우선)

### 가치 1: 학생 정보 재입력 최소화(학년/졸업 주기 반영)

- 가설: 교리교사의 연간 재입력·정리 시간과 학년 전환 오류가 감소한다
- 근거: 매년 출석부 재작성·재입력, 학년 전환 시 수기 수정이 반복됨
- 검증: 연간 입력 업무 시간, 학년 전환 오류 건수 전후 비교 → 미완료

### 가치 2: 매주 필요한 정보를 한곳에서 빠르게 조회

- 가설: 출석 확인·축일·멤버 현황이 한곳에 있으면 매주 반복 사용하게 된다
- 근거: 장위동에서 출석 조회뿐 아니라 축일 검색에도 활용 — "주간 운영 허브"로 사용
- 검증: 주간 활성률, 축일/멤버 조회 빈도, 출석 기록 대비 조회 비율 → 미완료

### 가치 3: 행사/공지 운영 간소화

- 가설: 캠프/피정 등 행사 준비와 공지 전달의 누락·재작업이 감소한다
- 근거: 행사 체크리스트·역할·물품·연락이 수기로 분산되어 누락 발생
- 검증: 누락 건수/재작업 시간 전후 비교 → 미완료

### 가치 4: 협업 기반 데이터 일관성

- 가설: 동일 학생/출석 기록의 불일치가 감소한다
- 근거: 여러 파일/메신저/수기로 기록이 분산되어 최신성이 깨짐
- 검증: 동일 일자 불일치 건수/최신본 충돌 빈도 측정 → 미완료

### 가치 5: 통계/보고 신뢰도 향상(사제 의사결정 지원)

- 가설: 통계/보고 준비 시간이 줄고 사제 의사결정 리드타임이 감소한다
- 근거: 분산 기록으로 통계 산출·공지 준비가 수작업이며 데이터 불일치가 보고 신뢰도를 낮춤
- 검증: 보고 준비 시간/오류율, 사제 승인 리드타임 전후 비교 → 미완료

### 가치 6: 신뢰/보안 시그널 (2026-05-07 신규, R25 대응)

- 가설: 보안 페이지·데이터 정책 명문화·약관 가시성이 도입 게이트키퍼(교구 청소년국 직원, 사제, 학부모)의 보안 인식 우려를 해소한다
- 근거: **2026-05-02 서울대교구 청소년국 중고등부 간사 피드백** — "서버 기반 데이터 소유권/해킹 시 개인정보 유출 책임" 우려로 사용 꺼림. 도입 게이트키퍼급 페르소나 첫 보안 인식 시그널 (`4_risk/risks.md` R25, `0_feedback/entries/2026-05-02-cheongsonyeon-ganji.md`)
- 검증: 신뢰 시그널 정비 전후 도입 거부 사유 중 "보안 우려" 비율, 도입 전환율 비교 → 미완료

## 해자 가설 (2026-04-24 신규)

### 가설

- **회의록 자동화 기능**이 주일학교 운영에서 **전환 비용(switching cost)을 발생시키는 구조적 해자**이다
- 기존 가치 1~5는 "편의성" 차원에 머무르나, 회의록 자동화는 **필수 업무를 대체**하여 이탈 시 데이터 손실 발생

### 근거

- 교리교사의 회의록 작성은 **주간 필수 업무** (장현도 2026-04-24: "회의록같은거 무조건쓸 수밖에 없고 적기만 하면 캘린더에 자동연동이 됌 → 나는 무조건 쓸것 같은데")
- 필수 업무의 자동화 → 사용자가 자체적으로 동일 효과를 내기 어려움 (녹음 입력 + AI 분류 + PDF 생성 + 캘린더 자동 연동이 하나의 파이프라인)
- 회의록 데이터 축적 → 과거 회의록 참조 니즈 → 이탈 비용 증가
- 기존 가치와의 상승작용:
    - **가치 2 (빠른 조회) 강화**: 회의록 공지 → 캘린더 자동 반영으로 "매주 여는 도구" 포지셔닝 완성
    - **가치 3 (행사/공지 운영) 강화**: 회의록 작성만으로 공지 누락 자동 방지
    - **가치 4 (협업 데이터 일관성) 강화**: 회의록이 단일 소스가 되어 분산 기록 문제 해소
    - **가치 5 (보고 신뢰도) 강화**: AI 정리된 회의록 PDF가 사제 보고 자료로 직접 전환

### 기능 구조 (단일 시스템)

```
[입력]  사용자 메모 OR 녹음(Clova/Daglo 텍스트 추출)
   ↓
[AI 분석]  미사/교리/공지/안건 자동 분류·정리
   ↓
[PDF 생성]  회의록 PDF 초안
   ↓
[사용자 컨펌]  수정 가능, OK 시 다운로드
   ↓
[캘린더 자동 연동]  공지 섹션 날짜 파싱 → 구글 캘린더 자동 등록
```

### 검증

- 회의록 기능 파일럿 후 **프로 플랜 전환율 / 베이직 이탈률** 측정
- 회의록 작성 빈도(주 1회 이상 vs 미만)와 유지율 상관관계
- "엑셀/수기 회의록 → 우리 제품" 전환 시점의 다른 기능 사용률 변화
- 완료 기준: 파일럿 5곳에서 회의록 주간 활성률 60%+ 확인

## 협력 채널

- 가설: 교구 청소년국 사목부서(중고등부/학교사목부 중심)가 초기 협력 채널로 유효하다. **단, 청소년국은 협력·홍보 채널이지 의사결정자는 아니다 (2026-05-09 재정의)**
- 근거: 청소년국이 본당·학교 지원 역할을 수행하며 교리교사 접점(자료실/공지)을 보유
- 검증: 협력 요청 발송 후 응답/교육 세션 진행 여부 → 미완료
- **의사결정자 분리 (2026-05-09 신규)**: **양업시스템 연동 등 교구 차원 도입 의사결정은 주교가 최종 권한 보유** — 청소년국은 협력·홍보 채널이지만 교구 시스템 연동/예산 결정 권한은 없음. 청소년국 묶음 처리만으로는 양업시스템 협업의 격상 트리거가 발화하지 않음. 주교 접점 확보가 진짜 격상 조건. 상세: `4_risk/risks.md` R2 사제 동의 지연 (주교 = 최상위 게이트키퍼)
- **R25 영향 (2026-05-07)**: 청소년국 직원이 도입 게이트키퍼로 작동 시 보안 인식 우려가 도입 차단 시그널이 될 수 있음 (협력 채널과 게이트키퍼의 양면적 역할). 협력 채널 가설은 게이트키퍼 단계 신뢰 시그널 정비 후 재검증. 상세: `4_risk/risks.md` R25, `1_market/market.md` 도입 게이트키퍼

## 수익 모델

- 가설: 무료 + 광고 + 유료 구독의 3단 수익 구조가 적합하다
- 근거: 가톨릭 봉사 문화에서 진입 장벽(가격)이 핵심 저항 요인 → 무료 진입 + 광고로 운영비 충당 + 사용 정착 후 유료 전환의 단계적 모델이 도입 거부감을 최소화
- 검증: 광고 수익/CTR, 무료→유료 전환율, 유료 기능 활성도 → 미완료

### 무료(1단계)

- 포함: 학생/출석/기본 통계·보고/출석 UX 개선/출석 유형 구분/대량 작업
- 근거: 핵심 운영 기능은 도입 장벽을 낮추고 사용 습관을 만드는 데 필수
- 검증: 무료 가입 대비 4주차 활성률, 주간 출석 기록 수행률 → 미완료

### 광고 (보류)

- **상태 (2026-04-24)**: 보류. 재도입 여부·시점 미정
- 거부 이력: AdSense 승인 거부 (사유: 운영/관리자 도구 성격, 콘텐츠 페이지 비중 부족)
- 보류 사유: 광고 수익(R11 추정 월 1~3만원) vs 신뢰 손실(R10) ROI 불확실. 직장 병행 운영으로 단기 자금 부담 감내 가능
- 재도입 검토 트리거: 콘텐츠 페이지(행사 안내/후기 등) 비중 충분 확보 + 운영비 부족이 실제 발생할 때
- **배치 원칙 (재도입 시 적용)**: 하단 고정 배치만 허용. 핵심 기능 흐름 내 광고 절대 금지
- 검증: 광고 도입 결정 시점부터 CTR/수익/이탈률 측정 가능

### 유료 구독(3~4단계) (2026-04-24 재편 / 2026-05-09 시그니처 후보 재검토)

- **3단계 베이직 (시그니처: 알림/공지 발송)**: 알림/공지 발송(SMS/카톡) + 행사 운영 + 다음달 축일 관리 + 교사 공유 캘린더(수동 등록, 구글 캘린더 연동)
- **시그니처 후보 재검토 (2026-05-09)**: 단체 타입별 페르소나 분석 결과, **알림 시그니처 보편성 1/4** (초등부 단독 적합) vs **명단 이관/엑셀 자동화 보편성 3.5/4** (전 타입 적용). 명단 이관이 보편 시그니처 후보 1순위. 단 **데이터 이관 기능 자체가 큰 기능** — 즉시 교체 X, 차후 별도 SDD 기획 후 결정. 현 단계는 가설 등록 + 알림 시그니처 검증 데이터 수집 병행. 검증 항목: 알림 월 사용률 30% 미만 + 명단 이관 WTP 인터뷰 3건 이상 + 단체 타입별 사용 분포
- **4단계 프로 (시그니처: 회의록 자동화 — 해자)**: 회의록 기능(메모/녹음 입력 → AI 분류 → PDF → 캘린더 자동 연동) + 등록제(전자서명) + 개인정보 제공 동의(**전자서명 연동** — 2단계 수동 체크에서 업그레이드) + 결제 모듈
- **업셀 구조**: 베이직=수동 운영(캘린더 직접 입력, 알림 직접 발송) → 프로=회의록 작성만으로 전 과정 자동화
- **2030세대 유의**: 요노(YONO) 소비 성향 — "있으면 좋은 기능"으로는 결제하지 않음. **회의록 자동화(프로)가 유료 전환 핵심 트리거이자 구조적 해자**
- 검증: 유료 기능별 활성도, 전환율, 업그레이드율 → 미완료

## 결제 주체 (2026-02-20 재정의 / 2026-05-09 자금원 명확화)

- **본당/모임 단위 구독**이 1차 모델: 관리자(부장 교리교사)가 본당 단위로 결제
- 보조 모델: 개인 프리미엄 (Slack/Notion 패턴, 승인 불필요, 전환 빠름, ARPU 낮음)
- **자금원 가설 (2026-05-09)**: 본당 단위 결제의 1차 자금원은 **본당 예산(주임신부 결재 → 본당 운영비)**. 사목회는 본당 전체 서포트(전례·예산·재정 위원회) 역할이지 교리 운영 직접 주체 아님 — 결제 주체에서 제외. 학부모회비/교사 자비는 본당별 보조 자금원 가능성 (검증 필요)
- **단체 타입별 자금원 차등 가설**:
    - 초등부: 본당 예산(교리교사회 운영비) 1차 / 학부모회비·자모교사 자비 2차
    - 중고등부: 본당 예산 1차 / 교사 자비 2차 (R18 청년 봉사 비중 ↑)
    - 청년: 청년회 회비/예산 1차 / 본인 결제 2차
    - 군종: **군종교구 차원 예산** (별도 의사결정 라인) — 2026-03-27 목포해군성당 DM 양호 관계 1건 실증, R25 게이트 우회 사례

### 리스크

- 본당 예산 사용 시 설득 필요: 사제/사무실이 현행 방식에 불편함을 느끼지 않음
