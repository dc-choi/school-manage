# PRD: 학생 등록 관리

> SDD 작성자가 작성하는 제품 요구사항 문서입니다.
> 사업 에이전트의 `docs/business/` 문서를 기반으로 작성합니다.

## 배경/문제 요약

- **참고**: `docs/business/6_roadmap/roadmap.md` (4단계 등록 관리), `docs/business/STATUS.md` (파일럿 현황)
- **문제**:
  - 주일학교 등록 시즌(3월)에 학생의 등록 여부를 시스템에 기록할 수 없음
  - 교사가 종이 명단이나 별도 엑셀로 등록 현황을 관리해야 하는 이중 관리 발생
  - 현재 Student 모델에는 등록 관련 필드가 전혀 없음 (이름, 세례명, 성별, 나이 등 기본 정보만 존재)
  - 로드맵 4단계에 "등록 관리: 연/학기 등록제, 등록 여부/보상 기준 관리(전자서명 연동)"가 있으나, 전자서명·결제까지 포함된 풀 스펙으로 당장 적용 불가
- **현재 상태**: 등록 여부를 기록할 방법 없음. 교사가 수기 또는 엑셀로 별도 관리
- **목표 상태**: 학생 목록에서 등록 여부를 관리하고, 연도별 등록 이력을 추적할 수 있음

## 목표/성공 기준

- **목표**:
  - 등록 시즌에 학생의 등록 상태를 시스템 내에서 관리
  - 연도별 등록 이력 추적 (몇 년째 다니는 학생인지 파악)
  - 등록/미등록 학생 필터링으로 현황 즉시 파악
- **성공 지표**:
  - 등록 기능 사용 단체 수 (WAU 중 등록 기능 활용 비율)
  - 등록 시즌 이중 관리(엑셀+시스템) 감소 — 정성 피드백
- **측정 기간**: 배포 후 4주 (2026년 3월 등록 시즌)

## 사용자/대상

- **주요 사용자**:
  - 주일학교 교리교사: 학기 초 등록 처리 및 현황 관리
  - 주일학교 책임자: 학년별 등록 현황 파악
- **사용 맥락**:
  - 매년 3월 등록 시즌: 재학생 등록 확인 + 신규 학생 등록
  - 학기 중: 중도 등록/미등록 학생 관리
  - 연말: 연간 등록 현황 정리

## 범위

### 포함

- 별도 Registration 테이블로 연도별 등록 이력 관리
- 학생 목록에서 일괄 등록 처리 (체크박스 + 일괄 등록 버튼)
- 등록/미등록 필터링
- 등록 취소
- 등록 현황 요약 (등록 N명 / 미등록 M명)
- 엑셀 Import 시 "등록 여부" 컬럼 추가 (신규 학생 생성 + 등록 동시 처리)

### 제외

- 전자서명 연동 (4단계)
- 등록비 결제 (4단계)
- 보상/시상 기준 자동 산출 (4단계)
- 등록 양식 PDF 생성
- 학기(상/하반기) 단위 등록 — 연도 단위만

## 사용자 시나리오

1. **등록 시즌 일괄 처리**: 교사가 학생 목록에서 등록한 학생들을 체크 → "등록" 버튼 클릭 → 해당 연도 등록 완료
2. **엑셀로 신규 학생 + 등록 동시 처리**: 교사가 엑셀 양식에 학생 정보 + "등록 여부" 컬럼(O/X) 작성 → 업로드 → 학생 생성과 동시에 등록 처리
3. **등록 현황 확인**: 교사가 학생 목록에서 "미등록" 필터 → 아직 등록하지 않은 학생 확인 → 연락 필요 학생 파악
4. **등록 취소**: 실수로 등록한 학생 또는 등록 취소 학생 → 체크 후 "등록 취소" → 미등록 상태로 복귀
5. **연도별 이력 확인**: 학생 상세에서 과거 등록 이력 확인 (2024, 2025, 2026 등록 여부)

## 요구사항

### 필수 (Must)

- [ ] 별도 Registration 테이블: studentId + year 조합으로 연도별 등록 이력 저장
- [ ] 학생 목록에서 등록 상태 표시 (현재 연도 기준 등록/미등록)
- [ ] 학생 목록에서 체크박스 선택 → 일괄 등록 처리
- [ ] 등록 취소 (일괄)
- [ ] 등록/미등록 필터링
- [ ] 등록 현황 요약 표시 (등록 N명 / 미등록 M명)
- [ ] 엑셀 Import 시 "등록 여부" 컬럼 추가 (9번째 컬럼, 기존 8컬럼 뒤에 배치)
- [ ] 엑셀 등록 여부 입력값: O/X 또는 빈 값 → O이면 현재 연도 등록 처리, 그 외 미등록
- [ ] 엑셀 템플릿에 "등록 여부" 컬럼 추가
- [ ] GA4 이벤트: `student_registration` (등록 건수), `student_registration_cancel` (취소 건수)

### 선택 (Should)

- [ ] 학생 상세 페이지에서 연도별 등록 이력 표시

### 제외 (Out)

- 전자서명/결제 연동
- 보상 기준 관리
- 등록 양식 생성/출력
- 등록 마감일 설정
- 등록 알림 (SMS/카톡)

## 제약/가정/리스크

- **제약**:
  - 현재 학생은 하나의 그룹(group_id)에만 소속 (1:N) — 계정 모델 전환 전
  - 등록은 계정(account) 단위가 아닌 학생(student) 단위
- **가정**:
  - 대부분의 주일학교가 연도 단위로 등록제를 운영함 (학기 단위는 드묾)
  - 등록 연도는 현재 연도가 기본값 (교사가 수동 변경할 필요 없음)
  - 기존 학생(등록 이력 없는)은 미등록 상태로 표시됨
- **리스크**:
  - 계정 모델 전환(P0) 시 Registration 테이블의 관계가 변경될 수 있음 → 현재 studentId 기반으로 설계하면 영향 최소화

## 의존성
