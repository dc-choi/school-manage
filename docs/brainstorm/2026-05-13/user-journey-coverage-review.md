# 브레인스토밍 결과 보고서

## 주제

비즈니스 문서가 **유저 전체 여정(Awareness → Acquisition → Activation → Retention → Reactivation → Referral → Revenue → Offboarding)**을 잘 커버하는지 검토. 갭 발견 시 사업 기획 아이디어로 메우는 것이 목적.

## 토론 요약

3라운드 + Decision Tree 결과:

- **R1**: planner가 Activation/Referral/Offboarding 갭 진단 → 아이디어 1(온보딩 카드) 2(교사 인계) 3(사제 공유 링크) 제안. critic이 (i) 통계 유의성 (ii) 우선순위 간섭 (iii) R15/R25 충돌 (iv) 미검증 수요로 모두 강타. → 모두 보류/축소.
- **R2**: critic 메타 비판 — 사용자 원 질문("여정 커버리지")을 "STATUS 할 일 목록"으로 환원 = 회피. **진짜 갭은 Awareness/Acquisition** (인스타 4월 -78%, 비팔로워 도달 -11%p, link_in_bio 단일 채널). planner 수용 → 아이디어 4(대안 채널 2종) + 5(인터뷰 모수 보강) 신규.
- **R3**: critic이 (a) 4-B 공유 마찰 ≠ 공유 동기 (b) 4-A 사제 53일 단절 패턴 (c) **여름방학 갭 신규 식별 (CRITICAL)** (d) 3개월 캐파 불가능을 지적. planner 전면 수용 → 단일 트랙 + 명시적 컷오프 + 사용자 결정 2건 도출.

## 주제 결론

> **사용자 원 질문에 대한 답**: 비즈니스 문서는 Retention/Activation 측정은 정밀하나, **Awareness/Acquisition 갭(인스타 단일 채널 + 타겟 적합성 미측정)과 여름방학 계절성 갭(R18 누락)이 명백한 미커버 영역**. 신규 채널 발굴(4-A/4-B)과 R18 여름방학 섹션 추가가 즉시 필요. 1인 개발 3개월 캐파 한계로 "안 할 것"을 명시한 단일 트랙 운영이 합당.
