# PRD: 컨텍스트 배너

## 배경/문제 요약

- 참고: `docs/brainstorm/2026-03-16/acquisition-retention.md` (아이디어 2)
- 참고: `docs/business/STATUS.md` (전환율 분석, WAU 전환 유력 목록)
- 문제: 학생 등록까지 완료했지만 첫 출석을 기록하지 않는 단체가 퍼널 병목(학생→출석 전환율 73%, 이탈 27%). 전환 유력 6곳(사창동초 44명, 불광동 벤야민 29명, 잠원동 17명 등)이 출석 미시작 상태로 정체 중
- 현재 상태: 학생 등록 후 출석 화면으로의 유도가 없음. 사용자가 스스로 출석 메뉴를 찾아야 함
- 목표 상태: 퍼널 병목 구간에 주일 맥락 배너를 노출하여 "다음 행동(첫 출석)"을 유도, 첫 출석 완료 시 자동 소멸

## 목표/성공 기준

- **목표**: 학생 등록 후 출석 미시작 단체의 첫 출석 전환율 향상 → WAU 24→30 기여
- **성공 지표**:
  - 배너 노출 단체 중 2주 내 첫 출석 완료 전환율: 50%+
  - WAU 전환 유력 6곳 중 4곳+ WAU 전환
- **측정 기간**: 출시 후 4주

## 사용자/대상

- **주요 사용자**: 학생 1명 이상 등록했지만 출석 기록이 0건인 단체의 교리교사
- **사용 맥락**: 대시보드 진입 시. 주일 전후(토~월)에 특히 유효

## 범위

### 포함

- 대시보드 상단 컨텍스트 배너 (조건부 노출)
- 주일 날짜를 포함한 CTA 문구
- 클릭 시 출석 화면 이동
- 첫 출석 완료 시 배너 자동 소멸
- GA4 이벤트 추적 (노출/클릭)

### 제외

- 백엔드 API 추가 (기존 데이터로 판단)
- 온보딩 전체 체크리스트 (셀프 온보딩에서 이미 구현)
- 여러 퍼널 구간별 다중 배너 (첫 출석 미시작 구간만)
- 배너 닫기/다시 보지 않기 (첫 출석 완료가 유일한 소멸 조건)

## 사용자 시나리오

1. **첫 출석 유도**: 교리교사가 학생 5명을 등록한 후 대시보드에 진입 → "이번 주 일요일(3월 22일)에 첫 출석을 기록해보세요" 배너 표시 → 클릭 → 출석 화면으로 이동 → 출석 기록 → 다음 대시보드 진입 시 배너 미표시
2. **주중 노출**: 수요일에 대시보드 진입 → 다가오는 주일 날짜가 포함된 배너 표시 → 주일 출석 준비 동기 부여
3. **배너 미표시 — 이미 출석 있음**: 기존 사용자(출석 기록 1건+)가 대시보드 진입 → 배너 미표시 (정상 대시보드)
4. **배너 미표시 — 학생 없음**: 학생 0명인 단체 → 배너 미표시 (온보딩 체크리스트가 안내)

## 요구사항

### 필수 (Must)

- [ ] 학생 1명+ & 출석 0건인 단체에만 배너 노출
- [ ] 대시보드 상단에 배너 배치 (온보딩 체크리스트 아래, 통계 위)
- [ ] 배너 문구에 다가오는 주일(일요일) 날짜 포함
- [ ] CTA 클릭 시 출석 화면으로 이동
- [ ] 첫 출석 기록 완료 시 배너 자동 소멸
- [ ] 프론트엔드에서 기존 API 데이터로 조건 판단 (별도 API 불필요)
- [ ] GA4 이벤트: 배너 노출(`context_banner_shown`), 클릭(`context_banner_clicked`)

### 선택 (Should)

- [ ] 배너 소멸 시 GA4 이벤트(`context_banner_completed`) — 첫 출석 전환 추적용

### 제외 (Out)

- 백엔드 API 변경/추가
- 배너 닫기(dismiss) 버튼
- 다중 배너 (그룹 미생성, 학생 미등록 등)
- 푸시 알림/이메일 연동

## 제약/가정/리스크

- **제약**: 프론트엔드만 변경. 기존 API 데이터(학생 수, 출석 기록 수)로 조건 판단
- **제약**: 셀프 온보딩 체크리스트와 공존 (체크리스트는 그룹 0개 시 표시, 배너는 학생 1명+ & 출석 0건 시 표시 — 상호 배타적)
- **가정**: "학생 1명+ & 출석 0건"이 퍼널 병목 구간의 적절한 판단 기준
- **가정**: 다가오는 주일 날짜가 문구에 포함되면 행동 트리거로 작용 (주일 맥락 인식)
- **리스크**: 매주 동일 배너 반복 노출 시 배너 피로(banner blindness) 가능 → 첫 출석 완료가 소멸 조건이므로 전환 안 되면 지속 노출

## 의존성

- **내부**: 기존 API — 대시보드 통계(학생 수, 출석 기록 수), 셀프 온보딩 체크리스트 로직
- **외부**: 없음

## 롤아웃/검증

- **출시 단계**: 전체 배포 (조건 미충족 단체에는 미노출이므로 기존 사용자 영향 없음)
- **이벤트/로그**: `context_banner_shown`, `context_banner_clicked`, `context_banner_completed`
- **검증 방법**: GA4에서 배너 노출 → 클릭 → 첫 출석 전환 퍼널 분석. 전환 유력 6곳 WAU 전환 추적
