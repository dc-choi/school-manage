# 영수증·세금계산서 발행 정책 (MVP 필수)

> 메인 보고서: [pg-subscription.md](./pg-subscription.md)
>
> 2026-04-30 격상: "PG 계약 후 처리 가능"에서 **PG 본계약 전 완비 필수 기능**으로 격상

## 격상 사유

가톨릭 본당은 봉헌금/장부 회계 문화가 강하다. 본당 예산으로 결제하든 개인 카드로 선결제 후 정산받든, **영수증·증빙 없이는 사제 결재가 불가능**하다. 봉헌금 페이 실패 사례(`docs/business/4_risk/risks.md` R1)와 동일한 구조적 행동 비용이 영수증 부재에서 발생한다.

따라서 영수증/세금계산서 발행은 **PG 본계약 전 완비 필수 기능**이다. MVP 결제 모듈 sandbox 개발 단계부터 발행 플로우를 함께 구현한다.

## MVP 발행 플로우 (필수 구현)

| 결제 주체         | 발행 형식                                    | 발행 시점                                  | 발행 채널                                       |
| ----------------- | -------------------------------------------- | ------------------------------------------ | ----------------------------------------------- |
| 개인 카드         | 현금영수증 (지출증빙용 또는 소득공제용 선택) | 결제 완료 즉시                             | 결제 완료 페이지 PDF + 이메일 자동 발송         |
| 사업자 (본당법인) | 전자세금계산서                               | 결제 완료 즉시 (사업자번호 사전 등록 필요) | 토스페이먼츠 자동 발행 + 이메일                 |
| 모든 결제         | 카드 매출전표 사본                           | 결제 완료 즉시                             | 마이페이지 영수증함에서 다시 보기/이메일 재발송 |

## MVP 결제 화면 요구사항

- 결제 시 "현금영수증 발행" 옵션 (휴대폰번호/사업자번호 입력)
- 결제 완료 후 영수증 PDF 자동 다운로드 + 이메일 자동 발송
- 마이페이지 "영수증함" 메뉴: 과거 결제 영수증 재발송/PDF 재다운로드 가능
- 본당 사업자번호로 결제 시 세금계산서 발행 옵션 노출

## 발행 주체

- 위클리랩(개인사업자) → 결제자 (개인 또는 본당법인)
- 부가세 신고: 위클리랩 책임
- 세금계산서 발행 시 부가세 별도 표기

## 토스페이먼츠 자동 발행 기능 활용

- 현금영수증: 토스페이먼츠 결제 위젯에서 자동 발급 옵션 지원
- 전자세금계산서: 사업자등록번호 입력 시 자동 발행 API 사용
- 별도 영수증 발행 시스템 자체 구현 불필요 (PG 자동 처리)

## 영수증 부재 시 예상 영향

| 시나리오                         | 결과                                                                  |
| -------------------------------- | --------------------------------------------------------------------- |
| 본당 예산 결제 + 영수증 미발행   | 사제 결재 거부 → 결제 취소/환불 → 사용자 이탈                         |
| 개인 카드 선결제 + 영수증 미발행 | 본당 사무실 정산 불가 → 사용자 부담 증가 → 봉헌금 페이 실패 패턴 재현 |
| 세금계산서 미발행 (사업자 결제)  | 본당 회계 처리 불가 → 결제 자체 진행 불가                             |

따라서 영수증 발행 플로우 부재는 단순 "기능 누락"이 아니라 **결제 전환의 구조적 차단**이다.

## 인터뷰 검증 항목 (영수증 관련)

자세한 질문은 [pg-subscription-interview.md](./pg-subscription-interview.md) 영수증 질문 섹션 참조 (질문 6~8).
