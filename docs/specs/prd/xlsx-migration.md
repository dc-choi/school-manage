# PRD: xlsx 라이브러리 보안 취약점 대응 (라이브러리 교체)

> 상태: Done | 작성일: 2026-04-16

## 배경/문제 요약

- 참고: `docs/specs/README.md` BUGFIX 섹션 P1
- **문제**: 프론트엔드에서 사용 중인 `xlsx@^0.18.5` (SheetJS Community Edition, npm 배포본) 에 GitHub Advisory 기준 고심각도 취약점 2건이 미패치 상태로 노출되어 있다.
    - **Prototype Pollution** (GHSA-4r6h-8v6p-xvw6, High)
    - **Regular Expression Denial of Service (ReDoS)** (GHSA-5pgg-2g8v-p4x9, High)
    - npm 레지스트리 배포본은 더 이상 패치되지 않으며, 공식 권고는 CDN 또는 다른 라이브러리로 이동
- **현재 상태**:
    - `apps/web/package.json` 의 `xlsx@^0.18.5` 의존
    - 사용 지점 3개 (모두 클라이언트):
        - `features/student/utils/excel-template.ts` — 학생 등록 양식 `.xlsx` 다운로드 (헤더 + 셀 메모/스타일)
        - `features/student/utils/excel-import.ts` — 사용자가 업로드한 `.xlsx` 파싱 (첫 시트 → 행 단위)
        - `features/student/components/StudentImportModal.tsx` — 위 두 유틸리티 사용
    - 백엔드 의존 없음
- **목표 상태**:
    - 미패치 취약점이 노출된 `xlsx` 의존 제거
    - 학생 일괄 등록(다운로드/업로드) UX 동작 호환 유지
    - 대체 라이브러리: **`exceljs`** (단일 라이브러리로 read/write, 셀 메모/스타일 직접 지원)
    - **동적 import**로 분리하여 메인 번들 크기 회귀 방지

## 목표/성공 기준

- **목표**: 클라이언트 번들에서 `xlsx` 의존 완전 제거 + 학생 일괄 등록 동작 호환 유지
- **성공 지표**:
    - `pnpm ls xlsx --filter @school/web` 결과 0건
    - GitHub Dependabot/SonarCloud 의 `xlsx` 관련 경보 0건
    - 기존 사용자가 다운로드한 양식으로 업로드 시 정상 등록 (회귀 없음)
- **측정 기간**: 머지 후 즉시 검증 (CI + 로컬 시나리오 테스트)

## 사용자/대상

- **주요 사용자**: 학생 일괄 등록을 사용하는 교리교사 (조직 관리자/교사 권한)
- **사용 맥락**: 새 학년/학기 시작 시 명단 일괄 입력. 양식 다운로드 → 엑셀에서 작성 → 업로드 → 미리보기 → 확정 등록의 한 번의 흐름

## 범위

### 포함

- `xlsx` → **`exceljs`** 교체 (확정 — 라이브러리 비교 결과는 PR 본문에 첨부)
- `excel-template.ts` / `excel-import.ts` 를 **동적 import 패턴** (`await import('exceljs')`)으로 전환하여 메인 번들에서 ExcelJS 분리
- `excel-template.ts` 동작 호환:
    - 9개 컬럼 헤더 (`학년/이름/세례명/성별/전화번호/축일/나이/비고/등록 여부`)
    - 컬럼별 너비 (`!cols`)
    - 헤더 셀 스타일 (배경 `#4472C4`, 흰색 굵은 텍스트, 가운데 정렬)
    - 헤더 셀 메모(comment) — 입력 가이드
    - 시트명 `학생목록`, 파일명 `학생_등록_양식.xlsx`
- `excel-import.ts` 동작 호환:
    - 업로드 파일에서 첫 시트 파싱
    - 헤더 1행 제외, 빈 행 필터링
    - 행 인덱스(엑셀 1-based) 보존
    - 셀 값 문자열 정규화 (기존 `cellToString` 동작 유지)
- 기존 단위 테스트가 있다면 그대로 통과, 없다면 핵심 시나리오 테스트 추가

### 제외

- 백엔드 측 엑셀 파일 재검증 추가 (별도 BUGFIX P2 항목 — `서버측 Excel 파일 재검증 없음`)
- 엑셀 입력 검증 강화 (별도 BUGFIX P2 — `입력 검증 강화`)
- 새로운 입력 컬럼/필드 추가 (다른 P1 — `학생 추가 필드`에서 다룸)
- 통계/리포트 등 다른 도메인의 엑셀 내보내기 기능 신규 도입

## 사용자 시나리오

1. **양식 다운로드**: 학생 페이지 → "엑셀로 등록" 모달 → "양식 다운로드" 클릭 → `학생_등록_양식.xlsx` 저장. 헤더 셀에 마우스 오버 시 입력 가이드 메모 표시.
2. **업로드/미리보기**: 양식 작성 후 모달에서 파일 선택 → 클라이언트 파싱 → 행별 검증 결과(성공/오류) 미리보기 표시.
3. **확정 등록**: 검증 성공 행만 일괄 등록. 오류 행은 사용자에게 안내.

## 제약/가정/리스크/의존성

- **제약**:
    - 클라이언트 단독 처리(서버 파싱 없음) 유지 — SSR/Node 의존 도입 금지
    - 번들 크기 회귀를 최소화 (현재 `xlsx` 도 무거운 편이므로 동급 이하 목표)
- **가정**:
    - 사용자가 보유한 기존 양식 파일은 표준 `.xlsx` (Office Open XML)
    - 셀 메모는 "있으면 좋음" 수준 — 대체 라이브러리에서 호환성 확보 가능 (`exceljs` 의 `cell.note` 지원)
- **리스크**:
    - **R1 (중)**: ExcelJS 의 셀 스타일/메모 적용 결과가 xlsx 와 미세하게 달라질 수 있음 → 동작 호환만 보장하고 시각 차이는 PR 본문에 명시
    - **R2 (저, 완화)**: ExcelJS 번들 크기 ~250KB gzipped → **동적 import 로 메인 번들에서 분리**. 모달 첫 진입 시 1회 추가 다운로드, 이후 캐싱
    - **R3 (저)**: 사용자가 비표준(.xls/구버전) 파일 업로드 시 파싱 실패 — 기존도 미보장이므로 동일하게 명시 에러
    - **R4 (저)**: ExcelJS 동적 청크 로딩 실패(네트워크) 시 모달 동작 불가 → 에러 토스트로 안내
- **내부 의존성**: 학생 일괄 등록 모달 외 영향 없음
- **외부 의존성**: 신규 라이브러리(`exceljs` 등) 추가
