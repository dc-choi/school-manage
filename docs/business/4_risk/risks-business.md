# 리스크 — 수익화/결제 + 법규/인프라

`risks.md` 분리 — 결제 모델·광고·후원·개인정보·PG·법규 등 사업 운영 리스크. 인덱스: `risks.md`.

## 리스크 8: 결제 모델 전환 실패 (2026-02-24 재정의)

- 가설: 본당/모임 단위 구독이 1차 모델이나, 관리자 대표 결제 구조가 형성되지 않으면 유료 전환이 지연된다
- 근거: 본당 예산 결제는 승인/결재 절차로 리드타임이 길어질 수 있으며, 개인 프리미엄은 ARPU 불확실
- 검증: 본당 vs 개인 결제 분포, 승인 리드타임, 전환율/해지율
- 대응: 본당 구독 1차 + 개인 프리미엄 보조 (Slack/Notion 패턴)

## 리스크 9: 광고로 인한 사용자 이탈

- 가설: 광고가 UX를 해쳐 이탈률이 증가할 수 있다. 2030세대는 직관성에 민감하여 광고 이탈 리스크가 더 높음
- 검증: 광고 전후 주간/월간 이탈률, 위치별 불만 신고 건수
- 대응: 핵심 기능(출석 체크) 흐름 외부에 광고 배치

## 리스크 10: 부적절한 광고 노출 (광고 도입 보류로 잠정 비활성)

- 가설: 종교/교육 환경에 맞지 않는 광고 노출 시 신뢰도 하락
- 근거: AdSense 일반 카테고리는 도박/주류/타종교 광고 노출 가능. 가톨릭/주일학교 맥락에서 노출 시 사제·학부모의 신뢰도 손실 직접 발생
- 검증: 필터링 적용 후 부적절 광고 신고 건수 — **광고 도입 결정 시 측정 가능 (현재 보류)**
- 대응: 광고 네트워크 카테고리 필터링 + 사용자 신고 기능

## 리스크 11: 광고 수익 불확실성 (광고 도입 보류로 잠정 비활성)

- 가설: 틈새 시장의 낮은 트래픽(주 1회 사용)으로 광고 수익이 운영 비용을 충당하지 못할 수 있다
- 근거: 주간 운영 도구 특성상 평균 세션 빈도가 주 1~2회 수준 → AdSense CPM(국내 평균 ₩300~1,500) 기준 MAO 50곳으로는 월 1~3만원 수준 추정. 운영비 40만원 충당 불가. **광고 수익 vs 신뢰 손실 ROI 불확실로 광고 도입 자체 보류 (`2_bm/bm.md` 광고 섹션)**
- 검증: 월간 광고 수익/운영 비용 비율, CTR, CPM — **광고 도입 결정 시 측정 가능 (현재 보류)**
- 대응: 광고는 보조 수익원, 핵심은 유료 구독. 단기 자금은 개인 부담 + 도네이션으로 충당

## 리스크 15: 개인정보보호법 미준수 (법적 컴플라이언스)

- 가설: 개인정보 수집·이용 동의 절차 미흡 시 법적 리스크(최대 5천만원 과태료)
- 근거: 학생 이름·성별·학년·출석 기록 수집 중. **동의 절차 구현 완료 (2단계)**
- 검증: 소급 동의 미수집, 제3자 제공 동의 미확보, 법률 자문 미완
- 대응: **법률 자문 필요** (동의 절차는 구현됨, 법적 적정성 검토 미완)
- **R25와의 차이 (2026-05-07 분리)**: R15는 법적 미준수에 따른 규제·과태료 리스크(공급자 책임). 보안 인식 기반 도입 거부는 별도 트랙 R25 참조 (`risks-pmf.md`)

## 리스크 21: 업종/통신판매업 미비로 PG 심사 지연

- 가설: 현재 사업자 등록은 정보통신업 단일 업종으로, PG사 심사는 전자상거래 소매업(525101) 또는 통신판매 중개업(525103)을 요구하므로 업종 추가와 통신판매업 신고를 선행하지 않으면 3단계 베이직 플랜(SMS/카톡 유료 전환)의 결제 연동이 지연된다
- 근거:
    - 현재 홈택스 사업자 등록 현황: 주업종 "정보통신업"만 등록 (일반과세자, 개인사업자)
    - 국내 PG 3사(토스페이먼츠/나이스/NHN KCP) 공통 심사 요건: 전자상거래 소매업 업종 등록 + 통신판매업 신고번호 제출 필수
    - 통신판매업 신고가 [업종 정정 → PG 가계약 → 에스크로 확인증 발급 → 정부24 신고 → 본계약] 체인으로 연결되어 있어 전체 리드타임 최소 3~5주 소요 (단계별 상세: `6_roadmap/roadmap.md` PG/SMS 결제 인프라 계약 체크리스트)
    - 사이트 하단(Footer) 사업자번호·통신판매업 신고번호 미기재 시 PG 최종 승인 거절 가능
- 검증:
    - 업종 정정 승인까지 소요 기간 (홈택스, 통상 1~2일) — **2026-04-30 접수·당일 승인 완료 (1일 미만, 측정 결과: 통상보다 빠름)**
    - PG 가계약 → 통신판매업 신고증 수령까지 리드타임
    - 심사 보완 요청 횟수 및 반려 사유
- 대응:
    - 홈택스에서 525101·525103을 부업종으로 **동시 추가** (비용 없음, 미래 중개 플랫폼 전환 대비)
    - 토스페이먼츠 가계약 → 에스크로 확인증 수령 → 정부24 통신판매업 신고 순서로 병렬 진행
    - 사이트 Footer에 사업자번호·통신판매업 신고번호·이용약관·개인정보처리방침 사전 구축 (PG 심사 보완 방어)
    - 행정 체크리스트: `6_roadmap/roadmap.md` 3단계 의존성 섹션

## 리스크 22: 중개 플랫폼 전환 시 여행업/관광진흥법 등록 요건

- 가설: 로드맵 확장 방향인 "버스 대절·장소 연결 중개 플랫폼"은 단순 IT 플랫폼을 넘어 관광진흥법상 여행업 등록 대상으로 분류될 수 있으며, 무등록 알선으로 간주 시 과태료·사업 정지 리스크가 발생한다
- 근거:
    - 관광진흥법상 "운송 수단(버스 포함) 또는 숙박·관광지 알선"은 여행업 범위에 포함 가능
    - 단순 '정보통신업'이나 '통신판매 중개업'만으로는 운송 알선의 법적 근거가 불충분할 수 있음 (지자체 해석에 따라 차이)
    - 여행업 자본금 요건은 소기업 기준 약 1,500만 원 이상 (지자체별 상이), 과거 대비 완화되었으나 여전히 고정비 부담
    - 중개 모델에서 총거래액(GMV)과 실제 매출(수수료)의 괴리로 세무 소명 리스크 병존
- 검증:
    - 중개 플랫폼 실제 착수 시점(확장 단계 진입 시) 관할 지자체 관광과 사전 질의
    - 유사 서비스(버스 대절 중개, 장소 연결)의 법적 분류 사례 리서치
    - 약관 초안에 "중개자 책임 한계" 조항 반영 여부
- 대응:
    - 사업 확장 시점까지는 "자기 서비스 직접 판매"(전자상거래 소매업) 지위를 유지하며, 중개 전환 확정 전까지 버스/장소 연결 기능 착수 금지
    - 중개 전환 확정 시 [관광진흥법 자문 → 자본금 요건 확인 → 여행업 등록 → 약관·정산 로직 재설계] 순서로 진행
    - 플랫폼 이용약관에 "중개자 지위·실제 이행 책임은 공급자(버스/장소)" 조항 명시하여 연대 책임 리스크 감쇄
    - 재무/회계: 중개 전환 시 매출 인식을 GMV에서 수수료로 변경, 세무사와 정산·세금계산서 발행 주체 설계

## 리스크 23: 임시 후원 수령 채널의 매출 누락 + 신뢰도 저하

- 가설: 카카오페이 후원 링크 차단 우회로 운영자 개인 명의 계좌번호를 후원 페이지에 안내 문구 없이 직접 노출하는 임시 조치는 ① 사업자 매출 누락·세무 소명 리스크, ② 입금 주체 식별 불가, ③ "정식 결제 시스템 부재" 인상에 따른 신뢰도 저하의 3중 리스크를 누적시킨다
- 근거:
    - 2026-04-30 적용: 후원 페이지에 임시 안내 없이 운영자 개인 계좌번호만 노출
    - 사업자(school_back, 2026-03-24 등록) 매출과 개인 계좌 입금이 분리되어 사업자 장부 미반영 → 세무 소명 시 매출 누락 의심 가능
    - 입금자명만으로 후원자 식별 시 동명이인·약어 입력 등으로 후원 영수증 발급 또는 사용자 컨택 불가
    - 카카오페이 차단 원인 진단을 별도 트랙으로 운영하지 않기로 결정(2026-04-30) → PG 본계약(**5월 말~6월 중순 예상**, 산식: 5/6 가계약 → 1~2주 → 통신판매업 신고 3~7일 → 본계약 1~2주) 완료까지 임시 조치 유지
    - **2026-04-30 사업자 명의 계좌·체크카드 발급 → ① 매출 누락 리스크 해소 경로 확보**, 단 후원 페이지 노출 계좌 교체 전까지 ① 리스크 잔존 (사용자 "곧 교체" 결정, `STATUS.md` 오픈 이슈 별도 트래킹)
- 검증:
    - 임시 조치 적용 기간(2026-04-30 ~ PG 본계약 완료) 동안 입금 건수와 후원자 매칭 실패 비율
    - PG 본계약 완료 후 후원 전환율 회복 정도 (정식 카카오페이/네이버페이 연동 후)
    - 세무사 자문 결과 (개인 계좌 후원의 사업자 매출 회복 처리 가능 여부)
    - **사업자 계좌 교체 완료일 (R23 ① 해소 시점 기록)** — `STATUS.md` 오픈 이슈 "후원 노출 계좌 사업자 명의 교체" 트래킹
- 대응:
    - **단기(우선)**: 후원 페이지 노출 계좌를 개인 → 사업자 명의로 즉시 교체 → ① 매출 누락 리스크 해소 (`STATUS.md` 오픈 이슈 "후원 노출 계좌 사업자 명의 교체" 트래킹). 교체 후에도 ②·③ 잔존
    - PG 본계약 완료 시 임시 조치 전체 제거 (`STATUS.md` 오픈 이슈 별도 트래킹)
    - 사업자 계좌 교체 전 임시 운영 기간 입금 내역 별도 기록 (사업자 매출 회복 처리 대비)
    - 임시 안내 문구("결제 시스템 점검 중, 정식 결제 도입 예정") 추가는 별도 작업 필요 시 SDD 작성자 핸드오프 — 현재는 미적용 (2026-04-30 사용자 결정)
