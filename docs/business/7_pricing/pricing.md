# 가격 정책

## 가격 기준(가설/근거/검증)

### 가설

- 디다케 웹진(월 3,600원/연 36,000원) 대비 1.4-4.2배 수준이 허용 범위이다. **베이직 (1.36배 런칭가, 1.66배 정가)** = "엑셀+카톡" 대체 수동 도구, 프로(4.2배) = 회의록 자동화 해자 + AI 비용 반영 상위 가격대
- **anchor 가격 전략 (2026-05-09 도입)**: 정가 5,990원 표기 + 런칭 4,900원 실결제. 5,990원 anchor로 "비싸지 않다" 인식 + 실결제 4,900원이 2026-05-08 카톡 인터뷰 대역(2,900-3,900원)과 -1,000원 차이로 근접. 9로 끝나는 가격(5,990/4,900) = 좌측 자릿수 효과(left-digit pricing)

### 근거

- "매주 주일에 여는 도구"로서 출석 + 축일 + 멤버가 한곳에 — 엑셀이 대체할 수 없는 통합 가치
- 가톨릭 전용 주간 운영 도구가 시장에 없으므로 대체 불가한 가치가 있고, 가격 정당성이 존재
- 1단계 무료는 PMF 검증/사용자 확보 전략이며, 유료 전환은 2-3단계에서 진행
- 시장 가격 비교: `docs/business/1_market/competitive-analysis.md` 참조
- **신뢰/보안 가정 (2026-05-07 추가, R25 대응)**: 신뢰 시그널 정비 전 단계에서는 보안 인식 기반 도입 거부(`4_risk/risks.md` R25)가 가격 수용 측정에 노이즈로 작용 가능. 가격 검증 시 "보안 우려"로 인한 거부 비율 분리 측정 필요. 신뢰 시그널 정비 후 영향 재측정

### 검증

- 방법: 가격 구간별 결제 전환율/해지율 비교
- 완료 기준: 가격 수용 범위 1차 도출

## 무료 플랜(1-2단계)

- 가격: 0원 (광고 정책: 보류, `2_bm/bm.md` 광고 섹션 참조)
- 포함: 학생 관리, 출석 관리(기록/조회/정렬/자동 저장/간편 체크/유형 구분), 통계/보고(기본 집계), 대량 작업
- 근거: 핵심 운영 기능은 도입 장벽을 낮추고 사용 습관을 만드는 데 필수
- 검증: 무료 가입 대비 4주차 활성률, 주간 출석 기록 수행률

## 베이직 플랜(3단계) (2026-04-24 재편 / 2026-05-09 anchor 가격 도입)

- 가격: **정가 월 5,990원 / 런칭가 월 4,900원** (연: 정가 59,900원 / 런칭가 49,000원, 광고 정책: 보류)
- **가격 표기 방식**: 정가 5,990원에 취소선 anchor 적용 → 런칭가 4,900원 강조 표시 (할인가 anchor)
- **런칭 할인 종료 기준**: 미정 (PG 본계약 후 첫 100건 결제 또는 6개월 중 빠른 시점 — 가설)
- **시그니처**: 알림/공지 발송 (SMS/카톡)
- 포함: 무료 전체 + 알림/공지 발송 + 행사 운영 + 다음달 축일 관리 + 교사 공유 캘린더(수동 등록, 구글 캘린더 연동)
- 근거: 수동 운영 지원 도구군. 알림은 외부 비용(SMS/카톡 건당)이 있지만 "엑셀+카톡" 대체 가치로 유료 전환 유도
- 사용량: SMS/카톡 월 200건 포함 (가설), 초과분 애드온
- **WTP 리스크 (가정)**: 행사 운영 기능은 파워 유저가 AI로 일회성 도구를 직접 제작 가능(`4_risk/risks.md` R20). 베이직 플랜의 "행사 운영" 가치만으로는 구독 전환 동력이 약할 수 있음. 시그니처(알림/공지)가 전환 핵심 트리거여야 하며, 행사 운영은 부가 혜택으로 포지셔닝
- 검증: 무료→베이직 전환율, 알림 발송 건수 분포, 캘린더 활성도, **행사 운영 기능 월 사용률 vs 구독 지속율 상관 (R20 검증)**, **단체 타입별 시그니처 적합도 (`1_market/personas.md` — 알림 1/4 vs 명단 이관 3.5/4)**

## 프로 플랜(4단계) (2026-04-24 재편)

- 가격: **월 15,000원 / 연 150,000원** (광고 정책: 보류. 2026-04-24 10,000→15,000 인상, AI 비용 반영)
- **시그니처**: 회의록 자동화 (해자)
- 포함: 베이직 전체 + **회의록 자동화** + 등록제(전자서명) + 개인정보 제공 동의(전자서명) + 결제 모듈
- 회의록 자동화: 메모/녹음 입력 → AI 분류(미사/교리/공지/안건) → PDF 컨펌 → 캘린더 자동 연동 (단일 통합 시스템)
- 근거: 회의록 자동화가 교리교사 주간 필수 업무를 대체 → 전환 비용(switching cost) 발생 → **구조적 해자** (상세: `2_bm/bm.md` 해자 가설)
- AI 비용 구조: Clova/Daglo STT + Claude API 분류 (변동비 발생 → 프로 가격 인상 정당화 근거)
- 사용량: AI 회의록 월 4회 포함 (가설), 초과분 애드온 or 토큰 기반 과금
- 검증: 베이직→프로 업그레이드율, 회의록 작성 빈도, AI 월 사용량, 회의록 기반 이탈률 저하

## 유료 애드온: NFC 출석 — 보류 (2026-04-24)

- 상태: **보류**. 임팩트 불명확 + 하드웨어 모델 검증 선행 필요
- 기존 가설 (참고용 보존): NFC 출석은 유료 애드온으로 제공, 기본 출석은 무료 유지
- 기존 근거: 출석은 핵심 가치로 무료 유지 필요. NFC는 추가 비용과 프리미엄 편의/재미 요소
- 재평가 기준: 3단계 베이직 플랜 PMF 확인 후 다시 검토

## PG 선정 (2026-04-24 추가)

### 가설

- 토스페이먼츠를 1순위로 선정한다. 개발자 친화적 문서/SDK와 플랫폼 정산 API가 3단계 베이직(결제 연동)·미래 중개 플랫폼 모두에 적합하다.

### 근거

- 비교 대상: 토스페이먼츠 / 포트원(구 아임포트) / 나이스페이먼츠·NHN KCP
- 선정 기준 (개발 리소스 1인, 플랫폼 확장 가능성 고려):
    - **연동 난이도**: 토스페이먼츠 가장 낮음 (공식 문서 최적화) — 1인 개발 시 행정 부담 최소화
    - **플랫폼 정산**: 토스페이먼츠 전용 정산 API 제공, 미래 중개 전환 시 재설계 비용 낮음
    - **네이버페이/카카오페이 연동**: PG 본계약 후 별도 심사(1-2주 추가 리드타임 필요)
- 대안 유보 사유:
    - 포트원: 다중 PG 전환 유연성은 우수하나 초기 복잡도 과다
    - 나이스/KCP: 대형 플랫폼 안정성은 우수하나 심사 보수적·초기 리드타임 길음

### 검증

- PG 본계약 완료까지 실제 리드타임 (가계약 → 에스크로 → 통신판매업 신고 → 본계약)
- 첫 결제 연동 후 1개월간 결제 성공률, 환불 처리 리드타임
- 수수료율 체감 — 거래 규모 확장 시 재협상 또는 다른 PG로 이전 가능성 (포트원 SDK 유연성 재검토 트리거)
- 재선정 트리거: (a) 월 거래액이 수수료 0.1% 차이가 유의미해지는 구간 진입, (b) 중개 플랫폼 전환 확정
- **진척 (2026-04-30)**: 부업종 525101·525103 당일 승인 완료, PG 가계약 5/6(수) 토스페이먼츠 신청 예정, 본계약 5월 말-6월 중순 예상. 상세: `6_roadmap/roadmap.md` PG/SMS 결제 인프라 계약 체크리스트, `4_risk/risks.md` R21
