# 코드 리뷰 표준

## 목적

머지 전 품질·보안·유지보수성 보장. 본 룰은 **언제·어떻게** 리뷰할지를 정의한다.

## 리뷰 트리거 (필수)

- 코드를 작성/수정한 직후
- 공유 브랜치에 커밋하기 전
- 보안 민감 코드 변경 시 (인증/권한/결제/사용자 데이터)
- 아키텍처 변경 시
- PR 머지 전

## 사전 요구사항

리뷰 요청 전:
- 모든 자동 검사(CI) 통과
- 머지 충돌 해결됨
- 베이스 브랜치와 최신 동기화

## 리뷰 체크리스트

작업 완료 전:
- [ ] 코드가 잘 명명되어 읽기 쉬움
- [ ] 함수 집중 (<50줄)
- [ ] 파일 응집 (<800줄)
- [ ] 깊은 중첩 없음 (<4단계)
- [ ] 에러를 명시적으로 처리
- [ ] 하드코딩된 비밀/자격증명 없음
- [ ] `console.log`/디버그 문 없음
- [ ] 새 기능에 테스트 존재
- [ ] 테스트 커버리지 80% 이상

## 보안 트리거 → 자동 위임

**다음 변경 발생 시 `security-reviewer` 에이전트 호출 필수:**

- 인증/권한 코드
- 사용자 입력 처리
- DB 쿼리 (Prisma raw, `$queryRaw`)
- 파일 시스템 작업
- 외부 API 호출
- 암호화 작업
- 결제/금융 코드

## 심각도 매트릭스

| 등급 | 의미 | 액션 |
|---|---|---|
| **CRITICAL** | 보안 취약점/데이터 손실 위험 | **BLOCK** — 머지 전 반드시 수정 |
| **HIGH** | 버그 또는 중요한 품질 문제 | **WARN** — 머지 전 수정 권장 |
| **MEDIUM** | 유지보수 우려 | **INFO** — 수정 검토 |
| **LOW** | 스타일/사소한 제안 | **NOTE** — 선택 사항 |

> **보고 원칙**: 등급에 관계없이 모든 발견 항목을 사용자에게 그대로 노출한다. 에이전트가 자의로 MEDIUM/LOW를 삼키거나 요약/생략하지 않는다. 수정 여부 판단은 사용자가 한다.

## 에이전트 위임 매트릭스

| 에이전트 | 담당 영역 |
|---|---|
| `security-reviewer` | OWASP Top 10, 시크릿 노출, 인증/권한, 입력 검증 |
| `typescript-reviewer` | TS 타입 안전, async 정합, 에러 전파, idiom |
| `database-reviewer` | Prisma 스키마, 인덱스, N+1, 트랜잭션 잠금 순서 |
| `silent-failure-hunter` | 빈 catch, `.catch(()=>[])`, 위험한 fallback |
| `design-reviewer` | UI/UX, 접근성, 디자인 시스템 일관성 |
| `performance-analyzer` | 번들 크기, 렌더 비용, 쿼리 성능 |

같은 PR에 여러 영역이 걸치면 **병렬 호출 권장** (Task tool 한 메시지에 다중 호출).

## 자주 잡히는 문제

### 보안
- 하드코딩된 자격증명 (API 키, 비밀번호, 토큰)
- SQL 인젝션 (`$queryRaw`에 문자열 결합)
- XSS (escape 안 된 사용자 입력)
- 경로 순회 (sanitize 안 된 파일 경로)
- 인증 우회

### 코드 품질
- 큰 함수(>50줄) → 더 작게 분리
- 큰 파일(>800줄) → 모듈 추출
- 깊은 중첩(>4단계) → early return
- 에러 처리 누락 → 명시적 처리
- mutation 패턴 → 불변 연산
- 테스트 누락 → 커버리지 추가

### 성능
- N+1 쿼리 → `include`/`select` 또는 배치
- 페이지네이션 누락 → cursor 또는 LIMIT 추가
- 무한 쿼리 → 제약 추가
- 캐싱 누락 → 비싼 연산 캐시

## 승인 기준

- **승인**: CRITICAL/HIGH 없음
- **경고**: HIGH만 존재 (주의해서 머지)
- **차단**: CRITICAL 발견
