# NFC 출석 v2: 라즈베리파이 프리미엄

> 상위 문서: `docs/brainstorm/2026-03-07-nfc-attendance/README.md`

## 개요

라즈베리파이 기반 NFC 출석 리더기. v1(ESP32)의 모든 기능에 더해 7인치 터치 디스플레이로 출석 현황을 실시간 시각화한다. Python 또는 Node.js로 개발 가능.

## 구성 요약

```
[NTAG215 카드] → tap → [PN532 NFC 모듈] → I2C → [라즈베리파이 5]
                                                       │
                                          WiFi ← ──────┤──→ [7인치 터치 디스플레이]
                                                       │         (DSI 리본 케이블)
                                    [USB 스피커] ← USB ─┘
```

## 하드웨어 부품

### 라즈베리파이 5 (4GB)

- **역할**: 전체 제어 + 디스플레이 출력 (리눅스 OS 기반 미니 컴퓨터)
- **프로세서**: Broadcom BCM2712, 쿼드코어 Cortex-A76, 2.4GHz
- **RAM**: 4GB (8GB도 가능하나 4GB면 충분)
- **WiFi**: 802.11ac 듀얼밴드 내장
- **I/O**: GPIO 40핀, USB 3.0 x2, USB 2.0 x2, HDMI x2, DSI
- **전원**: USB-C (5V 3A 이상 권장)
- **OS**: Raspberry Pi OS (Debian 기반 리눅스)
- **가격**: ~125,000원 (4GB) / ~180,000원 (8GB)
- **구매처**: 디바이스마트, 아이씨뱅큐, 엘레파츠
- **프로그래밍**: Python, Node.js, 또는 원하는 언어

### PN532 NFC RFID Module V3

- v1과 동일 모듈 사용
- **통신**: I2C (라즈베리파이 GPIO 핀 사용)
- **읽기 거리**: 스펙 상 최대 5~7cm, 실사용 시 밀착~3cm (교통카드와 동일한 터치 방식)
- **가격**: 5,000~12,000원
- **주의**: 전원은 반드시 5V 핀에서 공급 (3.3V 핀은 전류 부족으로 PN532 오작동 가능)

### 7인치 터치 디스플레이

- **역할**: 출석 현황 실시간 시각화 (오늘 출석 목록, 출석률 등)
- **해상도**: 800 x 480
- **터치**: 정전식 (손가락 터치 지원)
- **연결**: DSI 리본 케이블 (라즈베리파이 DSI 포트에 직접 연결)
- **전원**: 라즈베리파이에서 DSI 케이블로 공급 (별도 전원 불필요)
- **가격**: ~70,000~105,000원
- **구매처**: 디바이스마트, 아이씨뱅큐, 엘레파츠

### USB 스피커 또는 USB 사운드카드 + 스피커

- **역할**: TTS 음성 재생
- **배경**: 라즈베리파이 5는 3.5mm 오디오 잭이 **없음** (Pi 4까지는 있었음)
- **방법 1**: USB 스피커 (플러그앤플레이, 가장 간단)
- **방법 2**: USB 사운드카드(DAC) + 3.5mm 스피커 연결
- **방법 3**: HDMI 오디오 (모니터 내장 스피커 사용 시)
- **가격**: USB 스피커 ~5,000~10,000원 / USB DAC ~3,000원 + 스피커 ~2,000원
- **추천**: USB 스피커가 가장 간단 (드라이버 설치 불필요, 꽂으면 바로 인식)

### Micro SD 카드 (32GB)

- **역할**: 라즈베리파이 OS + 소프트웨어 설치
- **권장**: 32GB 이상, Class 10 / U1 이상 속도
- **가격**: ~10,000원

### USB-C 전원 어댑터

- **사양**: 5V 3A 이상 (라즈베리파이 5 권장 사양, 27W PD 지원 권장)
- **주의**: 일반 충전기(5V 2A)로는 전력 부족 경고 발생 가능
- **가격**: ~15,000원

### 케이스 (디스플레이 일체형)

- **역할**: 라즈베리파이 + 디스플레이를 하나로 고정, 세워서 사용
- **가격**: ~15,000~25,000원

### NTAG215 PVC 카드

- 상위 문서 참조 (v1/v2 공통)

## 하드웨어 연결

### 전체 배선도

```
라즈베리파이 5 GPIO        PN532 NFC 모듈
════════════════════       ═══════════════
 Pin 4  (5V)    ─────────→  VCC  ⚠️ 반드시 5V 사용
 Pin 6  (GND)   ─────────→  GND
 Pin 3  (GPIO2, SDA1) ───→  SDA  (데이터)
 Pin 5  (GPIO3, SCL1) ───→  SCL  (클럭)


라즈베리파이 5              7인치 터치 디스플레이
════════════════════       ═══════════════
 DSI 포트 ←── 리본 케이블 ──→ DSI 입력
                            (전원도 리본 케이블로 공급)


라즈베리파이 5              USB 스피커
════════════════════       ═══════════════
 USB 2.0 포트 ←── USB ────→ USB 플러그
                            (플러그앤플레이, 설정 불필요)
```

### PN532 DIP 스위치 설정

v1과 동일:

| 스위치 1 | 스위치 2 | 모드 |
|---------|---------|------|
| **ON**  | **OFF** | **I2C (사용)** |
| OFF     | ON      | SPI |
| ON      | ON      | UART (HSU) |

### 연결 시 주의사항

1. **PN532 전원은 반드시 5V (Pin 4)**: 라즈베리파이의 3.3V 레귤레이터는 PN532에 충분한 전류를 공급하지 못함. 3.3V로 연결하면 랜덤 리셋, 통신 실패 발생
2. **I2C 활성화 필요**: 라즈베리파이 설정에서 I2C 인터페이스를 활성화해야 함 (`sudo raspi-config` → Interface Options → I2C → Enable)
3. **디스플레이 연결**: DSI 리본 케이블은 방향이 있음. 양쪽 커넥터의 금속 면을 맞춰서 삽입
4. **전원 어댑터**: 디스플레이 + PN532 + USB 스피커를 동시에 구동하므로 5V 3A 이상 필수

### 연결 확인 방법

터미널에서 I2C 디바이스 스캔:

```bash
sudo apt install i2c-tools    # 설치 (최초 1회)
sudo i2cdetect -y 1            # I2C 버스 스캔
```

정상 연결 시 주소 `0x24`에 PN532가 표시됨.

## 동작 흐름

```
1. 전원 ON → 라즈베리파이 부팅 (30~60초) → OS 시작
2. NFC 출석 프로그램 자동 실행 (systemd 서비스)
3. 디스플레이: 출석 대기 화면 표시
4. 학생이 카드 터치
5. PN532 → 라즈베리파이: 카드 UID 전달
6. 라즈베리파이 → API 서버: HTTP POST { tagId, deviceId }
7. API 서버 → 라즈베리파이: { studentName, audioUrl, attendanceData }
8. USB 스피커: TTS 오디오 재생
9. 디스플레이: 출석 현황 업데이트 (학생명, 시간, 누적 출석 등)
10. 4번으로 돌아가 다음 카드 대기
```

## 기술 스택 (소프트웨어)

- **OS**: Raspberry Pi OS (Bookworm)
- **언어**: Python 또는 Node.js (현재 프로젝트와 기술 스택 통일 시 Node.js 추천)
- **NFC 라이브러리**: pn532pi (Python) 또는 nfc-pcsc (Node.js)
- **HTTP**: requests (Python) 또는 fetch/axios (Node.js)
- **오디오**: aplay/mpv (CLI) 또는 node-speaker (Node.js)
- **디스플레이 UI**: 웹 브라우저 키오스크 모드 (Chromium) 또는 Electron
- **자동 시작**: systemd 서비스로 부팅 시 자동 실행

### 디스플레이 UI 구현 방식

라즈베리파이에서 Chromium 브라우저를 키오스크 모드로 실행하면, 기존 웹앱(React)을 그대로 활용하거나 별도 출석 현황 페이지를 만들어 띄울 수 있다. 별도 GUI 프레임워크 학습 불필요.
