# 기능 설계: 게스트 대시보드

> 비인증 상태에서 `/`에 대시보드 레이아웃을 노출하고, 데이터 영역에 로그인 유도 안내를 표시합니다.

## 연결 문서

- PRD: `docs/specs/prd/guest-dashboard.md`
- Task: `docs/specs/target/functional/tasks/guest-dashboard.md`
- Development: `docs/specs/target/functional/development/guest-dashboard-*.md`

## 흐름/상태

### 사용자 플로우

1. 비인증 유저가 `/` 접속
2. 대시보드 레이아웃(사이드바, 헤더, 필터, 카드 영역) 렌더링
3. 데이터 영역(통계 테이블, 차트, 랭킹, 축일 카드)에 "로그인이 필요합니다" + CTA 표시
4. 전례 시기 카드는 공개 데이터로 정상 표시
5. 헤더에 로그인/회원가입 버튼 표시
6. 사이드바 네비게이션 클릭 시 `/login`으로 이동
7. 인증 완료 후 기존 대시보드 그대로 동작

### 상태 전이

```
[비인증] → / → 게스트 대시보드 (레이아웃 + 로그인 유도)
[비인증] → 로그인 CTA 클릭 → /login
[인증 완료] → / → 기존 대시보드 (변경 없음)
```

### LoginRequiredCard (신규 컴포넌트)

데이터 영역을 대체하는 로그인 유도 안내:

```
┌─────────────────────────┐
│                         │
│   로그인이 필요합니다      │
│                         │
└─────────────────────────┘
```

- Card 컴포넌트 사용, 중앙 정렬
- 텍스트만 표시, 별도 CTA 버튼 없음 (헤더의 로그인/회원가입 버튼으로 유도)

### 헤더 게스트 모드

```
인증 시:  [메뉴] 안녕하세요, {name}님!    [계정] [로그아웃]
게스트 시: [메뉴] 주일학교 출석부            [로그인/회원가입]
```

- 계정 정보(프로필 링크) 숨김
- 기존 로그아웃 버튼 자리에 "로그인/회원가입" 표시 (같은 위치, `LogIn` 아이콘 + 텍스트)
- 클릭 시 `/login`으로 이동

## 권한/보안

- **접근 제어**: `/` 라우트는 인증 불요. 데이터 API는 기존 인증 유지 (게스트는 API 호출 안 함)
- **감사/로그**: GA4 `guest_dashboard_viewed` 이벤트

## 예외/엣지 케이스

| 상황 | 처리 방법 |
|------|----------|
| 인증 로딩 중 (`isLoading`) | 기존 로딩 스피너 표시 |
| 인증 → 비인증 전환 (로그아웃) | 게스트 대시보드로 전환 (리다이렉트 없음) |
| 사이드바 네비 클릭 (게스트) | `/login`으로 이동 (from state 포함) |
| 전례 시기 API 실패 | 기존 동작 유지 (카드 숨김) |

## 성능/제약

- 게스트 모드에서 tRPC 데이터 쿼리 호출하지 않음 (불필요한 401 방지)
- LiturgicalSeasonCard만 공개 API 호출

## 측정/모니터링

- **이벤트**: `guest_dashboard_viewed` (게스트 대시보드 진입 시)
- **전환 추적**: 게스트 → 로그인/회원가입 CTA 클릭

## 테스트 시나리오

### 정상 케이스

1. **TC-1**: 비인증 유저가 `/` 접속 → 대시보드 레이아웃 렌더링, 데이터 영역에 "로그인이 필요합니다" 표시
2. **TC-2**: 비인증 유저가 로그인 CTA 클릭 → `/login`으로 이동
3. **TC-3**: 인증된 유저가 `/` 접속 → 기존 대시보드 그대로 동작
4. **TC-4**: 비인증 유저에게 전례 시기 카드 정상 표시
5. **TC-5**: 비인증 유저에게 축일 카드 숨김

### 예외 케이스

1. **TC-E1**: 비인증 유저가 사이드바 "출석부" 클릭 → `/login`으로 이동
2. **TC-E2**: 인증 → 로그아웃 → 게스트 대시보드로 전환 (리다이렉트 없음)
