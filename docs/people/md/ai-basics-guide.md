# AI 기초 사용 가이드

`ai-setup-guide.md`에서 설치를 완료한 후 읽는 문서.
기본 사용법을 익힌 후 `ai-usage-guide.md`에서 실전 활용법(슬래시 커맨드, 규칙 파일)을 학습한다.

## 기본 인터페이스

Claude Code(터미널) 또는 Conductor(GUI)를 열면 **채팅창**이 보인다.
자연어로 질문하거나 명령을 입력한다.

### 입력 방식 2가지

**1. 자연어 질문/요청** — 일상 대화처럼 입력
```
"이 프로젝트가 뭘 하는 건지 설명해줘"
"Student 도메인 코드 구조를 알려줘"
"최근 커밋 10개 보여줘"
```

**2. 슬래시 커맨드** — 미리 정의된 워크플로우 실행
```
/biz status        → 사업 현황 확인
/test              → 테스트 실행
/commit            → 변경사항 커밋
/sdd 0             → 새 기능 개발 시작
```

> `/`를 입력하면 사용 가능한 커맨드 목록이 뜬다.

## 도구 승인 이해하기

AI가 파일을 수정하거나 명령을 실행할 때 **승인 요청**이 뜬다.
이것은 안전장치다. AI가 멋대로 작업하지 않도록 사람이 확인한다.

| 버튼 | 의미 | 언제 누르나 |
|------|------|-----------|
| **Allow** | 이번 한 번 실행 허용 | 내용 확인 후 OK일 때 |
| **Deny** | 실행 거부 | 잘 모르겠거나 위험해 보일 때 |
| **Allow Always** | 같은 유형 자동 허용 | 익숙해진 후 반복 작업에 |

**안전 기준:**
- **읽기 작업** (Read, Grep, Glob) → 항상 안전. 편하게 Allow
- **쓰기 작업** (Edit, Write) → 변경 내용을 확인하고 Allow
- **명령 실행** (Bash) → 어떤 명령인지 확인하고 Allow
- **잘 모르겠으면** → Deny. AI가 다른 방법을 제안한다

> Deny를 눌러도 아무 문제 없다. AI가 화나지 않는다.

## 첫 대화 연습

설치 확인겸 다음을 순서대로 해본다. **읽기 전용**이라 안전하다.

### 1. 프로젝트 파악

```
이 프로젝트가 뭘 하는 거야? 간단히 설명해줘.
```
→ AI가 CLAUDE.md와 README.md를 읽고 프로젝트 개요를 설명해준다.

### 2. 현재 상황 확인

```
/biz status
```
→ 사업 현황(WAU, 전환 퍼널, 이탈 모니터링)을 보여준다.

### 3. 파일 탐색

```
docs/business/ 폴더에 뭐가 있어?
```
→ AI가 폴더 구조를 보여주고 각 파일의 역할을 설명한다.

### 4. 도움 요청

```
/help
```
→ 사용 가능한 명령어 목록을 보여준다.

## AI에게 잘 물어보는 법

### 좋은 질문 vs 나쁜 질문

| 나쁜 질문 | 좋은 질문 |
|----------|----------|
| "코드 고쳐줘" | "StudentListPage에서 검색이 안 되는데, 관련 코드를 분석해줘" |
| "콘텐츠 만들어" | "출석 기능을 소개하는 카드뉴스를 content-templates.md 템플릿대로 만들어줘" |
| "다 해줘" | "첫 번째로 백엔드 API부터 만들고, 완료되면 프론트엔드 작업하자" |

### 핵심 팁

1. **구체적으로** — "학생 목록 페이지" > "그 페이지"
2. **단계별로** — 한 번에 하나씩 요청, 완료 확인 후 다음
3. **맥락 제공** — "STATUS.md 참고해서" 같이 참조 파일 알려주기
4. **모르면 모른다고** — "이 에러 메시지가 뭔지 모르겠어" → AI가 설명해준다
5. **거절해도 OK** — AI 제안이 맘에 안 들면 "그건 아니고, 이렇게 해줘"

## 역할별 다음 단계

### 프로덕트 엔지니어

```
"Student 도메인 전체 구조를 설명해줘.
schema.prisma부터 프론트엔드 페이지까지 추적해줘."
```
→ AI가 코드베이스를 탐색하며 아키텍처를 설명해준다.

```
/test
```
→ 전체 테스트를 실행하고 결과를 요약해준다.

### 콘텐츠 마케터

```
"현재 인스타그램에 게시된 콘텐츠 목록을 보여줘.
docs/content/instagram/feed/ 폴더 확인해줘."
```
→ AI가 기존 콘텐츠 목록을 정리해준다.

```
"카드뉴스 템플릿이 어떻게 생겼는지 보여줘.
.claude/rules/content-templates.md 읽어줘."
```
→ AI가 템플릿 구조를 설명하고 예시를 보여준다.

## 자주 하는 실수

**"AI가 이상한 답을 해요"**
→ 질문이 모호할 가능성. 더 구체적으로 다시 물어본다.
→ "방금 답변이 맞는지 확인해줘" 라고 요청할 수도 있다.

**"승인 창이 떠서 무서워요"**
→ 위의 "도구 승인 이해하기" 섹션 참조. 잘 모르면 Deny.

**"파일을 잘못 수정했어요"**
→ Git이 있으므로 되돌릴 수 있다.
→ "방금 수정한 파일 원래대로 되돌려줘" 요청.

**"AI가 너무 많이 한 번에 해요"**
→ "한 단계씩 천천히 하자. 먼저 OO부터." 라고 범위를 좁혀준다.
