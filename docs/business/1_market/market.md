# 시장

> 문제 배경/핵심 문제는 `docs/business/README.md`를 참고하세요.

## 비전

- **가톨릭 교회 내 모든 모임을 위한 그룹 운영 플랫폼** (출석·멤버·행사·통계 일관 관리)
- 1차 검증: 주일학교 중고등부 교리교사 → 이후 레지오, 성가대, 청년회 등 확장

## 타겟 사용자
### 가설
- **1차 사용자 (현재 검증)**: 서울대교구 233개 본당 주일학교 교리교사(중고등부 우선)
- **2차 사용자**: 초등부 교리교사, 주일학교 외 모임 운영자(레지오, 성가대, 청년회 등)
- **3차 사용자**: 전국 교구 가톨릭 모임 운영자
- 협력 채널: 지구/교구 청소년국 사목부서 → 이후 본당 사목위원회 확대
- 최종 의사결정자: 사제 (주일학교), 모임별 책임자 (기타 모임)

### 근거
- 청소년국이 본당·학교 지원 역할을 명시하고, 교리교사 접점(자료실/공지)이 존재
- 주일학교 운영의 핵심 실행은 교리교사이며, 최종 결정은 사제가 한다는 사용자 합의

### 검증
- 목적: 사용자/의사결정 구조 확정 — 도입 요청/승인/결제 기록과 역할/권한 로그로 추정
- 완료 기준: 도입 주체/승인 경로 유형 정리

## 사용자 특성 (2030세대)

> 파일럿 본당 교리교사 연령대: 20~30대

### 핵심 특성
- **디지털 네이티브**: 새 도구 시도 장벽이 낮으나, 직관적이지 않으면 즉시 삭제
- **첫인상 결판 + Silent Churn**: 첫 경험에서 가치를 못 느끼면 재시도 없이 조용히 이탈
- **커스터마이징 선호**: 자기 방식대로 정보를 구성 (예: 비고란 활용)
- **SNS 유입에 익숙**: 인스타그램 → 랜딩 → 가입 흐름이 자연스러운 세대
- **요노(YONO) 소비**: 유료 전환 시 확실한 가치 증명 필요. "가톨릭 전용" 포지셔닝이 가치 공감 유발 가능

### 행동-보상 원칙

> 2030세대는 "행동 → 확실한 보상" 연결이 빠를수록 움직인다. 온보딩, 기능 설계 전반에 적용 (상세 사례: `docs/business/3_gtm/gtm.md`)

### 이탈자 복귀 가능성
- ~~**A (구조 혼란)**~~: 계정 모델 전환 완료로 해소 (2026-03)
- **B (대기)**: 바빠서 못 씀 → 시간 여유 시 자연 복귀 (높음)
- **C (가치 미체감)**: 가치를 못 느낌 → 1:1 통화 후 판단 (낮음)
- 대체재 0개 + 매주 반복 니즈로 일반 앱보다 복귀 창이 넓음

### 사용자 파악 방법
- **행동 데이터 (GA4)**: 이탈 지점 이벤트 추적 (26개 구현 완료)
- **이탈 감지 알림**: 14일 미활동 단체 자동 감지 + 운영자 이메일 (구현 완료)
- **구조적 관찰**: 가입 후 첫 행동 이탈 지점 분석 (전환 퍼널 N=66 단체)

### 파일럿 근거
- 고덕동: 그룹 1개 생성 후 미사용 → **첫인상 결판** 확인
- 신당동/금호동: 계정 모델 혼란으로 이탈 → **직관성 민감** 확인
- 가재울/창동/길음: 가입 후 바로 출석 입력, 정착 → **첫인상 성공** 확인
- 성남동: 비고란 세밀 활용 → **커스터마이징 욕구** 확인
- 월곡동/창5동: 인스타 → 랜딩 → 가입 → **SNS 유입** 확인

### 제품 시사점

| 특성 | 시사점 | 현재 대응 |
|------|--------|----------|
| 첫인상 결판 | 첫 경험에서 막히면 영구 이탈 | 계정 모델 전환 구현 완료 (개인 계정 + 합류) |
| 직관적 UX 필수 | "본당 찾아 합류" 흐름이 자연스러워야 함 | 구현 완료. 조직 입력 UX 개선까지 완료 |
| SNS 유입 | 인스타 → 랜딩 퍼널 유효 | 이탈율 81.8% → 12.5% 개선 완료 |

### 역할 분리 관찰 (2026-04-19 누적 시작)
- 1계정 운영 본당(성북동 미카엘라)에서 "서기는 주로 조회, 입력자는 교사" 언급 — 계정 모델 전환 후에도 1계정 공유 운영이 잔존
- 시사점: 조회 중심 UX와 입력 중심 UX가 분리될 수 있음. 팀 전환 유인(추가 계정 부여)이 약한 조직은 단일 계정 유지 → 팀 전환 GTM 재고 필요
- 검증: 팀 사용 본당 vs 1계정 본당의 조회/입력 행동 비교, 역할 분리 언급 누적 관찰

## 사용자 니즈 신호
### 가설
- 핵심 니즈는 "출석 기록"보다 **"매주 필요한 정보(출석 확인 + 축일 + 멤버 현황)를 한곳에서 빠르게 조회"**하는 것
- 출석 조회 + 가톨릭 정보가 통합된 도구는 엑셀이 대체할 수 없다

### 근거
- 장위동 4년 사용 관찰: "누가 나왔는지 빠른 확인"이 핵심 가치, 축일 검색에도 활용
- 피드백에서 통계/현황, 출석 UX, 대량 처리, 축일/등록/동의 관리 요구 반복(상세: `docs/business/0_feedback/feedback.md`)

### 검증
- 목적: 핵심 니즈 상위 항목 도출 — 기능 사용률/처리 시간 기준 상위 5개 항목 확정

## 시장 규모

### 가설
- **주일학교 (1차)**: 서울대교구 233개 본당 × 중고등부 → 최소 233개
- **본당 내 기타 모임 (2차)**: 레지오, 성가대, 청년회 등 → 본당당 5~10개 추정
- 18개 지구는 협력/확산 채널, 청소년국 중고등부/학교사목부는 초기 협력 채널

### 근거
- 서울대교구 본당현황: 성당 233개 — 중서울 85, 동서울 75, 서서울 73 ([본당현황](https://aos.catholic.or.kr/pro10317?upCode=1,2,3))
- 청소년국 8개 사목부서 운영 ([청소년국](https://aos.catholic.or.kr/con35)), 포털에 교리교사 접점 존재 ([SSO](https://sso.eduseoul.or.kr/))

### 검증
- 목적: 운영 여부/규모 실측 — 가입/활성 주일학교 수, 반 수, 계정 수 DB 집계로 분포 확인

## 경쟁 환경

> 상세: `docs/business/1_market/competitive-analysis.md` (2026-02-13 조사, 03-16 브랜치스 추가)

### 핵심 발견
- **가톨릭 전용 주간 운영 도구가 시장에 존재하지 않음** (출석 + 축일 + 멤버 통합)
- 개신교 ChMS 6~7개 존재하나 가톨릭 전용은 공백
- 가톨릭 하상 = 신자 앱, 양업시스템 = 사무실 도구 (교리교사 접근 불가)

### 경쟁 구도

| 카테고리 | 대표 서비스 | 우리와의 관계 |
|---------|-----------|------------|
| 개신교 ChMS | 오직교적, 만나ERP 등 | 용어/구조 불일치, 기능 과잉 |
| **개신교 직접 경쟁자** | **브랜치스 (95교회, 사업화)** | **문제 인식 유사. 가톨릭 도메인 장벽으로 방어** |
| 가톨릭 공식 앱 | 하상, 양업시스템 | 운영 도구가 아닌 신자 앱/사무실 도구 |
| 범용 출석 앱 | Attendable, Wasso | 교회 특화 기능 없음 |
| 수기/오피스 | 엑셀, 카카오톡+수기 | **현재 최대 대체재** — 분산/유실 |
| 해외 ChMS | Planning Center 등 | 영어 전용, 한국 가톨릭 구조 불일치 |

### 가설
- 대체재는 "엑셀/한글 + 카카오톡 + 수기"이며, 전용 SaaS 침투율 0%에 가까움
- 개신교 ChMS는 용어/구조가 달라 전환 대상이 아님

### 근거
- 경쟁사 조사(2026-02-13)에서 가톨릭 전용 출석 관리 도구 0개 확인
- 현장 조사에서 수기 → 엑셀 → 카카오톡 공유 워크플로우가 가장 흔함

### 검증
- 목적: 대체재 사용 패턴과 전환 비용 파악
- 방법: 본당 10곳 이상 현행 툴/워크플로우 조사
- 완료 기준: 대체재 유형 3개 이상 분류

## 차별화

### 가설
- **"매주 일요일에 이거 하나 열면 다 되는" 도구**가 핵심 차별점 — 출석 조회 + 가톨릭 정보(축일/전례력) + 멤버 현황 통합

### 차별점

| 차별점 | 설명 | 경쟁사 대비 |
|--------|------|-----------|
| 주간 운영 허브 (유일) | 출석 + 축일 + 멤버 한곳에 | 엑셀은 출석만, 축일은 별도 검색 |
| 가톨릭 전용 (유일) | 의무축일, 학년 전환, 가톨릭 용어 | 시장에 0개 — 대체 불가 |
| 핵심에 집중 | 운영에 필요한 것만 | 개신교 ChMS는 90% 불필요 기능 |
| 웹 기반 + 합리적 가격 | 설치 없이 접근, 낮은 가격대 | 개신교 ChMS 월 7,700~39,000원 |
| 학년 자동 전환 | 연간 재작성 불필요 | 엑셀/수기는 매년 재작성 |

### 근거
- 장위동 4년 사용 관찰: "빠른 조회 + 축일 검색"이 실제 사용 이유
- 경쟁사 분석에서 가톨릭 전용 도구 0개 확인
- 엑셀은 축일/전례력/멤버 현황을 한곳에서 제공하지 못함

### 검증
- 목적: 통합 관리의 효과 확인
- 방법: 연간 입력 시간 절감, 출석 누락 감소, 보고 준비 시간 전후 비교
- 완료 기준: 개선 효과 1차 수치 확보
