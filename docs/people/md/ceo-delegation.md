# 대표 위임 체크리스트

대표가 사업 개발(본당 방문, 사제 네트워크, 파트너십)에 전념할 수 있도록
기술·콘텐츠 역할을 4주 내 완전 위임하기 위한 체크리스트.

## 온보딩 전 준비물

### 프로덕트 엔지니어용

- [ ] `.env.example` 최신화 확인 (모든 필수 변수 포함)
- [ ] fresh machine에서 `pnpm install && pnpm build && pnpm dev` 테스트
- [ ] 프로덕션 DB 읽기 전용 접근 준비
- [ ] GitHub "starter issue" 라벨 + 첫 버그 2-3개 태깅

### 콘텐츠 마케터용

- [ ] Instagram 현재 분석 데이터 export (기준선 지표)
- [ ] 프로덕트 스크린샷 5-10장 준비 (주요 화면별)
- [ ] 브랜드 가이드 보강: 로고 파일, 컬러 팔레트, 승인된 스크린샷
- [ ] 본당 방문 에피소드 3-5개 정리 (콘텐츠 소재)

### 공통

- [ ] 10분 프로덕트 데모 영상 녹화 (1회, 재사용)
- [ ] "위클리스쿨 한 장 요약" 작성: 비전, 지표, 페르소나, 경쟁 포지션
- [ ] Conductor 라이선스 2개 준비 (각 팀원용)
- [ ] AI 가이드 3종 전달: `ai-setup-guide.md` (설치) → `ai-basics-guide.md` (기초) → `ai-usage-guide.md` (실전)

## 주차별 위임 타임라인

### Week 1: 기초 세팅

| 위임 항목 | 대상 | 대표 역할 |
|----------|------|----------|
| 로컬 개발환경 구축 | 엔지니어 | 질의응답 대응 |
| 프로덕트 전체 사용 | 마케터 | 데모 영상 제공 |
| 사업 문서 읽기 | 양쪽 | 질문 대응 |

**대표 투입 시간**: ~4시간 (합동 1h + 질의응답 3h)

### Week 2: 첫 성과물

| 위임 항목 | 대상 | 대표 역할 |
|----------|------|----------|
| 첫 버그 수정 PR | 엔지니어 | 페어 프로그래밍 2시간, 배포 시연 |
| 첫 카드뉴스 게시 | 마케터 | 콘텐츠 리뷰 1시간, DM 응대 시연 |

**대표 투입 시간**: ~4시간 (페어 2h + 리뷰 1h + 시연 1h)

### Week 3: 독립 수행

| 위임 항목 | 대상 | 대표 역할 |
|----------|------|----------|
| SDD 기반 기능 개발 | 엔지니어 | PRD 비동기 리뷰 30분, PR 리뷰 30분 |
| 주 2건 콘텐츠 + DM 응대 | 마케터 | 비동기 피드백 30분 |

**대표 투입 시간**: ~1.5시간 (비동기 리뷰)

### Week 4: 인수인계 완료

| 위임 항목 | 대상 | 대표 역할 |
|----------|------|----------|
| 프로덕션 배포 + 운영 | 엔지니어 | 지식이전 세션 2시간 |
| Instagram 완전 인수 | 마케터 | 지식이전 세션 1.5시간 |

**대표 투입 시간**: ~3.5시간 (지식이전 세션)

## 4주 후 역할 경계

### 대표가 유지하는 것

- 사업 전략 (`docs/business/`) + 로드맵 우선순위 결정
- **사제 네트워크** — 본당 방문, 관계 관리 (최고 레버리지 활동)
- 파트너십 — 교구 청소년국, 양업 시스템 통합
- 가격/재무 결정 — 구독 전환, 광고 수익, 비용 관리
- 기능 우선순위 — STATUS.md 지표 기반 의사결정
- PRD 리뷰 — 새 기능의 방향성 확인 (30분/건)

### 대표가 더 이상 하지 않는 것

- 코드 작성 / 대부분의 PR 리뷰
- Instagram 콘텐츠 기획·제작·게시
- Instagram DM 응대
- 프로덕션 배포·모니터링
- SDD 문서 작성 (PRD 리뷰만)
- 일상 버그 리포트 처리
- 콘텐츠 캘린더 관리

## 성공 지표

### 4주 후 확인 사항

| 지표 | 기준 |
|------|------|
| 엔지니어 독립성 | 버그 접수→배포 4시간 내 단독 처리 |
| 마케터 독립성 | 콘텐츠 기획→게시 대표 리뷰 없이 단독 수행 |
| 대표 시간 확보 | 기술·콘텐츠 업무 주당 2시간 이하 (비동기 리뷰만) |
| 서비스 연속성 | 배포 장애 없음, DM 24시간 응대 유지 |

### 위임 실패 시그널

- 엔지니어가 Week 3에도 대표 페어 프로그래밍 요청
- 마케터가 Week 3에도 모든 콘텐츠 대표 리뷰 요청
- 대표가 주당 8시간 이상 기술/콘텐츠에 투입
- 프로덕션 장애 시 대표만 대응 가능

위 시그널 발생 시 온보딩 기간 연장 또는 역할 범위 조정 검토.
