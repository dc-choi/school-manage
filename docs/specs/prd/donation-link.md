# PRD: 도네이션 링크 + 앱 내 후원 안내

> SDD 작성자가 작성하는 제품 요구사항 문서입니다.
> 사업 에이전트의 `docs/business/` 문서를 기반으로 작성합니다.

## 배경/문제 요약

- 참고: `docs/business/2_bm/bm.md` (자금원 구조, 도네이션 가설)
- 문제: 현재 서버 운영비(Lightsail 월 ~$10)를 개인이 전액 부담하고 있으며, 사용자가 후원할 수 있는 경로가 없음
- 현재 상태: 후원 채널 없음. 개인 자금 전액 부담
- 목표 상태: 카카오페이 송금 링크를 통해 사용자가 자발적으로 서버 운영비를 후원할 수 있는 경로 제공

## 목표/성공 기준

- **목표**: 앱 내에서 후원 안내를 노출하고, 카카오페이 송금 링크로 연결하여 자발적 후원 유도
- **성공 지표**: 후원 링크 클릭 수, 월 후원 금액
- **측정 기간**: 출시 후 4주

## 사용자/대상

- **주요 사용자**: 로그인한 교리교사 (관리자/일반 모두)
- **사용 맥락**: 설정 페이지 또는 사이드바에서 후원 안내를 발견하고 외부 송금 링크로 이동

## 범위

### 포함

- 앱 내 후원 안내 UI (사이드바 푸터 + 설정 페이지 섹션)
- 카카오페이 송금 링크 외부 연결
- 후원 링크 클릭 GA4 이벤트 트래킹

### 제외

- PG 결제 연동 (사업자 등록 불필요한 범위로 제한)
- 후원 금액 앱 내 표시/관리
- 후원자 목록/보상 기능
- 랜딩 페이지 내 후원 안내 (인증 사용자 대상으로 한정)

## 사용자 시나리오

1. **사이드바에서 후원 발견**: 교리교사가 출석 기록 후 사이드바 하단의 후원 링크를 발견 → 클릭 → 설정 페이지 후원 섹션으로 이동 → 카카오페이 링크 클릭 → 외부 송금 페이지로 이동
2. **설정 페이지에서 후원**: 교리교사가 설정 페이지에서 "후원하기" 섹션을 확인 → 후원 메시지를 읽고 → 카카오페이 링크 클릭 → 외부 송금 페이지로 이동

## 요구사항

### 필수 (Must)

- [ ] 설정 페이지에 후원 안내 섹션 추가 (메시지 + 카카오페이 링크)
- [ ] 사이드바 푸터에 후원 링크 추가
- [ ] 후원 메시지: "이 서비스는 교리교사가 만든 봉사 프로젝트입니다. 후원금은 서버 운영비로 사용됩니다"
- [ ] 외부 링크로 연결 (`target="_blank"`, `rel="noopener noreferrer"`)
- [ ] 후원 링크 클릭 시 GA4 이벤트 전송 (`donation_link_click`)

### 선택 (Should)

- [ ] 모바일 햄버거 메뉴에서도 후원 링크 접근 가능

### 제외 (Out)

- 앱 내 결제/송금 처리
- 후원 금액 추적 대시보드
- 후원자 리워드/뱃지
- 게스트(비인증) 사용자 대상 후원 안내

## 제약/가정/리스크

- **제약**: 카카오페이 송금 링크는 환경변수로 관리 (링크 변경 시 재배포 필요)
- **가정**: 가톨릭 봉사 문화 특성상 "운영비 후원" 포지셔닝이 거부감 없이 수용될 것
- **리스크**: 후원 안내가 과도하면 사용 경험을 해칠 수 있음 → 최소한의 비침투적 UI로 제한

## 의존성

- **내부**: 없음 (기존 UI 컴포넌트 활용)
- **외부**: 카카오페이 송금 링크 생성 (운영자가 사전 생성)

## 롤아웃/검증

- **출시 단계**: 즉시 전체 배포 (A/B 테스트 불필요)
- **이벤트/로그**: `donation_link_click` (location: `sidebar` | `settings`)
- **검증 방법**: GA4에서 클릭 수 확인, 4주 후 월 후원 금액 집계

## 오픈 이슈
