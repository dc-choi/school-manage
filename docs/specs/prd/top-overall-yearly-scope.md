# PRD: 전체 우수 출석 TOP 5 연간 스코프 고정 (S-1)

> 상태: Approved | 작성일: 2026-06-01 | 구현 완료: 2026-06-01

> quick 플로우 — 명백한 버그 수정. 필수 섹션만 작성.

## 배경/문제 요약

- 문제: 대시보드 "전체 우수 출석 학생 TOP 5" 카드가 데스크탑에서 전원 같은 점수(2점)로 표시되어 순위가 무의미함.
- 현재 상태: `useDashboardStatistics`가 `topOverall` 쿼리에 대시보드의 기간 필터(`month`/`week`)를 그대로 전달. 데스크탑 대시보드는 기본 필터가 "현재 월/현재 주차"로 초기화되므로, `topOverall`이 단일 주 범위로 조회됨. 출석 점수는 누적 합산(◎=2/○=1/△=1)이고 `@@unique([studentId, date])` + 주당 일요일 1개 제약상 단일 주 최대 점수가 2점 → ◎ 출석자들이 전부 2점으로 붕괴.
- 모바일은 대시보드에서 TOP 5를 숨기고(`hidden md:grid`), `/statistics`(연간 기본값)에서만 노출하므로 정상으로 보임.
- 목표 상태: 카드 제목 "전체(누적)" 의미에 맞게 항상 연도 스코프로 조회하여, 기간 필터와 무관하게 연간 누적 순위를 표시.

## 목표/성공 기준

- **목표**: TOP 5가 연간 누적 점수 기준으로 다양한 점수/순위를 표시.
- **성공 지표**: 데스크탑 대시보드 기본 진입 시 TOP 5 점수가 단일 값(2점)으로 붕괴되지 않음. 모바일/데스크탑 동일 결과.
- **측정 기간**: 배포 직후 육안 + 회귀 테스트.

## 사용자/대상

- **주요 사용자**: 교사/관리자 (대시보드 + 통계 페이지 조회자).
- **사용 맥락**: 데스크탑 대시보드 진입 시 우수 출석 학생 순위 확인.

## 범위

### 포함

- `useDashboardStatistics`의 `topOverall` 쿼리에서 `month`/`week` 제외 → `{ year, limit: 5 }`.

### 제외

- `byGender`, `groupStatistics`의 기간 필터(의도된 기간 스코프 — 변경 없음).
- 백엔드 `getDateRange` / `GetTopOverallUseCase` (다른 유스케이스 공유 — 변경 없음).
- 점수 산식, 카드 UI/제목 변경.

## 요구사항

### 필수 (Must)

- [ ] `topOverall` 쿼리가 `month`/`week`에 종속되지 않고 연도 스코프로 고정.
- [ ] `byGender`/`groupStatistics`는 기존 기간 필터 동작 유지.
- [ ] 회귀 테스트로 호출 인자 검증.

### 제외 (Out)

- 카드 제목을 기간 종속으로 바꾸는 대안(데이터를 정답으로 보는 방향)은 채택하지 않음.

## 제약/가정/리스크/의존성

- **제약**: 프론트 1파일 수정.
- **가정**: "전체 우수 출석"의 의도된 의미는 연간 누적 순위.
- **리스크**: 낮음. 통계 페이지에서 기간 필터 변경 시 TOP 5는 연간 고정으로 유지됨(의도된 동작).
