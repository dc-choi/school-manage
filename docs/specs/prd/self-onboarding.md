# PRD: 셀프 온보딩 (최소 가이드)

## 배경/문제 요약

- 참고: `docs/business/STATUS.md` (고덕동/돈암동 이탈 재해석), `docs/business/3_gtm/gtm.md` (온보딩)
- 문제: 가입 후 "다음에 뭘 해야 하는지 모름" + "그룹/멤버 구조를 이해 못함"으로 이탈 발생
  - 고덕동: 그룹 1개 생성 후 멈춤 (멤버 등록 미진행)
  - 돈암동: 멤버 1명만 입력 후 멈춤 (그룹/멤버 구조 미이해)
  - 신당동: 인스타 경유 가입 후 진행 멈춤 (유사 패턴 추정)
  - 신정동: 인스타 경유 가입 후 진행 멈춤 (유사 패턴 추정)
- 현재 상태: 가입 후 대시보드에 빈 차트/테이블만 표시. 안내 문구 없음. 빈 상태 메시지는 "등록된 그룹이 없습니다." 수준
- 목표 상태: 가입 직후 "그룹 만들기 → 멤버 등록 → 출석 체크" 흐름을 자연스럽게 안내하여 첫 주 안에 핵심 루프 진입

## 목표/성공 기준

- **목표**: 신규 가입 사용자가 그룹 생성 → 멤버 등록 → 출석 기록까지의 핵심 흐름을 자력으로 완료
- **성공 지표**:
  - 가입 후 7일 내 "그룹 생성 + 멤버 1명 이상 등록" 완료율: 현재 ~25% (4곳 중 1곳만 진행) → 80% 이상
  - 가입 후 7일 내 "첫 출석 기록" 완료율 증가
- **측정 기간**: 출시 후 4주

## 사용자/대상

- **주요 사용자**: 방금 가입한 신규 사용자 (그룹/멤버/출석 데이터가 없는 상태)
- **사용 맥락**: 가입 직후 또는 가입 후 첫 방문 시. 서비스 구조(그룹 → 멤버 → 출석)를 모르는 상태

## 범위

### 포함

- 대시보드 온보딩 체크리스트 (신규 사용자에게만 표시)
- 빈 상태 페이지 안내 강화 (그룹/멤버/출석 페이지)
- GA4 온보딩 이벤트 추적

### 제외

- 투어/오버레이/코치마크 (과도한 개입)
- 온보딩 건너뛰기/다시 보지 않기 (체크리스트는 완료 시 자동 소멸)
- 백엔드 API 변경 (프론트엔드만 변경)
- 샘플 데이터 자동 생성

## 사용자 시나리오

1. **시나리오 1 — 첫 방문 (대시보드)**: 사용자가 가입 후 대시보드에 진입한다. 빈 차트 대신 "시작하기 가이드" 체크리스트가 표시된다. "그룹 만들기"가 첫 번째 단계로 강조되어 있고, 클릭하면 그룹 추가 페이지로 이동한다.

2. **시나리오 2 — 그룹 생성 후**: 그룹을 만든 뒤 대시보드로 돌아오면 체크리스트의 1단계가 완료 표시되고, 2단계 "멤버 등록"이 강조된다. 클릭하면 멤버 추가 페이지로 이동한다.

3. **시나리오 3 — 멤버 등록 후**: 멤버를 등록한 뒤 체크리스트의 2단계가 완료 표시되고, 3단계 "출석 체크"가 강조된다.

4. **시나리오 4 — 모든 단계 완료**: 출석 기록까지 완료하면 체크리스트가 "준비 완료!" 메시지로 변경되고, 이후 대시보드 진입 시 체크리스트가 표시되지 않는다 (일반 대시보드 표시).

## 요구사항

### 필수 (Must)

- [ ] 대시보드에 온보딩 체크리스트 표시 (온보딩 미완료 사용자에게)
- [ ] 체크리스트 3단계: ① 그룹 만들기 → ② 멤버 등록 → ③ 출석 체크
- [ ] 각 단계에 간결한 설명과 CTA 버튼 (해당 페이지로 이동)
- [ ] 완료된 단계는 체크 표시 + 비활성화
- [ ] 현재 진행할 단계 강조 (다음 미완료 단계)
- [ ] 모든 단계 완료 시 체크리스트 자동 소멸 (대시보드 정상 표시)
- [ ] 실시간 상태 반영: 기존 API 데이터(그룹 수, 멤버 수, 출석 기록 수)로 완료 여부 판단

### 선택 (Should)

- [ ] 빈 상태 페이지 안내 강화: 그룹/멤버/출석 페이지에서 데이터 없을 때 다음 단계 안내 문구 추가
- [ ] GA4 온보딩 이벤트: 체크리스트 표시/단계 클릭/완료 이벤트 추적

### 제외 (Out)

- 백엔드 API 추가/변경
- 투어/오버레이/팝업
- 온보딩 진행 상태 서버 저장 (프론트에서 API 데이터 기반 계산)
- 관리자용 온보딩 통계 대시보드

## 제약/가정/리스크

- **제약**: 프론트엔드만 변경. 기존 API 데이터로 온보딩 상태 판단 (별도 onboarding API 없음)
- **가정**:
  - 그룹 0개 = 온보딩 미완료 사용자로 간주
  - 그룹 1개 이상 + 멤버 0명 = 2단계 진행 중
  - 그룹 1개 이상 + 멤버 1명 이상 + 출석 0건 = 3단계 진행 중
  - 출석 1건 이상 = 온보딩 완료
- **리스크**: 기존 사용자가 모든 데이터를 삭제한 경우 체크리스트가 다시 표시될 수 있음 → 수용 가능 (데이터 없으면 안내가 유용)

## 의존성

- **내부**: 기존 API — `statistics.weekly` (그룹/멤버/출석 데이터 포함), `group.findAll`, `student.findAll`
- **외부**: 없음

## 롤아웃/검증

- **출시 단계**: 전체 사용자 대상 즉시 배포 (그룹 0개인 사용자에게만 표시되므로 기존 사용자 영향 없음)
- **이벤트/로그**: `onboarding_checklist_shown`, `onboarding_step_clicked`, `onboarding_completed`
- **검증 방법**: GA4에서 가입 후 7일 내 그룹/멤버/출석 생성 전환율 비교 (before/after)
