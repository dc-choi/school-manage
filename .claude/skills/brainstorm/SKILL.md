# 브레인스토밍 실행

$ARGUMENTS 주제에 대해 아래 워크플로우를 순서대로 실행해라.

## Phase 1: 아이디어 생성

biz-planner 서브에이전트에게 위임해라:
- 주제: "$ARGUMENTS"
- 요청: 아이디어 2~3개 제안

biz-planner의 결과를 기록한다.

## Phase 2: 3라운드 토론

Round 1부터 Round 3까지 반복해라. 각 라운드마다:

1. biz-critic 서브에이전트에게 위임해라:
   - 입력: 직전 biz-planner 발언 전체
   - 요청: 각 아이디어 비판

2. biz-planner 서브에이전트에게 위임해라:
   - 입력: 직전 biz-critic 비판 전체
   - 요청: 반론 및 개선안 제시

3. 라운드 요약을 기록해라:
   - 합의된 사항
   - 미합의 사항
   - 아이디어 변경 내역

3라운드가 끝나기 전에 Phase 3으로 넘어가지 마라.

## Phase 3: Decision Tree 정리

3라운드 토론 결과를 바탕으로 각 아이디어에 Decision Tree를 적용해라:

```
Q1. 사용자 가치가 명확한가? (파일럿 데이터/피드백 근거)
 ├─ NO → 기각
 └─ YES ↓

Q2. 1인 개발로 3개월 내 MVP 가능한가?
 ├─ NO → 스코프 축소 후 재평가
 └─ YES ↓

Q3. 가톨릭 전용 도구로서 차별점이 있는가?
 ├─ NO → 피벗 필요
 └─ YES ↓

Q4. 수익 모델/성장에 기여하는가?
 ├─ NO → 조건부 (성장 기여 보완 필요)
 └─ YES → 승인
```

## Phase 4: 결과 저장 및 사용자 결정 요청

1. 전체 과정을 `docs/business/brainstorm-result.md` 파일로 저장해라.
   파일 구조는 brainstorm-rules 스킬의 "최종 보고서 형식"을 따라라.

2. 사용자에게 결과를 요약하고 최종 판단을 요청해라:
   - 승인된 아이디어 중 우선순위
   - 조건부/피벗 아이디어의 보완 방향
   - 다음 단계 (STATUS.md 반영, SDD 핸드오프 등)
