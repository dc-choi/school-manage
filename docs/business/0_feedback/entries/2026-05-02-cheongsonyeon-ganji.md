# 2026-05-02 서울대교구 청소년국 간사 (보안/데이터 소유권)

**요약**: 5/2(토) 청소년국 동반 모임 대면 대화. 서버 기반 SaaS의 데이터 소유권/해킹 시 개인정보 유출을 우려해 "사용을 꺼린다" 워딩. 개인 차원 의견 (청소년국 차원 검토 아님). 도입 게이트키퍼급 페르소나 첫 보안 인식 시그널.

**맥락**

- 대상: 서울대교구 청소년국 중고등부 간사 (교구 직원)
- 수집 방법: 대면 (청소년국 동반 모임 자리)
- 의견 형태: 개인 차원 (청소년국 공식 입장 아님)
- 채널 형성: 사용자(개발자) 본인이 교리교사 + 청소년국 활동 겸직 → 동반 모임 자연 노출
- 관련 이전 피드백: 없음 — 보안 인식 기반 도입 거부 첫 시그널

**원문/주요 발언**

> 서버 기반이라 데이터가 결국 우리 것이 아닌 것 같다.
> 해킹이라도 당하면 학생들/교사들 개인정보 유출 책임이 누구한테 있는지 모르겠다.
> 그래서 (도입을) 꺼리게 된다.

**제품 시사점**

- **신규 페르소나**: 교구 청소년국 직원 = 도입 결정 게이트키퍼 (사제 네트워크 외 라인). 청소년국이 본당 추천을 매개할 경우 진입 차단 가능
- **R15와 분리**:
    - R15 = 법적 컴플라이언스 (과태료/규제) — 동의 절차 구현 완료, 법률 자문 미완
    - R25 = 사용자 인식 기반 도입 거부 — 신뢰 시그널 부재가 트리거
- **신뢰 시그널 정비 필요**:
    - 보안 페이지 (암호화·백업·접근 제어 명시)
    - 데이터 소유권 명문화 (이용약관)
    - 개인정보처리방침 가시성 (Footer 정비 오픈 이슈와 묶음)
- 우선순위 후보: P2 (5/6 협력 신부 미팅 어젠다와 무관, 별도 트랙)
- 로드맵 단계: 2단계 신뢰 시그널 정비 + 3단계 PG 본계약 시점 보안 인증 검토
