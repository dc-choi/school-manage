# GTM

## 포지셔닝

> 핵심 근거: 가톨릭 교리교사/모임 운영자가 매주 여는 주간 운영 도구가 시장에 없음 (상세: `docs/business/1_market/competitive-analysis.md`)

### 범용 메시지
"매주 일요일, 이거 하나 열면 됩니다. 출석·축일·멤버가 한곳에."

### 교리교사 메시지 (1차 검증)
"누가 나왔는지, 이번 주 축일이 뭔지, 달력에서 바로 확인하세요."

### 사제/책임자 메시지
"통계/보고 자동화로 의사결정 리드타임 단축"

### 경쟁 차별 메시지 (내부용)
- 엑셀은 출석만, 축일은 따로 검색 → **한곳에 다 있는 도구가 없었음**
- 가톨릭 하상 ≠ 운영 도구 (신자 개인 앱)
- 개신교 ChMS ≠ 가톨릭 (용어/구조 불일치, 기능 과잉)

| 콘텐츠 유입      | 실무용 자료 (출석부 템플릿/체크리스트) | 미구현                                       |

## 전환 경로

### 인스타그램 → 가입
```
인스타그램 게시물/하이라이트 (인지 + 설명)
    ↓
프로필 링크 클릭
    ↓
/landing (서비스 설득, 퍼블릭)
    ↓
/signup 또는 /login
    ↓
/ (대시보드)
```

- **A. 인스타그램 자체를 랜딩으로 활용**: 프로필 소개 + 하이라이트(기능소개/FAQ/사용후기)로 서비스 설명 → 추가 개발 없이 즉시 가능
- **B. 로그인 페이지에 서비스 소개 추가**: 기능 안내 3개 + 서비스 소개 텍스트 (**완료**, 2026-02-11)
- A + B 병행 → 인스타에서 충분히 이해한 뒤 로그인 페이지에서 한 번 더 확인
- ~~**C. 랜딩 페이지 도입**~~: `/landing` 경로로 기존 앱 내 구현, 인스타 프로필 링크 연결 (**완료**, 2026-02-12)

### 교사 네트워크 → 가입
```
기존 사용자 (장위동 레퍼런스)
    ↓ 카카오톡으로 랜딩 페이지 링크 공유
관심 있는 교리교사
    ↓ /landing 확인 → /signup
회원가입
```

- 별도 에셋 불필요 — 랜딩 페이지가 설명/설득을 담당
- "이거 써보는데 괜찮아" + 랜딩 링크 하나면 충분

### 계정 모델 안내 (완료, 2026-02-11)
- **주일학교(모임) 하나당 계정 하나** — 개인 계정이 아님을 가입 시 명확히 안내
- 로그인 페이지에 공유 계정 안내 문구 추가 완료

## 세일즈 모션

```
모임 운영자 직접 접촉 (1차: 교리교사)
    ↓
2~4주 파일럿 (운영자 주도)
    ↓
책임자 공유/동의 (필요 시)
    ↓
운영자 개인 결제 중심 유료 전환
```

## 리텐션

- **주간 허브**: 매주 일요일 출석 확인 + 축일 조회 + 멤버 현황 → 자연스러운 주간 재방문
- 월간: 통계/리포트 확인
- 핵심: 출석 "기록"이 아닌 "조회"가 재방문 동기 — 가톨릭 정보(축일/전례력)가 결합되면 엑셀로 돌아갈 이유가 없어짐
