# 2026-04-24 장현도 프란치스코 형 (주일학교 교사, 카카오톡 대화)

**요약**: 과거 출석 엑셀 업로드 요청으로 시작된 대화. 대화 중 "회의록→캘린더 자동 연동" 아이디어 발화 — 해자 후보로 재검토 트리거.

**피드백 항목**

| 피드백 | 분류 | 대응 |
|--------|------|------|
| 과거 출석 엑셀 일괄 업로드 | 데이터 관리 | 로드맵 2단계 잔여 작업 추가 |
| 교사 공유 캘린더 | 일정/행사 | 3단계 베이직 신규 |
| 회의록(미사/교리/공지/안건)→캘린더 자동 연동 | 일정/행사 | **해자**. 4단계 프로 시그니처 (단일 통합 시스템) |
| AI 회의록 자동 정리 (녹음→AI 요약) | 일정/행사 | 4단계 프로 시그니처 일부. AI 비용 리스크 |
| 알림 발송 (공지/리마인드) | 알림/공지 | 로드맵 4단계→3단계(베이직) 이동 |

**맥락**
- 장현도: 주일학교 초등부 교사, 최동철 지인이자 파일럿 사용자 유형
- 실제 회의록 포맷 공유: 섹션=미사/교리/공지/안건, 공지 포맷="4/25 어린이미사없음 : 입니다"
- 공지 작성 원칙: 날짜만 기재 ("찾을때 편해서")
- 본인 WTP 시그널: "저렇게 적어놨는데 구글캘린더에 박힌다? 난 쓴다", "돈내고 쓸만하겠네"

**해자 논의 (최동철 관점)**
- "굳이 왜 써야 함?" — 단순 출석부만으로는 해자 부재
- 학생 정보 관리도 "어차피 핸드폰에 있음" 반박 가능 → 해자 불충분
- 해자 없이는 마케팅 포인트 확보 불가
- 합의: 유입 ↔ 체류 병행. 이번 대화에서 **체류(해자) 강화 아이디어 수집**

**패턴 분석**
- **강제 체류 트리거**: 회의록은 교사 주간 필수 업무 → 자동화 시 이탈=데이터 손실 → 전환 비용 발생
- 기존 해자(가치 2 빠른 조회)는 **편의성**, 회의록 자동화는 **필수 업무 대체** → 전환 비용 차원에서 더 강한 해자
