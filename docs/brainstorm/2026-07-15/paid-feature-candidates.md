# 브레인스토밍 결과 보고서

## 주제

Free Core를 유지하고 기존 1,200,000원 단계를 제거한 뒤, 실제 피드백과 현재 구현 상태를 근거로 어떤 신규 기능을 유료 후보로 탐색할 것인가?

이번 문서는 첫 번째 유료 기능 기획 배치다. 후보를 많이 쌓기 전에 각 후보의 증거 상한, 무료 경계, 개인정보 위험, 개발 진입 게이트를 같은 형식으로 검증하는 기준도 함께 만든다.

## 한 줄 결론

**첫 탐색 후보는 `고정 형식 회의 공지 → 검토 가능한 ICS 일정 후보`로 좁힌다. 정형 보고서는 문제 탐색 보류, 과거 출석 이관은 장기 보류로 둔다. 어느 후보도 판매, 결제, 실제 데이터 접수, 개발 또는 로드맵 진입이 승인된 상태는 아니다.**

활성 가격 정책에서는 1,200,000원 단계를 제거했다. 2026-09-01부터 2026-11-29일까지는 Free Core와 Paid Parish Operations의 100,000원/300,000원 실제 수납을 우선 검증하고, 기존 계약과 손익 게이트를 모두 통과한 경우에만 별도 구매자에게 600,000원 연간 지원을 제안한다.

## 확정 전제

- 기존 사용자와 신규 사용자 모두 현재 핵심 기능을 Free Core로 사용한다.
- 자발적 운영 지원 여부로 기능, 서비스 기간, 배지 또는 우선 지원을 차등하지 않는다.
- Paid Parish Operations는 소프트웨어 기능 접근권이 아니라 고정 범위 진단과 운영 결과를 판매하는 별도 사업 실험이다.
- 기존 1,200,000원 관리형 단계는 활성 가격과 90일 실행 계획에서 제거한다.
- Future Paid Add-ons는 기존 핵심을 다시 잠그지 않고 신규 자동화 결과에만 선택 과금하는 장기 후보군이다.
- 이 문서의 후보는 사용자 결정 전까지 사업 문서, 가격표, 로드맵 또는 TARGET에 추가하지 않는다.

정책 근거는 [부분 유료화 보고서](partial-monetization.md), 90일 실행 기준은 [수익성 검토](revenue-viability.md)를 따른다.

## 무료/유료 경계

### Free Core로 유지할 것

- 학생과 그룹의 기본 관리
- 출석 입력과 조회
- 기본 통계와 현재 공개된 보고 화면
- 학생 명단 XLSX 등록과 중복 확인
- 관리자 양도와 현재 역할 체계
- 개인정보 보호, 권한 확인, 삭제와 보안 같은 기본 신뢰 요건
- 권한이 확인된 조직 관리자가 자기 조직의 원시 기록을 기계 판독 가능한 형태로 돌려받을 수 있다는 데이터 이동 정책 원칙

마지막 데이터 이동 항목은 정책 원칙이며 현재 고객용 Export가 구현됐다는 뜻이 아니다. 최소 범위는 학생/멤버 기본 데이터, 그룹 소속, 출석 날짜와 상태, 등록 기록이다. 계산 통계, 데이터 검증, 오류 진단, 제출 양식 가공과 외부 연동은 이 최소 범위에 포함하지 않는다.

### Future Paid Add-ons가 될 수 있는 것

- 기존 원시 기록을 검증하고 계산해 특정 제출 양식으로 만드는 결과
- 사용자가 반복 작성하는 고정 형식 문서에서 일정 후보를 추출하고 검수 가능한 파일로 만드는 자동화
- 안전한 출처 추적, 재실행과 롤백을 포함해 과거 데이터를 옮기는 별도 이관 결과
- SMS처럼 실제 외부 원가가 발생하는 신규 사용량 기능

유료 경계는 신규 결과에만 둔다. 데이터 접근을 막거나 현재 무료 기능을 축소해 결제를 유도하지 않는다.

## 토론 요약

### Round 1

#### 기획자 발언

기획자는 피드백과 구현 상태를 바탕으로 세 후보를 제안했다.

1. 고정 형식 회의록을 일정 후보로 바꾸는 반복형 기능
2. 과거 출석 엑셀을 옮기는 일회성 이관
3. 연간 또는 학기 데이터를 고정 양식으로 만드는 보고서

현재 구현된 학생 명단 Import, 중복 확인, 관리자 양도와 기본 통계는 유료로 다시 묶지 않는다고 선을 그었다.

#### 비판자 발언

- 수동 선결제가 증명하는 것은 사람이 납품한 결과의 WTP이지 자동화 제품 수요가 아니다.
- 기존 발화자와 관계 밖 한 곳의 결제만으로 독립 반복 수요를 주장할 수 없다.
- 90일 Paid Parish Operations와 표본, SKU, 계약, 원장과 시간을 분리해야 한다.
- 개인정보, 회계, 환불과 검수 절차가 없으면 선결제와 원본 수령을 금지해야 한다.
- 출석 Import에는 작업 출처 추적과 롤백 기반이 없고, 고객 Export도 미구현이다.
- 일회성 이관과 보고서 결제를 SaaS 경제성으로 승격하면 안 된다.

#### 라운드 요약

- 합의된 사항: Free Core 유지, 수동 결제의 증거 상한 고정, 기존 90일 실험과 분리, 개인정보와 계약 절차 선행, 상업 신호 뒤 기술 검토 실시
- 미해결 사항: 회의 일정 SKU, 가격 단위, 무료 데이터 이동 범위, 반복형 기능과 일회성 서비스의 서로 다른 성공 기준
- 아이디어 변경: 회의 일정은 반복형 제품 후보, 이관과 보고서는 일회성 또는 계절성 후보로 분리
- 다음 라운드에서 다룰 것: 회의 일정의 정확한 4주 SKU와 제품 진입 게이트, 이관과 보고서의 유지 여부

### Round 2

#### 기획자 발언

기획자는 회의 일정 후보를 4주 묶음으로 축소했다. 주 1건, 최대 4건의 고정 형식 텍스트에서 일정 후보를 만들고 사용자가 직접 가져갈 ICS 파일을 제공하는 수동 결과 실험을 제안했다. Google OAuth, 직접 캘린더 쓰기, 녹음, AI 요약과 실제 개인정보는 제외했다.

결과 WTP, 제한 프로토타입과 운영 제품을 서로 다른 게이트로 분리하고, 이관과 보고서는 활성 실험에서 제외했다.

#### 비판자 발언

- 적격 예산권자 5곳 중 2곳 결제는 결과 WTP 신호일 뿐 프로토타입 승인 근거가 아니다.
- 입력 길이, 일정 후보 수, 수정 횟수, 납기, 납품 형식과 시간 상한이 고정돼야 한다.
- 15,000원은 제품 가격이 아니라 단일 실험 가격 가설로만 써야 한다.
- 무료 데이터 이동의 정책 범위와 현재 미구현 상태를 분리해야 한다.
- 현재는 12개월 기대수익 예측이 불가능하며 가정별 손익 시나리오와 손익분기점 역산만 가능하다.
- 이관은 장기 보류, 보고서는 문제 탐색 보류로 강등해야 한다.

#### 라운드 요약

- 합의된 사항: 회의 일정만 활성 탐색 후보로 유지, 4주 단위 고정 SKU, 2/5 결제의 증거 상한, 이관과 보고서 보류
- 미해결 사항: 15,000원과 60분의 경제성, 작은 표본에서의 제품 판단, 프로토타입 시간 예산
- 아이디어 변경: 회의 일정 범위를 검수표와 ICS 파일까지로 축소하고 결과 WTP, 프로토타입, 운영 제품 게이트를 분리
- 다음 라운드에서 다룰 것: 적자 학습 실험과 양수 손익 실험 분리, 신규 독립 코호트 요구, 작은 표본의 판정 방식

### Round 3

#### 기획자 발언

기획자는 15,000원 실험을 다음 두 방식으로 분리했다.

- 학습 실험: 최대 60분의 수동 작업과 적자를 허용하되 총손실 예산을 사용자가 사전 승인
- 양의 공헌이익 실험: 내부 시간단가와 직접비를 먼저 고정하고 허용 노동시간을 역산. 현실적인 납품 시간이 나오지 않으면 판매 금지

동일 초기 2곳의 사용과 갱신은 운영 제품 근거가 아니라 확장 유료 파일럿 후보까지만 인정하고, 운영 제품 전 신규 독립 코호트 Gate D를 추가했다. 작은 표본의 80% 기준은 삭제하고 성공 건수, 전체 건수, 실패 유형과 수정 정도를 기록하도록 변경했다.

#### 비판자 발언

- 내부 시간단가가 15,000원 이상이면 60분 인건비만으로 매출이 소진되므로 15,000원, 60분, 양의 공헌이익을 동시에 요구할 수 없다.
- 같은 2곳의 효용과 갱신은 신규 고객 획득, 셀프 온보딩과 반복 가능한 지원을 증명하지 않는다.
- 작은 표본의 80%는 거짓 정밀도를 만든다. 분자, 분모와 실패 유형을 기록해야 한다.
- 보안 사고 0건은 성공률이 아니라 한 건 발생 시 즉시 중단 조건이다.
- 40시간은 프로토타입 승인 예산이 아니라 거절 상한이며 실제 투입은 별도 사용자 승인이 필요하다.
- 무료 Export는 보고서 후보의 재진입 조건이지 회의 일정 수동 실험의 선행 조건은 아니다.

#### 라운드 요약

- 합의된 사항: 1순위 후보와 Gate A 설계만 조건부 승인, 판매와 개발은 비승인, 학습과 이익 실험 분리, Gate D 추가, 작은 표본의 비율 판정 제거
- 남은 사용자 결정: 내부 시간단가, 실험 목적, 최대 손실 또는 노동 예산, 계약과 환불, 시작 시점, 프로토타입 시간, Gate D 규모, 무료 Export 우선순위
- 최종 변경: Gate C를 확장 파일럿 후보 단계로 강등하고, 운영 제품 판단 전에 신규 독립 코호트를 요구
- 다음 단계에서 다룰 것: 사용자가 1순위 방향을 승인하면 Gate 0의 사업 조건을 결정. 그 전에는 판매, 데이터 접수와 개발을 하지 않음

## 아이디어 목록

### 1. 고정 형식 회의 공지 → 검토 가능한 ICS 일정 후보

**상태: 1순위, 조건부 활성. 미래 결과 WTP 실험 설계만 승인 가능하다.**

- 사용자 문제: 교리교사가 회의 공지에 적은 날짜와 일정을 다시 캘린더에 입력해야 한다.
- 직접 근거: 실제 `미사/교리/공지/안건` 형식과 캘린더 자동 연동 WTP 발화 1건
- 구매 결과: 회의 공지를 해석해 사용자가 검수할 일정 후보와 직접 가져오기용 ICS 파일을 받는다.
- 유료 경계: 현재 출석 달력과 기본 통계는 무료로 유지하고, 신규 일정 변환 결과만 선택 과금 후보로 둔다.
- 기술 최소 범위: 고정 입력, 날짜/시간 후보 추출, 검수와 수정, ICS 생성
- 제외: 녹음, STT, AI 요약, PDF, 자유 형식 문서, Google OAuth, 직접 캘린더 쓰기, 다중 캘린더, 알림 발송
- 증거 상한: 수동 결제는 일정 결과물의 WTP만 증명한다. 제품 가격, 자동화 정확도, 셀프 온보딩, 갱신 또는 시장성을 증명하지 않는다.

#### Gate A 초안 SKU

다음 숫자는 관측값이나 출시 약속이 아니라 폐기 가능한 실험 범위를 고정하기 위한 신규 가정이다.

| 항목 | 고정 초안 |
| --- | --- |
| 기간 | 결제일부터 4주 |
| 입력 | 주 1건, 최대 4건 |
| 형식 | 미사/교리/공지/안건 네 섹션 일반 텍스트 |
| 길이 | 건당 최대 2,000자 |
| 일정 후보 | 건당 최대 5개, 4주 총 20개 |
| 출력 | 일정 후보 검수표와 사용자 직접 가져오기용 ICS 파일 |
| 수정 | 입력 1건당 취합 수정 요청 1회 |
| 납기 | 각 입력 접수 후 2영업일 이내 |
| 개인정보 | 학생/보호자 이름, 연락처, 건강정보 등 실제 개인정보 입력 금지 |
| 제외 | 녹음, STT, AI 요약, PDF, 자유 형식, Google OAuth, 직접 등록, 다중 캘린더, 긴급 납품 |
| 가격 | 4주 총 15,000원, 단일 실험 가격 가설. 출시 가격이 아님 |

15,000원은 현재 승인된 가격이 아니다. 실행 전 다음 중 하나를 선택해야 한다.

1. 학습 실험: 최대 60분 작업을 허용하되 예상 적자와 총손실 한도를 사전 승인
2. 양의 공헌이익 실험: 내부 시간단가를 확정한 뒤 아래 식으로 허용 노동시간을 역산하고, 납품이 불가능하면 판매하지 않음

`허용 노동분 = (실수령액 - 직접비 - 실패 영업 배분 - 환불 충당 - 지원비) / 내부 시간단가 × 60`

| Gate C, 기존 구매자 검증 | 초기 2곳의 사용 주차, 입력/수정/실패 건수, 셀프 입력, 지원시간, 실제 갱신과 계약별/전체 손익 기록 | 신규 코호트 확장 유료 파일럿 후보 | 운영 제품, 3개월 MVP, 로드맵 진입 |

모든 단계에서 입력 전체 건수와 `무수정 승인/표시 수정/의미 수정/실패/창업자 해석 필요` 건수를 함께 기록한다. 개인정보 노출, 무권한 접근 또는 잘못된 외부 전송이 한 건이라도 발생하면 즉시 중단한다. 40시간은 승인 예산이 아니라 프로토타입 거절 상한이다.

### 2. 정형 연간/학기 보고서

**상태: 2순위, 문제 탐색 보류. 유료 실험과 개발은 비승인이다.**

- 사용자 문제 가설: 기본 기록을 연간 또는 학기 제출 양식에 다시 집계하는 수고를 줄인다.
- 현재 근거: 동일 사용자 1곳의 연간 보고 연동과 데일리 합계 발화. 공통 공식 양식과 독립 예산권자 WTP는 없음.
- 미래 최소 범위: 하나의 공통 양식, 성별/학년/출석 집계, 누락과 이상값 목록, 결과 스냅샷, PDF 또는 XLSX
- 무료 경계: 권한이 확인된 조직 관리자가 원시 기록을 기계 판독 형식으로 돌려받는 경로는 유료 보고서와 무관하게 제공한다는 정책 원칙
- 유료 경계: 원시 기록의 검증, 계산, 오류 진단과 특정 제출 양식 가공
- 현재 차단 사유: 고객용 원데이터 Export 미구현, 공통 제출 양식 부재, 독립 결제자 수요 부재

재진입 조건:

- 무료 원데이터 Export와 안전한 권한 확인, 생성, 전달과 감사 경로 구현
- 서로 다른 본당 2곳 이상에서 동일한 공식 제출 양식 확인
- 독립 예산권자가 같은 가공 결과에 대한 구매 의사를 확인
- `무료는 원시 기록의 기계 판독 가능한 반환, 유료는 검증/계산/오류 진단/제출 양식 가공`이라는 경계를 유지

본당별 맞춤 양식만 발생하거나 무료 데이터 이동과 유료 가공 결과를 구분하지 못하면 후보를 폐기한다.

### 3. 과거 출석 엑셀 이관

**상태: 3순위, 장기 보류. 진단 판매, 실제 이관과 개발은 비승인이다.**

- 사용자 문제 가설: 기존 엑셀의 과거 출석을 다시 입력하지 않고 새 시스템으로 옮긴다.
- 현재 근거: 직접 요청 1건. 학생 명단 Import와 중복 확인은 구현됐지만 과거 출석 Import는 미구현.
- 미래 최소 범위: 하나의 고정 XLSX 구조, 학생/날짜/상태 검증, 미리보기와 오류 파일, 작업 식별자, 출처 추적, 멱등 재실행, Import 데이터만 안전하게 되돌리는 롤백
- 무료 경계: 현재 학생 명단 등록과 중복 확인은 계속 무료
- 유료 경계: 검증된 과거 출석 이관 결과와 한정된 데이터 처리 책임
- 현재 차단 사유: 독립 수요 부족, 미성년자 개인정보, 공통 스키마 미확인, 출처 추적과 롤백 기반 부재

재진입 조건:

- 기존 요청자를 제외한 독립 본당 2곳 이상이 같은 문제를 직접 요청
- 표본 파일에서 공통 열과 변환 규칙이 확인되고 본당별 맞춤 매핑이 불필요
- 합성 데이터로 작업 식별자, 출처 추적, 멱등 재실행과 롤백 검증 완료
- 개인정보 처리 권한, 안전한 전달/보관/삭제, 검수/환불과 오등록 책임 확정
- 노동 조정 공헌이익이 양수인 동일 범위 상품 설계

데이터 준비 진단이 결제돼도 이관 제품 수요로 합산하지 않는다. 위 조건 전에는 실제 파일이나 개인정보를 받지 않는다.

Decision Tree상 지금 개발할 기능은 없다. 1순위도 Gate 0 사용자 결정 전에는 판매, 결제, 실제 데이터 접수와 프로토타입을 시작하지 않는다.

## 12개월 경제성 판단

현재 가격, 적격 조직 수, 전환율, 갱신률, 사용 빈도, 개발시간, 유지보수와 지원량이 모두 미관측이다. 따라서 12개월 매출 예측은 작성하지 않는다.

향후 Gate A 결과가 생겨도 다음 원칙을 지킨다.

- 관측값과 가정을 별도 열로 기록
- 2/5 결과를 연간 전환율로 외삽하지 않음
- 개발/유지보수, 실패 영업, 결제 수수료, 환불, 지원과 재작업 포함
- 기대수익 대신 가정별 손익 시나리오와 손익분기 고객 수를 역산
- 12개월 시나리오는 제품 승인 근거가 아니라 추가 검증 비용의 상한을 결정하는 보조 자료로만 사용

## 인터뷰 항목

### 회의 일정 후보

1. 최근 4주 동안 회의록이나 공지를 몇 주 작성했는가?
2. 실제 문서는 어떤 섹션과 형식을 쓰며 날짜, 시간과 장소는 어떻게 적는가?
3. 회의 공지를 누가 어느 캘린더에 다시 입력하며 한 주에 몇 분이 드는가?

5. 고정 4개 섹션과 건당 2,000자 제한을 받아들일 수 있는가?
6. Google 계정 직접 연동 없이 검수표와 ICS 파일만 받아도 실제로 사용하는가?
7. 결제자는 교사 개인인가, 본당 예산권자인가? 승인 절차와 예산 시기는 언제인가?
8. 4주 사용 뒤 같은 조건으로 다시 결제하게 만드는 결과는 무엇인가?

3. 한 번 작성하는 데 걸리는 시간과 가장 자주 발생하는 오류는 무엇인가?

## 반복 기획 카드

다음 유료 기능 배치도 후보마다 아래 항목을 채운다.

1. 해결할 반복 문제와 직접 발화 원문
2. 독립 요청자 수와 예산권자 수
3. 현재 구현과 미구현 경계
4. Free Core에 남길 부분
5. 구매자가 돈을 내고 받는 고정 결과
6. 수동 실험이 증명할 수 있는 최대 범위
7. 개인정보, 회계, 환불과 책임 조건
8. 정확한 SKU, 시간과 손실 상한
9. 결과 WTP, 프로토타입, 기존 코호트와 신규 코호트 게이트
10. 실패와 재진입 조건

한 배치에서는 2개에서 3개 후보만 깊게 검토한다. 후보 수를 늘리는 것보다 약한 근거를 개발 약속으로 바꾸지 않는 것이 우선이다.

## 누락된 관점

- 직접 WTP 발화자가 아닌 독립 본당 예산권자의 반응
- 주임신부 또는 실제 결재권자의 구매 기준
- 관계 밖 신규 고객의 판매와 셀프 온보딩 비용
- 회의록 형식이 다른 본당에서도 고정 입력이 가능한지
- ICS 가져오기 실패와 모바일 캘린더별 호환성
- 개인정보, 회계와 환불 조건에 대한 전문 검토
- 무료 원데이터 Export의 보안, 감사와 운영 비용
- 실제 공식 연간/학기 보고 양식의 공통성
- 출석 이관 파일의 공통 열과 롤백 요구
- 제한 프로토타입의 소스 기반 시간 견적
