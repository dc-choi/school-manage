# 브레인스토밍 결과 보고서

## 주제

PG 선택 — 정기결제 지원 + 다양한 결제 수단을 갖춘 PG 추천

## 토론 요약

3라운드 토론을 통해 핵심 사실 하나가 드러났다: **국내 PG에서 카카오페이/네이버페이/토스페이 등 간편결제는 정기결제(빌링)를 지원하지 않는다.** 토스페이먼츠 빌링 API는 신용·체크카드와 계좌이체만 지원하며, 간편결제 정기결제는 각 결제사와 별도 계약이 필요하다.

이 사실로 인해 사용자가 요청한 "정기결제 + 다양한 결제 수단" 두 요구사항은 동시에 만족시킬 수 없다는 구조적 제약이 확인됐다. 결과적으로 정기결제는 **토스페이먼츠 단일 PG (카드/계좌이체)**로 가되, 카카오페이는 정기결제에서 제외하고 도네이션 일회성 결제 옵션은 차후 재검토하는 방향으로 합의됐다.

부수적으로 도출된 핵심 결정:

- MVP 결제 주체는 **개인 카드 단일** (인터뷰 결과에 따라 변경 가능)
- 인터뷰 표본을 얼리어답터 3 + 신규 2 + **사제 1**로 다각화하고 **실결제 테스트 질문** 추가
- 손익분기 기준을 비현실적인 "MAO 200 + 40% 전환" 대신 보수 시나리오 "MAO 200 + 15% 전환"으로 하향
- 행정 트랙은 **블로커 체인**(홈택스→토스 가계약→통신판매업→Footer→카드심사→본계약)과 독립 트랙(인터뷰·CTA·sandbox 개발) 분리

## 아이디어 목록

### 아이디어 1: 토스페이먼츠 단일 PG (정기결제: 카드 + 계좌이체)

- **한 줄 요약**: 토스페이먼츠 빌링 API로 신용카드/계좌이체 정기결제만 지원
- **사용자 가치**: 카드 1회 등록 후 자동 청구. 결제 마찰 최소
- **실현 방법**: 빌링키 발급 → 매월 자동 청구 → 결제 실패 시 7일 유예 후 무료 플랜 다운그레이드
- **차별점**: 1인 개발자 기준 연동 난이도 가장 낮음. 토스페이먼츠 공식 문서 품질 국내 최상

### 아이디어 2: 카카오페이 완전 제외 (옵션 A 채택)

- **한 줄 요약**: 카카오페이를 정기결제 트랙에서 완전 배제
- **사용자 가치**: 결제 UX 단순화. "카카오페이 → 카드 재등록" 같은 추가 마찰 회피
- **실현 방법**: 결제창에 카드/계좌이체 2개 탭만 노출
- **차별점**: 1인 개발 인지 부하 절감. 카카오페이 후원 링크 차단 이슈(STATUS.md, 27일 미해결) 해소 전에는 옵션 B(도네이션 일회성)도 보류

### 아이디어 3: MVP 결제 주체 = 개인 카드 단일 (조건부)

- **한 줄 요약**: Account 레벨 구독만 구현, 본당 결제는 2차 릴리즈
- **사용자 가치**: 데이터 모델 충돌 회피. 3개월 MVP 타임라인 준수
- **실현 방법**: 결제 모듈 sandbox에서 개인 카드 단일 트랙으로 개발
- **차별점**: 인터뷰 결과 "본당 결제 선호 과반"이면 계좌이체 인보이스 모델로 즉시 전환

### 아이디어 4: 인터뷰 5+1 + 실결제 테스트

- **한 줄 요약**: 교리교사 5명 + 사제 1명 인터뷰 + 인터뷰 종료 시점 실결제 5,000원 요청
- **사용자 가치**: 의향-행동 갭(20~50%) 보정. WTP 실증
- **실현 방법**: 얼리어답터 3 + 신규 도입 2 + 5/6 사제 미팅 어젠다에 결제 주체 질문 통합
- **차별점**: 단순 의향 조사 대신 "지금 1개월치 결제하시겠어요?"로 실 행동 측정

### 아이디어 5 (기각): 카카오페이 동시 심사 통합 트랙

- **한 줄 요약**: 토스 본계약과 카카오페이 간편결제 심사 동시 진행 → **기각**
- **기각 사유**: 토스페이먼츠 빌링은 카카오페이 정기결제를 지원하지 않음 → 전환율 향상 가설의 전제 자체가 성립 불가

### 아이디어 6 (기각): MAO 200 + 40% 전환 손익분기 모델

- **한 줄 요약**: MAO 200곳 + 베이직 전환율 40% 달성 시 손익분기 → **기각**
- **기각 사유**: SaaS freemium 평균 전환율 5~10%, 40%는 비현실적. 보수 시나리오 15%(MAO 200 기준 월 15만원)로 하향

## 실행 계획 (블로커 체인 + 독립 트랙) — 2026-04-30 수정

### 블로커 체인 (순차, 5~9주)

| 단계 | 액션                                                                                        | 예상 시점                     |
| ---- | ------------------------------------------------------------------------------------------- | ----------------------------- |
| 1    | 홈택스 업종 추가 (525101·525103)                                                            | T+0 (지금)                    |
| 2    | 약관 3종 + 영수증 정책 초안 작성 (이용약관·개인정보처리방침·환불약관)                       | T+0~1주                       |
| 3    | **1차 Footer 배포** (사업자번호·상호·대표자·연락처·약관 3종 링크)                           | T+1주                         |
| 4    | 토스페이먼츠 가계약 신청 (1차 Footer + 약관 노출 상태에서 진행)                             | T+1~2주                       |
| 5    | 에스크로 확인증 수령                                                                        | T+2~3주                       |
| 6    | 정부24 통신판매업 신고                                                                      | T+3~4주                       |
| 7    | **2차 Footer 갱신** (통신판매업 신고번호 추가)                                              | T+4주                         |
| 8    | 영수증/세금계산서 발행 플로우 구현 (sandbox 기준 결제 → 자동 발행 → 다운로드/이메일 재발송) | T+3~5주 (sandbox 개발과 병렬) |
| 9    | 카드사 심사 통과 + 토스페이먼츠 본계약                                                      | T+6~8주                       |

> **Footer 우선 배포 이유**: PG 가맹 심사는 사이트 검수를 포함한다. 사업자 정보·약관 3종이 Footer에 노출되지 않은 상태에서 가계약 신청 시 보완 요청이 발생하여 리드타임이 길어진다 (R21 근거). 통신판매업 신고번호는 신고 완료 후 받으므로 2차 갱신으로 분리한다.

### 독립 트랙 (블로커 체인과 병렬)

- 인터뷰 5+1 진행 (T+0~3주, 5/6 사제 미팅 포함, 영수증 관련 질문 추가)
- CTA 랜딩 구축 (T+2주 시작, 1차 Footer 배포 후)
- 결제 모듈 sandbox 개발 (T+1주 시작, 영수증 발행 플로우 포함, 본계약 후 live 전환)

### 1인 개발자 인지 부하 관리

블로커 체인 진행 중에는 인터뷰 1건 + 결제 모듈 sandbox 2개 트랙만 병렬 운영. CTA 랜딩과 사제 미팅은 1차 Footer 배포 후 착수.

## 누락된 관점 (후속 브레인스토밍 주제)

1. **카카오페이 후원 링크 차단 원인 진단** (27일 미해결, 단기 자금 영향)
2. **결제 실패 자동 재시도 정책 상세 설계** (간격, 횟수, 알림 채널) — 결제 모듈 개발 단계
3. **카드 만료 90일 전 재등록 안내 UX** — Involuntary churn 방지
4. **환불 일할 계산 정책** — 월 중 해지 시 처리
5. **포트원 V2 재검토 트리거** — 월 거래액이 수수료 0.1% 차이가 유의미해지는 구간 진입 시 (pricing.md 명시)
6. **본당 사업자등록 보유 비율 조사** — 세금계산서 발행 대상 사전 파악 (인터뷰 결과로 보완)

## 사용자 결정 대기

다음 3가지를 결정해 주세요:

### 결정 1. 인터뷰 결제 주체 검증 결과에 따른 MVP 모델 변경 권한

인터뷰에서 "본당 결제 선호 4명 이상"이 나오면 MVP 결제 모델을 **개인 카드 단일 → 계좌이체 인보이스**로 즉시 전환할 권한을 위임하시겠습니까?

- (a) 인터뷰 결과 즉시 변경 권한 부여 → 빠른 피벗
- (b) 결과 보고 후 별도 결정 → 의사결정 통제 강화

### 결정 2. 카카오페이 후원 링크 차단 이슈 처리 우선순위

27일 미해결 상태입니다. 옵션 B(카카오페이 도네이션 일회성) 검토를 위해선 원인 확인이 선행돼야 합니다.

- (a) 이번 주 내 원인 확인 (카카오페이 정책 / 링크 만료 / 계정 이슈) 우선 처리
- (b) PG 본계약 이후 별도 처리

### 결정 3. 홈택스 업종 추가 즉시 착수 여부

블로커 체인의 첫 단계입니다. PG 본계약까지 5~9주 리드타임을 시작합니다.

- (a) 지금 즉시 착수 → MAO 50 도달 시점에 PG 본계약 완료 가능
- (b) MAO 50 도달 후 착수 → PG 본계약 + 베이직 출시까지 추가 5~9주 지연
