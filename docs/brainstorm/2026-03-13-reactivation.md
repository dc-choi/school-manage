# 브레인스토밍 결과 보고서

## 주제
재방문율 향상 전략 — WAU 8곳(22%), 이탈 계정 15곳+ 상황에서 기존 가입자의 재활성화 및 리텐션 강화 방안

## 토론 요약

### Round 1
- 기획자 3개 아이디어 제안: (1) 직접 콜 재활성화 — 이탈 계정 15곳+에 전화/카톡, (2) 첫 출석 한 번만 온보딩 — 학생 1명 체험으로 Aha Moment, (3) 전례력 기반 재진입 훅 — 부활절 등 시점에 맞춘 재방문 메시지
- 비판자 핵심 지적: 아이디어 1 실질 타겟은 4~5곳이지 15곳+가 아님. 아이디어 2는 이탈 원인 파악이 선행돼야 함. 아이디어 3은 연 4~6회로 너무 느리고 5월 목표에 기여 미흡
- 합의: 아이디어 1 타겟 축소(15곳→5곳), 카톡 먼저 접촉. 아이디어 2는 아이디어 1 결과 반영 후 설계. 아이디어 3은 "이탈 감지 자동화"로 재정의

### Round 2
- 기획자 개선: 아이디어 1 성남동 제외/고덕동 추가, 이탈 원인 6유형 분류. 아이디어 2 타겟을 "출석 0건 계정 전체"로 재정의, 7일 진행도 제거. 아이디어 3 전례력→행동 기반 자동 알림으로 전면 변경
- 비판자 핵심 지적: 아이디어 3 WAU 8곳 규모에서 자동화 정당화 어려움. 4주차 동시 배포 시 효과 측정 분리 불가. 이메일 오픈율 문제
- 합의: 아이디어 3 현 시점 제외(WAU 20곳+ 시 재검토). 실행 순서 병렬→순차 변경. 주요 논점 전부 합의 도달

### Round 3
- 기획자 최종 개선: 카톡 48시간 무응답→통화 전환 기준 확정. 창동/월곡동 참고 포함. 배너 케이스 스터디 3건 목표. 딥링크(학생 등록 직행) 추가. IDOR/Refresh Token 완료 후 배너 착수. 성공 기준: WAU +1곳(2주 내)
- 비판자 최종 지적: 3주 순차 의존 구조의 플랜 B 필요. 보안 부채와 일정 충돌. 실험 종료 조건 미정. 배너 모수 3~5곳으로 통계 검증 불가
- 최종 합의: 플랜 B 확정(3곳 미달 시 1주 연장). 보안 우선 착수. 케이스 스터디 수준으로 기대치 조정

## 아이디어 목록

### [아이디어 1] D유형 본당 팔로업 인터뷰

- 한 줄 요약: 학생 미등록/이탈 위험 5곳과 1:1 카톡→통화로 이탈 원인 분류, 팔로업 스크립트 표준화
- 사용자 가치: 이탈 원인을 파악해 반복 이탈 방지. 흑석동/길음동에서 이미 검증된 방식
- 실행 방법:
  - 타겟: 정릉4동(G), 흑석동(G), 고덕동(C), 보라동(D), 과천(B). 참고: 창동(H, 입력 부담), 월곡동(H, 온보딩 실패)
  - 카톡 먼저 → 48시간 무응답 시 통화 전환
  - 이탈 원인 6유형 분류: 입력 부담/계정 혼란/바빠서/기능 부족/가치 체감 부족/기타
  - 3곳 이상 반복 패턴을 배너 카피 키워드로 채택
- 플랜 B: 1주차 3곳 미달 시 1주 연장. 2주차에도 미달 시 수집된 패턴 수 기준 진행
- 차별점: 1:1 팔로업은 범용 앱/밴드에서 불가능. 계정별 이탈 감지는 전용 도구만 가능
- 판정: **승인**

### [아이디어 2] D유형 대상 컨텍스트 배너 (케이스 스터디)

- 한 줄 요약: 출석 0건 계정에 컨텍스트 배너 노출, 클릭 시 학생 등록 화면 딥링크로 첫 행동 유도
- 사용자 가치: 퍼널 병목 "학생→출석" 35% 이탈 구간 공략
- 실행 방법:
  - 노출 대상: 가입 14일 이내 + 그룹 생성 완료 + 학생 미등록 + 이탈 확정 아닌 계정
  - 콘텐츠 A가안(몰라서): "학생을 등록하면 출석을 기록할 수 있어요" + 학생 등록 딥링크
  - 콘텐츠 B가안(귀찮아서): "학년이 준비돼 있어요. 학생만 추가하면 바로 시작" + 학생 등록 딥링크
  - GA4 행동 데이터(학생 등록 페이지 방문 여부)로 A/B 유형 판단
  - 측정: 배너 노출 후 7일 내 첫 출석 기록 생성 여부 (케이스 스터디 3건)
- 착수 조건: IDOR + Refresh Token 완료 후
- 차별점: 주일학교 맥락 온보딩은 엑셀/구글 시트/밴드에 없음
- 판정: **승인** (보안 부채 완료 후 착수)

### [아이디어 3] 이탈 감지 자동 알림

- 한 줄 요약: 마지막 출석 후 7일/14일 경과 시 자동 알림 발송 + 개발자 이탈 위험 감지
- 사용자 가치: 이탈 전 자동 개입으로 Silent Churn 방지
- 현 시점 판정: **조건부 — WAU 20곳+ 도달 시 재검토**
- 제외 사유: WAU 8곳 규모에서 수동 모니터링(주 1회 DB 쿼리 + STATUS.md 갱신)이 충분

## Decision Tree 결과

```
Q1. 사용자 가치가 명확한가? (파일럿 데이터/피드백 근거)
 ├─ NO → 기각
 └─ YES ↓

Q2. 1인 개발로 3개월 내 MVP 가능한가?
 ├─ NO → 스코프 축소 후 재평가
 └─ YES ↓

Q3. 가톨릭 전용 도구로서 차별점이 있는가?
 ├─ NO → 피벗 필요
 └─ YES ↓

Q4. 수익 모델/성장에 기여하는가?
 ├─ NO → 조건부 (성장 기여 보완 필요)
 └─ YES → 승인
```

| 아이디어 | Q1 사용자 가치 | Q2 3개월 MVP | Q3 차별점 | Q4 성장 기여 | 판정 |
|---------|--------------|-------------|----------|------------|------|
| 팔로업 인터뷰 | YES (흑석동/길음동 통화 검증) | YES (개발 불필요) | YES (계정별 이탈 감지) | YES (WAU 직접 기여) | **승인** |
| 컨텍스트 배너 | YES (퍼널 병목 35% 이탈) | YES (배너 1주 구현) | YES (주일학교 맥락 온보딩) | YES (전환율 개선) | **승인** |
| 이탈 감지 자동 알림 | YES | YES | YES | 조건부 (현 규모 불필요) | **조건부** |

## 실행 계획

### 3주 순차 실행

| 주차 | 집중 작업 | 산출물 |
|------|---------|-------|
| 1주차 | 팔로업 인터뷰 5곳 실행 | 이탈 원인 6유형 분류 결과, 반복 패턴 Top 3 |
| 2주차 | 인터뷰 결과 분석 + 배너 콘텐츠/필터링 확정 | 배너 A/B 가안 중 확정, 노출 조건 확정 |
| 3주차 | 배너 구현 + GA4 UTM 추적 + 배포 | 배너 노출 시작, WAU 변화 추적 |

### 선행 조건
- IDOR + Refresh Token 완료 → 배너 착수 (보안 부채 우선)

### 성공 기준

| 기준 | 수치 |
|------|------|
| 인터뷰 완료 | 3곳 이상 (2주 내) |
| 이탈 원인 반복 패턴 | 1개 이상 도출 |
| 배너 케이스 스터디 | 3건 |
| WAU 변화 | 배너 노출 후 2주 내 +1곳 이상 |

## 사용자 판단 필요 항목

1. **IDOR/Refresh Token 완료 시점**: 현재 "미작성" 상태. 일정 확정 필요
2. **인터뷰 대상 우선순위**: 5곳(정릉4동/흑석동/고덕동/보라동/과천) 중 연락처/관계 기반으로 1주차 우선 3곳 선정
3. **전체 실행 승인 여부**: 위 계획대로 진행할지, 수정할 부분이 있는지
