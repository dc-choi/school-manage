## 배경/문제 요약

- 참고: `docs/business/0_feedback/feedback-categories.md` 출석 섹션, `docs/specs/functional-design/attendance-management.md`
- 문제:
    - 출석 화면은 "주간 리듬 안정화"라는 핵심 가치의 진입점이지만, 다수 파일럿(장위동·흑석동·길음동·성북동)에서 정보 위계와 입력 동선에 누적 피드백이 발생했다.
    - 정렬 옵션이 없어 학생 수 30+ 단체에서 특정 학생 탐색에 시간이 든다.
    - 자동 저장이 셀 단위 즉시 전송 구조라 빠른 연속 입력 시 요청 폭주 가능성이 있고, 현재 latency 계측 데이터가 없다.
- 현재 상태: 테이블/달력/미사·교리 분리 표시 구현 완료 (단일 정렬, 셀 단위 즉시 저장). 모바일 톤은 #295에서 처리됨.
- 목표 상태: (1) 정렬 옵션으로 학생 탐색 시간 단축, (2) 시각/구조 리뉴얼로 정보 위계 명확화, (3) 자동 저장 성능 계측 및 보강.

## 목표/성공 기준

- **목표**: 출석 화면 진입~첫 입력까지의 동선 단축 + 자동 저장 latency P95 안정화
- **성공 지표**:
    - 정렬 사용율: 출석 페이지 진입 세션 중 정렬 클릭 발생 비율 (베이스라인 → 개편 후 비교)
    - 자동 저장 P95 latency: 계측 도입 후 베이스라인 확보 → 보강 후 비교
    - 정성 피드백: MAO 상위 5곳 응답 (긍정/부정/혼합)
- **측정 기간**: 출시 후 4주

## 사용자/대상

- **주요 사용자**: 교리교사 (출석 입력 담당)
- **사용 맥락**: 매주 토/일 미사·교리 직후 모바일/데스크톱에서 입력. 학생 30+ 단체에서 특정 학생 탐색 빈도 높음.

## 범위

### 포함

- **정렬 옵션 2종**: 등록 순(기본) / 가나다 (출석 **모달** 학생 목록 헤더 — 모바일 메인 동선이 달력→모달이라 모달에 적용. 테이블 뷰 미적용. 우수 출석은 단일 날짜 모달 의미 약해 제외)
- **시각/구조 리뉴얼**: 정보 위계 재설계, 디자인 시스템 일관성, 와이어프레임 재합의 (FD 단계)
- **자동 저장 성능 보강**:
    - latency 계측 도입 (요청 시작~응답)
    - 빠른 연속 입력 시 폭주 방지 (디바운스/배치/큐 중 1택, FD에서 결정)
    - 실패/재시도 UX 명확화

### 제외

- 학생 셀프 체크 (하드웨어 의존, NFC 보류와 정합 — 별도 SDD)
- 행사 출석 구분 (후속 검토)
- 모바일 톤/폰트 (#295 처리 완료)
- 데일리 통계 대시보드 (별도 P2 TARGET)

## 사용자 시나리오

1. **시나리오 1**: 교리교사가 토요일 미사 후 출석 화면 진입 → 가나다 정렬 → "김민준" 학생 즉시 탐색 → ◎ 입력 → 자동 저장 표시 확인 후 다음 셀
2. **시나리오 2**: 30명 단체에서 5분 내 30셀 연속 입력 → 자동 저장이 부하 없이 전부 성공, 요청 큐 폭주 없음
3. **시나리오 3**: 네트워크 일시 끊김 → 실패 셀 가시화 + 재시도 버튼으로 복구

## 제약/가정/리스크/의존성

- **제약**:
    - `(student_id, date)` UNIQUE + atomic upsert 동작 유지
    - 디바운스/배치 적용 시 마지막 입력 손실 0건 보장
- **가정**: 학생 30~150명 단체가 주 타겟. 5분 내 30셀 입력 시 P95 < 1s 달성 가능.
- **리스크**:
    - 시각 리뉴얼 학습 부담 → 4년 사용자(장위동 등)의 거부 반응 가능. 1주 베타 또는 단계 출시 검토.
    - 디바운스 적용 시 마지막 입력 손실 우려 → 단위·통합 테스트 강제.
- **내부 의존성**: `attendance.update` 자동 저장 로직, `group.attendance` 조회, 디자인 시스템 토큰
- **외부 의존성**: 없음

## 롤아웃/검증

- **출시 단계**: 메인 머지 → 전체 노출. 자동 저장 동작 변경이 있을 경우 1주 베타 채널 검토.
- **이벤트**:
    - `attendance_sort_clicked` (sort_type)
    - `attendance_save_latency_ms` (P50/P95 집계)
    - `attendance_save_failed` (error_code)
- **검증**: GA4 + 서버 로그 + MAO 상위 5곳 정성 피드백
