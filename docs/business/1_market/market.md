# 시장

> 문제 배경/핵심 문제는 `docs/business/README.md`를 참고하세요.

## 비전

- **가톨릭 교회 내 모든 모임을 위한 그룹 운영 플랫폼** (출석/멤버/행사/통계 일관 관리) — **내부 장기 나침반. 외부 노출 금지** (2026-06-12 전략: 현 단계 외부 포지셔닝은 "가톨릭 교리교사의 주일을 운영하는 도구", 격상은 회의록 출시 트리거)
- 1차 검증: 주일학교 중고등부 교리교사. 전 그룹 확장은 회의록 출시 이후 재검토

## 타겟 사용자

> 페르소나 매트릭스(단체 타입 × 운영/의사결정 페르소나) + 잠재 고객 우선순위: `personas.md`

### 가설

- **1차 사용자 (현재 검증)**: 서울대교구를 시작으로 전국 가톨릭 중고등부 교리교사. 비서울 도입 사례도 관측됐지만 최신 지역별 MAO는 재집계 전까지 복제하지 않는다
- **2차 사용자 (회의록 출시 이후 재검토)**: 초등부 교리교사, 주일학교 외 모임 운영자(레지오, 성가대, 청년회 등) — 1차 교리교사 PMF 검증과 회의록 출시가 진입 판단 조건 (2026-06-12 전략: 병기 금지, 우선순위 분리)
- **현재 유료 검증 세그먼트**: 2026-09-01부터 2026-11-29까지 실제 본당 예산 결정자 5명을 검증 모수로 사용하고, 최소 2명은 기존 개인 관계 밖에서 선정한다. 사용자 단체 타입보다 예산 권한과 운영 부담을 우선 선발 기준으로 둔다
- **장기 SaaS 세그먼트 예외, 현재 보류**: 3단계 베이직 알림/공지는 초등부 P1 적합도가 높다는 과거 가설이다. 2026-11-30 재승인 전에는 초등부 Basic 결제 코호트를 운영하지 않는다 (`personas.md`, `7_pricing/pricing.md`)
- **3차 사용자**: 전국 교구 가톨릭 모임 운영자. 목포해군성당은 군종 도입/사용 사례 1건이며 보안 승인 여부는 측정되지 않았다
- 협력 채널: 지구/교구 청소년국 사목부서 → 이후 본당 사목위원회 확대
- 최종 의사결정자: 본당 차원 = 본당 사제(주임신부), 교구 차원(양업시스템/교구 예산) = 주교 (2026-05-09 위계 분리)
- **도입 게이트키퍼 (2026-05-07 추가)**: 교구 청소년국 직원(간사) — 협력 채널이자 도입 게이트키퍼의 양면적 역할 (사제/교리교사 추천 매개 vs 보안 인식 차단 가능). 보안 인식 우려가 도입 차단 시그널이 될 수 있음 (R25). 2026-05-02 서울대교구 청소년국 중고등부 간사 피드백으로 첫 시그널 확보. **단 청소년국은 협력 채널일 뿐 의사결정 권한 없음 — 교구 차원 도입은 주교 접점 확보가 게이트** (2026-05-09)

### 근거

- 청소년국이 본당/학교 지원 역할을 명시하고, 교리교사 접점(자료실/공지)이 존재
- 주일학교 운영의 핵심 실행은 교리교사이며, 최종 결정은 사제가 한다는 사용자 합의

### 검증

- 목적: 사용자/의사결정 구조 확정 — 도입 요청/승인/결제 기록과 역할/권한 로그로 추정
- 완료 기준: 도입 주체/승인 경로 유형 정리

## 사용자 특성 (2030세대)

> 파일럿 본당 교리교사 연령대: 20-30대

### 핵심 특성

- **디지털 네이티브**: 새 도구 시도 장벽이 낮으나, 직관적이지 않으면 즉시 삭제
- **첫인상 결판 + Silent Churn**: 첫 경험에서 가치를 못 느끼면 재시도 없이 조용히 이탈
- **커스터마이징 선호**: 자기 방식대로 정보를 구성 (예: 비고란 활용)
- **SNS 유입에 익숙**: 인스타그램 → 랜딩 → 가입 흐름이 자연스러운 세대
- **요노(YONO) 소비**: 유료 전환 시 확실한 가치 증명 필요. "가톨릭 전용" 포지셔닝이 가치 공감 유발 가능

### 행동-보상 원칙

> 2030세대는 "행동 → 확실한 보상" 연결이 빠를수록 움직인다. 온보딩, 기능 설계 전반에 적용 (상세 사례: `docs/business/3_gtm/gtm.md`)

### 이탈자 복귀 가능성

- **A (구조 혼란)** [해소]: 계정 모델 전환 완료로 해소 (2026-03)
- **B (대기)**: 바빠서 못 씀 → 시간 여유 시 자연 복귀 (높음)
- **C (가치 미체감)**: 가치를 못 느낌 → 1:1 통화 후 판단 (낮음)
- 전용 대체재가 뚜렷하지 않고 매주 반복 니즈가 있어 복귀 창이 넓을 수 있다는 가설이며, 실제 복귀율로 검증한다

### 사용자 파악 방법

- **행동 데이터 (GA4)**: 이탈 지점 이벤트 추적 구현. 최신 이벤트 목록은 `apps/web/src/lib/analytics.ts`를 따른다
- **이탈 감지 알림**: 미구현. 기존 이탈 감지 cron과 이력 테이블은 2026-05-07 제거됐다. 재구축 시 R24-A0/A1과 입력 지연을 분리하고, 일수만으로 영구 이탈을 판정하지 않는다
- **구조적 관찰**: 동일 단체 코호트의 생성부터 첫 출석까지 분석. 현재 계측은 사용자/단체 이벤트가 섞여 있어 유효 전환율은 미측정이다

### 파일럿 근거

- 고덕동: 그룹 1개 생성 후 장기 미사용 관찰. 첫 경험이 원인인지는 미검증
- 신당동/금호동: 계정 모델 혼란으로 이탈 → **직관성 민감** 확인
- 가재울/창동/길음: 가입 후 바로 출석 입력, 정착 → **첫인상 성공** 확인
- 성남동: 비고란 세밀 활용 → **커스터마이징 욕구** 확인
- 월곡동/창5동: 인스타 → 랜딩 → 가입 → **SNS 유입** 확인

### 제품 시사점

| 특성           | 시사점                                  | 현재 대응                                   |
| -------------- | --------------------------------------- | ------------------------------------------- |
| 첫인상 결판    | 첫 경험에서 막히면 장기 미사용 위험      | 계정 모델 전환 구현 완료 (개인 계정 + 합류) |
| 직관적 UX 필수 | "본당 찾아 합류" 흐름이 자연스러워야 함 | 구현 완료. 조직 입력 UX 개선까지 완료       |
| SNS 유입       | 인스타 → 랜딩 가입 사례 관측            | 확인 가입 2건, 동일 귀속 퍼널 전환율은 미측정 |

### 역할 분리 관찰 (2026-04-19 누적 시작)

- 1계정 운영 본당(성북동 미카엘라)에서 "서기는 주로 조회, 입력자는 교사" 언급 — 계정 모델 전환 후에도 1계정 공유 운영이 잔존
- 시사점: 조회 중심 UX와 입력 중심 UX가 분리될 수 있음. 팀 전환 유인(추가 계정 부여)이 약한 조직은 단일 계정 유지 → 팀 전환 GTM 재고 필요
- 검증: 팀 사용 본당 vs 1계정 본당의 조회/입력 행동 비교, 역할 분리 언급 누적 관찰

## 사용자 니즈 신호

### 가설

- 핵심 니즈는 "출석 기록"보다 **"매주 필요한 정보(출석 확인 + 축일 + 멤버 현황)를 한곳에서 빠르게 조회"**하는 것
- 출석 조회 + 가톨릭 정보가 통합된 도구는 엑셀이 대체할 수 없다

### 근거

- 장위동 4년 사용 관찰: "누가 나왔는지 빠른 확인"이 핵심 가치, 축일 검색에도 활용
- 피드백에서 통계/현황, 출석 UX, 대량 처리, 축일/등록/동의 관리 요구 반복(상세: `docs/business/0_feedback/feedback.md`)

### 검증

- 목적: 핵심 니즈 상위 항목 도출 — 기능 사용률/처리 시간 기준 상위 5개 항목 확정

## 시장 규모와 수익성 연결

- **검증 거점(SOM 후보)**: 2026-02-13 조사 기준 서울대교구 233개 본당 × 중고등부 → 최대 233개 단체. 현재 수치는 시장 계산 전 재확인하며, 233개 전체를 매출 모수로 간주하지 않는다
- **전국 1차 시장(SAM)**: 전국 가톨릭 중고등부가 타겟이나, 전국 본당 수와 중고등부 운영률은 본 문서에서 미검증이다. 검증 전 재무 시나리오는 서울대교구 233개만 보수적 기준으로 사용한다
- **본당 내 기타 모임 (2차 확장)**: 레지오, 성가대, 청년회 등 본당당 5-10개 추정. 1차 PMF 전에는 매출/BEP 계산에서 제외한다. 18개 지구와 청소년국은 초기 협력/확산 채널이다

## 경쟁 환경

> 상세: `docs/business/1_market/competitive-analysis.md` (2026-02-13 조사, 03-16 브랜치스 추가)

### 핵심 발견

- **2026-02-13 조사 범위에서 가톨릭 전용 주간 운영 도구를 발견하지 못함** (출석 + 축일 + 멤버 통합)

- 가톨릭 하상 = 신자 앱, 양업시스템 = 사무실 도구 (교리교사 접근 불가)

### 경쟁 구도

| 카테고리               | 대표 서비스                   | 우리와의 관계                                   |
| ---------------------- | ----------------------------- | ----------------------------------------------- |
| 개신교 ChMS            | 오직교적, 만나ERP 등          | 용어/구조 불일치, 기능 과잉                     |
| **개신교 직접 경쟁자** | **브랜치스 (95교회, 사업화)** | **문제 인식 유사. 가톨릭 도메인 장벽으로 방어** |
| 가톨릭 공식 앱         | 하상, 양업시스템              | 운영 도구가 아닌 신자 앱/사무실 도구            |
| 범용 출석 앱           | Attendable, Wasso             | 교회 특화 기능 없음                             |
| 수기/오피스            | 엑셀, 카카오톡+수기           | **현재 최대 대체재** — 분산/유실                |
| 해외 ChMS              | Planning Center 등            | 영어 전용, 한국 가톨릭 구조 불일치              |

### 가설

- 대체재는 "엑셀/한글 + 카카오톡 + 수기"라는 가설이며, 전용 SaaS 침투율은 현재 미측정이다
- 개신교 ChMS는 용어/구조가 달라 전환 대상이 아님

### 근거

- 경쟁사 조사(2026-02-13)에서 가톨릭 전용 출석 관리 도구 0개 확인
- 현장 조사에서 수기 → 엑셀 → 카카오톡 공유 워크플로우가 가장 흔함

### 검증

- 목적: 대체재 사용 패턴과 전환 비용 파악
- 방법: 본당 10곳 이상 현행 툴/워크플로우 조사
- 완료 기준: 대체재 유형 3개 이상 분류 — **검증 완료 (2026-05-05): 5종 분류** (엑셀/카톡, 개신교 ChMS, 가톨릭 공식 앱, 범용 출석 앱, 해외 ChMS). 상세: `competitive-analysis.md`

## 차별화

### 가설

- **"주일 아침 미사 직전에도, 이거 하나면 됩니다"** (2026-06-12 브랜드 약속)가 핵심 차별점 — 출석 조회 + 가톨릭 정보(축일/전례력) + 멤버 현황 통합

### 차별점

### 근거

- 장위동 4년 사용 관찰: "빠른 조회 + 축일 검색"이 실제 사용 이유
- 경쟁사 분석에서 가톨릭 전용 도구 0개 확인
- 엑셀은 축일/전례력/멤버 현황을 한곳에서 제공하지 못함
