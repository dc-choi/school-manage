## 배경/목표

### 배경

- **장기 비전**: 가톨릭 교회 내 모든 모임을 위한 그룹 운영 플랫폼
- **현재 문제**: "학생"이라는 용어가 주일학교 전용 느낌을 줌
- **결정**: PMF 검증은 중고등부로 유지하되, 용어는 범용화

### 목표

- "학생" → "멤버"로 용어 변경
- 다양한 모임(성가대, 레지오, 청년회 등)에서도 자연스럽게 사용 가능하도록 준비

### 범위

| 포함 | 제외 |
|------|------|
| "학생" → "멤버" 용어 변경 | 브랜딩(서비스명) 변경 - 추후 별도 진행 |
| UI 텍스트만 변경 | DB 스키마/API 변경 없음 |

## 구현 방식

### 방식 A: 직접 문자열 변경 (선택)

각 파일에서 하드코딩된 문자열을 직접 변경합니다.

**장점**: 단순, 빠른 구현
**단점**: 추후 다국어 지원 시 재작업 필요

### 방식 B: i18n 도입 (미선택)

다국어 라이브러리(react-i18next 등)를 도입하여 문자열을 외부화합니다.

**장점**: 다국어 지원 준비, 문자열 일괄 관리
**단점**: 추가 작업량, 현재 필요하지 않음

> **결정**: 현재는 방식 A로 진행. 다국어 지원이 필요해지면 그때 i18n 도입.

7. **TC-7**: 삭제/복구 확인 메시지에 "멤버"가 표시된다
