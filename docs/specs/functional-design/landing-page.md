# 기능 설계: 랜딩 페이지

> 인스타그램·사제 추천 유입 사용자가 서비스를 이해하고 가입을 결심하도록 설득하는 `/landing` 퍼블릭 페이지.

## 연결 문서

- PRD: `docs/specs/prd/school-attendance.md`, `docs/specs/prd/landing-faq.md`
- 전환 퍼널 분석: `docs/business/STATUS.md`
- 포지셔닝/가치 제안: `docs/business/3_gtm/gtm.md`, `docs/business/2_bm/bm.md`

## 기능 범위

| -------------------------------------- | ----------------------------------------------------------------------------------------------------- | --------- |
| 기본 랜딩 페이지                       | 풀스크린 스크롤 + 인터랙티브 데모                                                                     | 구현 완료 |

## 페이지 구조 (11요소)

표준 앱 랜딩 11요소에 정렬된 sticky Nav + 8섹션 본문 + Footer. FadeIn 애니메이션(IntersectionObserver).

| #   | 섹션 ID         | 내용                                                                                             |
| --- | --------------- | ------------------------------------------------------------------------------------------------ |
| ①   | (Nav)           | sticky 헤더 — 로고 + 앵커(기능/후기/FAQ) + 우상단 `시작하기` CTA. 모바일은 앵커 숨김             |
| ②   | `hero`          | 헤드라인 + 서브카피 + `시작하기` primary CTA + `데모 체험` 보조. 데스크톱 lg에서 좌우 2단        |
| ③   | (Hero 우측)     | `HeroPreview` 인라인 SVG (출석 화면 폰 목업). 모바일 숨김                                        |
| ④   | `social-proof`  | countData 1줄(`{본당}/{선생님}/{학생}` 누적) + 모임 유형 5카드(주일학교/청년/레지오/군종/성가대) |
| ⑤   | `features`      | 핵심 기능 4카드 (출석 체크 / 구성원과 축일 관리 / 통계 자동 / 달력)                              |
| ⑥   | `why-choose-us` | 차별화 4카드 — 주간 도구로 설계 / 가톨릭에 특화 / 종이와 엑셀 대신 / 모바일에서 즉시             |
| ⑦   | `demo`          | InteractiveDemo (달력 + 출석 체크 체험, 로컬 데이터)                                             |
| ⑧   | `reviews`       | 사용 사례 4카드 (안 B — 익명 시나리오 톤, 인용부호·별점 없음)                                    |
| ⑨   | `faq`           | "자주 묻는 질문" 아코디언 (단일 열기) + JSON-LD                                                  |
| ⑩   | `cta`           | 마무리 카피 + `지금 시작해보기` 큰 CTA + 보조 로그인 링크                                        |
| ⑪   | (Footer)        | `~/components/layout/Footer` 재사용 — 사업자 정보 + 약관 3종 + 인스타 + 문의 메일                |

## Hero 콘텐츠

| 요소              | 값                                                                               |
| ----------------- | -------------------------------------------------------------------------------- |
| h1                | "매주 일요일, 이거 하나면 됩니다"                                                |
| 서브카피          | "**출석, 축일, 구성원 현황**까지. / 가톨릭 모임을 위한 주간 도구."               |
| Primary CTA       | `시작하기` → `/signup` + `landing_cta_click({position:'hero'})`                  |
| 보조 CTA          | `데모 체험` → `#demo` 앵커 스크롤                                                |
| 우측 시각요소     | `HeroPreview` 인라인 SVG (Tailwind primary 토큰 호환). 데스크톱(lg ≥1024px) 한정 |
| 풀스크린          | `min-h-[calc(100dvh-3.5rem)]` (Nav 56px 보정 + iOS Safari `100vh` 이슈 회피)     |
| AuthLayout 동기화 | 데스크톱/모바일 h1·서브 = 랜딩 Hero와 동일. 모바일 h1에 `text-balance` 적용      |

## Reviews 섹션 — 사용 사례 카드 (안 B, 2026-05-09)

가짜 별점/인용 노출 금지. `feedback-categories.md` 기반 익명 시나리오. 향후 사용자 동의 확보 시 안 A(실제 후기)로 점진 전환 가능.

| 카드             | 시나리오 카피                                                   |
| ---------------- | --------------------------------------------------------------- |
| 주일학교 운영    | 매주 학생 70명 출석을 폰 한 대로. 종이 출석부 정리는 끝         |
| 다중 교사 협업   | 교리교사 여럿이 같은 모임을 공유. 누가 무엇을 입력했는지 한곳에 |
| 통계 자동 정리   | 출석률과 마지막 출석일을 따로 계산할 필요 없어요                |
| 모바일 즉시 사용 | 앱 설치 없이 폰에서 탭 한 번. 새 교사도 바로 시작               |

- 데스크톱 4컬럼 / 모바일 가로 스크롤(`snap-x snap-mandatory`)
- 부모 section에 `overflow-x-clip` 방어 적용

## Target Groups (Social Proof로 통합, 2026-05-09)

비로그인 청년/레지오/군종/성가대 운영자 후킹 5카드. countData 1줄과 함께 ④ Social Proof 섹션으로 통합 재배치 (이전 구조의 ③.5 단독 섹션 제거).

| 카드     | 아이콘 (lucide-react) | 1줄 카피                        |
| -------- | --------------------- | ------------------------------- |
| 주일학교 | BookOpen              | 매주 학생 출석을 한 곳에서      |
| 청년     | Sparkles              | 청년 모임 참석 기록을 간편하게  |
| 레지오   | Heart                 | 주회 출석과 활동을 빠르게       |
| 군종     | Shield                | 장병들의 미사 참여를 한눈에     |
| 성가대   | Music                 | 연습 출석을 카톡 대신 한 곳에서 |

- 반응형: 모바일 2열, 데스크톱 5열
- countData(`account.count`) 0/실패 시 통계 줄만 숨김. 5카드는 항상 표시

## FAQ 콘텐츠

7항목 아코디언. JSON-LD `FAQPage` 스키마 출력으로 SEO 강화.

| #   | 질문                   | 답변 요약                                                       |
| --- | ---------------------- | --------------------------------------------------------------- |
| 1   | 비용이 드나요?         | 모든 기능 비용 없이, 핵심 기능 유지                             |
| 2   | 개인정보는 안전한가요? | 최소 수집, 제3자 미제공                                         |
| 3   | 선생님 여러 명이면?    | 각자 가입 → 같은 모임에 합류 + 관리자 승인                      |
| 4   | 가입하면 뭘 먼저?      | "학년/그룹 → 학생/멤버 → 출석 체크" (라벨 분기 안내)            |
| 5   | 스마트폰에서도?        | 웹 기반, 앱 설치 불필요                                         |
| 6   | 가톨릭 주일학교만?     | 주일학교, 청년, 레지오, 군종, 성가대 등 가톨릭 모임 운영에 사용 |
| 7   | 기존 데이터 이전?      | 직접 입력, 대량 등록 준비 중                                    |

- UI: shadcn/ui Accordion (단일 열기, FadeInSection)

## 예외/엣지 케이스
