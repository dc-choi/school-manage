# 브레인스토밍 결과 보고서

## 주제

가톨릭 소식 추적 도구 — 교리교사가 교구 공지, 교황 소식, 전례 변경, 교육 자료를
체계적으로 받아보지 못하는 문제를 해결하는 방법

## 주제 피벗 선언

3라운드 토론 결과, 원래 주제인 "가톨릭 소식 추적"은 **기각**됐다.

- 외부 크롤링: 저작권 리스크 + 1인 개발 유지보수 과중
- 사용자 요청: 파일럿 24곳 피드백에 소식/자료 요청 없음
- 결론: 핵심 페인포인트는 "소식 수집"이 아니라 **"주간 운영 행동 유도"**

피벗 방향: **보유 데이터(전례력 DB + 세례명 축일 DB + 출석 데이터) 기반 운영 컨텍스트 제공**

"가톨릭 소식 추적"은 3단계 이후 교구 파트너십 확보 시 재접근 검토.

## 토론 요약

### Round 1
- 원안 3개 제시: (1) 앱 내 뉴스 탭, (2) 위클리 이메일, (3) 자료 피드
- 비판: 3개 모두 파일럿 요청 없음, 크롤링 과중, 2단계 미착수 7개가 우선
- 합의: 아이디어 3 기각, 외부 크롤링 제거, 앱 내 데이터만 사용

### Round 2
- 수정안: (1) 주간 컨텍스트 카드 (앱 내 데이터), (2) 이탈 감지 알림 통합 이메일
- 비판: 활성/비활성 동일 이메일 불일치, 스케줄러 복잡도, 법적 리스크
- 합의: 이탈 경고와 주간 요약 분리, 이탈 경고만 단독 구현

### Round 3
- 최종 수정: (1) 컨텍스트 카드 v2, (2) 이탈 경고 이메일 v2
- 비판: 방학 예외 처리 누락, GA4 판단 기준 부재, 수신 거부 백엔드 필요
- 합의: 방학 토글 공유, GA4 기준값 사전 정의, 우선순위 확정

## 아이디어 목록

### 주간 컨텍스트 카드 v2

- **한 줄 요약**: 홈 화면 접이식 카드로 전례 시기 + 세례명 축일자 + 미출석 학생을 표시
- **사용자 가치**: 수업 전 여러 앱/문서를 열어 맥락을 조합하는 반복 행정 제거
- **실현 방법**: 기존 전례력 DB + 세례명 축일 DB + 출석 데이터. 외부 의존 없음
- **차별점**: 전례력 + 세례명 + 출석 데이터 결합은 브랜치스 포함 경쟁사 불가
- **선행 조건**: 컨텍스트 배너 완료 후 착수 (배너: 미기록 상태 / 카드: 출석 시작 후)
- **측정 기준**: 카드 클릭률 30% (4주), 미달 시 카피 교체 후 2라운드

### 이탈 경고 이메일 v2

- **한 줄 요약**: 7일/14일 미활동 모임에만 자동 이메일 발송, Silent Churn 방지
- **사용자 가치**: 비활성 패턴을 인식하지 못하고 조용히 이탈하는 문제 해결
- **실현 방법**: GitHub Actions schedule + 기존 SMTP + 수신 거부 테이블
- **차별점**: 전례력 기반 방학 판단 + 가톨릭 모임 맥락의 이탈 경보
- **선행 조건**: (1) 법률 자문 완료, (2) 수신 거부 백엔드, (3) 배너 4주 데이터
- **측정 기준**: 오픈율 20% / 14일 내 재활동 전환율 15%, 미달 시 재검토

### 기각된 아이디어

| 아이디어 | 사유 |
|----------|------|
| 앱 내 뉴스 탭 (원안 1) | 외부 크롤링 저작권 + 유지보수 + 카톡 대비 속도 열위 |
| 위클리 이메일 (원안 2) | 이탈 경고에 흡수. 주간 요약은 데이터 확보 후 별도 결정 |
