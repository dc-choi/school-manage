## 포지셔닝

> 핵심 근거: 가톨릭 교리교사/모임 운영자가 매주 여는 주간 운영 도구가 시장에 없음 (상세: `docs/business/1_market/competitive-analysis.md`)

### 범용 메시지
"매주 일요일, 이거 하나 열면 됩니다. 출석·축일·멤버가 한곳에."

### 교리교사 메시지 (1차 검증)
"누가 나왔는지, 이번 주 축일이 뭔지, 달력에서 바로 확인하세요."

### 사제/책임자 메시지
"통계/보고 자동화로 의사결정 리드타임 단축"

### 경쟁 차별 메시지 (내부용)
- 엑셀은 출석만, 축일은 따로 검색 → **한곳에 다 있는 도구가 없었음**
- 가톨릭 하상 ≠ 운영 도구 (신자 개인 앱)
- 개신교 ChMS ≠ 가톨릭 (용어/구조 불일치, 기능 과잉)

## 전환 경로

### 인스타그램 → 가입
```
인스타그램 게시물/하이라이트 → 프로필 링크 → /landing → /signup 또는 /login → / (대시보드)
```

- **A + B 병행**: 인스타 자체를 랜딩으로 활용(프로필+하이라이트) + 로그인 페이지 서비스 소개 (완료, 2026-02-11)
- 랜딩 페이지 `/landing` 구현 완료 (2026-02-12)

### 교사 네트워크 → 가입
```
기존 사용자 → 카카오톡으로 랜딩 링크 공유 → /landing → /signup → 회원가입
```
- 별도 에셋 불필요 — 랜딩 페이지가 설명/설득을 담당

### 계정 모델 (2026-02-20 전환 결정)
- **전환: 개인 계정 + 본당/모임 합류 구조**
  - 각자 가입 → 본당/모임을 찾아 합류하거나 새로 생성
  - 기존 사용자가 같은 본당 교리교사를 초대 → 바이럴 루프
- 전환 이유: 공유 계정 모델이 사용자 행동과 불일치
- SDD 작업 등록됨

## 세일즈 모션

```
모임 운영자 직접 접촉 → 개인 가입·모임 생성 → 같은 본당 교리교사 초대 (바이럴)
    → 2~4주 파일럿 → 책임자 공유/동의 → 본당 단위 유료 전환
```

## 온보딩

- 목표: 첫 주 안에 "멤버 등록 + 1회 출석 기록" 완료
- **2030세대 원칙**: 각 단계에서 "행동 → 즉각적 결과"가 보여야 함. 첫 경험에서 막히면 영구 이탈 (상세: `docs/business/1_market/market.md`)

### 온보딩 퍼널 전환율 (2026-03-01, N=25)

```
가입 100% → 그룹 생성 76% → 학생 등록 60% → 첫 출석 40%
         -24%             -16%             -20%
```

- **최대 이탈 구간**: 가입 → 그룹 생성 (-24%), 학생 등록 → 첫 출석 (-20%)
- 반복 사용(출석 5건+)까지 도달: 28%, 안착(완전): 12%

### 온보딩 마찰 지점 (2026-03-01)

| 마찰 유형 | 해당 계정 | 추정 원인 |
|----------|---------|---------|
| 그룹 → 학생 중단 | 과천(김민건), 양재2동 | 학생 등록 UX 마찰 또는 시간 부족 |
| 학생 → 출석 중단 | 성남동 | 그룹별 출석 구조가 워크플로우와 불일치 (피드백 이벤트에서 확인) |
| 가입 → 방치 | 한강(김지민), 박민규 외 | 탐색 목적 가입 또는 관심만 |

> **즉시 활성화 패턴**: 창5동성당은 가입 당일 학생 39명 등록 + 출석 26건 완료. "기존에 엑셀/수기로 출석을 관리하던 교리교사"가 가입하면 당일 안착하는 패턴 (S유형)

### 진입점: 회원가입
- 오픈 가입 방식 (2026-01-25 구현 완료) — 누구나 즉시 가입 가능, 스팸 발생 시 승인제 전환 검토
