# 가격 정책

## 확정 실험 가격, 2026-09-01부터 2026-11-29까지

이번 90일에는 기존/신규 모든 조직의 현재 핵심 기능을 Free Core로 유지하고 소프트웨어 기능 접근권이 아닌 자발적 운영 지원과 Paid Parish Operations를 검증한다. 자발적 운영 지원 여부에 따른 기능/기간/우선 지원 차등은 없다. Basic/Pro 가격은 장기 SaaS 가설로만 보존하며 판매하거나 신규 기능을 개발하지 않는다.

| 상품 | 가격 | 제공시간 상한 | 이번 실험에서의 의미 |
| --- | ---: | ---: | --- |
| 자발적 운영 지원 | 1인 30,000원 | 해당 없음 | 별도 기능과 서비스 기간을 약속하지 않는 비용 회수 실험 |
| 유료 운영 진단 | 100,000원 | 2시간 | 1장 진단서, 범위와 제외 항목, 일정과 선행 조건. 후속 계약 시 전액 반영 |
| 학기 도입 패키지 | 300,000원 | 4시간 | 명단 1개 파일 정리와 등록 지원, 기존 기능 세팅, 교육과 인수인계 |
| 연간 지원 패키지 | 600,000원 | 8시간 | 300,000원 패키지의 전액 선입금, 4시간 이내 납품, 양수 공헌이익 확인 뒤 다른 구매자에게 제안 |

- 같은 구매자에게 여러 가격을 동시에 제시하지 않는다.
- 유료 운영 진단 또는 범위가 명확한 학기 패키지는 전액 선입금 뒤 시작한다.
- 맞춤 기능 개발, 과거 출석 전체 이관, 외부 연동, 맞춤 보고서, 법률 의견과 적법성 보증은 포함하지 않는다.
- 600,000원 이상 선납은 관리형 서비스의 초기 신호일 뿐 SaaS나 갱신을 증명하지 않는다.
- 상세 납품물과 개인정보 선행 조건은 `docs/brainstorm/2026-07-15/revenue-viability.md`, 무료/유료 경계는 `docs/brainstorm/2026-07-15/partial-monetization.md`를 따른다.

## 베이직 플랜(3단계) (2026-04-24 재편 / 2026-05-09 anchor 가격 도입)

- 가격: **정가 월 5,990원 / 런칭가 월 4,900원** (연: 정가 59,900원 / 런칭가 49,000원, 광고 정책: 보류)
- **가격 표기 방식**: 정가 5,990원에 취소선 anchor 적용 → 런칭가 4,900원 강조 표시 (할인가 anchor)
- **런칭 할인 종료 기준**: 미정 (PG 본계약 후 첫 100건 결제 또는 6개월 중 빠른 시점 — 가설)
- **시그니처**: 알림/공지 발송 (SMS/카톡)
- 포함: 무료 전체 + 알림/공지 발송 + 행사 운영 + 다음달 축일 관리 + 교사 공유 캘린더(수동 등록, 구글 캘린더 연동)
- 근거: 수동 운영 지원 도구군. 알림은 외부 비용(SMS/카톡 건당)이 있지만 "엑셀+카톡" 대체 가치로 유료 전환 유도
- 사용량: SMS/카톡 월 200건 포함 (가설), 초과분 애드온
- **WTP 리스크 (가정)**: 행사 운영 기능은 파워 유저가 AI로 일회성 도구를 직접 제작 가능(`4_risk/risks.md` R20). 베이직 플랜의 "행사 운영" 가치만으로는 구독 전환 동력이 약할 수 있음. 시그니처(알림/공지)가 전환 핵심 트리거여야 하며, 행사 운영은 부가 혜택으로 포지셔닝. **브랜드 약속 경계 (2026-06-12)**: 약속("주일 아침 미사 직전에도, 이거 하나면 됩니다")은 속도/편의이지 기능 범위가 아님 — 플랜 기능 나열을 외부 카피에서 약속처럼 쓰지 않는다
- 검증: 초등부 P1 별도 코호트의 무료→베이직 전환율, 알림 발송 건수 분포, 캘린더 활성도, **행사 운영 기능 월 사용률 vs 구독 지속율 상관 (R20 검증)**, **단체 타입별 시그니처 적합도 (`1_market/personas.md` — 알림 1/4 vs 명단 이관 3.5/4)**. 4주 알림 사용률 30% 미만이면 3단계 착수 전 시그니처 교체 검토. 중고등부 무료 핵심 PMF와 동일 분모로 합산하지 않음

## 프로 플랜(4단계) (2026-04-24 재편)

- 가격: **월 15,000원 / 연 150,000원** (광고 정책: 보류. 2026-04-24 10,000→15,000 인상, AI 비용 반영)
- **시그니처**: 회의록 자동화 (해자)
- 포함: 베이직 전체 + **회의록 자동화** + 등록제(전자서명) + 개인정보 제공 동의(전자서명) + 결제 모듈
- 회의록 자동화: 메모/녹음 입력 → AI 분류(미사/교리/공지/안건) → PDF 컨펌 → 캘린더 자동 연동 (단일 통합 시스템)

- AI 비용 구조: Clova/Daglo STT + Claude API 분류 (변동비 발생 → 프로 가격 인상 정당화 근거)
- 사용량: AI 회의록 월 4회 포함 (가설), 초과분 애드온 or 토큰 기반 과금
- 검증: 베이직→프로 업그레이드율, 회의록 작성 빈도, AI 월 사용량, 회의록 기반 이탈률 저하

## 유료 애드온: NFC 출석 — 보류 (2026-04-24)

## 결제 주기 구조 (가설/근거/검증) (2026-05-14 추가)

### 가설

- 주일학교 교사의 업무 사이클(전례력/학년)에 결제 주기를 정렬해 **월간 요금제를 제공하지 않고** 학기(6개월)/연(1년) 2단 구조로 가면, 갱신율과 제품 몰입이 월간 대비 높다. **단, 무료→유료 1차 전환이 검증된 이후 적용하는 후처리 카드** — 현 단계(1단계 무료, PMF 미확정, 인터뷰 "비쌈" 시그널 직후)에서 월간을 제거하면 몰입 효과는 없고 진입장벽만 생긴다.

### 근거

- 위노트(상담교사 SaaS) 사례: 월간 요금제 미제공 → 월 기준 이탈 구조 자체를 제거. 결제 옵션 6개월/1년, 99% 연간 선택. "월 단위로 결제를 고민하게 만드는 것 자체가 제품 사용에 방해"라는 포지셔닝.
- 핵심 통찰: 결제 주기는 가격 전략이 아니라 **이탈 의사결정 지점(decision point) 제거 = 리텐션 설계**다. 월간 결제는 매달 "계속 쓸까?"를 묻게 해 결제 성공 여부와 무관하게 제품 몰입을 깬다.
- 기존 "연 10개월 할인"(플랜 구성)과 "연 결제 = 현금흐름 갭 완화"(정산 주기 가정)는 할인/현금흐름 프레임. 본 가설은 **월간 미제공 자체가 리텐션 기능**이라는 별도 프레임으로, 자금 확보 최우선 원칙(연간 선불 = 현금 선유입 + sunk cost lock-in)과 정합.

### 검증

- PG 본계약 후 첫 결제 모수에서: (a) 6개월 vs 1년 선택 비율 (b) 월간 옵션 요청 빈도 (c) anchor 가격 + 장기 결제 동시 제시 시 거부율 (d) 연간 갱신율(renewal rate)과 무료→유료 전환율 — 위노트가 공개하지 않은 지표

## PG 후보 (2026-04-24 조사 스냅샷)

### 가설

- 토스페이먼츠를 1순위 후보로 둔다. 신청 전 공식 문서, 실제 심사 조건, 수수료, 정산 기능을 다시 비교해 최종 선정한다.

### 근거

- 비교 대상: 토스페이먼츠 / 포트원(구 아임포트) / 나이스페이먼츠/NHN KCP
- 2026-04-24 당시 선정 기준 (개발 리소스 1인, 플랫폼 확장 가능성 고려):
    - **연동 난이도**: 토스페이먼츠 문서/SDK가 1인 개발에 유리하다는 내부 판단
    - **플랫폼 정산**: 미래 중개 전환에 필요한 기능 제공 여부를 공식 문서와 계약으로 재확인
    - **간편결제 연동**: 지원 범위, 별도 심사, 실제 리드타임을 계약 전 확인
- 대안 유보 사유도 당시 판단이다. 포트원의 다중 PG 유연성, 나이스/KCP의 심사와 리드타임을 같은 체크리스트로 다시 비교한다

### 검증

- PG 본계약 완료까지 실제 단계와 리드타임. 과거 가계약 → 에스크로 → 통신판매업 신고 → 본계약 순서가 현재도 적용되는지 먼저 확인
- 첫 결제 연동 후 1개월간 결제 성공률, 환불 처리 리드타임
- 수수료율 체감 — 거래 규모 확장 시 재협상 또는 다른 PG로 이전 가능성 (포트원 SDK 유연성 재검토 트리거)
- 재선정 트리거: (a) 월 거래액이 수수료 0.1% 차이가 유의미해지는 구간 진입, (b) 중개 플랫폼 전환 확정
