# PRD: 데일리 통계 대시보드

> 상태: Draft | 작성일: 2026-05-07

## 배경/문제 요약

- 참고: `docs/business/0_feedback/entries/2026-04-19-sungbuk-mikaela-2.md`, `docs/business/6_roadmap/roadmap.md` 2단계
- 문제: 통계 페이지 "학년별 통계" 테이블은 **주간 평균 / 월간 평균 / 연간 평균** 출석만 보여준다. 특정 날짜의 학년별 출석 수("오늘/지난 주일 우리 본당 학년별 몇 명 왔지?")를 한 화면에서 확인할 수 없다.
- 현재 상태: `statistics.groupStatistics` 출력은 학년 단위로 weekly/monthly/yearly만 포함.
- 목표 상태: 학년별 통계 테이블에 **일간 출석** 컬럼이 추가되어, 특정 날짜의 학년별 출석 수와 출석률을 같은 표에서 확인 가능.

## 목표/성공 기준

- **목표**: 서기/조회 역할 사용자가 학년별 통계에서 특정 날짜의 학년별 출석을 별도 페이지 이동 없이 확인.
- **성공 지표**: 출시 후 30일 내 "데일리 토탈 부재" 유사 피드백 재발 0건.
- **측정 기간**: 출시 후 30일.

## 사용자/대상

- **주요 사용자**: 본당/모임 관리자, 서기(조회 위주) 역할.
- **사용 맥락**: 일요일 미사 직후 또는 주중에 모바일/데스크톱에서 빠르게 확인.

## 범위

### 포함

- `GroupStatisticsTable`에 **"일간 출석"** 컬럼 추가 (모바일 표시)
- `GroupStatisticsTable`에 **"일간 출석률"** 컬럼 추가 (데스크톱 표시)
- 총계 행에 일간 합산 추가
- 통계 페이지 필터에 **날짜(day) 선택** 추가 (기본: 가장 최근 출석일 또는 오늘)
- 기존 헤더 명명 일관화: `주간 평균` → **주간 평균 출석**, `월간 평균` → **월간 평균 출석**, `연간 평균` → **연간 평균 출석**
- `statistics.groupStatistics` 입력에 `day` 추가, 출력에 `daily` 필드 추가

### 제외

- 시계열 차트, 학년 x 날짜 매트릭스
- 학생 단위 일별 출석 분포
- 신규 페이지 또는 신규 라우트
- 출석률/출석 수 외 추가 지표(◎/○/△ 마크 분포 등)

## 사용자 시나리오

1. **서기 — 일요일 미사 후**: 모바일로 통계 페이지 진입 → 날짜 필터가 가장 최근 출석일로 기본 설정 → 학년별 통계에 "일간 출석" 컬럼이 함께 표시되어 학년별 출석 수 즉시 확인.
2. **관리자 — 과거 일자 점검**: 날짜 필터로 특정 일요일 선택 → 학년별 일간 출석 표시 + 총계 행에서 그 날 전체 출석 합 확인.

- [ ] 날짜 필터 기본값: **가장 최근 출석일**(없으면 오늘)

### 제외 (Out)

## 제약/가정/리스크/의존성

- **제약**: 기존 `statistics.groupStatistics` API/스키마 확장으로 처리. 신규 procedure 추가하지 않음.
- **가정**: 기존 학년 스냅샷·졸업 필터 정책 그대로 적용.
- **리스크**: 컬럼 1~2개 추가로 모바일 가로 폭 부담. 모바일은 일간 출석 1개만 노출 + 출석률은 데스크톱 한정.
- **내부 의존성**: 기존 `GetGroupStatisticsUseCase`, `Attendance.groupId`, `GroupSnapshot`.
- **외부 의존성**: 없음.

## 롤아웃/검증

- **출시 단계**: 단일 PR 머지 후 전체 노출.
- **이벤트**: 신규 GA4 이벤트 없음. 기존 `statistics_viewed` 재사용.
- **검증**: 출시 후 7일 내 GA4에서 통계 페이지 평균 체류 시간 변화 + 성북동 직접 피드백 1건.
