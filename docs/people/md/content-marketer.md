# 콘텐츠 마케터 온보딩 플랜

향후 콘텐츠 마케터가 합류할 때 사용하는 4주 온보딩 템플릿이다. 현재 담당자가 채용됐거나
Instagram 운영이 위임됐다는 뜻은 아니다. 2026-07-15 감사 기준 상태이며, 최신 운영 상태는
`docs/business/STATUS.md`를 따른다.
기준 인재는 2~4년차급 실무형 미들이며, 연차보다 동등한 수행 근거를 우선한다.
선발 기준과 합류 전 강점/공백 기록은 `role-profiles.md`를 사용해 이 4주 계획에 연결한다.

> 90일 사업 검증 모드 채택은 콘텐츠 마케터 채용이나 Instagram 재개 승인이 아니다.
> `STATUS.md`와 `docs/business/3_gtm/instagram.md`에 재개 승인이 기록되기 전까지 콘텐츠 생산은 0이며 수신 DM 대응만 수행한다.

## 적용 모드

| 모드 | 허용 업무 |
| --- | --- |
| 동결 | 제품과 기존 콘텐츠 학습, 수신 DM 대응 연습, 기존 콘텐츠 검수 |
| 재개 승인 후 | 승인된 캘린더에 따른 제작, 게시, DM, 지표 보고 |

- 재개 결정이 `STATUS.md`에 기록되기 전에는 Week 2~4의 제작과 게시 과제를 기존 콘텐츠 검수로 대체한다.
- 현재 수치, 게시 주기, KPI를 이 문서에 복제하지 않고 당일 단일 출처에서 확인한다.
- 콘텐츠 초안은 AI가 보조하지만 사실 확인과 게시 책임은 담당자가 진다.

## Week 1: 제품과 사업 맥락

### Day 1~2: 제품 체험

1. 테스트 계정으로 가입, 조직, 그룹, 학생, 출석, 대시보드, 전례력 흐름을 사용한다.
2. `docs/content/README.md`에서 전체 제작 이력을 확인한다.
3. `docs/business/3_gtm/instagram.md`의 공개 유지 피드와 과거 제작물을 구분한다.
4. `docs/content/instagram/feed/019.md`의 장기 사용자 인터뷰를 읽되 단일 사례를 전체 성과로 일반화하지 않는다.

### Day 3~4: 단일 출처 읽기

1. `docs/business/README.md`: 현재 포지셔닝과 문서별 단일 출처
2. `docs/business/STATUS.md`: 기준일이 있는 운영 상태와 오픈 이슈
3. `docs/business/1_market/market.md`: 현재 검증 타깃과 의사결정 구조
4. `docs/business/2_bm/bm.md`: 가치 제안과 현행 수익 구조
5. `docs/business/3_gtm/gtm.md`: 채널과 온보딩 원칙
6. `docs/business/3_gtm/instagram.md`: 동결, 재개 게이트, 운영 원칙

### Day 5: 콘텐츠 규칙과 도구

- `.claude/rules/content.md`: 톤, 사실 확인, 개인정보, 금지 표현
- `.claude/rules/content-templates-feed.md`: 피드 카드뉴스 템플릿
- `.claude/rules/content-templates-reel.md`: 릴스 템플릿
- `.claude/rules/content-templates-combined.md`: 피드와 릴스 통합 템플릿
- `docs/content/brand/README.md`: 사용 승인된 브랜드 에셋과 적용 상태
- `docs/people/md/ai-usage-guide.md`: AI 도구 사용법

도구 체험은 읽기 작업부터 시작한다. `/biz status`로 현재 상태를 확인하고,
`/biz content`를 이용한 신규 초안 생성은 재개 승인 후에만 수행한다.

### 사실 확인 가드

- 실제 기능은 `docs/specs/README.md`와 코드에서 확인한다.
- 현재 수치와 운영 상태는 `STATUS.md`에서 확인하고 기준일을 함께 쓴다.
- 시장에 경쟁 도구가 없다는 식의 절대 표현은 최신 조사 없이 사용하지 않는다.
- 중고등부가 현재 검증 타깃이다. 전 그룹 확장 비전은 회의록 출시 전 외부에 노출하지 않는다.
- 학생과 교사 개인정보를 콘텐츠, 스크린샷, 생성형 AI 입력에 넣지 않는다.

- [ ] 현재 포지셔닝과 PMF 타깃 설명 가능
- [ ] 단체 지표와 사용자 지표를 구분해 설명 가능

## Week 2: 첫 산출물과 채널 안전

### 제작 워크플로

1. `STATUS.md`와 `instagram.md`에서 운영 모드를 확인한다.
2. 동결이면 기존 콘텐츠 1건을 골라 기능, 수치, 개인정보, 금지 표현을 검수하고 종료한다.
3. 재개 승인 상태이면 주제와 공개 범위를 대표에게 승인받는다.
4. `/biz content instagram [주제]`로 초안을 만든다.
5. `docs/specs/README.md`와 코드로 기능을 확인한다.
6. 수치, 출처, 개인정보, 금지 표현, 브랜드 규칙을 검수한다.
7. `docs/content/instagram/`의 형식별 디렉토리에 초안을 저장하고 대표와 에셋 검수를 받는다.
8. 승인 후 게시하고, 실제 상태에 맞게 콘텐츠 인덱스를 갱신한다.

### 채널 접근

- 개인별 역할 권한을 우선하고 로그인 정보를 문서나 메신저에 남기지 않는다.
- MFA와 권한 회수 담당자를 확인한다.
- 동결 중에는 Insights 조회와 수신 DM만 다룬다.
- 답변하기 어려운 가격, 계약, 법률, 개인정보 요청은 대표에게 이관한다.

### Week 2 체크포인트

- [ ] 동결 중 기존 콘텐츠 검수 1건 또는 재개 후 신규 초안 1건 완료
- [ ] 계정 권한, MFA, 회수 절차 확인
- [ ] DM 이관 기준을 예시로 설명 가능
- [ ] 재개 승인 시에만 게시했음을 확인

## Week 3: 독립 수행 연습

동결 중에는 콘텐츠 캘린더와 신규 초안을 만들지 않는다. 대신 기존 콘텐츠 1건의 사실, 톤,
개인정보를 점검하고 수신 DM 처리 시나리오를 연습한다. 재개 승인 후에는
`instagram.md`의 당시 실험안과 승인된 캘린더에 맞춰 제작과 게시를 수행한다.

### 지표 보고 원칙

| 지표 | 기록 원칙 |
| --- | --- |
| 게시물 도달 | 게시 후 동일한 관찰 기간으로 비교 |
| 프로필 방문과 외부 링크 클릭 | 같은 기간과 같은 분모 사용 |
| 가입 | UTM과 동일 사용자 퍼널이 있을 때만 채널 전환으로 귀속 |
| DM | 문의, 지원, 영업 의도를 구분하고 개인정보는 집계에서 제외 |

- 누적 외부 링크 클릭을 가입 수로 간주하지 않는다.
- Instagram 지표와 제품 DB의 단체 지표를 직접 나누지 않는다.
- 목표값과 재개 실험 조건은 `instagram.md`에서 읽는다.

### Week 3 체크포인트

- [ ] 운영 모드에 맞는 과제만 수행
- [ ] 기존 콘텐츠 검수 또는 승인된 신규 산출물 완료
- [ ] 수신 DM 1건을 기준에 맞게 처리하거나 모의 연습

## 핵심 메시지와 톤

- 외부 포지셔닝: 가톨릭 교리교사의 주일을 운영하는 도구
- 브랜드 약속: 주일 아침 미사 직전에도, 이거 하나면 됩니다
- 핵심 가치: 출석 기록 자체보다 명단, 출결, 축일 정보를 빠르게 조회하는 경험
- 공감적이고 비판하지 않는 톤을 사용한다.
- 기능보다 사용자가 얻는 결과를 말한다.
- 고객 노출 카피에 "무료"를 쓰지 않고 "바로 시작하기"를 사용한다.
- 기술 용어, 과장, 경쟁사 비방, 검증되지 않은 시장 단정을 피한다.
