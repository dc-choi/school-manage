# NFC 출석 v1: ESP32 기본형

> 상위 문서: `docs/brainstorm/2026-03-07-nfc-attendance/README.md`

## 개요

ESP32 마이크로컨트롤러 기반 NFC 출석 리더기. WiFi를 통해 API 서버에 HTTP 요청을 보내고, 서버 TTS 음성을 스피커로 재생한다.

## 구성 요약

```
[NTAG215 카드] → tap → [PN532 NFC 모듈] → I2C → [ESP32 DevKitC]
                                                       │
                                          WiFi ← ──────┤
                                                       │
                                    [MAX98357A 앰프] ← I2S ──┘
                                          │
                                    [스피커 (4Ω 3W)]
```

## 하드웨어 부품

### ESP32 DevKitC V4 (WROOM-32)

- **역할**: 전체 제어 (WiFi, NFC 통신, 오디오 재생)
- **프로세서**: Xtensa 32-bit LX6 듀얼코어, 240MHz
- **WiFi**: 802.11 b/g/n 내장
- **전원**: USB Micro-B (5V)
- **GPIO**: 30+ 핀 (I2C, SPI, I2S 등 지원)
- **가격**: 5,000~12,000원
- **구매처**: 디바이스마트, 아이씨뱅큐, 쿠팡, 알리익스프레스
- **프로그래밍**: Arduino IDE 또는 PlatformIO (C/C++)

### PN532 NFC RFID Module V3

- **역할**: NFC 카드의 고유 UID 읽기
- **칩**: NXP PN532
- **지원 카드**: NTAG215, Mifare Classic, Mifare Ultralight 등 (13.56MHz)
- **통신 모드**: I2C, SPI, UART (DIP 스위치로 선택)
- **읽기 거리**: 스펙 상 최대 5~7cm, 실사용 시 밀착~3cm (교통카드와 동일한 터치 방식)
- **전원**: 3.3V~5V
- **가격**: 5,000~12,000원
- **구매처**: 디바이스마트, 가치창조기술(VCTEC), 쿠팡
- **주의**: 저가 호환품은 오류 발생 가능. Elechouse V3 호환 제품 권장

### MAX98357A I2S 앰프 모듈

- **역할**: ESP32의 디지털 오디오 신호(I2S)를 아날로그로 변환 + 증폭
- **출력**: 최대 3.2W (4Ω 부하)
- **입력**: I2S 디지털 오디오 (DIN, BCLK, LRC 3선)
- **전원**: 2.5V~5.5V
- **가격**: ~3,000원

### 소형 스피커

- **사양**: 4Ω 3W (MAX98357A에 최적)
- **용도**: TTS 음성 재생 ("홍길동, 출석 완료!")
- **가격**: ~2,000원

### NTAG215 PVC 카드

- 상위 문서 참조 (v1/v2 공통)

## 하드웨어 연결

### 전체 배선도

```
ESP32 DevKitC           PN532 NFC 모듈
═══════════════         ═══════════════
 3.3V (3V3)  ─────────→  VCC
 GND         ─────────→  GND
 GPIO 21     ─────────→  SDA (데이터)
 GPIO 22     ─────────→  SCL (클럭)


ESP32 DevKitC           MAX98357A 앰프
═══════════════         ═══════════════
 GPIO 25     ─────────→  DIN  (오디오 데이터)
 GPIO 26     ─────────→  BCLK (비트 클럭)
 GPIO 27     ─────────→  LRC  (채널 선택)
 GND         ─────────→  GND
 VIN (5V)    ─────────→  VIN


MAX98357A 앰프           스피커 (4Ω 3W)
═══════════════         ═══════════════
 (+) 출력     ─────────→  (+)
 (-) 출력     ─────────→  (-)
```

### PN532 DIP 스위치 설정

PN532 모듈 뒷면에 2개의 DIP 스위치가 있다. I2C 모드로 설정해야 한다:

| 스위치 1 | 스위치 2 | 모드 |
|---------|---------|------|
| **ON**  | **OFF** | **I2C (사용)** |
| OFF     | ON      | SPI |
| ON      | ON      | UART (HSU) |

### 연결 시 주의사항

1. **전원 분리**: ESP32의 3.3V 핀으로 PN532에 전원 공급. MAX98357A는 5V(VIN) 사용
2. **선 길이**: 점퍼 와이어는 20cm 이내 권장 (I2C 통신 안정성)
3. **접지 공유**: ESP32, PN532, MAX98357A 모두 GND를 공유해야 함
4. **GPIO 충돌 없음**: I2C(21,22)와 I2S(25,26,27) 핀이 겹치지 않음

### 연결 확인 방법

Arduino IDE 시리얼 모니터에서 I2C 스캔 실행:
- PN532 응답 주소: `0x24` (정상 연결 시)
- 응답 없음: 배선 또는 DIP 스위치 확인

## 동작 흐름

```
1. 전원 ON → ESP32 부팅 (1초) → WiFi 연결
2. NFC 대기 상태 (PN532 폴링)
3. 학생이 카드 터치
4. PN532 → ESP32: 카드 UID 전달
5. ESP32 → API 서버: HTTP POST { tagId, deviceId }
6. API 서버 → ESP32: { studentName, audioUrl }
7. ESP32 → MAX98357A → 스피커: TTS 오디오 재생
8. 2번으로 돌아가 다음 카드 대기
```
