# 브레인스토밍 결과 보고서

## 주제
유입 전략과 이탈 방지 — WAU 24 → 30 달성을 위한 사용자 활성화/리텐션 전략

## 토론 요약

### Round 1
- 기획자가 3개 아이디어 제안: 사제 보고서 PDF, 7일 온보딩 배너, 운영자용 이탈 감지 알림
- 비판자 핵심 지적: PDF 양식 미검증, 7일 기준이 주1회 사용자와 불일치, 운영자 수동 개입 확장 불가
- 합의: PDF→텍스트 축소, 7일→주일 주기, 운영자→교리교사 직접 넛지, 전례력 예외 필요

### Round 2
- 기획자 개선: 복사 버튼 1개(1주), 주일 맥락 배너(3일), 교리교사 직접 이메일+전례력 예외(3~4일)
- 비판자 핵심 지적: 장위동 64% 편중으로 통계 빈약, 이탈 원인 미파악, 인터뷰 선행 필요
- 합의: 범위 축소 타당, 아이디어2 = 로드맵 항목, 문구 변경 MVP 제외

### Round 3
- 비판자: 아이디어1 조건부 동의, 아이디어2 동의, 아이디어3 조건부 동의(인터뷰 선행)
- 기획자: 모든 조건 수용. 배너 소멸 = "첫 출석 완료" 단일 조건 확정
- 신규 유입 직접 공략 아이디어 누락 → 다음 브레인스토밍 주제로 설정
- 최종 착수 순서 합의: 아이디어2 → 아이디어1 → (인터뷰 병렬) → 아이디어3

## 아이디어 목록

### [아이디어 1] 출석 현황 공유 텍스트
- 한 줄 요약: 출석 화면에서 버튼 하나로 사제 보고용 텍스트를 클립보드에 복사
- 사용자 가치: 수기 보고 제거, 사제 의사결정 리드타임 단축 (bm.md 가치5)
- 실현 방법: "공유 텍스트 복사" 버튼 → "이번 주 출석: N명/전체: M명(O%)|결석: P명|[날짜]" 클립보드 복사
- 차별점: 밴드/엑셀은 데이터 가공→전송 3단계. 이 기능은 확인→전송 2단계
- 검증: GA4 `share_text_copied` 이벤트. 클릭률 30%+ → 수요 존재. 6~8주 후 인터뷰로 실제 사제 전달 확인
- 판정: **조건부 승인** (간접 기여 가설로 운영)

### [아이디어 2] 컨텍스트 배너 — 출석 미시작 구간 집중
- 한 줄 요약: 학생 등록 후 첫 출석 미시작 단체에 주일 날짜 넣은 단일 CTA 배너 노출
- 사용자 가치: 퍼널 병목 27% 이탈 구간 직접 공략 (전환 유력 6곳 포함 11곳 대상)
- 실현 방법: 학생 1명+ & 출석 0건 조건 → "이번 주 일요일([날짜])에 첫 출석을 기록해보세요" 배너 → 클릭 시 출석 화면 → 첫 출석 완료 시 소멸
- 차별점: 범용 앱은 퍼널 상태 인식 불가. 주일 워크플로우 맥락 인식은 가톨릭 특화
- 검증: GA4 배너 노출/클릭 이벤트. 전환 유력 6곳의 WAU 전환 여부 추적
- 판정: **승인** (STATUS.md 2단계 미착수 항목, WAU 30 직접 기여)

### [아이디어 3] 이탈 감지 알림 — 전례력 인지 + ADMIN 발송
- 한 줄 요약: 14일 미활동 + 전례력 미휴무 조건 시 관리자에게 자동 넛지 이메일 발송
- 사용자 가치: Silent Churn 조기 감지 및 복귀 트리거 (흑석동 15일 미활동 사례)
- 실현 방법: 매일 크론잡 → 14일 미활동 & 전례력 예외 아닌 단체 → ADMIN 이메일 발송 → 출석 화면 직접 링크
- 차별점: 밴드/엑셀에 이탈 감지 없음. 전례력 예외는 가톨릭 도메인 특화
- 검증: 발송 후 7일 내 활동 재개 여부 추적
- 판정: **조건부 승인** (인터뷰 선행)
