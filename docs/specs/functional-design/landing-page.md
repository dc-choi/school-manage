## 기본 설계 (구현 완료)

> 사업 근거: `docs/business/STATUS.md` 전환 퍼널 분석 (이탈율 81.8%)

### 배경

- 로그인 페이지가 서비스 설명 없이 바로 행동을 요구 → 이탈 발생
- AuthLayout 개선만으로는 설득 구간 부족 → 전용 랜딩 페이지 필요

### 사용자 플로우

```
인스타그램 프로필 링크 → /landing
    ↓ (풀스크린 스크롤)
① 히어로: 후킹 헤드라인 + 서브카피 + 스크롤 가이드
    ↓
② 공감: Pain Points 3건
    ↓
③ 기능 소개: 핵심 기능 4개 (아이콘 + 설명)
    ↓
④ 인터랙티브 데모: 달력 + 출석 체크 체험 (로컬 데이터)
    ↓
⑤ 사회적 증거 + CTA → /signup 또는 /login
```

### 상태 전이

```
[비인증 방문] → /landing (설득) → /signup (전환) → / (가치 체감)
[인증된 사용자가 /landing 접근] → / 대시보드로 리다이렉트
```

### 페이지 구조

5단 풀스크린 스크롤 (각 섹션 min-h-screen). FadeIn 애니메이션 (IntersectionObserver).

```
┌──────────────────────────────┐
│  ① Hero (풀스크린)             │
│  - 후킹 헤드라인 + 서브카피      │
│  - 스크롤 가이드               │
├──────────────────────────────┤
│  ② Pain Points               │
│  - 공감 유발 3건               │
├──────────────────────────────┤
│  ③ Features                  │
│  - 기능 카드 4개               │
├──────────────────────────────┤
│  ④ Interactive Demo           │
│  - 달력 + 출석 체크 체험         │
├──────────────────────────────┤
│  ⑤ Social Proof + CTA        │
│  - "{N}개 단체" + CTA + 로그인  │
├──────────────────────────────┤
│  Footer                      │
└──────────────────────────────┘
```

### 라우팅

| 경로 | 컴포넌트 | 인증 | 비고 |
|------|----------|------|------|
| `/landing` | LandingPage | public | 인증 시 `/` 리다이렉트 |

### API/인터페이스

신규 API 없음. `account.count` (기존) 재사용 → 사회적 증거 표시.

### 예외/엣지 케이스

| 상황 | 처리 방법 |
|------|----------|
| account.count 실패 / 0 | 사회적 증거 영역 숨김 |
| 인증된 사용자 접근 | `/` 리다이렉트 |

### 측정/모니터링

| 이벤트 | 설명 |
|--------|------|
| `landing_view` | 랜딩 페이지 진입 |
| `landing_section_view` | 섹션 뷰포트 진입 (section: hero/pain-points/features/demo/cta, 1회) |
| `landing_cta_click` | CTA 버튼 클릭 (위치: bottom) |
| `landing_login_click` | 로그인 링크 클릭 |

### 테스트 시나리오

1. `/landing` 접근 → 5단 섹션 정상 표시
2. CTA "지금 시작해보기" 클릭 → `/signup` 이동
3. "이미 계정이 있으신가요?" 클릭 → `/login` 이동
4. 사회적 증거에 account.count 값 동적 표시
5. 인터랙티브 데모: 달력 날짜 클릭 → 출석 체크 → 달력 반영
6. 인증된 사용자 `/landing` 접근 → `/` 리다이렉트
7. account.count 실패/0 → 사회적 증거 숨김

## 포지셔닝 개선 2차 — "주간 허브" 리프레이밍 (로드맵 1단계)

> 사업 근거: `docs/business/2_bm/bm.md` (가치 제안 — "매주 일요일, 이거 하나 열면 다 되는 가톨릭 모임 운영 도구")
> 사업 근거: `docs/business/3_gtm/gtm.md` (포지셔닝 메시지 — "한곳에" 테마)
> 핵심 관찰: 출석 "기록"보다 "빠른 조회"가 핵심 가치 (장위동 4년 사용)

### 배경

- 1차 포지셔닝 개선에서 "주일학교 교리교사를 위한 / 출석·멤버 관리 도구"로 타겟은 명확해짐
- 그러나 BM 가치 제안이 "출석 관리 도구" → **"주간 운영 허브"**로 리프레이밍됨
- 핵심 가치 2("매주 필요한 정보를 한곳에서 빠르게 조회")와 축일 조회가 랜딩에 전혀 반영되지 않음
- 랜딩이 "기록하는 도구"로만 보임 → "매주 여는 도구" 느낌이 안 남

### 목표

- 방문자가 "매주 일요일에 이거 하나만 열면 돼"라는 **주간 습관**을 연상
- 축일·멤버·출석이 **한곳에** 있다는 통합 가치 전달
- 5단 구조, 인터랙티브 데모, 레이아웃 유지 (콘텐츠/메시지만 교체)

### 변경 범위

| 섹션 | 변경 내용 | 비고 |
|------|----------|------|
| ① Hero | 헤드라인 + 서브카피 교체 | "주간 허브" 포지셔닝 |
| ② Pain Points | 1건 교체 | 축일 조회 마찰 추가 |
| ③ Features | 1건 리프레이밍 | 축일·조회 반영 |
| ④ Interactive Demo | 변경 없음 | |
| ⑤ CTA | 변경 없음 | 이전 개선에서 이미 간소화됨 |
| AuthLayout | 히어로 메시지 동기화 | 랜딩 기준 |

**① Hero**

| 요소 | 현재 | 변경 후 |

- 근거: GTM 범용 메시지("매주 일요일, 이거 하나 열면 됩니다") 기반
- "도구" → "주간 습관"으로 프레이밍 전환

## 메타 태그 + 브랜딩 정비 (로드맵 2단계)

> 사업 근거: `docs/business/3_gtm/gtm.md` (전환 경로 — 인스타그램/카톡 링크 공유가 주요 유입 채널)
> 사업 근거: `docs/business/STATUS.md` (전환 퍼널 — 이탈율 개선)

### 배경

- 인스타그램과 카톡이 주요 유입 채널이지만, OG 태그가 없어 링크 공유 시 미리보기가 빈약함
- `<meta description>` 없음 → 검색엔진에서 서비스 설명 미표시
- `<title>`이 "주일학교 관리 프로그램"으로 브랜드명("주일학교 출석부")과 불일치
- favicon이 JPEG 형식(비표준) → 일부 브라우저/OS에서 표시 불안정
- footer 저작권 연도 "2022"로 고정 → 현재 연도 미반영

### 목표

- 인스타/카톡 링크 공유 시 **제목 + 설명 + 썸네일**이 표시되어 클릭율 향상
- 검색엔진에서 서비스 설명이 노출되어 자연 유입 기반 마련
- 브랜드명/아이덴티티를 메타데이터에 일관되게 반영
