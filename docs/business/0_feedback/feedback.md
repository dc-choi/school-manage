## 외부 피드백

### 2026-01-25 성북동 미카엘라 (사업 관점)

**요약**: 서비스 니즈는 있으나 장기 BM 고려 필요

**핵심 논점**
- 사업성 미지수: 봉헌금 페이도 앱 전환 실패 사례 → 교회 디지털 전환 어려움
- 결제 주체 불명확: 본당 돈이면 설득이 관건
- 윗선 니즈 부재: 사제/사무실은 현행(한글/엑셀)이 불편하지 않음. 프로세스 전환이 오히려 불편
- 광고 수익 의문: 교회 커뮤니티 광고 효과 제한적

**BM 방향 제안**
1. 교회가 서비스 구매/정기결제
2. 서비스 내 수익화 (광고) - 효과 의문

**확장 방향 제안**
- 현재: 교사를 위한 학생출석부 앱
- 제안: 가톨릭 교회 내 모든 모임을 위한 앱
- 그룹 운영에 도움될 기능이 있으면 소구 가능성 있음

**성공 요건**
- 연간 보고와 연동 필요
- 무료 지속 시 경쟁/대체제 무관

**대응**
- PMF 우선: 1차 타겟(중고등부)에서 먼저 검증
- 비전 확장: 장기적으로 "가톨릭 교회 내 모든 모임을 위한 앱" 지향

### 2026-03-11 흑석동 (지인 경유)

**요약**: 대만족. 추가 필드 니즈 + 학생 참여형 출석 요청

**피드백 항목**

| 피드백 | 분류 | 대응 |
|--------|------|------|
| 부모님 연락처 별도 필드 요청 (현재 비고란 사용) | 학생 정보 | 추가 필드 니즈 — 성남동과 동일 패턴 (2곳 반복) |
| 생년월일 표기 요청 | 학생 정보 | 추가 필드 니즈 |
| 학생이 자기 출석률 확인하여 자극받게 | 출석 운영 | NFC 기반 유료 기능으로 대응 예정 |
| 계정 공유 (여러 교사가 동시 접속) | 인증/계정 | 계정 모델 전환 작업 중 |

**도입 사이클**: 지인 홍보 → 다이렉트 가입 → 다이렉트 도입 → 만족 중
**본당 공유**: 흑석동은 개인→단체 공유 완료, 과천은 미사용

**패턴 분석**
- **추가 필드 니즈**: 성남동(비고란 세밀 작성) + 흑석동(부모님 연락처/생년월일) — 2곳 반복, 트리거 조건 3곳 접근 중
- **학생 참여형 출석**: 길음동(수기 출석부로 학생 직접 체크) + 흑석동(셀프 출석 확인 요청) — 2곳 반복

### 2026-03-13 수색성당 박지안

**요약**: 계정 모델 이슈 실증 + 운영 기능 요청

**피드백 항목**

| 피드백 | 분류 | 대응 |
|--------|------|------|
| 여러단체 동시 가입 | 인증/계정 | 실제 문제는 TEACHER가 기존 모임에 합류 불가. 계정 모델 전환(2단계)으로 해결 |
| 선생님 명단 관리 | 조직/권한 | 3단계(교리교사 역할/권한 세분화)에서 대응 |
| 단체 해산 | 조직 관리 | 신규 항목. 계정 모델 전환 시 라이프사이클 관리로 커버 가능 |

**패턴 분석**: 수색성당은 3계정(최민석 ADMIN + 박지안 TEACHER + 중복) 생성. 금호동/신당동과 동일한 A유형(계정 모델 불일치).

## 출석 운영
- 출석 체크 간편화 — 길음동(수기 출석부로 학생 직접 체크), 흑석동(학생 셀프 출석 확인 요청)
- 출석부 자동 저장 (체크/페이지 전환 즉시 저장)
- 정렬 기준 (가나다/우수 출석)
- 의무축일/행사 출석 구분
- 출석부 UI 전면 개편 요구 (다수)
- 출석 페이지에서 전체 그룹에 속한 학생 확인 (2026-02-28 피드백 이벤트) — 2단계 독립 항목. 현재 1:N 구조에서도 구현 가능

> 출석 관련 피드백의 대부분은 장위동 초등부 파일럿에서 수집됨
> (초등부 전체 의견 일반화 아님, 초기 가치 체감이 이탈을 좌우한다는 레퍼런스)
