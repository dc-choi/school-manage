## 유스케이스

### UC-1: 컷오버 실행

**전제 조건**: 배포 이미지/정적 산출물이 준비됨

**주요 흐름**:
1. 사전 체크리스트를 확인한다.
2. 백엔드를 업데이트한다.
3. 프론트를 교체한다.
4. 스모크 테스트를 수행한다.

**결과**: 서비스가 정상 상태로 전환된다.

### UC-2: 롤백

**전제 조건**: 스모크 테스트 실패 또는 장애 발생

**주요 흐름**:
1. 이전 백엔드 이미지로 복구한다.
2. 이전 프론트 dist로 복구한다.
3. 상태를 재검증한다.

**결과**: 이전 버전으로 안정화된다.
