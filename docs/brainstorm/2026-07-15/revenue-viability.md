# 수익성 브레인스토밍 결과 보고서

## 주제

이 프로젝트는 근본적으로 돈이 되는가?

현재 사용자 가치, 결제자, 시장 상한, 가격, 판매비용을 분리해 검토하고 다음 네 가지 가능성을 판정한다.

1. 운영비를 스스로 충당하는 무료 사이드프로젝트
2. 창업자에게 의미 있는 소득을 주는 독립 SaaS
3. 본당 대상 제품화 서비스
4. 제품을 신뢰 자산으로 사용하는 가톨릭 조직 SI

## 한 줄 결론

**쓸모 있는 제품이라는 근거는 있지만, 현재 가격과 구매 구조로 반복해서 돈을 버는 독립 SaaS라는 근거는 없다.**

- 운영비 회수형 사이드프로젝트: 조건부 Go
- 현재 Basic/Pro 구조의 독립 SaaS: 현시점 No-Go
- 본당 대상 제품화 서비스: 판매 실험만 조건부 Go
- 제품 유래 SI: 현재 근거 부족, 후속 피벗 후보
- 투자형 또는 벤처형 사업: 현재 시장과 객단가로는 No-Go

월 직접비가 작아 지금 즉시 접을 필요는 없다. 다만 유료 기능을 더 만들기 전에 90일 동안 실제 수납과 창업자 시간을 측정해야 한다.

## 채택 결정, 2026-07-15

- 사용자는 **90일 사업 검증 모드**를 채택했다.
- 실행 기간은 **2026-09-01부터 2026-11-29까지**, 시작일과 종료일을 포함해 90일이다.
- 이번 기간에는 자발적 운영 지원과 본당 운영 지원 패키지의 실제 수납만 검증한다.
- Basic/Pro 신규 유료 기능 개발, 제품 유래 SI, 채용은 이번 검증의 실행 범위에 포함하지 않는다.
- 2026-11-29까지 데이터를 마감하고, 2026-11-30에 실제 현금, 개인 관계 밖 결제, 전체 영업과 납품시간을 기준으로 다음 모드를 다시 결정한다.

## 판정 전제

- 모든 매출 계산은 세금, 환불, PG 수수료, 알림과 AI 변동비, 창업자 노동비 차감 전이다.
- 현재 제품 현금 Burn 월 5만원은 서버와 도메인만 포함한다. 정상화 비용과 창업자 노동비가 아니다.
- MAO는 단체 수이고 MAU는 사용자 수다. 개인 후원 전환율의 분모로 MAO를 직접 사용하지 않는다.
- 월 100만원, 300만원, 500만원은 사업 크기를 비교하기 위한 매출 예시이며 사용자의 소득 목표로 가정하지 않는다.
- 외부 시장의 1,499곳은 초등부만 운영하는 본당도 포함한 넓은 상한이다. 현재 중고등부 타깃의 실제 시장은 이보다 작다.

### 판매비용 충돌

문서는 총마진 60%, CAC 회수 3개월을 가정한다. 월 결제 가격을 기준으로 한 허용 CAC는 Basic 8,820원, 혼합 14,274원, Pro 27,000원 정도다. 연 결제 가격을 월 환산하면 각각 7,350원, 11,895원, 22,500원으로 더 낮다.

Basic 연 49,000원의 첫해 공헌이익을 60%로 잡으면 29,400원이다. 창업자 내부 시간단가를 비교용으로 시간당 3만원만 잡으면 약 59분은 첫해 공헌이익 전체를 노동비로 소진하는 노동 조정 손익분기 상한이다. 3개월 CAC 회수 기준의 연 결제 Basic 영업시간 상한은 약 14.7분에 불과하다.

직접 접촉, 2주에서 4주 파일럿, 사제 승인과 증빙이 필요한 현재 판매 방식과 양립하기 어렵다. 핵심 문제는 서버비가 아니라 **기업형 구매 마찰에 소비자형 가격을 붙인 것**이다.

## 외부 비교가 주는 시사점

- [지저스온 공식 가격](https://www.jesuson.net/home-ux/solution/price.aspx)은 100명 기준 교적, 목회, 헌금, 회계 등 핵심 행정 기능을 월 7,700원부터 제공한다.
- [Planning Center People](https://www.planningcenter.com/people)는 구성원 데이터베이스, 보고, 기본 커뮤니케이션을 무료로 제공한다.
- [브랜치스 현재 공개 페이지](https://branches.team/)는 415개 교회, 3,547명 사용을 주장한다. 저장소의 2026-03-16 스냅샷은 95개 교회, 220명이었다.

이 비교는 위클리스쿨이 가격을 받을 수 없다는 뜻이 아니다.

1. 범용 명단과 출석만으로는 무료 대체재와 경쟁해야 한다.
2. 가톨릭 특화는 도입 이유가 될 수 있지만 그 자체가 높은 가격이나 해자를 증명하지 않는다.
3. 브랜치스의 빠른 확장은 인접 시장 수요를 보여주지만 유료 고객이나 매출을 증명하지 않는다.
4. 높은 가격은 기능 접근권이 아니라 데이터 이관, 인수인계, 보고, 교육처럼 결제자의 책임을 줄이는 관리 결과로만 검증해야 한다.

## 3라운드 토론

### Round 1

기획자는 세 모델을 제안했다.

1. 연 3만원에서 5만원의 운영 후원
2. 본당 연 30만원에서 60만원의 운영 패키지
3. 100만원에서 200만원의 가톨릭 조직 SI

비판자는 다음을 지적했다.

- 현재 Basic은 사람을 통해 판매할 수 없는 가격이다.
- 교리교사 가치가 사제와 사무실의 결제 가치로 연결되지 않는다.
- 무료 PMF와 유료 PMF가 서로 다르다.
- 시장, 채널, 보안 비용을 포함하면 독립 SaaS 근거가 없다.

합의:

- 현재 Basic을 사람을 통해 판매하는 전략은 폐기한다.
- 후원, 제품화 서비스, SI를 서로 다른 사업 가설로 분리한다.
- PG나 신규 기능보다 실제 수납을 먼저 본다.

### Round 2

기획자는 본당 패키지를 SaaS가 아닌 관리형 서비스로 재분류하고 첫해 120만원 상단 가격을 제안했다.

비판자는 다음을 추가했다.

- 연 120만원은 국내 ChMS 가격보다 높아 소프트웨어 가격으로 방어할 수 없다.
- 현재 Export, 맞춤 보고, 법률 검토가 미완료라 제공시간 8시간 가정은 낙관적이다.
- 제품이 SI 리드를 만들었다는 증거는 현재 0건이다.
- 유료 본당 10곳은 상품 검증이지 독립 사업 성공이 아니다.

합의:

- 당시에는 연 120만원을 관리형 서비스 상단 가격 실험으로 남겼으나, 후속 부분 유료화 결정에서 활성 단계에서 제거했다.
- 통제 가능한 납품물만 약속하고 맞춤 개발, 과거 출석 전체 이관, 적법성 보증은 제외한다.
- 성공한 계약뿐 아니라 실패 영업시간도 손익에 포함한다.
- 상품 검증과 독립 사업 검증을 분리한다.

### Round 3

최종안은 세 모델을 동시에 진행하지 않고 다음 순서로 수렴했다.

1. 한 번의 후원 실험으로 직접비 방어 가능성을 확인한다.
2. 본당 예산 결정자에게 유료 진단과 고정 범위 패키지를 순차 판매한다.
3. 본당 패키지가 실패한 경우에만 별도의 다음 90일 실험으로 제품 유래 SI를 검토한다.
4. 각 단계에서 입금 전에 개발하지 않는다.

남은 불확실성:

- 최신 활성 개인 수가 없어 후원 분모를 모른다.
- 본당 결제자가 실제로 줄이고 싶은 비용이나 책임이 확인되지 않았다.
- 고단가 패키지를 개인 관계 밖에서 팔 수 있는지 모른다.
- 제품이 SI 리드를 만드는지 모른다.
- 연간 갱신은 90일 안에 검증할 수 없다.

- 최신 활성 사용자 수와 실제 전환율로 75만원 도달 경로가 없음
- 후원 메시지가 향후 유료 기능 신뢰를 훼손함

이 실험은 사업성을 증명하지 않는다. 향후 12개월 직접비에 해당하는 현금을 자발적 지원으로 확보할 수 있는지만 검증한다. 수납 전에는 적용되는 회계 처리, 안내 약관, 환불, 서비스 종료 조건을 확인한다.

### 아이디어 2. 본당 연간 운영 지원 패키지

분류: 고단가 제품화 서비스, 관리형 서비스 탐색

순차 가격 실험:

| 단계 | 가격 | 제공시간 상한 | 원칙 |
| --- | ---: | ---: | --- |
| 유료 운영 진단 | 10만원 | 2시간 | 후속 계약 시 전액 차감 |
| 학기 도입 패키지 | 30만원 | 4시간 | 진단 후 또는 범위가 명확하면 직접 제안 |
| 연간 지원 패키지 | 60만원 | 8시간 | 30만원 납품 가치 확인 뒤 제안 |

유료 운영 진단의 독립 납품물:

- 현재 운영 흐름과 데이터 상태를 정리한 1장 진단서
- 도입 시 포함 범위와 제외 범위
- 예상 일정, 필요한 역할, 다음 단계 견적
- 개인정보 원본이 없는 상태에서 확인 가능한 위험과 선행 조건

진단 단계에서는 학생 개인정보 원본을 받지 않는다.

포함:

- 현재 학기 명단 1개 파일 정리와 등록 지원
- 기존 기능 범위 내 그룹, 계정, 권한 설정
- 운영자 교육 1회
- 인수인계용 운영 체크리스트
- 정해진 일정의 운영 점검 2회
- 계약된 지원시간 내 질의 대응

제외:

- 과거 출석 이력 전체 이관
- 고객별 기능 개발
- 외부 시스템 연동
- 맞춤형 사제 보고서와 데이터 Export
- 법률 의견, 적법성 보증, 보안 인증
- 사용률과 리텐션 같은 고객 행동 결과 보장

개인정보 선행 조건:

- 학생 명단 원본을 받기 전에 본당의 처리 권한과 필요한 절차를 확인
- 처리 항목, 목적, 접근자, 전달 방식, 보관 장소, 삭제 기한을 문서화
- 안전한 파일 전달과 접근 기록, 완료 후 삭제 확인 방법을 고정
- 위 조건을 확정하지 못하면 원본 파일을 받지 않고 명단 정리와 등록 대행을 진행하지 않음

90일 신호:

- 실제 예산 결정자 5명에게 유료 진단 또는 범위가 고정된 30만원 패키지 제안
- 최소 2명은 기존 개인 관계 밖의 구매자
- 진단 결제 또는 패키지 전액 선입금 합계 2건 이상
- 30만원 패키지 결제 1건 이상
- 개인 관계 밖 결제 1건 이상
- 실패 영업시간을 포함한 노동 조정 공헌이익이 양수

신호의 의미를 구분한다.

- 10만원 진단 결제: 유료 문제 진단 수요
- 30만원 패키지 결제: 일회성 제품화 서비스 수요
- 60만원 이상 실제 선납: 연간 관리형 서비스의 초기 신호
- 사람 개입이 거의 없는 소프트웨어 반복 결제: SaaS 신호

이번 90일 안에는 SaaS를 검증했다고 판정하지 않는다.

즉시 중단:

- 5명 중 유료 진단과 30만원 패키지 전액 선입금이 모두 0건
- 결제자 고통이 없고 교리교사 개인 편의만 확인됨
- 30만원 패키지도 거부되는데 더 높은 가격만 필요함
- 실제 제공시간이 상한을 넘고 재사용 가능한 범위가 없음
- 개인정보와 보안 요구가 현재 제공 범위를 계속 초과함

개인 관계 밖 결제가 0건이면 관계 기반 서비스 가능성만 남기고 독립 사업 경로는 기각한다.

### 아이디어 3. 제품 유래 SI 리드 자산

분류: 본당 패키지 실패 시 피벗 후보

제품 유래 리드로 인정하는 조건:

- 제품 사용, 제품 사용자 추천, 제품 랜딩을 통해 처음 유입
- 창업자의 기존 지인, 직장 관계, 기존 외주 관계가 아님
- 실제 예산 결정자와 유료 프로젝트 대화가 진행됨

현재 이 정의를 충족하는 유료 리드는 0건이다.

후보 상품:

- 1주에서 2주 고정 범위 데이터 운영 스프린트
- 가격 100만원에서 200만원
- 최대 제공시간 20시간
- 사전 정의한 입력 형식 1개의 데이터 정리와 검증 보고서
- 사전 정의한 입력과 출력 1개의 반복 보고 자동화
- 일회성 백업 설정과 복원 테스트 중 합의한 범위
- 제품 본체의 고객별 맞춤 개발 제외
- 지속 백업 운영, 무제한 외부 연동, 고객별 신규 시스템 개발 제외

검증:

- 제품 유래 적격 대화 10건 후 유료 계약 0건이면 가설 기각
- 첫 신호는 유료 계약 1건, 최소 검증은 2건
- 계약당 100만원 이상
- 실패 영업시간까지 포함한 노동 조정 공헌이익률 50% 이상
- 공통 범위가 50% 미만이면 제품화하지 않음

## 90일 실행안

세 모델을 동시에 운영하지 않는다. 운영 지원은 한 번의 저비용 요청으로 끝내고 본당 패키지를 주 실험으로 둔다. SI는 패키지 실패가 확정된 뒤 별도 실험으로만 시작한다.

이번 90일에는 후원과 본당 패키지만 검증한다. 제품 유래 SI는 이 결과가 실패하고 사용자가 피벗을 선택한 경우 별도의 다음 90일 실험으로 진행한다.

| 상대 일정 | 실제 일정 |
| --- | --- |
| 1일에서 7일 | 2026-09-01부터 2026-09-07까지 |
| 8일에서 21일 | 2026-09-08부터 2026-09-21까지 |
| 15일에서 42일 | 2026-09-15부터 2026-10-12까지 |
| 29일에서 60일 | 2026-09-29부터 2026-10-30까지 |
| 43일에서 75일 | 2026-10-13부터 2026-11-14까지 |
| 76일에서 90일 | 2026-11-15부터 2026-11-29까지 |

### 1일에서 7일

- 유료 기능 신규 개발 동결
- 최신 활성 사용자와 실제 연락 가능한 개인 수 재산출
- 본당 예산 결정자 5명 선정, 최소 2명은 개인 관계 밖에서 선정
- 창업자 내부 시간단가 확정
- 진단과 패키지 범위, 제외 항목을 한 장으로 고정
- 영업, 납품, 재작업 시간 기록 양식 확정
- 실제 수납 전 증빙, 회계 처리, 안내 약관, 환불, 서비스 종료 조건 확인
- 개인정보 원본 수령 전 처리 권한, 범위, 안전한 전달, 보관과 삭제 절차 확인

### 8일에서 21일

- 30,000원 단일 자발적 운영 지원 요청
- 실제 입금만 성공 처리
- 후원자 기능이나 배지 개발 금지
- 21일에 75만원 도달 가능성과 반감 여부 판정

### 15일에서 42일

- 실제 예산 결정자 5명에게 10만원 유료 진단을 순차 제안
- 문제가 이미 명확한 구매자에게는 범위가 고정된 30만원 패키지 전액 선입금을 직접 제안 가능
- 진단비는 후속 계약에 반영
- 무료 진단 납품으로 전환하지 않음
- 5명 중 진단 결제와 패키지 전액 선입금이 모두 0건이면 본당 패키지 중단

### 29일에서 60일

- 유료 진단 결제자 또는 직접 패키지 의향 구매자에게 30만원 학기 패키지 제안
- 한 건을 먼저 납품해 실제 시간과 범위를 측정
- 30만원 제안 거절 시 60만원을 제안하지 않음
- 입금 전 개발 금지

### 43일에서 75일

앞 단계가 성공한 경우에만 진행한다.

- 30만원 전액 선입금 1건, 4시간 이내 납품, 해당 건 노동 조정 공헌이익 양수를 모두 확인한 경우에만 별도 구매자에게 60만원 연간 지원 패키지 제안
- 같은 구매자에게 여러 가격을 동시에 제시하지 않음

### 76일에서 90일

다음을 한 표로 합산한다.

- 후원 실제 현금
- 유료 진단과 패키지 계약 수
- 개인 관계 밖 계약 비율
- 성공과 실패 영업시간
- 납품과 재작업 시간
- 직접비
- 노동 조정 공헌이익
- 반복된 구매 이유
- 범위 초과 요청
- 보안과 개인정보 요구

판정 조합:

| 결과 | 판정 |
| --- | --- |
| 운영 지원 75만원 이상, 본당 유료 신호 없음 | 비용 회수형 무료 사이드프로젝트 |
| 유료 진단만 결제, 30만원 패키지 없음 | 문제 진단 수요만 확인, 서비스 상품 재설계 또는 중단 |
| 30만원 패키지 결제, 60만원 이상 선납 없음 | 일회성 제품화 서비스 신호 |
| 60만원 이상 실제 선납 | 연간 관리형 서비스 초기 신호, 실제 갱신 전까지 반복 사업으로 보지 않음 |
| 운영 지원과 본당 결제가 모두 기준 미달 | 독립 사업 가설 종료, 유지 또는 중단 판단 |
| 개인 관계 밖 결제 0건 | 관계 기반 서비스만 확인, 반복 가능한 독립 사업 경로는 No-Go |

여러 조건을 동시에 충족하면 가장 높은 사업 단계와 운영 지원 결과를 함께 기록한다. 어떤 결과에서도 이번 90일만으로 SaaS를 Go로 바꾸지 않는다. 본당 패키지가 실패하고 사용자가 원할 때만 제품 유래 SI를 별도 90일 피벗 실험으로 시작한다.

## 장기 사업 게이트

유료 본당 10곳은 상품 검증일 뿐 독립 사업 증거가 아니다. 독립 사업 가능성은 아래를 모두 충족할 때 재평가한다.

- 유료 본당 30곳에서 50곳 또는 복수 교구 계약
- 실제 연간 갱신 발생
- 고객 다수가 기존 개인 관계 밖에서 유입
- 특정 고객 매출 비중 20% 미만
- 실패 영업과 창업자 노동을 차감한 연간 공헌이익 3,000만원에서 3,600만원의 경로
- 지원과 세팅 범위가 반복 가능
- 실제 보안, 개인정보, 운영 비용을 포함한 뒤에도 양수 손익

## People 계획에 미치는 영향

현재 인재상과 온보딩 문서는 미래 채용 템플릿으로는 유효하다. 그러나 수익성 검증 전에는 채용 계획이나 예정 인원으로 해석하면 안 된다.

최소한 다음이 확인되기 전에는 채용을 실행하지 않는다.

- 반복 매출과 실제 갱신
- 창업자 노동비 차감 후 양수 공헌이익
- 역할별 업무가 외주나 자동화보다 채용이 유리하다는 근거
- 해당 인건비를 감당할 현금과 지속 기간

## 사용자 결정, 2026-07-15

1. **비용 회수 모드**: 후원 실험만 진행하고 지속 가능한 무료 사이드프로젝트로 정의한다.
2. **90일 사업 검증 모드, 권장**: 후원으로 직접비를 방어하면서 본당 패키지 실제 결제를 검증한다.
3. **전략 자산 모드**: SaaS 유료화 기대를 낮추고 무료 제품이 제품 유래 SI를 만드는지만 검증한다.
4. **유지 또는 중단 모드**: 고단가 서비스를 판매할 의지가 없다면 독립 사업 가설을 종료하고 유지 가능한 범위로 축소하거나 서비스를 정리한다.

**2번 90일 사업 검증 모드를 채택했다.** 1번, 3번, 4번은 2026-11-30 최종 판정 때 선택 가능한 다음 경로로 보존한다.
