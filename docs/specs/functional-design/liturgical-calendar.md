# 기능 설계: 전례 시기/전례력 달력

> PRD를 기반으로 구체적인 기능 흐름과 인터페이스를 정의합니다.

## 연결 문서

- PRD: `docs/specs/prd/liturgical-calendar.md`, `docs/specs/prd/liturgical-enhancement.md`
- 사업 문서: `docs/business/6_roadmap/roadmap.md` (2단계)

## 기능 범위

| 기능 | 설명 | 상태 |
|------|------|------|
| 전례 시기 계산 API | 현재 날짜 기준 전례 시기 + 다가오는 축일 반환 | 구현 완료 |
| 대시보드 전례 카드 | 전례 시기 + 다가오는 축일 표시 | 구현 완료 |
| 성주간 시기 구분 | 주님 수난 성지주일~성수요일 시기명/전례색 구분 | 구현 완료 |
| 성삼일 시기 구분 | 성목요일~성토요일을 별도 시기/전례색으로 구분 | 구현 완료 |

## 전례 시기 계산

### 전례 시기 정의

가톨릭 전례력은 5개 시기 + 성주간/성삼일로 구성됩니다.

| 시기 | 전례색 | 시작 | 종료 |
|------|--------|------|------|
| 대림 시기 | 보라 | 대림 제1주일 (11/30에 가장 가까운 주일) | 12/24 |
| 성탄 시기 | 흰 | 12/25 | 주님 세례 축일 |
| 사순 시기 | 보라 | 재의 수요일 (부활 -46일) | 성지주일 전날 (부활 -8일) |
| **성주간** | **일별 상이** | **주님 수난 성지주일 (부활 -7일)** | **성수요일 (부활 -4일)** |
| **파스카 성삼일** | **일별 상이** | **성목요일 (부활 -3일)** | **성토요일 (부활 -1일)** |
| 부활 시기 | 흰 | 부활 대축일 | 성령 강림 대축일 (부활 +49일) |
| 연중 시기 | 초록 | 성탄 시기 종료 다음 날 ~ 사순 전날 + 성령 강림 다음 날 ~ 대림 전날 |

### 성주간 + 성삼일 일별 전례색

| 날 | 시기명 | 전례색 | 비고 |
|----|--------|--------|------|
| 주님 수난 성지주일 (부활 -7일) | 주님 수난 성지주일 | 빨강 (red) | 수난 봉독 |
| 성월~성수요일 (부활 -6~4일) | 성주간 | 보라 (purple) | 사순과 동일 전례색 |
| 성목요일 (부활 -3일) | 파스카 성삼일 — 주님 만찬 성목요일 | 흰 (white) | 성체성사 제정 |
| 성금요일 (부활 -2일) | 파스카 성삼일 — 주님 수난 성금요일 | 빨강 (red) | 미사 없음, 수난 예식 |
| 성토요일 (부활 -1일) | 파스카 성삼일 — 파스카 성야 | 흰 (white) | 모든 성야의 어머니 |

### 계산 알고리즘 요약

1. `calculateEaster(year)`로 부활 대축일 결정
2. 부활 기준으로 사순/성주간/성삼일/부활/성령강림 날짜 계산
3. **성주간**: 부활 -7일(성지주일) ~ 부활 -4일(성수) → 사순에서 분리
4. **성삼일**: 부활 -3일(성목) ~ 부활 -1일(성토) → 사순에서 분리
4. 대림 제1주일: 12/25 기준 직전 4번째 주일 (11/27~12/3 범위)
5. 성탄 시기 종료: 주님 세례 축일 결정
6. 현재 날짜가 속하는 시기 판별 + 연중 시기 주차 계산

### 연중 시기 주차 계산

- 전반부: 주님 세례 축일 다음 날(월)부터 시작, 재의 수요일 전까지
- 후반부: 성령 강림 다음 주 월요일부터 시작, 대림 전까지
- 후반부는 34주(그리스도왕 대축일)부터 역산

## 토요일 전례 표시 (특전미사 반영, 로드맵 2단계)

### 규칙

| 날짜 | 표시 전례 시기 |
|------|--------------|
| 일반 토요일 | **다음 날(일요일)의 전례 시기** |
| 성토요일 | **파스카 성삼일 — 파스카 성야** (예외) |

### 적용 위치

`GetSeasonUseCase.execute()`에서 날짜 보정 적용 (순수 계산 함수 `getLiturgicalSeason`은 변경 없음):

1. 오늘이 토요일인지 확인
2. 토요일이면 내일(일요일) 날짜로 `getLiturgicalSeason()` 호출
3. **단, 성토요일이면 보정하지 않음** — 성삼일 시기 그대로 반환

---

## 사용자 플로우

1. 사용자가 대시보드 진입
2. 전례 시기 API 자동 호출 (현재 연도 기준)
3. **토요일이면 일요일 전례 시기로 보정** (성토요일 제외)
4. 전례 카드에 현재 전례 시기명, 전례색, 다가오는 축일 3개 표시

## UI/UX

### 대시보드 전례 카드

통계 카드 영역 상단에 위치합니다. 기존 레이아웃 변경 없음.

| 요소 | 설명 |
|------|------|
| `●` 컬러 도트 | 전례색 표시 (보라/흰/초록/빨강) |
| 시기명 | "대림 제2주일", "연중 제15주일", "주님 수난 성지주일", "성주간", "파스카 성삼일 — ..." 등 |
| 다가오는 축일 | 오늘 이후 가장 가까운 축일 3개 (날짜 + 이름) |

## API/인터페이스

### tRPC 프로시저

| 프로시저 | 타입 | 인증 | 설명 |
|----------|------|------|------|
| `liturgical.holydays` | query | consented | 축일 목록 (기존) |
| `liturgical.season` | query | consented | 전례 시기 + 다가오는 축일 (기존, 로직 수정) |

### liturgical.season 응답 (변경 없음)

| 필드 | 타입 | 설명 |
|------|------|------|
| `season` | string | 전례 시기명 (성삼일: "파스카 성삼일 — 주님 만찬 성목요일") |
| `color` | string | 전례색 코드 ("purple" / "white" / "green" / "red") |
| `upcomingHolydays` | Holyday[] | 다가오는 축일 3개 |

## 예외/엣지 케이스

| 상황 | 처리 방법 |
|------|----------|
| 연말 (대림 시기 시작 후) | 다가오는 축일에 다음 해 축일 포함 |
| 12/31 (성탄 시기) | 현재 연도 축일 소진 시 다음 해 축일 계산하여 3개 채움 |
| 주님 세례 축일 경계 | 한국 규칙: 주님 공현(1/2~1/8 주일) 다음 주일, 공현이 1/7~8이면 다음 날(월) |
| 성령 강림 당일 | 부활 시기 (빨강 전례색 — 해당일만 특수) |
| **성토요일** | **토요일이지만 일요일 전례가 아닌 성삼일로 표시** |
| **토요일 + 다가오는 축일** | **보정된 날짜(일요일) 기준으로 다가오는 축일 필터** |

## 성능/제약

- DB 접근 없이 순수 계산 — 응답 시간 < 10ms
- 클라이언트 캐시: `staleTime: 1일` (전례 시기는 하루 단위 변경)

## 테스트 시나리오

### 정상 케이스

1. **TC-1**: 연중 시기 중 조회 → "연중 제N주일" + 초록 반환
2. **TC-2**: 대림 시기 중 조회 → "대림 제N주일" + 보라 반환
3. **TC-3**: 부활 당일 조회 → "부활 대축일" + 흰 반환
4. **TC-4**: 다가오는 축일 3개 정확히 반환 (날짜순)
5. **TC-5**: 주님 수난 성지주일 → "주님 수난 성지주일" + 빨강
6. **TC-6**: 성화요일 → "성주간" + 보라
7. **TC-7**: 사순 마지막 날 (성지주일 전날) → 사순 + 보라
8. **TC-8**: 성목요일 → "파스카 성삼일 — 주님 만찬 성목요일" + 흰
9. **TC-9**: 성금요일 → "파스카 성삼일 — 주님 수난 성금요일" + 빨강
10. **TC-10**: 성토요일 → "파스카 성삼일 — 파스카 성야" + 흰

### 예외 케이스

2. **TC-E2**: 모든 축일이 지난 후 → 다음 해 축일로 채움
3. **TC-E3**: 일반 토요일 → 일요일 전례 시기/색 반환 (UseCase 레벨)
