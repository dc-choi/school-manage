## 리스크 4: 출석/통계 UX 복잡성으로 인한 사용 저하

- 가설: 출석 체크, 정렬, 통계 조회가 복잡하면 일상 사용이 감소한다
- 근거: 출석 UX 개선 요구 반복 (상세: `docs/business/0_feedback/feedback.md`)
- 검증: 출석 처리 시간/클릭 수, 지연/누락률
- 대응: 출석/통계 화면 단순화, 자동 저장 제공

## 리스크 13: 개신교 ChMS의 가톨릭 시장 진입

- 가설: 개신교 ChMS 업체(오직교적 9,500+ 교회, 지저스온 등)가 용어/구조만 변경하여 가톨릭 진출 가능
- **구체적 위협 (2026-03-16)**: 브랜치스(branches.team) — 개신교 청년부/청소년부 모임 운영 특화, 95교회 220명. 문제 인식과 기능 범위가 가장 유사한 직접 경쟁자. 다만 가톨릭 도메인(전례력/교구 계층/세례명) 진입 장벽 존재
- 검증: 개신교 ChMS의 가톨릭 대응 기능 출시 여부 모니터링, 브랜치스 가톨릭 확장 동향
- 대응: 가톨릭 특화 UX(의무축일, 학년 전환, 가톨릭 용어)로 전환 비용을 높이고 선점

## 리스크 14: 범용 출석 앱이 "충분히 좋은" 대안이 됨

- 가설: Attendable, 구글 시트 등 무료 범용 도구가 "그냥 이거면 되지" 수준으로 인식될 수 있다
- 근거: 현행 대체재 패턴 1위가 엑셀+카톡(market.md 경쟁 환경) — 가톨릭 특화 기능 없이도 "최소 기능"은 충족. 2030세대 요노(YONO) 소비 성향상 "있어도 그만" 기능 추가만으로는 전환 동기 부족
- 검증: 이탈자 인터뷰에서 대체재 사용 여부 확인
- 대응: 가톨릭 특화 기능(의무축일, 학년 자동 전환, 통계/보고)으로 차별화

## 리스크 17: 개인 계정 전환 후 권한 관리 복잡성

- 가설: 소규모 모임(2-5명)에서 권한 관리가 과잉 설계로 느껴져 "엑셀이 편하다" 반응 가능
- 근거: STATUS 활성 모임 99곳 중 1-3명 운영 모임 다수 (1명 운영 15곳 등). 권한 분리 요구 0건 누적 — 소규모는 단순함 선호 시그널. 다만 팀 사용 본당(사창동 14·불광동 13·우면동 14·성산2동 10 등)은 별개 트랙
- 검증: 권한 설정 완료율, 권한 관련 문의/불만 건수
- 대응: 최소 권한 모델(관리자/교사 2단계), 소규모 모임은 전원 관리자 기본값

## 리스크 26: 가톨릭 전용 포지셔닝의 피벗 비용 (2026-06-12 등록)

- 가설: "가톨릭 교리교사의 주일을 운영하는 도구"로 언어/포지셔닝을 못박은 뒤(2026-06-12 브랜드 전략) 범용/개신교 확장이 필요해지면 브랜드 재구축 비용이 발생한다
- 근거: 브랜드 전략 토론(`docs/brainstorm/2026-06-12/brand-strategy.md` Round 1)에서 식별. 가톨릭 전용은 유일한 방어막(R13/R14 대응)이라 채택했으나, 전환 시 비용은 수용한 트레이드오프
- 검증: MAO 성장 정체가 시장 크기 한계로 진단되는 시점(전국 가톨릭 침투 후)에 재평가
- 대응: 디자인 레이어는 직접 종교 심볼 회피로 확장 옵션 보존(logo.md). 범용 분기는 법인 전환 시 별도 브랜드(루멘/피앗, `2_bm/bm-funding.md`)로 처리 — 위클리스쿨 브랜드 자체를 피벗하지 않는다

## 리스크 27: 파운더 현직 종속 브랜드 보이스 (2026-06-12 등록)

- 가설: 파운더 보이스 스펙트럼 중 현직 공감 문장("저도 매주 그 자리에 있습니다" 등 2/4번)은 파운더의 교리교사 현직 유지에 묶여 있어, 역할 변화 시 외부 카피 일괄 회수가 필요하다
- 근거: 브랜드 전략 토론 Round 2 비판자 식별 — "같은 처지" 포지셔닝은 현직 동안만 진정성 보장
- 검증: 파운더 역할 변화 발생 시 트리거. 연 1회 보이스 스펙트럼 유효성 점검
- 대응: 불변 기준 문장(1번 "4년 동안 했고 직접 만들었다", 3번 "교리교사가 만들었고 교리교사가 씁니다")을 카피 기본값으로 운용, 현직 공감 문장은 보조로만 사용 (`docs/brainstorm/2026-06-12/brand-strategy-draft.md` 보이스 스펙트럼)

## 리스크 20: 행사 운영 유료 기능 WTP 미입증

- 가설: 3단계 행사 운영 기능은 파워 유저가 AI로 일회성 도구를 직접 제작하는 수준이면 SaaS 구독 가치를 인정받지 못할 수 있다
- 근거: 2026-04-19 성북동 미카엘라 — 캠프 준비 페이지를 클로드로 직접 제작, 동기화 문제로 노션 연계. "비슷한 거 만들면 쓸래요?" 질문에 "걍 일회성으로 쓰는 거라" 답변. 일회성 도구 직접 제작 가능한 사용자층 확인
- 검증: 3단계 파일럿 시 행사 운영 기능 월 사용률, 사용 본당 유료 전환율, 외부 도구(노션/AI) 병행 사용 여부. **2026-05-09 페르소나 매트릭스 추가 검증**: 알림 보편성 1/4 vs 명단 이관 3.5/4 — 베이직 시그니처 자체 재검토 트리거 (`1_market/personas.md`)
- 대응: 행사 도구를 독립 유료 애드온이 아닌 메인 서비스에 흡수하는 경로 우선 검토 (예: 사진 출석 체크 → 출석부 직접 흡수). 일회성 아웃풋의 서비스 편입 경로 설계
