# PRD: 사이트 Footer + 약관 3종 (1차 배포)

> 상태: Draft | 작성일: 2026-05-06 | 기능명: `footer-terms-v1`

> 3단계 베이직 플랜 정기결제 도입의 PG 가맹 심사 선행 조건.
> 통신판매업 신고번호는 별도 2차 갱신(`footer-terms-v2`)으로 분리.

## 배경/문제 요약

- 참고: `docs/business/4_risk/risks.md` R21, `docs/business/6_roadmap/roadmap.md` PG/SMS 결제 인프라 계약 체크리스트(4번), `docs/brainstorm/2026-04-30/pg-subscription.md`
- **문제**: 토스페이먼츠 가계약(5/6 신청 예정) → PG 심사 단계에서 사이트 검수 포함. 사이트 Footer에 사업자 정보·약관 3종(이용약관·개인정보처리방침·환불약관)이 노출되지 않으면 심사 보완 요청이 발생해 리드타임이 길어진다 (R21).
- **현재 상태**: 사이트 Footer에 사업자 정보·약관 링크 없음. 약관 3종 페이지 자체가 없음.
- **목표 상태**: PG 심사 담당자가 사이트 검수 시 Footer에서 사업자 정보를 즉시 확인 가능하고, 약관 3종 링크로 각 약관 본문에 접근 가능. PG 심사 보완 요청 없이 통과.

## 목표/성공 기준

- **목표**: PG 가계약(토스페이먼츠) 사이트 검수 보완 요청 0건으로 1차 통과.
- **성공 지표**:
    - PG 심사 사이트 검수 보완 요청 횟수 = 0 (목표)
    - 1차 배포 → 가계약 신청까지 리드타임 ≤ 7일
- **측정 기간**: 1차 배포 → PG 가계약 본계약 완료(예상 5월 말~6월 중순)까지.

## 사용자/대상

- **주요 사용자**:
    - 1차: PG 심사 담당자 (토스페이먼츠) — 사이트 검수 + 약관 검토.
    - 2차: 일반 사이트 방문자(교리교사·사제·잠재 사용자) — 사업자 신뢰성 확인 + 약관 열람.
- **사용 맥락**:
    - PG 심사: 데스크톱 브라우저로 Footer 확인 + 약관 페이지 링크 클릭 → 본문 확인.
    - 일반 사용자: 모바일/데스크톱 모두. Footer는 페이지 하단에서 항시 노출.

## 범위

### 포함

- **Footer 영역**: 모든 공개 페이지(랜딩 + 약관) 하단 공통 컴포넌트.
    - 사업자 정보: 상호·대표자명·사업자등록번호·연락처(이메일)
    - 약관 3종 링크: 이용약관·개인정보처리방침·환불약관
- **약관 3종 페이지**: 정적 페이지 3개 (URL 경로는 FD에서 확정). 각 페이지에 시행일·개정 이력 표시.
- **공개 접근**: 인증 불필요. 비로그인 상태에서 열람 가능.
- **반응형**: 모바일/데스크톱 모두 지원 (모바일 1열 스택, 데스크톱 가로 정렬).

### 제외

- **통신판매업 신고번호 표시** → 2차 갱신(`footer-terms-v2`)에서 추가. 신고 완료 시점 미정(체크리스트 5번).
- **영수증/세금계산서 발행 정책** → 별도 PRD (TARGET FUNCTIONAL 3단계 P1 "영수증/세금계산서 자동 발행").
- **결제 모듈 자체** → 별도 PRD (TARGET FUNCTIONAL 3단계 P1 "베이직 플랜 정기결제").
- **회원가입/로그인 약관 동의 체크박스 + 동의 이력 저장** → 별도 작업. 본 PRD는 약관 본문 노출까지만 포함.
- **약관 다국어** → 한글만.

## 사용자 시나리오

1. **PG 심사 담당자가 가맹 심사 단계에서 사이트 검수**: 데스크톱 브라우저로 사이트 진입 → Footer에서 상호·사업자등록번호·대표자명·연락처 확인 → 약관 3종 링크 각각 클릭 → 본문 검수 → 사이트 검수 통과.
2. **잠재 사용자(사제·교리교사)가 사이트 신뢰성 확인**: 모바일에서 사이트 진입 → 페이지 하단까지 스크롤 → Footer 사업자 정보 확인 → 신뢰성 인지.
3. **사용자가 환불 정책 확인**: 약관 페이지 링크 직접 접근 또는 Footer 링크 → 환불약관 본문 확인.

## 요구사항

### 필수 (Must)

- [ ] 모든 공개 페이지 Footer에 사업자 정보 표시: 상호·대표자명·사업자등록번호·연락처(이메일).
- [ ] Footer에 약관 3종(이용약관·개인정보처리방침·환불약관) 링크 노출.
- [ ] 약관 3종 페이지 각각 정적 본문 + 시행일 + 개정 이력 표시.
- [ ] 비로그인 상태에서 약관 페이지 접근 가능.
- [ ] 모바일/데스크톱 반응형.
- [ ] 약관 본문은 한글로 작성 (운영자 또는 외부 자문으로 초안 확정).

### 선택 (Should)

- [ ] 약관 페이지 내 목차 (긴 약관의 가독성 향상).
- [ ] Footer 디자인이 기존 디자인 시스템과 일관성을 유지 (색·여백·타이포).
- [ ] 약관 본문 전체 텍스트 검색 가능 (브라우저 기본 Ctrl+F 동작 보장).

### 제외 (Out)

- 약관 동의 체크박스 + 동의 이력 DB 저장.
- 통신판매업 신고번호 표시.
- 약관 PDF 다운로드.
- 약관 변경 시 사용자 알림.
