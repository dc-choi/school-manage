# /feedback

사용자 피드백 기록 자동화. 신규 피드백을 수집할 때마다 반복되는 3단계(entries 생성 → 인덱스 추가 → 카테고리 갱신)를 한 번에 처리한다.

## 사용법

```
/feedback <이름> <한줄요약>
```

예:
```
/feedback 성북동-미카엘라-3 데일리 통계 요청
/feedback 장현도-2 회의록 AI 비용 부담
```

## 실행 단계

### 1. 파일명 생성

- 날짜: 오늘 (KST `YYYY-MM-DD`)
- 슬러그: `<이름>` 인자를 그대로 사용 (영문 소문자/하이픈/한글)
- 동일 대상 복수 피드백은 접미사 `-1`, `-2` (사용자가 직접 인자에 포함)

결과: `docs/business/0_feedback/entries/YYYY-MM-DD-<slug>.md`

### 2. entry 파일 스캐폴딩

```markdown
# YYYY-MM-DD <이름> 피드백

> 원본: <대화 링크/출처> (작성 시 채움)

## 핵심 요약

- <한줄요약>

## 맥락

- 대상: <본당/역할/파일럿 규모>
- 수집 방법: 카톡/DM/인터뷰/이벤트
- 관련 이전 피드백: <있으면 링크>

## 원문/주요 발언

> ...

## 제품 시사점

- 기능: 
- 우선순위 후보: P1/P2/P3 (판단 근거)
- 로드맵 단계: 1/2/3/4

## 후속 액션

- [ ] 로드맵/TARGET 반영 검토
- [ ] 카테고리 갱신 (feedback-categories.md)
- [ ] 사용자 승인 후 SDD 등록
```

### 3. feedback.md 인덱스 갱신

`docs/business/0_feedback/feedback.md`의 "외부 피드백 (날짜순)" 표 **최상단 행에 삽입**:

```markdown
| YYYY-MM-DD | <이름> | <한줄요약> | [entries/YYYY-MM-DD-<slug>.md](./entries/YYYY-MM-DD-<slug>.md) |
```

### 4. feedback-categories.md 카테고리 추정 + 갱신 제안

엔트리의 핵심 요약·제품 시사점·원문에서 키워드 매칭으로 카테고리 추정. **자동 반영 금지** — 사용자 승인 후 편집.

**카테고리 목록** (`feedback-categories.md` 섹션 기준):

| 카테고리 | 키워드 힌트 |
|----------|-------------|
| 인스타그램 콘텐츠 | 4컷/만화/콘텐츠/릴스 |
| 통계/현황 | 학년별/마지막/미사/토탈/대시보드 |
| 로그인/인증 | 자동/유지/로그인/세션 |
| 출석 운영 | 출석/체크/UI/정렬/셀프 |
| 학생 정보 | 부모/생년월일/연락처/비고/세례명 |
| 데이터 관리 | 중복/일괄/엑셀/마이그레이션 |
| 일정/행사 | 캘린더/축일/회의록/행사/등록제 |
| 동의/개인정보 | 동의/개인정보/서명 |
| 후원/결제 | 후원/카카오페이/결제/PG |
| NFC 출석 | NFC/하드웨어/태그 |

**추정 규칙**:
1. 한 피드백이 **복수 카테고리에 걸칠 수 있음** (예: 회의록 자동화 = 일정/행사 + 동의/개인정보)
2. 확신 수준(높음/보통/낮음) 함께 제시
3. 매칭 실패 시 "카테고리 미상 — 사용자 판단 필요"로 표시

**추가 제안 형식** (카테고리 섹션에 한 줄):
```markdown
- <핵심 요약> — <대상>(YYYY-MM-DD). <상세 한 줄>
```

**사용자 제시 예시**:
```
[feedback] 카테고리 추정:
  - 일정/행사 (확신 높음): "교사 공유 캘린더" 키워드 매치
  - 동의/개인정보 (보통): "개인정보 제공" 언급
승인할 카테고리 선택 시 해당 섹션에 한 줄씩 추가 (자동 반영 X, Edit 제안만).
```

### 5. 사용자에게 다음 단계 안내

```
[feedback] 스캐폴드 완료:
  - docs/business/0_feedback/entries/YYYY-MM-DD-<slug>.md
  - feedback.md 인덱스 +1행
다음 단계:
  1. entry 파일에 핵심 요약/맥락/원문 채우기
  2. 카테고리 판단 후 feedback-categories.md 직접 편집
  3. 로드맵 반영 여부 결정 → /bs 또는 사용자 승인 후 /sdd 등록
```

## 주의사항

- **자동 로드맵 반영 금지**: `.claude/rules/business.md` 브레인스토밍 반영 규칙과 동일. 사용자 승인 없이 TARGET/로드맵 수정 불가
- **원본 보존**: 카톡/DM 원문은 가능하면 그대로 붙여넣기. 요약만 두면 맥락 손실
- **파일명 충돌 감지**: 같은 날짜+이름이 이미 존재하면 `-1`/`-2` 자동 제안
- **민감정보**: 본명/연락처는 편집자 재량으로 필요 시 익명화
