# Feature: 배포 롤백 전략

> 이 문서는 기능의 **단일 정보원(Single Source of Truth)** 입니다.
> 기술적 구현 방법이 아닌 **"무엇을"** 만들지에 집중합니다.

## 개요

이전 이미지 태그/디제스트로 롤백하는 절차를 문서화한다.

## 배경

- 배포 실패 시 즉시 복구할 수 있어야 한다.
- 이전 태그를 유지해야 롤백이 가능하다.

## 사용자 스토리

### US-1: 이전 버전으로 되돌린다
- **사용자**: 운영자
- **원하는 것**: 이전 태그로 쉽게 롤백하기
- **이유**: 장애를 빠르게 복구하기 위해

### US-2: 롤백 기준을 명확히 한다
- **사용자**: 운영자
- **원하는 것**: 롤백 트리거와 절차를 문서화
- **이유**: 판단을 빠르게 하기 위해

## 기능 요구사항

### 필수 (Must Have)
- [ ] 이전 이미지 태그를 보관한다.
- [ ] 롤백 절차를 단계별로 문서화한다.
- [ ] 필요 시 digest 기반 롤백을 정의한다.

### 선택 (Nice to Have)
- [ ] 롤백 리허설을 수행한다.

## 엣지 케이스

| 케이스 | 예상 동작 |
|--------|----------|
| 이전 태그 누락 | 이미지 보관 정책 강화 |
| DB 변경 동반 | 마이그레이션 롤백 포함 |

## 인수 조건 (Acceptance Criteria)

- [ ] 롤백 절차가 문서화되어 있다.
- [ ] 이전 태그로 복구 가능하다.
