# 브레인스토밍 결과 보고서

## 주제
수익화 모델 구체화

## 토론 요약

3라운드에 걸쳐 기획자(biz-planner)와 비판자(biz-critic)가 토론했다.

### Round 1
- 기획자가 3개 아이디어 제안: 교구 보고서 PDF, 팀 플랜, 전례력 콘텐츠 구독
- 비판자 핵심 지적: PDF 법적 리스크, 팀 플랜 가격 미설계, 콘텐츠 무료 대체재 존재
- 합의: 3개 모두 방향 전환 필요

### Round 2
- 기획자 개선: PDF→공유 링크, 팀 플랜→Organization 구독 흡수, 콘텐츠→액션 기능 전환
- 비판자 핵심 지적: **3개 아이디어 모두 WAU 10명 달성에 직접 기여하지 않음**
- 기획자 반박: WAU 10명은 기능이 아니라 현장 활성화가 결판. 기능의 역할은 리텐션/세션 깊이
- 합의: 카카오 비즈니스 API 제거 → 교리교사 직접 발송 방식

### Round 3
- Decision Tree 최종 평가
- 우선순위 합의: 아이디어 3 > 1 > 2
- 핵심 미합의: 축일 연락 관행 검증(아이디어 3), 사제 링크 열람 의향(아이디어 1)

## 아이디어 목록

### [아이디어 1] 사제 보고용 공유 링크
- **한 줄 요약**: 교리교사가 통계 페이지를 읽기 전용 링크로 사제에게 공유
- **사용자 가치**: 보고 준비 시간 단축, 사제가 도구를 직접 쓰지 않아도 데이터 열람
- **실현 방법**: 토큰 기반 공개 링크 생성 → 사제 접속 → 출석/통계 열람
- **차별점**: 사제용 별도 계정 없이 가톨릭 구조(교리교사→사제 보고) 디지털화
- **계정 모델 전환 의존**: 높음

### [아이디어 2] Organization 단위 구독
- **한 줄 요약**: 본당/모임 단위 프리미엄 구독 모델
- **사용자 가치**: 협업 기능(공동 출석 입력, 역할 분리), 데이터 소유권 명확화
- **실현 방법**: Organization 생성 → 관리자 결제 → 멤버 초대 → 기능 활성화
- **차별점**: 가톨릭 모임 단위 과금 (Slack/Notion 패턴의 교회 특화 버전)
- **가격 구조**: 무료(개인) / 베이직 5,000원(Organization+협업) / 프로 10,000원(알림/등록/동의)
- **계정 모델 전환 의존**: 매우 높음

### [아이디어 3] 앱 내 알림 허브 MVP
- **한 줄 요약**: 세례명 축일 학생을 자동 매칭하여 교리교사가 직접 연락할 수 있게 지원
- **사용자 가치**: 매주 앱을 열 이유 생성, 축일 학생에게 연락하는 관행 디지털화
- **실현 방법**: 축일 DB 조회 → 학생 매칭 → "이번 주 축일자" 표시 → 카카오톡 공유 SDK로 직접 발송
- **차별점**: 가톨릭 전례력 + 출석 데이터 결합 — 범용 앱 대체 불가
- **계정 모델 전환 의존**: 없음

## 실행 타임라인

| 시기 | 액션 | 전제 조건 |
|------|------|----------|
| 즉시 (2월 말) | 아이디어 3 MVP 설계 착수 | 축일 연락 관행 검증 (1문항) |
| 즉시 (2월 말) | 아이디어 2 사전 DM 의향 조사 | 없음 |
| 3-4월 | 계정 모델 전환 SDD + 구현 | 보안 이슈 해결 병행 |
| 4-5월 | 아이디어 1 MVP 착수 | 계정 전환 완료 |
| 5월 이후 | 아이디어 2 Organization 구독 설계 | 계정 전환 + DM 조사 결과 |
