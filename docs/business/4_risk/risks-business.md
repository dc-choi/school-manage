## 리스크 8: 본당 패키지 수요 부재, 관계 의존, 노동 초과 (2026-07-15 재정의)

- 가설: 교리교사가 느끼는 편의가 본당 예산 결정자의 지불 이유로 이어지지 않거나, 개인 관계 밖에서는 판매되지 않거나, 영업과 수동 납품시간이 가격 상한을 넘을 수 있다
- 근거: 본당 예산 결제는 승인과 증빙 절차가 필요하지만 저가 Basic의 허용 고객획득비용은 작다. 기존 사제 직접 영업 경로는 닫혔고 수평 추천의 반복성도 확인되지 않았다
- 검증: 2026-09-01부터 2026-11-29까지 예산 결정자 5명, 개인 관계 밖 2명 이상에게 100,000원 진단 또는 300,000원 패키지를 제안하고 실제 수납, 실패 영업시간, 납품과 재작업시간, 노동 조정 공헌이익을 기록한다
- 중단: 5명 모두 진단과 300,000원 패키지를 결제하지 않거나, 개인 관계 밖 결제가 0건이거나, 납품 상한을 초과하고 재사용 가능한 범위가 없으면 독립 사업 경로를 기각한다
- 대응: 신규 유료 기능을 만들지 않고 고정 범위 서비스를 먼저 판매한다. 600,000원 제안은 300,000원 전액 선입금 1건, 4시간 이내 납품, 해당 건 양수 공헌이익을 확인한 뒤 다른 구매자에게만 진행한다
- 장기 구독의 본당과 개인 결제 모델은 2026-11-30 최종 판정 뒤 별도 재승인 전까지 보류한다

## 리스크 9: 광고로 인한 사용자 이탈

- 가설: 광고가 UX를 해쳐 이탈률이 증가할 수 있다. 2030세대는 직관성에 민감하여 광고 이탈 리스크가 더 높음
- 근거: 미수집 (광고 도입 자체 보류 상태이므로 1차 데이터 부재). 일반 시그널: `1_market/market.md` 사용자 특성 — "직관적이지 않으면 즉시 삭제" 패턴
- 검증: 광고 전후 주간/월간 이탈률, 위치별 불만 신고 건수 — **광고 도입 결정 시 측정 가능 (현재 보류)**
- 대응: 핵심 기능(출석 체크) 흐름 외부에 광고 배치

## 리스크 10: 부적절한 광고 노출 (광고 도입 보류로 잠정 비활성)

- 가설: 종교/교육 환경에 맞지 않는 광고 노출 시 신뢰도 하락
- 근거: AdSense 일반 카테고리는 도박/주류/타종교 광고 노출 가능. 가톨릭/주일학교 맥락에서 노출 시 사제/학부모의 신뢰도 손실 직접 발생
- 검증: 필터링 적용 후 부적절 광고 신고 건수 — **광고 도입 결정 시 측정 가능 (현재 보류)**
- 대응: 광고 네트워크 카테고리 필터링 + 사용자 신고 기능

## 리스크 11: 광고 수익 불확실성 (광고 도입 보류로 잠정 비활성)

- 가설: 틈새 시장의 낮은 트래픽(주 1회 사용)으로 광고 수익이 운영 비용을 충당하지 못할 수 있다
- 근거: 주간 운영 도구 특성상 평균 세션 빈도가 주 1-2회 수준 → AdSense CPM 가정 기준 MAO 50곳의 월 광고 수익은 1-3만원으로 추정된다. 비용 기준은 `2_bm/bm.md`를 따른다. 2026-05-14 기준 출석부 직접 현금 Burn은 월 5만원이고, Claude Max 월 35만원은 회사일/외부 SI와 공유하는 운영자 현금지출이라 출석부 단독 Burn에서 제외한다. 따라서 "운영비 40만원"과 직접비를 한 분모로 섞지 않는다. **광고 수익과 신뢰 손실의 ROI가 미검증이므로 광고 도입은 보류 상태다**
- 검증: 광고 수익/출석부 직접비 비율과 광고 수익/운영자 총 현금지출 비율을 분리하고 CTR, CPM을 함께 측정한다. **광고 도입 결정 시 측정 가능 (현재 보류)**
- 대응: 광고는 직접비 일부를 보전할 수 있는 보조 수익원으로만 평가한다. 핵심 수익 모델은 미확정이며 90일 운영 지원과 본당 패키지 검증 뒤 다시 판정한다. 운영자 전체 현금흐름은 사업 유닛 이코노믹스와 분리해 관리한다

## 리스크 15: 개인정보보호법 미준수 (법적 컴플라이언스)

- 가설: 개인정보 수집/이용 동의 절차가 법적 요건에 미달하면 과태료, 시정조치, 손해배상 등 리스크가 발생할 수 있다. 구체적 적용 범위와 금액은 법률 자문 전 확정하지 않는다
- 근거: 학생 이름/성별/학년/출석 기록 수집 중. **동의 절차 구현 완료 (2단계)**
- 검증: 소급 동의 미수집, 제3자 제공 동의 미확보, 법률 자문 미완
- 대응: **법률 자문 필요** (동의 절차는 구현됨, 법적 적정성 검토 미완)
- **90일 패키지 추가 경계**: 유료 진단 단계에서는 학생 개인정보 원본을 받지 않는다. 명단 정리와 등록 지원 전에 본당의 처리 권한, 항목과 목적, 접근자, 안전한 전달, 보관과 삭제 기한을 문서화하지 못하면 원본을 받지 않는다
- **R25와의 차이 (2026-05-07 분리)**: R15는 법적 미준수에 따른 규제/과태료 리스크(공급자 책임). 보안 인식 기반 도입 거부는 별도 트랙 R25 참조 (`risks-pmf.md`)

## 리스크 21: 통신판매업/PG 외부 상태 미확인으로 결제 지연

- 가설: 업종 추가는 완료됐지만 통신판매업 신고와 PG 외부 심사 상태를 확인하지 못하면 3단계 베이직 플랜의 결제 연동이 지연된다
- 적용 범위: 이 리스크는 보류된 Basic/Pro 제품 결제에 적용한다. 2026년 9월 90일 패키지 실험의 PG 연동 선행 조건으로 사용하지 않는다. 다만 계좌 수납 전에 해당 거래의 회계, 증빙, 안내 약관, 환불과 서비스 종료 조건은 별도로 확인해야 한다
- 근거:
    - 2026-04-30 부업종 525101/525103 추가와 승인은 완료됐다
    - 과거 체크리스트는 전자상거래 업종과 통신판매업 신고번호를 PG 심사 선행 조건으로 가정했다. 실제 신청 전 선택 PG의 최신 공식 요건을 다시 확인한다
    - 과거 체크리스트는 [PG 가계약 → 에스크로 확인증 → 정부24 신고 → 본계약] 순서를 사용했다. 현재도 같은 순서가 적용되는지는 선택 PG와 공식 신고 안내에서 확인해야 하며, 과거 3-5주 예상은 만료됐다
    - 1차 Footer의 사업자 정보/약관 링크는 완료됐고, 통신판매업 신고번호는 외부 상태 확인 뒤 2차 반영이 필요하다
- 검증:
    - 업종 정정 승인까지 소요 기간 — **2026-04-30 접수/당일 승인 완료**
    - PG 가계약 → 통신판매업 신고증 수령까지 리드타임
    - 심사 보완 요청 횟수 및 반려 사유
- 대응:
    - 홈택스 525101/525103 부업종 동시 추가 **완료**
    - 선택 PG의 최신 심사 요건과 정부24 신고 선행 조건을 확인한 뒤 실제 순서를 확정한다
    - 1차 Footer/약관 링크는 완료. 신고번호 확인 후 2차 Footer 반영
    - 행정 체크리스트: `6_roadmap/roadmap.md` 3단계 의존성 섹션

## 리스크 22: 중개 플랫폼 전환 시 여행업/관광진흥법 등록 요건

- 가설: 로드맵 확장 방향인 "버스 대절/장소 연결 중개 플랫폼"은 단순 IT 플랫폼을 넘어 관광진흥법상 여행업 등록 대상으로 분류될 수 있으며, 무등록 알선으로 간주 시 과태료/사업 정지 리스크가 발생한다
- 근거:
    - 관광진흥법상 "운송 수단(버스 포함) 또는 숙박/관광지 알선"은 여행업 범위에 포함 가능
    - 단순 '정보통신업'이나 '통신판매 중개업'만으로는 운송 알선의 법적 근거가 불충분할 수 있음 (지자체 해석에 따라 차이)
    - 여행업 등록 유형과 자본금 요건은 법령/지자체 해석에 따라 달라질 수 있어 착수 시점의 공식 기준과 법률 자문으로 확인해야 한다
    - 중개 모델에서 총거래액(GMV)과 실제 매출(수수료)의 괴리로 세무 소명 리스크 병존
- 검증:
    - 중개 플랫폼 실제 착수 시점(확장 단계 진입 시) 관할 지자체 관광과 사전 질의
    - 유사 서비스(버스 대절 중개, 장소 연결)의 법적 분류 사례 리서치
    - 약관 초안에 "중개자 책임 한계" 조항 반영 여부
- 대응:
    - 사업 확장 시점까지는 "자기 서비스 직접 판매"(전자상거래 소매업) 지위를 유지하며, 중개 전환 확정 전까지 버스/장소 연결 기능 착수 금지
    - 중개 전환 확정 시 [관광진흥법 자문 → 자본금 요건 확인 → 여행업 등록 → 약관/정산 로직 재설계] 순서로 진행
    - 플랫폼 이용약관에 "중개자 지위/실제 이행 책임은 공급자(버스/장소)" 조항 명시하여 연대 책임 리스크 감쇄
    - 재무/회계: 중개 전환 시 매출 인식을 GMV에서 수수료로 변경, 세무사와 정산/세금계산서 발행 주체 설계

## 리스크 23: 임시 후원 채널의 식별 한계 + 신뢰도 저하

- 가설: 현재 사업자 명의 계좌를 직접 노출해 향후 개인 계좌 매출 누락 경로는 해소했지만, 과거 개인 계좌 기간의 장부 반영과 입금 주체 식별 불가, 정식 결제 시스템 부재 인상에 따른 리스크가 남는다
- 근거:
    - 2026-04-30에는 운영자 개인 계좌를 노출했으나 2026-05-09 사업자 명의 계좌로 교체했다
    - 개인 계좌 사용 기간의 입금은 사업자 장부 회복 처리 여부를 별도로 확인해야 한다
    - 입금자명만으로 후원자 식별 시 동명이인/약어 입력 등으로 후원 영수증 발급 또는 사용자 컨택 불가
    - 카카오페이 차단 원인 진단을 별도 트랙으로 운영하지 않기로 결정(2026-04-30). 당시 PG 일정은 만료됐고 현재 외부 상태는 미확인이므로, 공식 후원/결제 경로가 확인될 때까지 임시 조치를 유지한다
    - **2026-04-30 사업자 명의 계좌/체크카드 발급 → 현재 채널의 매출 누락 해소 경로 확보 / 2026-05-09 후원 페이지 계좌 교체 완료. 과거 개인 계좌 기간은 별도 확인 필요**
- 검증:
    - 2026-09-08부터 2026-09-21까지 30,000원 자발적 운영 지원 요청의 실제 입금, 입금자 매칭, 문의와 환불 처리시간
    - 임시 조치 적용 기간(2026-04-30 - PG 본계약 완료) 동안 입금 건수와 후원자 매칭 실패 비율
    - 공식 후원/결제 경로 도입 전후 후원 전환율과 입금자 매칭률
    - 세무사 자문 결과 (개인 계좌 후원의 사업자 매출 회복 처리 가능 여부)
    - **사업자 계좌 교체 완료일**: **2026-05-09** (사업자 명의 계좌/카드 04-30 발급 후 9일 만에 후원 페이지 교체 완료)

    - 사업자 계좌 교체 전 임시 운영 기간 입금 내역 별도 기록 (사업자 매출 회복 처리 대비)
    - 임시 안내 문구("결제 시스템 점검 중, 정식 결제 도입 예정") 추가는 별도 작업 필요 시 SDD 작성자 핸드오프 — 현재는 미적용 (2026-04-30 사용자 결정)
