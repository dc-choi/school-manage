# 리스크

## 리스크 목록

### 리스크 1: 보수적 문화로 인한 도입 저항
- 가설: 가톨릭 조직 문화 특성상 새로운 디지털 도구 도입이 느리거나 거부될 수 있다
- 근거: "가톨릭은 보수적"이라는 인식, 기존 방식(수기/엑셀) 유지 경향. 봉헌금 페이 서비스 상용화 실패 사례 (2026-01-25 피드백)
- 검증: 도입 제안→승인 전환율, 보류/거절 기록
- 대응: 소규모 파일럿과 성공사례 공유로 신뢰 기반 추천 유도

### 리스크 2: 사제 동의 지연
- 가설: 최종 의사결정자인 사제의 동의가 지연되면 파일럿 이후 전환이 늦어진다
- 근거: 도입과 결제의 최종 승인이 사제에게 있음
- 검증: 파일럿 종료 후 동의까지 리드타임, 보류/거절 코드
- 대응: 사제 보고용 요약(효과/통계) 템플릿 제공으로 동의 리드타임 단축

### 리스크 3: 첫인상 실패로 인한 영구 이탈
- 가설: 2030세대는 첫 경험에서 가치를 못 느끼면 재시도하지 않고 영구 이탈한다
- 근거: 고덕동 — 그룹 1개 생성 후 미사용 지속. ~~신당동/금호동 — 계정 모델 혼란~~ (계정 모델 전환으로 해소). 가재울/별내 등 S유형은 첫 경험 성공 시 바로 활성 전환
- 검증: 첫 세션 이탈률, 가입 후 첫 출석 기록까지 도달률 — 현재 모임→첫 출석 누적 전환율 44% (03-16, N=66)
- 대응: 온보딩 각 단계에서 즉각적 결과 제공 (행동-보상 원칙, 상세: `docs/business/1_market/market.md`)

### 리스크 4: 출석/통계 UX 복잡성으로 인한 사용 저하
- 가설: 출석 체크, 정렬, 통계 조회가 복잡하면 일상 사용이 감소한다
- 근거: 출석 UX 개선 요구 반복 (상세: `docs/business/0_feedback/feedback.md`)
- 검증: 출석 처리 시간/클릭 수, 지연/누락률
- 대응: 출석/통계 화면 단순화, 자동 저장 제공

### 리스크 5: B2B2C 구조로 인한 최종 사용자 정착 실패
- 가설: 도입 결정 주체와 실제 사용자가 분리되어 있으면 최종 사용자(교리교사) 정착이 어려울 수 있다
- 검증: 도입 경로별 4주 리텐션 및 활성도 비교
- 대응: 최종 사용자의 즉시 효용(출석/학생 관리) 강조

### 리스크 6: 학생/보호자 정보의 민감성
- → R15(개인정보보호법)로 통합. 동의서 제출 누락/지연, 권한 변경 요청 건수로 검증

### 리스크 7: 교리교사 교체로 인한 인수인계 공백
- **구조적 해소 (2026-02-20)**: 개인 계정 모델 전환으로 계정 유지, 본당 내 역할(관리자)만 이전하면 됨. 교체 후 유지율로 검증 예정

### 리스크 8: 결제 모델 전환 실패 (2026-02-24 재정의)
- 가설: 본당/모임 단위 구독이 1차 모델이나, 관리자 대표 결제 구조가 형성되지 않으면 유료 전환이 지연된다
- 근거: 본당 예산 결제는 승인/결재 절차로 리드타임이 길어질 수 있으며, 개인 프리미엄은 ARPU 불확실
- 검증: 본당 vs 개인 결제 분포, 승인 리드타임, 전환율/해지율
- 대응: 본당 구독 1차 + 개인 프리미엄 보조 (Slack/Notion 패턴)

### 리스크 9: 광고로 인한 사용자 이탈
- 가설: 광고가 UX를 해쳐 이탈률이 증가할 수 있다. 2030세대는 직관성에 민감하여 광고 이탈 리스크가 더 높음
- 검증: 광고 전후 주간/월간 이탈률, 위치별 불만 신고 건수
- 대응: 핵심 기능(출석 체크) 흐름 외부에 광고 배치

### 리스크 10: 부적절한 광고 노출
- 가설: 종교/교육 환경에 맞지 않는 광고 노출 시 신뢰도 하락
- 검증: 필터링 적용 후 부적절 광고 신고 건수
- 대응: 광고 네트워크 카테고리 필터링 + 사용자 신고 기능

### 리스크 11: 광고 수익 불확실성
- 가설: 틈새 시장의 낮은 트래픽(주 1회 사용)으로 광고 수익이 운영 비용을 충당하지 못할 수 있다
- 검증: 월간 광고 수익/운영 비용 비율, CTR, CPM
- 대응: 광고는 보조 수익원, 핵심은 유료 구독. 목표 미달 시 유료 전환 촉진

### 리스크 12: 가치 체감 부족으로 인한 빠른 이탈
- 가설: 진입 장벽을 낮춰도 핵심 가치가 체감되지 않으면 빠르게 이탈한다
- 근거: 장위동 초등부 안착 실패. 피드백 이벤트 응답 2건 — "행동 비용 대비 보상 불확실하면 아예 행동하지 않음" 패턴 실증 (상세: `docs/business/1_market/market.md` 행동-보상 원칙)
- 검증: 초기 핵심 기능 사용률과 2주 이탈률 상관
- 대응: 최소 입력 플로우 + 입력 즉시 결과 제공 ("행동 비용은 낮게, 보상 확실성은 높게")

### 리스크 13: 개신교 ChMS의 가톨릭 시장 진입
- 가설: 개신교 ChMS 업체(오직교적 9,500+ 교회, 지저스온 등)가 용어/구조만 변경하여 가톨릭 진출 가능
- **구체적 위협 (2026-03-16)**: 브랜치스(branches.team) — 개신교 청년부/청소년부 모임 운영 특화, 95교회 220명. 문제 인식과 기능 범위가 가장 유사한 직접 경쟁자. 다만 가톨릭 도메인(전례력/교구 계층/세례명) 진입 장벽 존재
- 검증: 개신교 ChMS의 가톨릭 대응 기능 출시 여부 모니터링, 브랜치스 가톨릭 확장 동향
- 대응: 가톨릭 특화 UX(의무축일, 학년 전환, 가톨릭 용어)로 전환 비용을 높이고 선점

### 리스크 14: 범용 출석 앱이 "충분히 좋은" 대안이 됨
- 가설: Attendable, 구글 시트 등 무료 범용 도구가 "그냥 이거면 되지" 수준으로 인식될 수 있다
- 검증: 이탈자 인터뷰에서 대체재 사용 여부 확인
- 대응: 가톨릭 특화 기능(의무축일, 학년 자동 전환, 통계/보고)으로 차별화

### 리스크 15: 개인정보보호법 미준수
- 가설: 개인정보 수집·이용 동의 절차 미흡 시 법적 리스크(최대 5천만원 과태료)
- 근거: 학생 이름·성별·학년·출석 기록 수집 중. **동의 절차 구현 완료 (2단계)**
- 검증: 소급 동의 미수집, 제3자 제공 동의 미확보, 법률 자문 미완
- 대응: **법률 자문 필요** (동의 절차는 구현됨, 법적 적정성 검토 미완)

### 리스크 16: 계정 모델 전환 시 기존 데이터 마이그레이션
- **구조적 해소 (2026-03)**: 계정 모델 전환 완료. 기존 계정 소유자 → 관리자 자동 전환, 신규 계정은 개인 계정 + 합류 구조로 운영 중. 장위동 4년 데이터 정상 유지 확인

### 리스크 17: 개인 계정 전환 후 권한 관리 복잡성
- 가설: 소규모 모임(2~5명)에서 권한 관리가 과잉 설계로 느껴져 "엑셀이 편하다" 반응 가능
- 검증: 권한 설정 완료율, 권한 관련 문의/불만 건수
- 대응: 최소 권한 모델(관리자/교사 2단계), 소규모 모임은 전원 관리자 기본값

### 리스크 18: 생업 이슈로 인한 도구 전환 시간 부족
- 가설: 교리교사는 봉사직이므로 도구 전환에 투자할 시간이 제한적이며, 이것이 D유형 이탈의 근본 원인일 수 있다
- 근거: 03-13 팔로업 인터뷰 결과 — 본업(생업)이 바빠 출석부 전환을 미루는 패턴 실증. S유형(즉시 안착)은 기존에 출석 관리를 적극적으로 하던 교리교사에 한정
- 검증: D유형 이탈 사유 중 "바빠서/시간 없어서" 비율, S유형 vs 나머지 사전 관리 방식 비교
- 대응: 최소 입력 온보딩 + 컨텍스트 배너로 행동 비용 최소화. 시간 투자 대비 즉각적 보상 강조

### 리스크 19: Silent Churn으로 인한 문제 파악 지연
- 가설: 2030세대는 불만이 있어도 피드백 없이 조용히 이탈하여, 원인 파악이 지연된다
- 근거: 피드백 이벤트 응답 2건에 그침. 고덕동은 피드백 없이 미사용 전환
- 검증: 피드백 수집률, 이탈 후 원인 파악까지 소요 시간
- 대응 (구현 완료): GA4 행동 데이터 기반 이탈 지점 추적(26개 이벤트) + 이탈 감지 자동 알림(14일 미활동, 운영자 이메일) + 피드백 수집 시 확실한 보상 설계

## 가정

### 가정 1
- 가설: 중고등부 교리교사가 1차 도입 주체가 될 가능성이 높다
- 근거: 중고등부가 1차 타겟이며 업무 복잡도가 높은 편
- 검증: 중고등부 계정 비중과 활성 사용률 측정

### 가정 2
- 가설: 교리교사 네트워크 추천이 초기 확산에 유효하다
- 근거: 주일학교 내 도구 선택이 신뢰 기반 추천에 영향
- 검증: 추천 기반 유입 비중/전환율 측정

### 가정 3 (실증 완료)
- 가설: 사제는 도입 자체에 우호적이며, 교리교사의 파일럿 결과를 기반으로 최종 결정한다
- 반론 (2026-01-25): 사제/사무실은 현행 방식에 불편 없을 가능성. "우호적"은 "반대하지 않음"과 다름
- **실증 (2026-03-14)**: 협력 신부님이 사제 네트워크에 적극 홍보 → 2일 +57 계정 획득. 방화3동 신부 직접 가입. "소극적 허용"이 아닌 **"적극적 추천"** 실증. 사제 네트워크가 최고 레버리지 GTM 채널로 작동
- 잔여 검증: 추천 사제 외 독립적 사제도 우호적인지, 유료 전환 시 사제 결재 동의율
