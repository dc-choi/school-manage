## 구조 해소 리스크 (참고용)

### 리스크 7: 교리교사 교체로 인한 인수인계 공백

- **구조적 해소 (2026-02-20)**: 개인 계정 모델 전환으로 계정 유지, 본당 내 역할(관리자)만 이전하면 됨. 교체 후 유지율로 검증 예정

### 리스크 16: 계정 모델 전환 시 기존 데이터 마이그레이션

- **구조적 해소 (2026-03)**: 계정 모델 전환 완료. 기존 계정 소유자 → 관리자 자동 전환, 신규 계정은 개인 계정 + 합류 구조로 운영 중. 장위동 4년 데이터 정상 유지 확인

## 가정

### 가정 1

- 가설: 중고등부 교리교사가 1차 도입 주체가 될 가능성이 높다
- 근거: 중고등부가 1차 타겟이며 업무 복잡도가 높은 편
- 검증: 중고등부 계정 비중과 활성 사용률 측정

### 가정 2

- 가설: 교리교사 네트워크 추천이 초기 확산에 유효하다
- 근거: 주일학교 내 도구 선택이 신뢰 기반 추천에 영향
- 검증: 추천 기반 유입 비중/전환율 측정

### 가정 3 (실증 완료)

- 가설: 사제는 도입 자체에 우호적이며, 교리교사의 파일럿 결과를 기반으로 최종 결정한다
- 반론 (2026-01-25): 사제/사무실은 현행 방식에 불편 없을 가능성. "우호적"은 "반대하지 않음"과 다름
- **실증 (2026-03-14)**: 협력 신부님이 사제 네트워크에 적극 홍보 → 2일 +57 계정 획득. 방화3동 신부 직접 가입. "소극적 허용"이 아닌 **"적극적 추천"** 실증. 사제 네트워크가 최고 레버리지 GTM 채널로 작동
- 잔여 검증: 추천 사제 외 독립적 사제도 우호적인지, 유료 전환 시 사제 결재 동의율

### 가정 4 (2026-04-16 검토 완료, 보류)

- 가설: 앱인토스(토스 미니앱) 입점은 현 시점 제품 조건에 부합하지 않아 배포 채널로 채택하지 않는다
- 근거:
    - **토스 로그인 강제**: 앱인토스는 토스 로그인만 허용. 기존 253개 계정의 본당/모임 승인 구조를 토스 계정으로 이전 불가 (사용자 확인)
    - **외부 링크/다중 출시 제한**: 미니앱은 기능 완결적이어야 하며 메인 웹앱과 핵심 기능이 같은 미니앱 동시 출시 금지. 부분 기능만 분리하는 방안은 검토 대상 아님
    - **만 19세 이상 정책**: 학생/학부모/일반 신자(C) 확장 시 차단. B2B 교리교사 타겟 유지 전제에서는 제약 미체감이나 확장 경로 막힘
    - **리소스 우선순위**: 2단계 MAO 50 목표 진행 중, 비게임 통합 개발 2~3개월 투입 대비 이득 낮음
    - **타겟 풀 미스매치**: 토스는 일반 C 사용자 채널, 우리 핵심 사용자는 가톨릭 교리교사. 사제 네트워크(GTM 채널)가 더 높은 레버리지 (가정 3 실증)
- 검증 (재검토 트리거): (a) 계정 모델이 토스 로그인과 양립 가능한 구조로 변경, (b) C 사용자 대상 부가 제품(축일/미사 등) 분리 결정, (c) 토스 정책 완화(외부 로그인 연동/B2B 카테고리 허용), (d) 2단계 완료 후 리소스 여유 확보
- 대응: 앱인토스 외 배포 채널(PWA 설치 유도, 사제 네트워크 GTM) 우선 유지. 본 결정은 재검토 트리거 발생 시 해제
