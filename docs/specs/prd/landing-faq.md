# PRD: 랜딩 페이지 FAQ

> SDD 작성자가 작성하는 제품 요구사항 문서입니다.
> 사업 에이전트의 `docs/business/` 문서를 기반으로 작성합니다.

## 배경/문제 요약

- 참고: `docs/business/3_gtm/gtm.md` (CS 섹션), `docs/business/STATUS.md` (전환 퍼널 분석)
- 문제: 랜딩 페이지에서 서비스 가치는 전달하지만, 잠재 사용자의 **구체적 궁금증**(무료 여부, 개인정보, 공유 계정 등)을 해소할 수단이 없음. 가입 전 망설이는 사용자가 답을 찾지 못하고 이탈하거나, 운영자에게 직접 문의 발생
- 현재 상태: FAQ 없음. CS는 구글폼/인스타 DM/구두로 대응 (운영자 1인)
- 목표 상태: 랜딩 페이지 하단에 FAQ 아코디언을 추가하여 반복 문의를 줄이고 전환율을 개선

## 목표/성공 기준

- **목표**: 가입 전 사용자의 주요 의문을 랜딩 페이지 내에서 해소
- **성공 지표**:
  - CS 반복 문의 감소 (정성 측정)
  - 랜딩 → 가입 전환율 유지 또는 개선 (GA4 `landing_cta_click` 대비 `sign_up` 비율)
- **측정 기간**: 출시 후 4주

## 사용자/대상

- **주요 사용자**: 랜딩 페이지를 통해 서비스를 처음 접하는 교리교사/모임 운영자
- **사용 맥락**: 인스타그램/카톡 링크 → `/landing` 방문 → 가입 여부 판단 중

## 범위

### 포함

- `/landing` 페이지에 FAQ 아코디언 섹션 추가 (CTA 섹션 아래)
- FAQ 항목 5~7개 (정적 콘텐츠)
- GA4 이벤트 추적 (FAQ 섹션 뷰, 개별 항목 열기)

### 제외

- 관리자 FAQ 편집 기능 (CMS)
- FAQ 전용 별도 페이지
- 백엔드 API 추가

## 사용자 시나리오

1. **시나리오 1 — 가입 망설임 해소**: 교리교사가 인스타 링크로 랜딩 페이지에 진입. 기능 소개와 데모를 보고 관심이 생겼지만 "무료인가?", "개인정보는 안전한가?" 궁금. CTA 아래 FAQ를 발견하고 해당 항목을 열어 확인 → 안심하고 가입
2. **시나리오 2 — 공유 계정 이해**: 방문자가 "선생님이 여럿인데 각각 가입해야 하나?"를 FAQ에서 확인 → 모임당 1계정 공유 방식 이해 → 가입

## FAQ 항목 초안

| # | 질문 | 답변 요약 |
|---|------|----------|
| 1 | 무료인가요? | 네, 모든 기능 무료. 앞으로도 핵심 기능은 무료 유지 |
| 2 | 개인정보는 안전한가요? | 개인정보 처리방침 준수, 최소 정보만 수집, 제3자 제공 없음 |
| 3 | 모임당 계정 하나인가요? | 네, 모임 하나당 1계정. 여러 선생님이 같은 계정 공유 |
| 4 | 다른 기기에서도 쓸 수 있나요? | 웹 기반이라 스마트폰/태블릿/PC 어디서든 사용 가능 |
| 5 | 가톨릭 주일학교만 쓸 수 있나요? | 주일학교에 최적화되어 있지만, 모임 출석 관리가 필요한 곳이면 사용 가능 |
| 6 | 기존 데이터를 옮길 수 있나요? | 현재 수동 입력만 지원. 대량 등록 기능은 준비 중 |

> **Note**: 최종 문구는 기능 설계 단계에서 확정
