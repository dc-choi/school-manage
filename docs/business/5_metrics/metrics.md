# 지표

## 측정 원칙

- 이 문서는 지표의 정의, 산식, 측정 조건만 관리한다. 현재 수치는 `STATUS.md`에서만 관리한다.
- 모든 수치에는 기준일, 집계 기간, 데이터 소스, 분석 단위, 분모를 함께 기록한다.
- 전환율은 같은 분석 단위와 같은 코호트로만 계산한다. 사용자 이벤트와 단체 이벤트를 직접 나누지 않는다.
- 운영 스냅샷과 GA4 스냅샷의 기준일이 다르면 결합 비율을 계산하지 않는다.
- GA4 일별 활성 사용자 합계는 중복 사용자를 포함하므로 MAU 또는 7일 고유 사용자로 부르지 않는다.

## North Star

### 공통 정의

- `WVO` (주간 가치활성 단체): KST 월요일 00:00부터 주일까지 한 주 동안 단체에서 핵심 조회 또는 출석 기록 행동이 1회 이상 발생한 경우
- `정착 WVO`: 최근 완료된 4개 주 모두 WVO였던 단체
- `MAO` (월간 출석입력 단체): 최근 30일 동안 `Attendance.createdAt` 기준으로 출석 기록을 1회 이상 입력한 단체
- `MAU` (월간 활성 사용자): GA4 rolling 30일 고유 사용자 `totalUsers`
- WVO와 MAO는 단체 단위, MAU는 사용자 단위다. 서로 다른 단위의 비율은 같은 기준일의 보조 진단으로만 사용한다.

### 핵심 가치 행동

핵심 가치는 출석 입력뿐 아니라 명단, 출결, 축일 정보를 빠르게 확인하는 행동을 포함한다.

| 구분 | 포함 행동 | 측정 조건 |
| --- | --- | --- |
| 조회 | 학생 목록, 출석 현황, 통계 화면 조회 | `organization_id`에 귀속 가능한 페이지 또는 이벤트 |
| 가톨릭 맥락 조회 | `liturgical_card_viewed`, `patron_feast_card_viewed` | `organization_id` 귀속 필요 |
| 운영 입력 | `attendance_recorded` | 서버 출석 기록과 대조 |

> 현재 조회 이벤트는 사용자 수로만 집계되는 구간이 있다. `organization_id` 귀속과 내부 계정 제외가 검증되기 전까지 WVO는 `미측정`으로 표시한다.

### NSM 1, 핵심 가치 정착

- 정의: **정착 WVO 단체 수**
- 보조 비율: 정착 WVO ÷ 최근 4주 중 한 번 이상 WVO였던 단체
- 목적: 출석 입력 여부만이 아니라 매주 필요한 정보를 확인하는 제품의 핵심 약속을 측정한다.

### NSM 2, 유료 가치 정착

- 정의: 월간 유료 단체 중 유료 기능을 1종 이상 사용한 단체의 비율
- 산식: 유료 기능 사용 단체 ÷ 결제 상태가 정상인 유료 단체
- 기능별 사용률도 함께 기록해 가격 저항과 기능 부적합을 분리한다.
- 상태: 장기 SaaS 지표다. 2026-09-01부터 2026-11-29까지의 운영 지원과 본당 패키지 실험에는 적용하지 않는다.

### 리포트 주기

- 운영 스냅샷: 주 1회, 동일 요일과 동일 컷오프
- GA4: 완료된 7일과 rolling 30일을 분리
- 재무: 월말 마감 후 월 1회
- 현재값은 `STATUS.md`에 스냅샷 기준일과 함께 기록

## 활성화 퍼널

### 기준 퍼널

단체 활성화 전환율은 같은 `organization_id` 코호트로 계산한다.

| 단계 | 정의 | 제한 시간 |
| --- | --- | --- |
| 단체 생성 | `organization_created` | 코호트 시작 |
| 첫 그룹 | 같은 단체의 첫 그룹 생성 | 생성 후 7일 |
| 첫 학생 | 같은 단체의 첫 학생 등록 | 생성 후 7일 |
| 첫 출석 | 같은 단체의 첫 출석 기록 | 생성 후 7일 |

- 현재 `sign_up`은 사용자 이벤트이고 `first_group_created`는 단체의 최초 그룹 이벤트다.
- `organization_created` 계측과 단체 식별자 연결이 검증될 때까지 `sign_up → first_group`을 전환율로 사용하지 않는다.
- `first_*` 주간 이벤트 수는 퍼널 재발화 여부를 보는 진단값일 뿐 전환율의 분자나 분모가 아니다.
- 사용자 가입 퍼널이 필요하면 동일 `user_id`의 GA4 sequence funnel을 별도로 만든다.

### 출석 입력 지연

MAO는 입력 시점 기준이므로 실제 출석일과 다를 수 있다.

| 지표 | 정의 | 해석 |
| --- | --- | --- |
| 출석 입력 지연 일수 | KST 날짜(`Attendance.createdAt`) - `Attendance.date` | 1일 이내는 근접 입력, 7일 초과는 사후 입력 후보 |
| 근접 입력률 | 지연 1일 이하 출석 비율 | 높을수록 현장 입력에 가까움 |
| 사후 입력률 | 지연 7일 초과 출석 비율 | 50% 이상이면 행동 분류 재검토 |
| 입력 세션 집중도 | 한 입력 세션에서 저장한 출석 건수 | 일괄 입력과 휴면을 구분하는 보조값 |

단체별 과거 판정은 `4_risk/risks-pmf.md`의 R24와 `STATUS.md`의 최신 검증 상태를 참조한다. 지연 입력만으로 운영 중단을 판정하지 않는다.

| 실제 요청 인원 | 운영 지원 메시지를 실제 전달한 활성 개인 수 | 요청 인원과 도달 실패를 분리 기록 |
| 자발적 운영 지원 현금 | 환불을 반영한 실제 입금액 | 25명 또는 750,000원 이상이면 비용 회수 신호 |
| 자발적 운영 지원 상단 목표 | 실제 입금 인원과 환불 차감 후 현금 | 30명 또는 900,000원 |
| 첫 요청 반응 | 처음 요청한 활성 사용자 20명 중 실제 입금 인원 | 5명 미만이면 운영 지원 실험 중단 |
| 운영 지원 관리시간 | 요청, 문의, 입금자 매칭, 환불에 사용한 시간 | 현금과 함께 기록, 과도하면 비용 회수 실패 |
| 활동 영향 | 요청 전후 같은 단위의 활성 사용자와 사용 단체 변화, 반감 피드백 | 유의한 하락 또는 신뢰 훼손 시 중단 |
| 예산 결정자 제안 | 가격과 범위를 전달하고 결제 여부를 회신받은 실제 결정자 | 5명, 이 중 개인 관계 밖 2명 이상 |
| 유료 신호 | 100,000원 진단 또는 300,000원 패키지 실제 입금 | 합계 2건 이상 |
| 학기 패키지 결제 | 300,000원 패키지 전액 선입금 | 1건 이상 |
| 개인 관계 밖 결제 | 기존 지인, 직장, 외주 관계가 아닌 구매자의 실제 입금 | 1건 이상 |
| 연간 관리형 서비스 신호 | 300,000원 전액 선입금, 4시간 이내 납품, 해당 건 양수 공헌이익 확인 뒤 별도 구매자에게 받은 600,000원 이상 실제 선납 | 초기 신호, 갱신 전에는 반복 사업으로 보지 않음 |
| 재작업시간 | 계약 범위 오류나 납품 수정에 사용한 시간 | 노동 조정 공헌이익에 포함 |
| 노동 조정 공헌이익 | 실제 수납액 - 환불액 - 직접비 - 내부 시간단가 × 전체 투입시간 | 양수 |

- 전체 투입시간에는 성공과 실패 상담, 견적, 후속 연락, 납품, 재작업을 모두 포함한다.
- 100,000원 진단은 문제 진단 수요, 300,000원 결제는 일회성 제품화 서비스, 600,000원 이상 선납은 관리형 서비스의 초기 신호다.
- 사람 개입이 거의 없는 반복 소프트웨어 결제와 실제 갱신이 없으면 SaaS 신호로 기록하지 않는다.

### 독립 사업 재평가 게이트

90일의 1건에서 2건 결제는 상품 신호일 뿐 독립 사업 증거가 아니다. 다음을 모두 확인한 뒤 독립 사업 가능성을 다시 평가한다.

- 유료 본당 30곳에서 50곳 또는 복수 교구 계약
- 실제 연간 갱신
- 고객 다수가 기존 개인 관계 밖에서 유입
- 특정 고객 매출 비중 20% 미만
- 실패 영업과 운영자 노동을 차감한 연간 공헌이익 30,000,000원에서 36,000,000원의 경로
- 지원과 세팅 범위의 반복 가능성, 보안과 개인정보 비용 반영 후 양수 손익
