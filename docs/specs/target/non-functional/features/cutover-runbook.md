# Feature: 주말 일괄 전환(컷오버) 런북

> 이 문서는 기능의 **단일 정보원(Single Source of Truth)** 입니다.
> 기술적 구현 방법이 아닌 **"무엇을"** 만들지에 집중합니다.

## 개요

주말 일괄 전환(big-bang cutover) 실행 절차와 롤백 기준을 문서화한다.

## 배경

- 모노레포/아키텍처 전환은 한 번에 전환할 계획이다.
- 배포 순서와 롤백 기준이 명확해야 운영 리스크를 줄일 수 있다.

## 사용자 스토리

### US-1: 컷오버를 안전하게 수행한다
- **사용자**: 운영자
- **원하는 것**: 사전 체크 → 배포 → 검증 → 롤백 절차가 명확한 런북
- **이유**: 장애 발생 시 빠르게 복구하기 위해

### US-2: 전환 범위를 통제한다
- **사용자**: 개발자
- **원하는 것**: 아키텍처/기능/타입/배포 단계가 순서대로 정리된 계획
- **이유**: 전환 실패를 최소화하기 위해

## 기능 요구사항

### 필수 (Must Have)
- [ ] 컷오버 실행 전 체크리스트가 존재해야 한다.
- [ ] 배포 순서(백엔드 → 프론트 → 스모크 테스트)가 명시되어야 한다.
- [ ] 롤백 기준과 절차가 문서화되어야 한다.
- [ ] 사전 준비물(이미지 태그, dist, DB 변경)을 명시해야 한다.

### 선택 (Nice to Have)
- [ ] 배포 시간/책임자/연락 체계 기록

## 엣지 케이스

| 케이스 | 예상 동작 |
|--------|----------|
| DB 마이그레이션 실패 | 즉시 롤백 절차 실행 |
| 부분 배포 실패 | 동일 버전으로 재정렬 후 재시도 |
| 스모크 테스트 실패 | 이전 버전으로 롤백 |

## 인수 조건 (Acceptance Criteria)

- [ ] 사전/사후 체크리스트가 존재한다.
- [ ] 배포/검증/롤백 절차가 단계별로 정리되어 있다.
- [ ] 필요한 아티팩트(이미지 태그, dist)가 명시되어 있다.
