# 사용자 피드백

> 문제 배경/핵심 문제는 `docs/business/README.md`를 참고하세요.

## 통계/현황
- 전체 학생 수, 남/여 학생 수, 학년별 학생 수
- 학생 마지막 출석일자
- 오늘 미사 출석 수

## 출석 운영
- 출석 체크 간편화
- 출석부 자동 저장 (체크/페이지 전환 즉시 저장)
- 정렬 기준 (가나다/우수 출석)
- 의무축일/행사 출석 구분
- 출석부 UI 전면 개편 요구 (다수)

> 출석 관련 피드백의 대부분은 장위동 초등부 파일럿에서 수집됨
> (초등부 전체 의견 일반화 아님, 초기 가치 체감이 이탈을 좌우한다는 레퍼런스)

## 데이터 관리
- 학생 일괄 추가/삭제

## 일정/행사
- 이번달/다음달 축일자
- 축일 관리
- 등록 관리 (등록제)

## 동의/개인정보
- 개인정보 제공 동의 관리

## NFC 출석
- NFC 기반 출석 (재미/편의성)
