## 자금 조달 전략 (2026-03-17 확정)

### 운영 비용

> **출석부 전용 한계비용**과 **공유 도구**를 분리한다. Claude Max는 회사일 + 외부 SI 일감 + 출석부가 공유하는 개발 도구로, 출석부를 중단해도 지속 지출되므로 출석부 단독 Burn에 귀속시키지 않는다 (2026-05-14 확인).

| 항목          | 월 비용 | 분류        | 비고                                                       |
| ------------- | ------- | ----------- | ---------------------------------------------------------- |
| 서버 + 도메인 | 5만원   | 출석부 전용 | 출석부 중단 시 사라지는 한계비용                           |
| Claude Max    | 35만원  | 공유 도구   | 회사일 + 외부 SI 일감 + 출석부 공유. 출석부 단독 Burn 제외 |

- **최근 확인 현금 Burn Rate = 2026-05-14 기준 월 5만원** (서버 + 도메인). 현재 월 원장은 `STATUS.md` 오픈 이슈에 따라 다시 마감하며, 아직 발생하지 않았거나 계약 전인 상용화 비용은 제외한다
- **정상화 고정비 = 최근 확인 현금 Burn + 등록면허세 월할액 + 계약상 PG 고정료**. PG 거래 수수료/SMS/카톡/AI 비용은 사용량 기반 변동비로 별도 계산하며, 계약상 PG 고정료가 0원이면 0원으로 확정한다
- Claude Max는 출석부와 무관하게 지속 지출되므로 현재 현금 Burn과 정상화 고정비 모두에서 제외한다
- 사업용 별도 자금 풀 없음 — 운영비는 운영자 직장 소득에서 충당 (2026-05-14 확인). 단기 자금원은 외부 SI 일감 (NAS 백업 / ERP 외주 / 장위동 홈페이지)
- 기존 외부 SI는 제품 매출이 아니며 제품이 만든 유료 리드의 증거로 사용하지 않는다

### 자발적 운영 지원, 기존 문서명 도네이션(가설/근거/검증)

- 가설: "운영비 후원" 포지셔닝이 가톨릭 봉사 문화와 부합하여 도입 가능하다
- 근거: 가톨릭 교회 문화의 후원/봉헌 습관과 별개의 자발적 지원 가능성. 실제 법적, 회계상 기부금 여부는 미확정
- 분류 주의: 문서의 `도네이션`, `기부`, `후원`은 사용자 메시지 가설이다. 실제 회계와 세무상 분류, 증빙 의무는 요청 전에 확인하고 확정된 분류로 안내한다
- 메시지: "이 서비스는 교리교사가 만든 봉사 프로젝트입니다. 후원금은 서버 운영비로 사용됩니다"
- **90일 실험**: 2026-09-08부터 2026-09-21까지 1인 30,000원 단일 금액으로 요청하고, 25명 또는 750,000원 이상 실제 입금을 비용 회수 신호로 본다. 30명 또는 900,000원은 상단 목표다
- **중단 기준**: 처음 요청한 활성 개인 20명 중 실제 입금이 5명 미만이면 확대 요청을 중단한다
- **대가 경계**: 후원자 전용 기능, 서비스 기간, 가동률을 약속하지 않는다. 이 실험은 사업성이나 다음 해 갱신을 증명하지 않는다
- 현재 채널: 앱 내 후원 페이지에 사업자 명의 계좌를 직접 노출. 카카오페이/Toss 정식 후원 링크는 현재 구현으로 간주하지 않는다
