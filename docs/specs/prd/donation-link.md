# PRD: 도네이션 링크 + 앱 내 후원 안내

> SDD 작성자가 작성하는 제품 요구사항 문서입니다.
> 사업 에이전트의 `docs/business/` 문서를 기반으로 작성합니다.

## 배경/문제 요약

- 참고: `docs/business/2_bm/bm.md` (자금원 구조, 도네이션 가설)
- 문제: 서버 운영비(Lightsail 월 ~$10)를 개인이 전액 부담 중이며, 후원 경로가 인증 사용자에게만 제한되어 있음
- 현재 상태: 후원 UI가 `/settings` 내부에 있어 인증 + 조직 가입 필수. 비인증 방문자는 후원 불가
- 목표 상태: 비인증 사용자(게스트)도 후원 페이지에 접근 가능. 로그인 없이 카카오페이 송금 가능

## 목표/성공 기준

- **목표**: 후원 접근 장벽 제거 — 로그인 없이도 후원 가능하도록 개선
- **성공 지표**: 후원 링크 클릭 수 (location별), 게스트 후원 클릭 비율
- **측정 기간**: 출시 후 4주

## 사용자/대상

- **주요 사용자**: 로그인한 교리교사 (ADMIN/TEACHER) + 비인증 방문자 (게스트)
- **사용 맥락**: 인증 사용자는 설정/사이드바에서, 게스트는 독립 후원 페이지에서 후원

## 범위

### 포함

- 앱 내 후원 안내 UI (사이드바 푸터 + 설정 페이지 섹션) — 기존 유지
- 비인증 접근 가능한 독립 후원 페이지 (`/donate`)
- 카카오페이 송금 링크 외부 연결
- 후원 링크 클릭 GA4 이벤트 트래킹 (게스트 location 추가)

### 제외

- PG 결제 연동 (사업자 등록 불필요한 범위로 제한)
- 후원 금액 앱 내 표시/관리
- 후원자 목록/보상 기능
- 랜딩 페이지 내 후원 안내 (별도 페이지로 분리)

## 사용자 시나리오

1. **인증 사용자 — 사이드바에서 후원**: 사이드바 "후원하기" 클릭 → `/settings#donation` 이동 → 카카오페이 클릭
2. **인증 사용자 — 설정에서 후원**: 설정 페이지 후원 섹션 → 카카오페이 클릭
3. **게스트 — 직접 URL 접근**: `/donate` URL 접근 → 후원 안내 확인 → 카카오페이 클릭
4. **게스트 — 랜딩에서 후원**: 랜딩 페이지 등에서 후원 링크 클릭 → `/donate` 이동 → 카카오페이 클릭

## 요구사항

### 필수 (Must)

- [x] 설정 페이지에 후원 안내 섹션 (기존 완료)
- [x] 사이드바/모바일 메뉴 후원 링크 (기존 완료)
- [ ] `/donate` 공개 라우트 — 인증 불필요, 독립 페이지
- [ ] 후원 메시지 + 카카오페이 버튼 (기존 DonationSection 재사용)
- [ ] GA4 이벤트: `donation_link_click` (location: `donate` 추가)

### 선택 (Should)

- [x] 모바일 메뉴 후원 링크 (기존 완료)

### 제외 (Out)

- 앱 내 결제/송금 처리
- 후원 금액 추적 대시보드
- 후원자 리워드/뱃지

## 제약/가정/리스크

- **제약**: 카카오페이 링크는 하드코딩 상수로 관리 (현행 유지)
- **가정**: 공유 링크 (`/donate`)를 통한 외부 유입이 후원 전환에 기여할 것
- **리스크**: 공개 페이지이므로 외부 노출 — 보안 위험 없음 (외부 링크만 연결)

## 의존성

- **내부**: 기존 DonationSection 컴포넌트, donation.ts 유틸
- **외부**: 카카오페이 송금 링크 (이미 생성 완료)

## 롤아웃/검증

- **출시 단계**: 즉시 전체 배포
- **이벤트/로그**: `donation_link_click` (location: `sidebar` | `settings` | `donate`)
- **검증 방법**: GA4에서 location별 클릭 수 비교. 게스트 대 인증 비율 확인
