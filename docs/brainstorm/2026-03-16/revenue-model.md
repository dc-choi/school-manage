# 브레인스토밍 결과 보고서

## 주제
현재 사업 계획 기반 수익 모델 — 기존 3단 구조(무료+광고+유료 구독)를 보완할 수익화 아이디어 도출 및 실행 계획 수립

## 토론 요약

### Round 1
- 기획자가 3개 아이디어 제안: 사제 보고서 유료 기능, 교리교사 단체 라이선스, 학기 시작 시즌 과금
- 비판자 핵심 지적: 세 아이디어 모두 검증 없이 시스템을 먼저 설계. 사제가 직접 사용하지 않는 구조에서 "가치 체감" 약함. 결제 주체 검증 미완 상태에서 B2B 설계는 순서 뒤집힘. ARPU 하한선 미달
- 합의: 수동 파일럿(카카오폼+계좌이체)으로 지불 의사 먼저 실측. 사제 보고서 → 사제 뷰(대시보드)로 피벗

### Round 2
- 기획자 개선: 사제 뷰를 "학기말 보고서 링크"로 축소, 파일럿 대상을 S확정으로 교체, A/B 가격을 시간 분리로 설계
- 비판자 핵심 지적: 사제 "추천"과 "사용"은 다른 행동. 공유 링크에 개인정보 노출 시 법률 리스크. S 후보에게 결제 제안 시 이탈 트리거. A/B 동시 제시는 앵커링 측정
- 합의: 실행 순서(아이디어2→3→1), S확정 장위동 제외, 시간 분리 A/B 설계

### Round 3
- 기획자 최종안: 아이디어1을 "통계 화면 로컬 내보내기"로 축소, WAU 30곳 우선 → 수익화 준비 병렬, 피벗 옵션 3개 사전 정의
- 비판자 최종 지적: 플랜 플래그 공통 선행 조건 미명시, 2단계 미착수 항목과 우선순위 경쟁, S확정 표본 분산 문제
- 최종 합의: WAU 30곳이 수익화보다 우선. 준비는 28곳부터 병렬. 표본 3+3+버퍼3+장위동분리

## 아이디어 목록

### [아이디어 1] 통계 화면 내보내기
- 한 줄 요약: 기존 통계 화면에 PDF/이미지 내보내기 버튼 추가. 사제 결재 근거 자료 제공
- 사용자 가치: 교리교사의 보고 준비 수작업 제거, 사제 결재 리드타임 단축
- 실현 방법: 기존 통계 화면 위에 내보내기 버튼 1개 추가. 데이터가 기기 밖으로 나가지 않아 개인정보 리스크 낮음
- 차별점: 엑셀은 통계 자동 집계+내보내기 불가. 가톨릭 교구 보고 맥락 포함

### [아이디어 2] S유형 지불 의사 수동 실험
- 한 줄 요약: S확정 3곳에 카카오폼+계좌이체로 연 60,000원 지불 의사 측정
- 사용자 가치: 가치 체감 완료된 사용자의 유료 전환 가능성 검증
- 실현 방법: 운영자 직접 연락 → A/B/C 분류(개인결제/사제결재/거부) → 계좌이체 수납 → DB 플래그 수동 변경
- 차별점: 가톨릭 주일학교 운영 도구 유료 전환 첫 시도. 결제 주체 오픈 이슈 동시 해소

### [아이디어 3] 시간 분리 가격 실험 + 상시 전환
- 한 줄 요약: 아이디어 2 성공 후, 학기 프로모션(60K) + 상시 정가(90K) 구조로 전환 확대
- 사용자 가치: YONO 소비 성향 대응. 학기 사이클에 맞춘 결제 타이밍으로 전환 저항 최소화
- 실현 방법: 시간 분리(1~2주 60K / 3~4주 90K), 상시 90K 정가 병행, 수동 처리 최대 5곳
- 차별점: 주일학교 학기제 반영 과금 구조. 개신교 ChMS에 없는 도메인 특화 전략

## 승인 조건

### 아이디어 1 (조건부 승인)
1. 파일럿 인터뷰에서 "사제에게 통계 공유 필요성" 1건 이상 확인 후 개발 착수

### 아이디어 2 (승인)
1. 플랜 플래그 제어 로직 코드 존재 여부 확인 (미존재 시 필드 추가 선행)
2. 제안 메시지 초안 → 1곳 선발송 어조 검증
3. 0/3 실패 시 피벗 옵션 사전 정의 완료

### 아이디어 3 (조건부 승인)
1. 아이디어 2에서 1곳 이상 전환 의사 확인 후 착수
2. 실험 목적: "가격 탄력성 측정"이 아닌 "전환 가능성 확인"
3. 수동 처리 한계: 동시 5곳 초과 금지

## 실행 계획

### 0단계: 선행 확인 (즉시, 개발 시간 0)

| 항목 | 내용 |
|------|------|
| 플랜 플래그 코드 확인 | 계정/조직 모델에 플랜 구분 필드 존재 여부 (30분) |
| 보고서 니즈 인터뷰 1건 | "사제 보고 시 어떤 형태로 자료 전달하나" 확인 |
| 장위동 분리 결정 | 수익화 실험 대상에서 장위동 명시적 제외 |

### 1단계: WAU 30곳 달성 (4월, 개발 우선)

| 우선순위 | 항목 | WAU 기여 |
|---------|------|---------|
| 1 | IDOR 구조적 해소 | 보안 리스크 해소 → 확장 가능 |
| 2 | 컨텍스트 배너 | 학생→출석 병목(73%) 공략 |
| 3 | 이탈 감지 알림 | G유형 조기 대응 |
| 4 | 통계 화면 내보내기 (아이디어 1) | 사제 결재 경로 준비 |

### 2단계: 수익화 준비 (WAU 28곳부터 병렬)

| 항목 | 내용 |
|------|------|
| 플랜 플래그 필드 추가 (미존재 시) | 최소 개발 1~2일 |
| 제안 메시지 초안 | 별내/창5동/성현동 중 1곳 선발 |
| 피벗 옵션 정의 | A: 성장 집중 / B: 광고 검증 / C: 결제 주체 변경 |

### 3단계: 수익화 실험 (WAU 30곳 달성 후)

**아이디어 2**: S확정 3곳 지불 의사 실험 (2주)
- 대상: 도입 6개월 이하 S확정 (장위동 제외)
- 판정: 1곳+ 전환 의사 → 아이디어 3 착수 / 0곳 → 피벗

**아이디어 3**: 가격 실험 (아이디어 2 성공 후)
- 대상: 아이디어 2 미참여 S확정 최대 5곳
- 목적: 첫 번째 결제 성공 여부

### S확정 표본 배분

| 구분 | 본당 | 용도 |
|------|------|------|
| 아이디어 2 | 별내/창5동/성현동 | 지불 의사 실험 |
| 아이디어 3 | 정릉4동/성북동/구룡 | 가격 실험 |
| 버퍼 | 가재울/야탑동/성산2동초 | 2차 투입 또는 아이디어 1 연계 |
| 분리 | 장위동 | 장기 사용자 전환 저항 별도 분석 |

### 0/3 실패 시 피벗 옵션

| 옵션 | 내용 |
|------|------|
| A | 유료화를 WAU 50곳 이후로 연기. 성장에만 집중 |
| B | 유료 전환 대신 광고 수익 모델 우선 검증 (AdSense 이미 적용) |
| C | 결제 주체를 교리교사 개인 → 사제/본당 단위로 전환 |

## 사용자 결정 대기

1. **실행 계획 전체 승인 여부**: WAU 30곳 우선 → 수익화 병렬 준비 → 달성 후 실험 순서
2. **아이디어 1**: 인터뷰 확인 후 4월 개발에 포함할지, 보류할지
3. **0/3 피벗 선호**: 실패 시 A/B/C 중 우선 검토 방향
4. **가격 기준점**: 연 60,000원(월 5,000원)으로 시작할지, 다른 금액을 검토할지
