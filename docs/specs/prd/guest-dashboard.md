# PRD: 게스트 대시보드

> 비인증 상태에서도 `/`에 대시보드 레이아웃을 노출하여 AdSense 크롤러가 콘텐츠를 인식하도록 합니다.

## 배경/문제 요약

- 참고: `docs/business/STATUS.md` (AdSense 하단 배치 — 오픈 이슈, 높음)
- 문제: `/` 접근 시 `ProtectedRoute`가 `/login`으로 리다이렉트 → AdSense 크롤러가 로그인 폼만 인식 → 콘텐츠 부족으로 승인 어려움
- 현재 상태: 비인증 유저 → `/` → `/login` 리다이렉트 (콘텐츠 없음)
- 목표 상태: 비인증 유저 → `/` → 대시보드 레이아웃 노출 + 데이터 영역에 "로그인이 필요합니다" 표시

## 목표/성공 기준

- **목표**: AdSense 크롤러가 `/`에서 충분한 콘텐츠를 인식하도록 대시보드 레이아웃을 공개
- **성공 지표**: AdSense 승인 통과
- **측정 기간**: 배포 후 2주 이내

## 사용자/대상

- **주요 사용자**: AdSense 크롤러, 비인증 방문자 (잠재 사용자)
- **사용 맥락**: 서비스 URL을 처음 방문했을 때 (직접 접속, 검색 엔진, 공유 링크)

## 범위

### 포함

- `/` 라우트의 인증 가드 조건부 해제
- 비인증 시 대시보드 레이아웃(필터, 카드, 테이블 구조) 노출
- 데이터가 출력되는 영역에 "로그인이 필요합니다" 안내 표시
- 로그인/회원가입 CTA 제공

### 제외

- 백엔드 API 변경 (공개 통계 API 등)
- 샘플/더미 데이터 표시
- `/landing` 페이지 변경
- 기존 인증 사용자의 대시보드 동작 변경

## 사용자 시나리오

1. **비인증 방문자**: `/` 접속 → 대시보드 레이아웃(필터, 통계 테이블, 차트 영역)이 보이되, 데이터 영역에는 "로그인이 필요합니다" + 로그인/회원가입 버튼 표시
2. **AdSense 크롤러**: `/` 접속 → HTML에 대시보드 구조(헤딩, 섹션, 텍스트)가 포함됨 → 콘텐츠 충분으로 판단
3. **인증된 사용자**: `/` 접속 → 기존과 동일한 대시보드 (변경 없음)

## 요구사항

### 필수 (Must)

- [ ] 비인증 시 `/`에서 `/login`으로 리다이렉트하지 않음
- [ ] 대시보드 레이아웃(헤더, 필터 UI, 테이블/차트 영역 구조)이 비인증 상태에서도 렌더링됨
- [ ] 데이터 출력 영역(통계 테이블, 차트, 랭킹)에 "로그인이 필요합니다" 안내 표시
- [ ] 로그인/회원가입으로 이동할 수 있는 CTA 버튼 제공
- [ ] 인증된 사용자의 기존 대시보드 동작에 영향 없음

### 선택 (Should)

- [ ] 전례 시기 카드는 공개 데이터이므로 비인증 시에도 표시 (축일 카드는 제외 — 조직 데이터 의존)
- [ ] GA4 이벤트: 게스트 대시보드 진입 추적

### 제외 (Out)

- 공개 통계 API 신규 개발
- 비인증 상태에서 실제 데이터 노출
- 랜딩 페이지와 대시보드 통합

## 제약/가정/리스크

- **제약**: 프론트엔드만 변경 (백엔드 API 추가 없음)
- **가정**: AdSense 크롤러는 JS 렌더링된 콘텐츠를 인식함 (Google 크롤러는 SPA 렌더링 지원)
- **리스크**: 대시보드 레이아웃만으로 AdSense가 "충분한 콘텐츠"로 판단하지 않을 수 있음 → 전례 시기 카드 공개 노출로 완화

## 의존성

- **내부**: 없음 (프론트엔드 UI 변경만)
- **외부**: Google AdSense 승인 기준

## 롤아웃/검증

- **출시 단계**: 즉시 배포
- **이벤트/로그**: GA4 `guest_dashboard_viewed` 이벤트
- **검증 방법**: 배포 후 AdSense 재신청 → 승인 여부 확인

## 오픈 이슈

- (해결됨) 전례 시기 카드: 공개 표시 / 축일 카드: 로그인 필요 (조직 데이터 의존)
