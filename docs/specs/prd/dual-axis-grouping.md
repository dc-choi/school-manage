# PRD: 학년/부서 두 축 그룹핑

> SDD 작성자가 작성하는 제품 요구사항 문서입니다.
> 사업 에이전트의 `docs/business/` 문서를 기반으로 작성합니다.

## 배경/문제 요약

- 참고: `docs/business/6_roadmap/roadmap.md` 2단계, `docs/specs/README.md` TARGET
- 문제: 현재 학생은 하나의 "학년" 그룹에만 소속 가능. 주일학교에서는 학년(1학년, 2학년, 3학년)과 부서(전례부, 봉사부, 성가대 등) 두 축으로 학생을 관리하는데, 부서 관리가 불가능하여 별도 수기 관리 필요.
- 현재 상태:
  - StudentGroup N:M 테이블은 계정 모델 전환에서 이미 생성됨
  - 그러나 Group에 타입 구분이 없어 모든 그룹이 "학년"으로만 취급됨
  - Student.groupId (레거시 단일 그룹 FK)가 아직 코드에서 사용 중
  - Group.accountId가 저장되지만 실제 미사용 (organizationId로 대체됨)
- 목표 상태: Group에 타입(학년/부서)을 도입하고, 레거시 필드를 정리하여 N:M 관계를 완전히 활용

## 목표/성공 기준

- **목표**: 학생을 학년 그룹과 부서 그룹에 동시 소속시키고, 각 축으로 조회/관리 가능하게 함
- **성공 지표**: 부서 그룹 생성 + 학생 배정이 가능하고, 출석/통계에서 부서별 조회 가능
- **측정 기간**: 배포 후 2주

## 사용자/대상

- **주요 사용자**: 교리교사 (출석 기록 및 학생 관리 담당)
- **사용 맥락**: 학기 초 반편성 시 학년 + 부서 배정, 주일 출석 시 부서별 활동 확인

## 범위

### 포함

- Group 타입 구분 (학년 GRADE / 부서 DEPARTMENT)
- 그룹 생성/수정 시 타입 선택
- 학생 생성/수정 시 학년 + 부서 다중 선택
- 학생 목록에서 학년별/부서별 필터링
- 출석 현황에서 부서별 조회
- 레거시 필드 정리 (Student.groupId 제거)
- StudentGroup 기반으로 전체 API 전환
- 그룹 상세에서 학생 추가/제거 (GRADE 그룹 자동 이동 포함)
- 그룹 타입 변경 허용 (수정 시 GRADE↔DEPARTMENT 변경 가능)

### 제외

- 부서별 역할/직책 관리 (부서장, 부원 등)
- 부서별 일정/활동 관리
- 반편성 자동화 (별도 P2 태스크)

## 사용자 시나리오

1. **학년 + 부서 그룹 생성**: 교리교사가 학년 관리 화면에서 "1학년" (학년 타입), "전례부" (부서 타입) 그룹을 각각 생성한다.
2. **학생 다중 그룹 배정**: 학생 생성/수정 시 학년 0~1개 + 부서 0~N개를 선택한다. (예: "2학년" + "전례부" + "성가대"). 학년 없이 등록도 가능.
3. **부서별 학생 조회**: 학생 목록에서 부서 필터를 선택하면 해당 부서 소속 학생만 표시된다.
4. **출석 현황 그룹별 조회**: 출석 현황에서 통합 드롭다운(학년 + 부서)으로 그룹을 선택하여 조회한다.
5. **엑셀 Import 학년 매칭 실패**: 학년명이 매칭되지 않으면 "학년 정보 없이 등록됩니다" 알림 → 사용자 확인 후 진행.
6. **그룹 상세에서 학생 이동**: 교리교사가 "중1" 그룹 상세에서 학생을 추가하면, 해당 학생이 "중2" 소속이었을 경우 자동으로 "중2"에서 제거되고 "중1"에 추가된다.

## 요구사항

### 필수 (Must)

- [ ] Group 모델에 type 필드 추가 (GRADE / DEPARTMENT)
- [ ] 기존 그룹 데이터를 GRADE로 마이그레이션
- [ ] 그룹 생성 시 타입 선택 UI
- [ ] 학생 생성/수정 시 학년(GRADE) 0~1개 + 부서(DEPARTMENT) 0~N개 선택
- [ ] Student.groupId 컬럼 제거 → StudentGroup 기반으로 전환
- [ ] 학생 목록에서 학년/부서 필터링
- [ ] 스냅샷 생성 시 StudentGroup 기반으로 그룹 정보 저장
- [ ] 그룹 상세에서 학생 추가/제거 기능
- [ ] GRADE 그룹 추가 시 기존 GRADE 자동 이동
- [ ] 그룹 타입 변경 허용 (수정 시)

### 선택 (Should)

- [ ] 출석 현황에서 부서별 탭 조회
- [ ] 학년 관리와 부서 관리 화면 분리 (탭 또는 필터)
- [ ] 엑셀 Import 시 부서 컬럼 지원

### 제외 (Out)

- 부서 내 역할/직책
- 부서별 활동 기록
- 자동 반편성

## 제약/가정/리스크

- **제약**: Student.groupId 제거 시 DB 마이그레이션 필요. 프로덕션 데이터 보존 필수. Group.accountId는 조직 미매칭 계정이 있어 유지.
- **가정**: 학년 그룹은 대부분 사용하지만 필수는 아님. 부서 그룹은 선택적으로 사용한다.
- **리스크**: Student.groupId 제거 시 스냅샷 생성 로직, 졸업 처리 등 6개 이상 UseCase 수정 필요. 영향 범위가 넓음.

## 의존성

- **내부**: 계정 모델 전환 완료 (StudentGroup N:M, organizationId 스코핑 — 완료됨)
- **외부**: 없음

## 롤아웃/검증

- **출시 단계**: 단일 배포 (마이그레이션 + 코드 변경 동시)
