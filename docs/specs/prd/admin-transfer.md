# PRD: 관리자 양도

> SDD 작성자가 작성하는 제품 요구사항 문서입니다.
> 사업 에이전트의 `docs/business/` 문서를 기반으로 작성합니다.

## 배경/문제 요약

- 참고: `docs/business/STATUS.md` (수색성당 계정 모델 이슈, 박지안 피드백)
- 문제: ADMIN은 역할을 변경하거나 양도할 수 없어, 탈퇴도 불가능하고 관리 책임을 넘길 수도 없다. 유일한 멤버인 ADMIN은 조직에 영구 잠금된다
- 현재 상태: 조직 생성자 → 자동 ADMIN, 합류 승인자 → 자동 TEACHER. 역할 변경 API 없음. ADMIN 계정 삭제 차단
- 목표 상태: ADMIN이 같은 조직의 TEACHER에게 관리자 권한을 양도할 수 있다. 유일한 멤버인 ADMIN은 조직을 떠날 수 있다

## 목표/성공 기준

- **목표**: ADMIN이 관리자 역할을 양도하거나, 유일 멤버인 경우 조직을 떠날 수 있도록 하여 영구 잠금 해소
- **성공 지표**: 양도 후 역할 교환 정상 동작, 유일 멤버 탈퇴 시 조직 소프트 삭제 정상 처리
- **측정 기간**: 배포 후 2주

## 사용자/대상

- **주요 사용자**: 조직의 ADMIN (교리교사 중 관리 담당자)
- **사용 맥락**: 관리 책임 이전(이임/퇴임), 잘못된 관리자 지정 수정, 탈퇴 전 권한 이전, 혼자 남은 조직 정리

## 범위

### 포함

- ADMIN → TEACHER 역할 교환 (양도, TEACHER 존재 시)
- 유일 멤버 ADMIN의 탈퇴 허용 (조직 소프트 삭제)
- 양도 후 기존 ADMIN은 TEACHER로 전환
- 양도/탈퇴 후 기존 ADMIN이 원하면 계정 삭제 가능

### 제외

- 다중 ADMIN (조직당 ADMIN은 1명 유지)
- TEACHER의 자발적 승격 요청
- 조직 간 관리자 이전

## 사용자 시나리오

1. **시나리오 1 — 관리 책임 이전**: 교리교사 A(ADMIN)가 이임하면서 교리교사 B(TEACHER)에게 관리자를 양도한다. A는 TEACHER가 되고, B가 새 ADMIN으로 합류 승인/거절 등을 관리한다.
2. **시나리오 2 — 탈퇴 전 양도**: ADMIN이 조직을 떠나려 하지만 "관리자 계정은 삭제할 수 없습니다" 에러. 먼저 TEACHER에게 양도 후 탈퇴한다.
3. **시나리오 3 — 잘못된 관리자 수정**: 수색성당처럼 처음 가입한 사람이 ADMIN이 됐지만 실제 관리자는 다른 사람. 양도로 수정한다.
4. **시나리오 4 — 유일 멤버 탈퇴**: ADMIN 혼자인 조직에서 떠나고 싶다. 양도 대상이 없으므로 조직을 소프트 삭제하고 탈퇴한다.

## 요구사항

### 필수 (Must)

- [ ] ADMIN이 같은 조직의 TEACHER를 선택해 관리자를 양도할 수 있다
- [ ] 양도 시 ADMIN→TEACHER, TEACHER→ADMIN으로 역할이 동시 교환된다
- [ ] 양도는 트랜잭션으로 원자적으로 처리된다
- [ ] 양도 전 확인 다이얼로그 표시 (되돌릴 수 없음 안내)
- [ ] 유일 멤버인 ADMIN이 계정 삭제 시 조직을 소프트 삭제하고 탈퇴할 수 있다
- [ ] 유일 멤버 탈퇴 시 확인 다이얼로그 표시 (조직 데이터 삭제 안내)

### 선택 (Should)

- [ ] 양도 완료 후 UI가 즉시 갱신되어 변경된 역할이 반영된다
- [ ] GA4 이벤트 기록 (admin_transferred, organization_dissolved)

### 제외 (Out)

- 다중 ADMIN 지원
- 양도 요청/승인 플로우 (즉시 양도)
- TEACHER→ADMIN 자발적 승격

## 제약/가정/리스크

- **제약**: 조직당 ADMIN은 반드시 1명 (불변 규칙)
- **가정**: TEACHER가 존재하면 양도, TEACHER 0명(유일 멤버)이면 조직 소프트 삭제 후 탈퇴
- **리스크**: 실수로 양도 시 되돌리려면 새 ADMIN에게 재양도 요청 필요 (확인 다이얼로그로 완화). 유일 멤버 탈퇴 시 조직 데이터 복구 불가 (소프트 삭제지만 재진입 경로 없음)

## 의존성

- **내부**: 계정 모델 전환 완료 (Organization + Account role 구조), 멤버 목록 API, 계정 삭제 UseCase
- **외부**: 없음

## 롤아웃/검증

- **출시 단계**: 전체 배포 (기능 단순, 단계 불필요)
- **이벤트/로그**: `admin_transferred` (양도자 ID, 수신자 ID), `organization_dissolved` (조직 ID)
- **검증 방법**: 양도 후 역할 교환 확인, 유일 멤버 탈퇴 후 조직 소프트 삭제 확인
