# 브레인스토밍 결과 보고서

## 주제
성령강림대축일 이벤트 인터랙티브 정적 웹페이지 — CPBC "성령칠은 뽑기" 레퍼런스 기반, 모노레포 apps/ 내 이벤트 페이지 제작

## 토론 요약

### Round 1
- **기획자**: 3개 아이디어 제안 — 은사 성격 진단(MBTI식), 수호성인 찾기(DB 매칭), 기도 생성기(템플릿 조합)
- **비판자**: 바이럴→CTA 전환 근거 부족. 신학적 검수 필수. 오픈 이슈 대비 우선순위 정당화 필요
- **합의**: 이벤트 목적은 브랜드 인지(전환 아님). 신학적 검수 필수. KPI는 공유 횟수+유입 수

### Round 2
- **기획자**: 기도 생성기 기각. 은사 진단→뽑기 전환. 수호성인 DB 10명 MVP. 기도 카드 3일 신규 제안
- **비판자**: CPBC와 차별화 부족. 수호성인은 앱 내 기능이 자연스러움. 기도 카드 바이럴성 최저. 은사 뽑기 하나에 집중 권고
- **합의**: 은사 뽑기 단일 집중. 옵션 A(출석부 CTA 연결). 시즌 한정 수명. AdSense 미배치

### Round 3
- **기획자**: 서브패스 배포. 선택 입력 개인화. 2단 KPI. 카카오 비대면 검수
- **비판자**: 잔여 약점 4개(개인화, KPI, 검수 일정, 모바일 QA) 제시 → 기획자 전부 수용
- **최종 합의**: 조건부 승인 → 3개 조건 충족으로 최종 승인

### CPBC 차별화 토론 (추가 라운드)
- **배경**: CPBC는 서울대교구 공식 채널. 동일 포맷은 카피캣 인식 리스크
- **전략 1 (모임 은사 분배)**: 보류 — 교리교사 수업 활용 가설 미검증, 렌더링 공수 과소평가
- **전략 2 (은사 + 성찰 질문)**: 조건부 승인 — 묵상-성찰-실천 가톨릭 교육 문법 활용. 은사별 성찰 질문 1개씩 (7개). 신부님 신학 검수 필수
- **전략 3 (CTA + 워터마크)**: 승인 — 결과 카드 하단 서비스명+URL 소형 회색 텍스트. CTA는 이미지 밖 배치 ("주일학교 출석부 — 교리교사용")

## 아이디어 목록

### [승인] 성령칠은 은사 뽑기
- 한 줄 요약: 성령강림대축일 시즌 은사 뽑기 이벤트 → 바이럴 → 출석부 브랜드 인지
- 사용자 가치: 가톨릭 특화 인터랙티브 콘텐츠. 개인화 공유(이름/세례명 선택 입력)
- 실현 방법: apps/ 하위 별도 정적 사이트 → Cloudflare Pages 배포. CSS keyframe 카드 뒤집기. 1주 공수
- CPBC 차별점: 은사 결과 + 성찰 질문 제공 (묵상-성찰-실천). 최소 워터마크 + 외부 CTA

### [기각] 수호성인 찾기
- 기각 이유: 가톨릭 하상 등 기존 서비스와 차별화 불가. 성령강림 시즌 연관 약함

### [기각] 기도 생성기
- 기각 이유: 전통 기도문 대비 수요 미검증. 바이럴성 최저. AI 없이 템플릿 조합 한계

### [기각] 기도 카드 공유
- 기각 이유: 바이럴성 낮음. iOS Safari 이미지 저장 호환 문제

## 사용자 확정 사항

| 항목 | 결정 |
|------|------|
| 배포 방식 | **Cloudflare Pages 별도 배포** (apps/ 하위 독립 정적 사이트) |
| AdSense/도네이션 선처리 | **병렬 처리** (이벤트와 독립적으로 진행) |
| 이름/세례명 입력 시점 | **카드 뒤집기 전** (개인화 극대화) |
| CPBC 차별화 전략 | 전략 2(성찰 질문) + 전략 3(최소 워터마크 + 외부 CTA) |

## 실행 계획

### 선행 조건
| 항목 | 비고 |
|------|------|
| 협력 신부님 검수 경로 확보 | 카카오 비대면. 성찰 질문 7개 검수 |

### 개발 일정 (1주)
| 항목 | 공수 |
|------|------|
| 뽑기 페이지 + CSS 카드 애니메이션 | 2일 |
| 은사 결과 + 성찰 질문 텍스트 검수 (신부님 협력) | 1일 |
| URL/카카오 공유 버튼 | 1일 |
| CTA 배치 + Cloudflare Pages 배포 설정 | 0.5일 |
| QA (안드로이드 실기 포함) | 0.5일 |

### 기술 사양
| 항목 | 확정 내용 |
|------|---------|
| 배포 | apps/ 별도 정적 사이트 → Cloudflare Pages |
| 애니메이션 | CSS keyframe 카드 뒤집기 |
| 모바일 최적화 | will-change, backface-visibility, transform-gpu |
| 개인화 | 이름/세례명 선택 입력 (카드 전), sessionStorage, 서버 저장 없음 |
| 결과 카드 | 은사명 + 설명 + 성찰 질문 + 소형 워터마크 |
| CTA | 이미지 밖 텍스트 링크 ("주일학교 출석부 — 교리교사용") |
| 공유 | URL 복사 + 카카오 공유 + 인스타 스토리 공유 (9:16 비율 결과 이미지) |
| 추적 | GA4 (별도 설정) |
| AdSense | 이벤트 페이지에 미배치 |

### KPI
| 기준 | 조건 | 결정 |
|------|------|------|
| 방문 100명 미만 | — | CTA 경로 재검토 |
| 방문 100명+, 공유 100회+ | — | 시즌2 제작 (대림절 등) |
| 방문 100명+, 공유 50회 미만 | — | 콘텐츠 개선 또는 폐기 |

### 이벤트 수명
- 성령강림대축일(6/7) 전후 2주 운영
- 종료 후 학습(공유 패턴, 유입 경로)을 대림절/사순절 이벤트에 적용

## 기각된 접근법
| 접근법 | 기각 이유 |
|--------|---------|
| 서브패스 배포 | 사용자 결정: 별도 배포가 기민함. 이벤트와 본 앱 독립 운영 |
| MBTI식 은사 유형 진단 | "은사는 유형이 아닌 소명" 신학적 반론 리스크 |
| 모임 단위 뽑기 (옵션 B) | 기존 가입자 전제 기능. 브랜드 인지 목적과 불일치 |
| 세 이벤트 동시 제작 | KPI 분산, 1인 개발 공수 초과, 학습 불가 |
| 모임 은사 분배 (차별화 전략 1) | 교리교사 수업 활용 가설 미검증, 렌더링 공수 과소평가 |

## 누락된 관점
- 대림절/사순절 시즌 이벤트 패턴: 이번 학습을 바탕으로 연간 이벤트 캘린더 수립
- 교리교사 수업 활용: 모임 은사 분배 기능 — KPI 결과 보고 후 시즌2에서 재검토
