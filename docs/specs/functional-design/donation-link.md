# 기능 설계: 도네이션 링크 + 앱 내 후원 안내

> PRD를 기반으로 구체적인 기능 흐름과 인터페이스를 정의합니다.

## 연결 문서

- PRD: `docs/specs/prd/donation-link.md`

## 흐름/상태

### 사용자 플로우 (공통 — 인증/비인증 동일)

1. 사이드바/모바일 메뉴 "후원하기" 클릭 또는 `/donate` URL 직접 접근
2. 후원 안내 메시지 + 카카오페이 버튼 확인
3. 카카오페이 버튼 클릭 → 새 탭에서 카카오페이 송금 페이지 열림
4. GA4 이벤트 전송 (`donation_link_click`, location: `donate`)

### 상태 전이

없음 — 외부 링크 연결만 수행하며, 앱 상태 변경 없음

## UI/UX

### 화면/컴포넌트

| 화면 | 설명 | 주요 요소 |
|------|------|----------|
| Sidebar 푸터 | 저작권 아래 후원 링크 | "후원하기" 텍스트 링크 → `/donate` 이동 |
| 모바일 메뉴 하단 | 네비게이션 아래 후원 링크 | "후원하기" 텍스트 링크 → `/donate` 이동 |
| **DonatePage** (`/donate`) | 독립 후원 페이지 (공개) | 후원 안내 + 카카오페이 버튼 + 돌아가기 버튼 |

> SettingsPage 내 DonationSection은 **제거** — `/donate` 페이지로 통일

### DonatePage 레이아웃 (2단계 개선)

```
┌─────────────────────────────────┐
│        주일학교 출석부             │
│                                 │
│ ♥ 후원하기                       │
│                                 │
│ 이 서비스는 교리교사가 만든        │
│ 봉사 프로젝트입니다.              │
│ 후원금은 서버 운영비로 사용됩니다.   │
│                                 │
│ [카카오페이로 후원하기]            │
│                                 │
│       ← 돌아가기                 │
└─────────────────────────────────┘
```

- MainLayout 없이 독립 페이지 (사이드바/네비 불필요)
- 하단에 돌아가기 버튼 (`navigate(-1)` — 브라우저 뒤로가기)
- 중앙 정렬, 최대 너비 제한

## 테스트 시나리오

### 정상 케이스

1. **TC-1**: `/donate` 접근 → 후원 안내 + 카카오페이 버튼 표시
2. **TC-2**: 카카오페이 버튼 클릭 → 새 탭에서 카카오페이 송금 페이지 열림 + GA4 이벤트
3. **TC-3**: 사이드바 "후원하기" 클릭 → `/donate` 이동
4. **TC-4**: 모바일 메뉴 "후원하기" 클릭 → 메뉴 닫힘 + `/donate` 이동
5. **TC-5**: `/donate` 하단 돌아가기 버튼 클릭 → 이전 페이지로 이동

### 예외 케이스

1. **TC-E1**: 카카오페이 URL 미설정 + `/donate` 접근 → `/` 리다이렉트
2. **TC-E2**: 카카오페이 URL 미설정 → 사이드바/모바일 메뉴 후원 링크 미노출
