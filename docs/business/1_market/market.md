# 시장

> 문제 배경/핵심 문제는 `docs/business/README.md`를 참고하세요.

## 비전

### 서비스 비전
- **가톨릭 교회 내 모든 모임을 위한 그룹 운영 플랫폼**
- 출석·멤버·행사·통계를 일관된 데이터로 관리
- 단순 데이터 아카이브 → 그룹 운영 지원 플랫폼

### 현재 목표 (PMF 검증)
- 1차 검증: 주일학교 중고등부 교리교사 대상
- 검증 후 가톨릭 교회 내 다른 모임(레지오, 성가대, 청년회 등)으로 확장

## 타겟 사용자
### 가설
- **1차 사용자 (현재 검증)**: 서울대교구 233개 본당 주일학교 교리교사(중고등부 우선)
- **2차 사용자**: 초등부 교리교사, 주일학교 외 모임 운영자(레지오, 성가대, 청년회 등)
- **3차 사용자**: 전국 교구 가톨릭 모임 운영자
- 초기 지역 범위: 서울대교구 233개 본당 중심으로 시작
- 협력 채널: 지구/교구 청소년국 사목부서(중고등부/학교사목부 중심), 이후 본당 사목위원회 등 확대
- 최종 의사결정자: 사제 (주일학교), 모임별 책임자 (기타 모임)
- 초기 내부 고객/파일럿: 작성자(교리교사)

### 근거
- 청소년국이 본당·학교 지원 역할을 명시하고, 교리교사 접점(자료실/공지)이 존재
- 주일학교 운영의 핵심 실행은 교리교사이며, 최종 결정은 사제가 한다는 사용자 합의

### 검증
- 목적: 사용자/의사결정 구조 확정
- 방법: 도입 요청/승인/결제 기록과 역할/권한 로그로 의사결정 구조 추정
- 데이터: 승인/결제 기록, 역할/권한 로그, 협력 요청/응답 기록
- 완료 기준: 도입 주체/승인 경로 유형 정리

## 사용자 특성 (2030세대)

> 파일럿 본당 교리교사 연령대: 20~30대 (2026-03-01 확인)

### 디지털 행동
- **디지털 네이티브**: 스마트폰 앱 사용에 거부감 없음. 새 도구 시도 장벽이 낮음
- **직관성에 민감**: 메뉴가 복잡하거나 화면이 지저분하면 바로 삭제. "고민 없이 쓸 수 있어야" 함
- **첫인상에서 결판**: 한번 써보고 별로면 삭제하고 다시 찾아오지 않음. "나중에 다시 해볼까"가 없는 세대
- **Silent Churn**: 불만이 있어도 피드백을 주지 않고 조용히 떠남. 2026-02 피드백 이벤트(구글폼, ~02-28 마감)에서 응답 2건에 그친 원인으로 추정. 보상 매력 부족도 복합 원인 — 보상이 행동 동기를 유발할 만큼 매력적이지 않았을 가능성
- **커스터마이징 선호**: 자기 방식대로 정보를 구성하고 싶어함 (예: 비고란에 아이 정보를 세밀하게 기입하는 행동)
- **SNS 기반 유입에 익숙**: 인스타그램 → 랜딩 → 가입 흐름이 자연스러운 세대

### 소비 성향
- **요노(YONO) 전환 중**: 욜로에서 "꼭 필요한 것만" 쓰는 방향으로 변화. 무료 도구에 민감하고, 유료 전환 시 확실한 가치 증명 필요
- **가치소비**: 돈을 쓰더라도 자기 가치관에 맞는 곳에 씀. "가톨릭 전용"이라는 포지셔닝이 가치 공감 유발 가능
- **확실한 보상 선호**: 행동 대비 보상이 불확실하면 아예 행동하지 않음. "행동 → 즉각적이고 확실한 결과" 연결이 빨라야 움직이는 세대

### 행동-보상 원칙

> 2030세대는 "행동 → 확실한 보상" 연결이 빠를수록 움직인다. 이 원칙은 피드백 수집뿐 아니라 온보딩, 기능 설계 전반에 적용됨

| 사례 | 행동 비용 | 보상 확실성 | 결과 |
|------|----------|-----------|------|
| 피드백 이벤트 (구글폼) | 높음 (폼 열기 → 생각 정리 → 작성) | 낮음 (추첨) | 응답 2건 — 안 함 |
| 가재울/창동/길음 출석 입력 | 낮음 (그룹 → 학생 → 체크) | 높음 (출석부 즉시 정리됨) | 가입 후 바로 사용 시작 |
| 성남동 비고란 기입 | 낮음 (자유 입력) | 높음 (입력한 정보 즉시 저장/조회) | 세밀하게 활용 |

### 이탈자 복귀 가능성

> 2030세대의 "첫인상 결판"은 대체재가 있는 앱에서 강하게 작동하지만, 우리 서비스는 대체재 0개 + 매주 반복 니즈라는 특수성이 있음

| 이탈 유형 | 본당 | 원인 | 복귀 가능성 | 트리거 |
|----------|------|------|-----------|--------|
| A (구조 혼란) | 신당동/금호동 | 계정 모델 불일치 | **중간~높음** | 계정 모델 전환 후 "바뀌었다" 재안내 |
| B (대기) | 돈암동 | 바빠서 못 씀 | **높음** | 시간 여유 생기면 자연 복귀 |
| C (가치 미체감) | 고덕동 | 써봤는데 가치를 못 느낌 | **낮음** | 1:1 통화로 원인 파악 후 판단 |

- **핵심**: "트리거 없이는 복귀 0, 트리거가 있으면 A/B유형은 복귀 가능"
- 대체재가 없고 매주 출석 니즈가 반복되므로 일반 소비 앱보다 복귀 창이 넓음

### 사용자 파악 방법

> 2030세대는 불만을 말하지 않고 떠나므로, 피드백 수집보다 행동 관찰이 더 정확한 진단법

| 방법 | 설명 | 현재 대응 |
|------|------|----------|
| 행동 데이터 (GA4) | 어디서 멈추는지 이벤트로 추적 | GA4 26개 이벤트 구현 완료 |
| 1:1 통화 | 미사용자에게 직접 물어보기 | 고덕동 3월 1주차 통화 계획 |
| 구조적 관찰 | 가입 후 첫 행동이 어디서 끊기는지 분석 | 전환 퍼널 분석 진행 중 |

> 2030세대 특성이 실제 파일럿 데이터와 일치하는지 검증

|------|------|------------------|
| 고덕동 | 그룹 1개 생성 후 미사용 지속 | **첫인상 결판** — 첫 경험에서 가치를 못 느끼면 재시도 안 함 |

## 사용자 니즈 신호
### 가설
- 교리교사의 핵심 니즈는 "출석 기록"보다 **"매주 필요한 정보(출석 확인 + 축일 + 멤버 현황)를 한곳에서 빠르게 조회"**하는 것이다
- 출석 기록 자체는 엑셀로도 가능하지만, 출석 조회 + 가톨릭 정보가 통합된 도구는 엑셀이 대체할 수 없다

### 근거
- 장위동 4년 사용 관찰: 출석 "기록"보다 "누가 나왔는지 빠른 확인"이 핵심 가치, 축일 검색에도 활용 (출석 도구가 아닌 가톨릭 정보 도구로도 사용)
- 사용자 피드백에서 통계/현황, 출석 UX, 대량 처리, 축일/등록/동의 관리 요구가 반복됨(상세: `docs/business/0_feedback/feedback.md`)

### 검증
- 목적: 핵심 니즈 상위 항목 도출
- 방법: 핵심 기능 사용률/처리 시간 기준 상위 5개 항목 도출, 전후 비교
- 데이터: 기능 사용 로그, 리포트/출석 처리 로그, 등록 기록
- 완료 기준: 상위 5개 항목 확정

## 경쟁 환경

> 상세: `docs/business/1_market/competitive-analysis.md` (2026-02-13 조사)

### 핵심 발견
- **가톨릭 교리교사/모임 운영자가 매주 여는 주간 운영 도구가 시장에 존재하지 않음** (출석 + 축일 + 멤버 통합)
- 개신교 ChMS는 6~7개 존재하나 가톨릭 전용은 공백
- 가톨릭 하상 = 신자 개인 앱 (출석 관리 기능 없음), 양업시스템 = 사무실 행정 도구 (교리교사 접근 불가)

### 경쟁 구도

| 카테고리     | 대표 서비스                              | 우리와의 관계                      |
|----------|-------------------------------------|------------------------------|
| 개신교 ChMS | 오직교적, 만나ERP, 지저스온 등                 | 용어/구조 불일치, 기능 과잉 (90% 불필요)   |
| 가톨릭 공식 앱 | 가톨릭 하상, 양업시스템                       | 운영 도구가 아닌 신자 앱/사무실 도구        |
| 범용 출석 앱  | Attendable, Wasso                   | 교회 특화 기능 없음 (의무축일, 학년 전환 등)  |
| 수기/오피스   | 엑셀, 구글시트, 카카오톡+수기                   | **현재 최대 대체재** — 분산/유실/수작업 문제 |
| 해외 ChMS  | Planning Center, Breeze, ChurchTrac | 영어 전용, 한국 가톨릭 구조와 불일치        |

### 가설
- 현재 가톨릭 교리교사의 대체재는 "엑셀/한글 + 카카오톡 + 수기"이며, 전용 SaaS 침투율은 0%에 가깝다
- 개신교 ChMS는 존재하나 가톨릭과 용어/구조가 달라 전환 대상이 아니다

### 근거
- 경쟁사 조사(2026-02-13)에서 가톨릭 전용 출석 관리 도구 0개 확인
- 현장 조사에서 수기 → 엑셀 → 카카오톡 공유 워크플로우가 가장 흔함

### 검증
- 목적: 대체재 사용 패턴과 전환 비용 파악
- 방법: 본당 10곳 이상 현행 툴/워크플로우 조사, 전환 비용 추산
- 완료 기준: 대체재 유형 3개 이상 분류

## 차별화

### 가설
- **"매주 일요일에 이거 하나 열면 다 되는" 도구**가 핵심 차별점이며, 출석 조회 + 가톨릭 정보(축일/전례력) + 멤버 현황이 한곳에 통합된 것이 엑셀/카카오톡이 대체할 수 없는 가치다

### 차별점

| 차별점              | 설명                          | 경쟁사 대비                   |
|------------------|-----------------------------|--------------------------|

### 근거
- 장위동 4년 사용 관찰: 출석 기록보다 "빠른 조회 + 축일 검색"이 실제 사용 이유
- 경쟁사 분석에서 가톨릭 전용 도구 0개 확인
- 엑셀은 출석 기록은 가능하지만, 축일/전례력/멤버 현황을 한곳에서 제공하지 못함

### 검증
- 목적: 통합 관리의 효과 확인
- 방법: 연간 입력 시간 절감, 출석 누락/불일치 감소, 보고 준비 시간 전후 비교
- 완료 기준: 개선 효과 1차 수치 확보
