# 브레인스토밍 결과 보고서

## 주제
리텐션 강화 전략 — WAU 30곳 달성 후 이탈 방지 및 안착률 향상

## 토론 요약

3라운드 토론을 통해 초기 3개 아이디어가 v3로 개선되었다.

**핵심 합의**:
- 이탈 10곳 중 8곳이 1인 사용이라는 데이터는 유효하나, "1인 사용 = 이탈 원인"이라는 등식은 성립하지 않음 (가재울 135학생 1계정이 반증)
- 이메일 채널 확장은 법률 자문(리스크 15) 완료 전까지 보류
- 병목을 "학생 등록"과 "출석 시작"으로 분리 설계
- 넛지 배너에 "나중에"/"준비 중" 옵션을 넣어 GA4로 이탈 원인 데이터 수집

**핵심 논쟁**:
- R1: "팀 합류 필요성"이 사용자 페인포인트인지 개발자 가정인지 → 인수인계 맥락으로 재설계
- R2: 이메일 법적 분류 문제 → 앱 내 배너 선행으로 합의
- R3: 트리거 조건 모호함 → 학생 5명/7일/출석 0건으로 확정

## 아이디어 목록

### 아이디어 3: 출석 시작 넛지 (v3) — 우선순위 1
- **한 줄 요약**: 학생 등록 후 출석을 시작하지 않은 단체에 인라인 배너로 전환 유도
- **사용자 가치**: 전환 유력 6곳(서판교 47명, 불광동 벤야민 29명 등)의 "출석 시작" 장벽 제거
- **실현 방법**: 트리거(학생 5명+ / 출석 0건 / 7일 경과) → 배너 노출 → "나중에"/"준비 중" 옵션 → GA4 추적 → 첫 출석 시 자동 제거. 기존 미전환 6곳 소급 적용
- **차별점**: 가톨릭 모임 맥락(그룹→학생→출석 계층) 안내는 범용 앱 불가

### 아이디어 2a: 이탈 앱 내 배너 (v3) — 우선순위 2
- **한 줄 요약**: 출석 기록이 있으나 14일 미활동 단체에 다음 로그인 시 앱 내 배너 노출
- **사용자 가치**: 이탈 위험 6곳(불광동MH, 정릉4동 등)의 Silent Churn 조기 개입
- **실현 방법**: 14일 미활동 + 출석 기록 있는 단체 → 로그인 시 배너 → 출석 기록량별 세그먼트 메시지 → GA4 추적. 아이디어 3 GA4 데이터로 메시지 최적화
- **차별점**: 교리교사 업무 패턴 기반 재활성화 메시지

### 아이디어 2b: 이탈 트리거 이메일 — 우선순위 3
- **한 줄 요약**: 3주 이상 활성 + 1주 미기록 단체에 안부 이메일 자동 발송
- **사용자 가치**: 앱을 열지 않는 단체에 외부 채널로 재방문 동기 제공
- **실현 방법**: 매주 월요일 배치 잡 → 세그먼트 조건 충족 단체에 단일 이메일 → 3회 미응답 시 중단
- **차별점**: 가톨릭 전례력 기반 주간 요약 (범용 앱 불가)

### 아이디어 1: 교체 시즌 인수인계 도우미 (v3) — 우선순위 4
- **한 줄 요약**: 12~2월 교체 시즌에 1인 단체에 "데이터 인수인계" 배너 노출
- **사용자 가치**: 교리교사 교체 시 4년치 데이터 유지 + 인수인계 간소화
- **실현 방법**: 12~2월 / 계정 1명 / 6개월 이상 활성 조건 → 이메일 초대 배너 → 기존 합류 기능 활용
- **차별점**: 조직 단위 데이터 보존은 엑셀/밴드/카카오톡 불가

## 누락된 관점

- **학생 등록 병목(71%)** 자체에 대한 넛지는 이번 토론에서 다루지 않았다. 출석 시작 넛지와 별도로 "그룹 생성 후 학생 미등록" 단체 20곳에 대한 넛지도 후속 검토 필요
- **유료 전환 시점**의 리텐션 전략 (현재는 모두 무료 사용 단계)
