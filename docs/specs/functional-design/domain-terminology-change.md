# 기능 설계: 도메인 용어 변경

> 그룹→학년, 멤버→학생으로 UI 라벨을 변경하여 사용자 이해도를 높입니다.

## 연결 문서

- 로드맵: `docs/business/6_roadmap/roadmap.md` (2단계)
- 현황: `docs/business/STATUS.md` (고덕동/돈암동 이탈 분석)
- Task: `docs/specs/current/functional/tasks/domain-terminology-change.md`
- Development: `docs/specs/current/functional/development/domain-terminology-change-frontend.md`

## 배경

### 문제

- 고덕동: 그룹 1개 생성 후 멈춤 — "다음에 뭘 해야 하는지 모름"
- 돈암동: 멤버 1명만 입력 후 멈춤 — "그룹/멤버 구조를 이해 못함"
- "그룹"과 "멤버"는 범용 소프트웨어 용어로, 주일학교 교리교사에게 직관적이지 않음

### 해결 방향

- "그룹" → "학년": 주일학교에서 반/학년이 일상 용어
- "멤버" → "학생": 교리교사가 관리하는 대상은 학생

### 변경 범위

| 구분 | 변경 여부 | 이유 |
|------|----------|------|
| UI 라벨 (한국어) | **변경** | 사용자 대면 텍스트 |
| URL 경로 | 유지 (`/groups`, `/students`) | 브라우저 히스토리, 북마크 호환 |
| API 프로시저명 | 유지 (`group.*`, `student.*`) | 내부 인터페이스, 변경 비용 대비 효과 없음 |
| DB 스키마 | 유지 (`group`, `student` 테이블) | 마이그레이션 불필요 |
| 코드 변수명 | 유지 | 내부 코드, 사용자 비노출 |
| GA4 이벤트명 | 유지 (`group_*`, `student_*`) | 기존 데이터 연속성 |

## 흐름/상태

사용자 플로우와 상태 전이는 기존과 동일합니다. **라벨만 변경**되며 동작은 변하지 않습니다.
