# 비즈니스 모델

## 비전
- **서비스 비전**: 가톨릭 교회 내 모든 모임을 위한 그룹 운영 플랫폼
- **현재 검증**: 주일학교 중고등부 대상 PMF 검증 → 검증 후 모임 전체로 확장

## 가치 제안
- 한 줄 정의(가설): **매주 일요일, 이거 하나 열면 다 되는 가톨릭 모임 운영 도구** — 출석 확인·축일·멤버 현황이 한곳에
- 시장 근거: 가톨릭 교리교사/모임 운영자가 매주 여는 주간 운영 도구가 시장에 없음 (경쟁사 분석 2026-02-13, 상세: `docs/business/1_market/competitive-analysis.md`)
- 핵심 관찰: 출석 "기록"보다 "빠른 조회"가 핵심 가치, 축일 검색 등 가톨릭 정보 도구로도 활용됨 (장위동 4년 사용)
- 현재 검증 대상: 주일학교 교리교사(중고등부 우선)

### 가치 1: 학생 정보 재입력 최소화(학년/졸업 주기 반영)
- 가설: 교리교사의 연간 재입력·정리 시간과 학년 전환 오류가 감소한다
- 근거: 매년 출석부 재작성·재입력, 학년 전환 시 수기 수정이 반복됨
- 검증:
  - 목적: 연간 재입력 시간 및 학년 전환 오류 감소 확인
  - 방법: 연간 입력 업무 시간, 학년 전환 오류 건수 전후 비교
  - 완료 기준: 시간/오류 변화 추세 확인

### 가치 2: 매주 필요한 정보를 한곳에서 빠르게 조회
- 가설: 출석 확인·축일·멤버 현황이 한곳에 있으면 매주 반복적으로 사용하게 된다
- 근거: 장위동에서 출석 조회뿐 아니라 축일 검색에도 활용 — "출석 도구"가 아닌 "주간 운영 허브"로 사용
- 검증:
  - 목적: 주간 재방문율과 출석 외 기능 사용률 확인
  - 방법: 주간 활성률, 축일/멤버 조회 빈도, 출석 기록 대비 조회 비율 측정
  - 완료 기준: 출석 외 기능 사용 비율 확인

### 가치 3: 행사/공지 운영 간소화
- 가설: 캠프/피정 등 행사 준비와 공지 전달의 누락·재작업이 감소한다
- 근거: 행사 체크리스트·역할·물품·연락이 수기로 분산되어 누락이 발생한다는 문제
- 검증:
  - 목적: 행사 준비 누락/재작업 감소 확인
  - 방법: 누락 건수/재작업 시간, 공지 전달 지연 전후 비교
  - 완료 기준: 개선 효과 확인

### 가치 4: 협업 기반 데이터 일관성
- 가설: 동일 학생/출석 기록의 불일치가 감소한다
- 근거: 여러 파일/메신저(카카오톡)/수기로 기록이 분산되어 최신성이 깨진다는 문제
- 검증:
  - 목적: 동일 기록 불일치 감소 확인
  - 방법: 동일 일자 불일치 건수/최신본 충돌 빈도 측정
  - 완료 기준: 불일치 유형/빈도 파악

### 가치 5: 통계/보고 신뢰도 향상(사제 의사결정 지원)
- 가설: 통계/보고 준비 시간이 줄고 사제 의사결정 리드타임이 감소한다
- 근거: 분산 기록으로 통계 산출·공지 준비가 수작업이며 데이터 불일치가 보고 신뢰도를 낮춘다
- 검증:
  - 목적: 보고 준비 시간/오류율 감소 확인
  - 방법: 보고 준비 시간/오류율, 사제 승인 리드타임 전후 비교
  - 완료 기준: 개선 효과 확인

## 고객 세그먼트
- **1차 (현재 검증)**: 서울대교구 233개 본당 주일학교 교리교사(중고등부 우선)
- **2차**: 초등부 교리교사, 주일학교 외 모임 운영자(레지오, 성가대, 청년회 등)
- **3차**: 전국 교구 가톨릭 모임 운영자
- 의사결정자: 사제(주일학교), 모임별 책임자(기타 모임)
- 초기 내부 고객/파일럿: 작성자(교리교사)

## 협력 채널
### 가설
- 교구 청소년국 사목부서(중고등부/학교사목부 중심)가 초기 협력 채널로 유효하다

### 근거
- 청소년국이 본당·학교 지원 역할을 수행하며 교리교사 접점(자료실/공지)을 보유한다는 시장 가설

### 검증
- 목적: 청소년국 협력 가능성 확인
- 방법: 협력 요청 발송 후 응답/교육 세션 진행 여부 기록
- 완료 기준: 협력 범위 1차 정의

## 수익 모델
- 가설: 무료 + 광고 + 유료 구독의 3단 수익 구조가 적합하다
- 근거: 핵심 운영은 무료로 제공하고, 광고로 운영 비용을 충당하며, 확장 기능에서 과금한다는 합의
- 검증:
  - 목적: 수익 구조의 현실성 확인
  - 방법: 광고 수익/CTR, 무료→유료 전환율, 유료 기능 활성도, 결제 완료율 확인
  - 완료 기준: 수익 구조 타당성 1차 판단

### 무료(1단계)
#### 가설
- 학생/출석/기본 통계·보고/출석 UX 개선/출석 유형 구분(의무축일/행사)/대량 작업은 무료로 제공한다

#### 근거
- 핵심 운영 기능은 도입 장벽을 낮추고 초기 사용 습관을 만드는 데 필수다

#### 검증
- 방법: 무료 가입 대비 4주차 활성률, 주간 출석 기록 수행률 측정
- 완료 기준: 무료 플랜 유지 기준선 확보

### 광고(2단계)
#### 가설
- 전체 사용자 대상 광고로 운영 비용을 일부 충당하며 무료 서비스 유지가 가능하다

#### 근거
- 무료 제공 지속을 위한 운영 비용 회수 필요

#### 검증
- 방법: 광고 CTR, 광고 수익/운영 비용 비율, 광고 후 이탈률 측정
- 완료 기준: 광고 수익성 판단

### 유료 구독(3~4단계)
#### 가설
- 행사/축일/조직·권한(3단계), 등록/공지/알림/개인정보 동의(4단계) 등 확장 기능은 유료 구독으로 과금한다

#### 근거
- 핵심 운영은 무료로 유지하되 확장 기능에서 과금하는 구조가 수용성이 높다

#### 검증
- 방법: 유료 기능별 활성도, 무료→유료 전환율, 업그레이드율 측정
- 완료 기준: 유료 전환 기준선 확보

## 결제 주체
### 가설 (2026-02-20 재정의)
- 계정 모델 전환(공유→개인)에 따라 결제 주체도 재정의 필요
- **본당/모임 단위 구독**이 자연스러운 1차 모델: 관리자(부장 교리교사)가 본당 단위로 결제
- 보조 모델: 개인 프리미엄 (개인이 추가 기능에 결제, Slack/Notion 패턴)

### 이전 가설
- ~~교리교사 개인 결제가 주를 이뤄야 유료 전환 속도가 안정적으로 유지된다~~
- 개인 계정 전환 전 공유 계정 모델 기준의 가설이었음

### 근거
- 개인 계정 모델에서는 "누가 결제하는가"가 더 명확해짐
- 본당 단위 구독: 관리자가 대표 결제, 승인 구조가 단순 (교리교사 합의)
- 개인 프리미엄: 승인 불필요, 전환 속도 빠름, 다만 ARPU 낮음

### 리스크 (2026-01-25 피드백)
- 본당 예산 사용 시 설득 필요: 사제/사무실이 현행(한글/엑셀) 방식에 불편함을 느끼지 않음
- 프로세스 전환 자체가 저항 요인: 새 시스템 도입이 오히려 불편
- 유사 사례: 봉헌금 페이 서비스도 앱 전환 어려움 → 교회 디지털 전환 한계

### 검증
- 방법: 결제 주체 분포, 승인 리드타임 측정
- 추가 검증: 계정 모델 전환 후 본당 단위 vs 개인 단위 결제 선호도 확인
- 완료 기준: 결제 주체별 전환율/해지율 비교

## 비용 구조
- 가설: 고정비는 클라우드(Compute/DB/Storage), 도메인, 운영 지원; 변동비는 온보딩/교육, 알림(SMS/Push) 연동 시 통신비
- 근거: 현 구조가 API/웹 호스팅 중심이며, 향후 알림·결제는 후순위 로드맵으로 별도 변동비 발생
- 검증:
  - 방법: 예상 트래픽 기준 인프라 견적, 온보딩 시간/인건비, 알림 통신 단가 확인
  - 완료 기준: 월간 비용 범위 산출

## 유닛 이코노믹스
- 가설: 월 ARPU 5,000~10,000원(모임 단위), CAC 회수 기간 3개월 이내, 인프라/지원 포함 총 마진 60%+
- 근거: 틈새 시장으로 대규모 판매비용이 낮고, 온보딩 후 유지 비용이 낮다는 가정
- 검증:
  - 방법: 파일럿 3곳 유료 전환율/지불 금액, 온보딩·지원 시간으로 CAC 추정
  - 완료 기준: CAC 회수 기간 1차 산출

## 오픈 이슈
- 결제 주체 검증: 계정 모델 전환 후 본당 단위 vs 개인 단위 선호도 확인
- 연 구독 할인 폭: 10개월 가격으로 연 구독 제공 (약 17% 할인)
- 연간 보고 연동: 교구 연간 보고 양식과 연동 시 사제 설득력 증가 가능성 검토
- 무료 지속 조건: 광고 수익으로 운영비 충당 가능 여부 검증 필요
