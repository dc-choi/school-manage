# 기능 설계: 사이트 Footer + 약관 3종 (1차 배포)

> 상태: Draft | 작성일: 2026-05-06 | 기능명: `footer-terms-v1`

> PRD `docs/specs/prd/footer-terms-v1.md`의 동작 명세 — PG 가맹 심사 선행 조건으로 Footer + 약관 3종 페이지 추가.
> 약관 본문 자체는 5단계 구현 시 코드(React 컴포넌트)에 직접 작성. FD에서는 본문 구조(섹션 목차)만 명시.

## 흐름/상태

### 사용자 플로우

1. **PG 심사 담당자가 사이트 검수**: 사이트 진입(`/landing` 등) → 페이지 하단 Footer에서 사업자 정보 확인 → 약관 3종 링크(이용약관/개인정보처리방침/환불약관) 각각 클릭 → 별도 페이지에서 본문 검수 → 검수 완료.
2. **일반 사용자가 신뢰성 확인**: 어느 공개 페이지에서든 하단 스크롤 → Footer 사업자 정보 노출.
3. **약관 페이지 직접 접근**: URL(`/terms` 등) 직접 입력 또는 외부 링크 클릭 → 비로그인 상태에서 정상 렌더링.

## 테스트 시나리오

### 정상 케이스

1. **TC-1 (Footer 노출)**: `/landing` 진입 → 페이지 하단 Footer 표시 확인 (사업자 정보 4항목 + 약관 링크 3개 + Copyright).
2. **TC-2 (약관 라우팅)**: Footer의 이용약관 링크 클릭 → `/terms` 이동 → 페이지 제목·시행일·본문 섹션 표시.
3. **TC-3 (3페이지 동작)**: `/terms`, `/privacy`, `/refund` 각각 직접 URL 접근 → 비로그인 상태에서 정상 렌더링.
4. **TC-4 (인증 후 Footer)**: 로그인 후 `MainLayout` 페이지 진입 → 동일 Footer 노출.
5. **TC-5 (반응형)**: 데스크톱(≥768px)에서 가로 정렬, 모바일(<640px)에서 1열 스택.

### 예외 케이스

1. **TC-E1 (페이지 간 스크롤 초기화)**: `/terms` 하단에서 `/privacy`로 이동 → 새 페이지 최상단으로 스크롤 초기화.
2. **TC-E2 (시맨틱 마크업)**: Footer는 `<footer>` 태그, 약관 페이지 본문은 `<main>` 또는 적절한 시맨틱 태그 사용 (스크린리더 호환성).
3. **TC-E3 (링크 외부 페이지 차단)**: 약관 페이지에서 외부 링크 없이 자체 본문만 표시 (개인정보처리방침의 보호책임자 이메일은 `mailto:` 링크 허용).
