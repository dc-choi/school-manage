# PRD: 전례력 개선 (특전미사 + 성주간/성삼일)

> SDD 작성자가 작성하는 제품 요구사항 문서입니다.
> 사업 에이전트의 `docs/business/` 문서를 기반으로 작성합니다.

## 배경/문제 요약

- 참고: `docs/business/6_roadmap/roadmap.md` (2단계), `docs/specs/prd/liturgical-calendar.md` (기존 PRD)
- 문제: 현재 전례력이 두 가지 가톨릭 전례 규정을 반영하지 못하고 있다.
  1. **특전미사**: 토요일 저녁 미사는 주일 미사 의무를 충족한다. 토요일에 앱을 열면 토요일 전례 시기가 표시되어, 주일학교 교리교사가 "내일 주일 전례 시기"를 확인하려면 별도 자료를 찾아야 한다.
  2. **성주간/성삼일**: 주님 수난 성지주일(빨강)이 "사순 제6주일(보라)"로, 성삼일이 사순(보라)로 일괄 처리되어 전례색과 시기명이 부정확하다.
- 현재 상태: 토요일에는 토요일 기준 전례 시기 표시. 성주간/성삼일은 사순 시기(보라)로 처리.
- 목표 상태: 토요일에 일요일 전례 시기 표시 (성토요일 제외). 성주간 + 성삼일을 정확한 시기명/전례색으로 구분.

## 목표/성공 기준

- **목표**: 가톨릭 전례 규정(특전미사, 성삼일)을 정확히 반영하여 "주간 허브" 신뢰도를 높인다.
- **성공 지표**: 성삼일 기간 + 매주 토요일에 전례 카드가 정확한 시기/전례색을 표시
- **측정 기간**: 배포 후 부활 시기 1회 경과 (검증 가능 시점)

## 사용자/대상

- **주요 사용자**: 주일학교 교리교사 (출석부 앱 기존 사용자)
- **사용 맥락**: 토요일 특전미사 전후에 앱을 열어 주일 전례 시기를 확인하거나, 성주간에 성삼일 전례 정보를 확인한다.

## 범위

### 포함

- 토요일에 일요일 전례 시기/축일 표시 (특전미사 반영)
- 성토요일은 예외 — 일요일이 아닌 "성토요일 / 파스카 성야"로 표시
- 성주간(주님 수난 성지주일~성수요일) 시기명 표시 + 성지주일 전례색(빨강) 반영
- 성삼일(성목요일~성토요일) 별도 시기 구분 + 각 날의 전례색 반영 (흰/빨/흰)
- 기존 `getLiturgicalSeason()` 계산 로직 수정
- 기존 테스트 업데이트

### 제외

- 미사 참례 기록 기능 (별도 TARGET P1)
- 성삼일 전용 UI 페이지
- 성주간 전용 UI 페이지
- 시간 기반 전환 (토요일 16시부터 등) — 날짜 단위로 처리

## 사용자 시나리오

1. **시나리오 1 — 토요일 특전미사 전**: 교리교사가 토요일에 앱을 열면, "연중 제7주일" (일요일 전례 시기)이 표시된다.
2. **시나리오 2 — 주님 수난 성지주일**: "주님 수난 성지주일"이 표시되고 전례색이 빨강이다.
3. **시나리오 3 — 성주간 중**: 성화요일에 앱을 열면 "성주간"이 표시되고 전례색이 보라이다.
4. **시나리오 4 — 성목요일**: "파스카 성삼일 — 주님 만찬 성목요일"이 표시되고 전례색이 흰색이다.
5. **시나리오 5 — 성금요일**: "파스카 성삼일 — 주님 수난 성금요일"이 표시되고 전례색이 빨강이다.
6. **시나리오 6 — 성토요일**: 토요일이지만 "파스카 성삼일 — 파스카 성야"가 표시되고 전례색이 흰색이다.

## 요구사항

### 필수 (Must)

- [ ] 일반 토요일에 일요일 전례 시기/전례색/축일 표시
- [ ] 성토요일은 예외 처리 — "파스카 성삼일" 시기로 표시
- [ ] 성주간(성지주일~성수요일) 시기명 + 성지주일 빨강 전례색
- [ ] 성삼일 3일 각각의 시기명과 전례색 정확히 구분
- [ ] 기존 전례 시기 계산 테스트 업데이트

### 선택 (Should)

- [ ] 성삼일 축일을 다가오는 축일 목록에 포함

### 제외 (Out)

- 시간 기반 전례일 전환 (16시 이후 등)
- 미사 참례 기록/횟수 관리
- 성주간 전용 UI 페이지

## 제약/가정/리스크

- **제약**: 한국 천주교 전례력 기준. 날짜 단위 처리 (시간 기반 전환 없음)
- **가정**: 토요일 = 일요일 전례 표시는 모든 토요일에 적용 (특전미사 시간대 구분 불필요)
- **리스크**: 성삼일 경계 계산 — 성목요일 저녁 미사부터 시작이지만, 날짜 단위로는 성목요일 전체를 성삼일로 처리해야 할지 판단 필요

## 의존성

- **내부**: 기존 `getLiturgicalSeason()` (`packages/utils/src/liturgical.ts`), `calculateEaster()` (`packages/utils/src/date.ts`)
- **외부**: 없음 (순수 계산)

## 롤아웃/검증

- **출시 단계**: 전체 사용자 동시 배포 (읽기 전용 기능)
- **이벤트/로그**: 기존 `liturgical_card_viewed` 재활용
- **검증 방법**: 성삼일 날짜 + 매주 토요일에 전례 카드 정확성 수동 검증, 단위 테스트

## 오픈 이슈
