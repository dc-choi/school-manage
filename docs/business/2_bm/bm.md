# 비즈니스 모델

## 비전
- **서비스 비전**: 가톨릭 교회 내 모든 모임을 위한 그룹 운영 플랫폼
- **현재 검증**: 주일학교 중고등부 대상 PMF 검증 → 검증 후 모임 전체로 확장

## 가치 제안
- 한 줄 정의(가설): **매주 일요일, 이거 하나 열면 다 되는 가톨릭 모임 운영 도구** — 출석 확인·축일·멤버 현황이 한곳에
- 시장 근거: 가톨릭 교리교사/모임 운영자가 매주 여는 주간 운영 도구가 시장에 없음 (상세: `docs/business/1_market/competitive-analysis.md`)
- 핵심 관찰: 출석 "기록"보다 "빠른 조회"가 핵심 가치, 축일 검색 등 가톨릭 정보 도구로도 활용됨 (장위동 4년 사용)
- 현재 검증 대상: 주일학교 교리교사(중고등부 우선)

### 가치 1: 학생 정보 재입력 최소화(학년/졸업 주기 반영)
- 가설: 교리교사의 연간 재입력·정리 시간과 학년 전환 오류가 감소한다
- 근거: 매년 출석부 재작성·재입력, 학년 전환 시 수기 수정이 반복됨
- 검증: 연간 입력 업무 시간, 학년 전환 오류 건수 전후 비교 → 미완료

### 가치 2: 매주 필요한 정보를 한곳에서 빠르게 조회
- 가설: 출석 확인·축일·멤버 현황이 한곳에 있으면 매주 반복 사용하게 된다
- 근거: 장위동에서 출석 조회뿐 아니라 축일 검색에도 활용 — "주간 운영 허브"로 사용
- 검증: 주간 활성률, 축일/멤버 조회 빈도, 출석 기록 대비 조회 비율 → 미완료

### 가치 3: 행사/공지 운영 간소화
- 가설: 캠프/피정 등 행사 준비와 공지 전달의 누락·재작업이 감소한다
- 근거: 행사 체크리스트·역할·물품·연락이 수기로 분산되어 누락 발생
- 검증: 누락 건수/재작업 시간 전후 비교 → 미완료

### 가치 4: 협업 기반 데이터 일관성
- 가설: 동일 학생/출석 기록의 불일치가 감소한다
- 근거: 여러 파일/메신저/수기로 기록이 분산되어 최신성이 깨짐
- 검증: 동일 일자 불일치 건수/최신본 충돌 빈도 측정 → 미완료

### 가치 5: 통계/보고 신뢰도 향상(사제 의사결정 지원)
- 가설: 통계/보고 준비 시간이 줄고 사제 의사결정 리드타임이 감소한다
- 근거: 분산 기록으로 통계 산출·공지 준비가 수작업이며 데이터 불일치가 보고 신뢰도를 낮춤
- 검증: 보고 준비 시간/오류율, 사제 승인 리드타임 전후 비교 → 미완료

## 고객 세그먼트
- **1차 (현재 검증)**: 서울대교구 233개 본당 주일학교 교리교사(중고등부 우선)
- **2차**: 초등부 교리교사, 주일학교 외 모임 운영자(레지오, 성가대, 청년회 등)
- **3차**: 전국 교구 가톨릭 모임 운영자
- 의사결정자: 사제(주일학교), 모임별 책임자(기타 모임)

## 협력 채널
- 가설: 교구 청소년국 사목부서(중고등부/학교사목부 중심)가 초기 협력 채널로 유효하다
- 근거: 청소년국이 본당·학교 지원 역할을 수행하며 교리교사 접점(자료실/공지)을 보유
- 검증: 협력 요청 발송 후 응답/교육 세션 진행 여부 → 미완료

## 수익 모델
- 가설: 무료 + 광고 + 유료 구독의 3단 수익 구조가 적합하다
- 근거: 핵심 운영은 무료 제공, 광고로 운영 비용 충당, 확장 기능에서 과금
- 검증: 광고 수익/CTR, 무료→유료 전환율, 유료 기능 활성도 → 미완료

### 무료(1단계)
- 포함: 학생/출석/기본 통계·보고/출석 UX 개선/출석 유형 구분/대량 작업
- 근거: 핵심 운영 기능은 도입 장벽을 낮추고 사용 습관을 만드는 데 필수
- 검증: 무료 가입 대비 4주차 활성률, 주간 출석 기록 수행률 → 미완료

### 광고(2단계)
- 전체 사용자 대상 광고로 운영 비용 일부 충당
- **배치 원칙 (2026-03-17 확정)**: **하단 고정 배치만 허용.** 핵심 기능 흐름 내 광고 절대 금지
- **현황**: AdSense 승인 완료, 미적용. 즉시 배치 가능
- 검증: 광고 CTR, 수익/비용 비율, 광고 후 이탈률 → 미완료

### 유료 구독(3~4단계)
- 3단계: 행사/축일/조직·권한 → 베이직
- 4단계: 등록/공지/알림/개인정보 동의 → 프로
- **2030세대 유의**: 요노(YONO) 소비 성향 — "있으면 좋은 기능"으로는 결제하지 않음. 가톨릭 특화 기능이 유료 전환 핵심 트리거
- 검증: 유료 기능별 활성도, 전환율, 업그레이드율 → 미완료

## 결제 주체 (2026-02-20 재정의)
- **본당/모임 단위 구독**이 1차 모델: 관리자(부장 교리교사)가 본당 단위로 결제
- 보조 모델: 개인 프리미엄 (Slack/Notion 패턴, 승인 불필요, 전환 빠름, ARPU 낮음)

### 리스크
- 본당 예산 사용 시 설득 필요: 사제/사무실이 현행 방식에 불편함을 느끼지 않음
- 프로세스 전환 자체가 저항 요인
- 유사 사례: 봉헌금 페이 서비스도 앱 전환 어려움 → 교회 디지털 전환 한계

### 검증
- 결제 주체 분포, 승인 리드타임, 본당 vs 개인 결제 선호도 → 미완료

## 비용 구조
- 고정비: 클라우드(Lightsail), 도메인, AI 구독료(Claude Max)
- 변동비: 온보딩/교육, 알림(SMS/Push) 연동 시 통신비
- 초기 투자: 맥미니(OpenClaw 자율 에이전트 전용 서버, ~100만원)
- 검증: 예상 트래픽 기준 인프라 견적, 온보딩 시간/인건비 → 미완료

## 유닛 이코노믹스
- 가설: 월 ARPU 5,000~10,000원(모임 단위), CAC 회수 기간 3개월 이내, 총 마진 60%+
- 근거: 틈새 시장으로 판매비용 낮고, 온보딩 후 유지 비용 낮음
- 검증: 파일럿 3곳 유료 전환율/지불 금액, 온보딩·지원 시간으로 CAC 추정 → 미완료

### 초기 투자
| 항목 | 금액 | 비고 |
|------|------|------|
| 맥미니 | ~100만원 | OpenClaw 전용 서버. 서버 모니터링 + 가톨릭 소식 자동 수집 |

### 자금원 구조
- **현재**: 개인 자금 전액 부담 (사이드 프로젝트)
- **단기**: AdSense(하단) + 도네이션으로 개인 부담 축소
- **중기**: 트래픽 성장 + 후원 안정화로 운영비 대부분 충당
- **장기**: 유료 멤버십(3단계) 도입으로 자체 수익 전환

### 도네이션(가설/근거/검증)
- 가설: "운영비 후원" 포지셔닝이 가톨릭 봉사 문화와 부합하여 도입 가능하다
- 근거: 가톨릭 교회 문화의 후원/봉헌 습관. 프리미엄 구독과 별개의 자발적 기부
- 메시지: "이 서비스는 교리교사가 만든 봉사 프로젝트입니다. 후원금은 서버 운영비로 사용됩니다"
- 채널: 카카오페이/Toss 후원 링크 + 앱 내 후원 안내
- 검증: 후원 전환율, 월 후원 금액, 후원→유료 전환 시 저항 여부 → 미완료

### 외부 투자
- 방침: 최소화. 엔젤 투자만 고려, VC/정부 지원은 규모가 커지기 전까지 보류
- 근거: 사이드 프로젝트 단계에서 외부 자금 의존도를 낮추고 자체 수익화 우선

### 개인사업자 등록
- 시기: 유료 전환(3단계) 착수 시 등록 (현재 불필요)
- 근거: AdSense/도네이션은 개인 수령 가능. PG 연동 시 사업자 필수

## 오픈 이슈
- 결제 주체 검증: 계정 모델 전환 후 본당 vs 개인 선호도 확인
- 연 구독 할인 폭: 10개월 가격으로 연 구독 제공 (약 17% 할인)
- 연간 보고 연동: 교구 연간 보고 양식과 연동 시 사제 설득력 증가 가능성 검토
- 무료 지속 조건: 광고+도네이션으로 운영비 충당 가능 여부 검증 필요
- 도네이션→유료 전환 저항: "후원 기반 운영"에서 유료 멤버십 전환 시 사용자 반응 모니터링 필요
