# 리스크

## 리스크 목록
### 리스크 1: 보수적 문화로 인한 도입 저항
- 가설: 가톨릭 조직 문화 특성상 새로운 디지털 도구 도입이 느리거나 거부될 수 있다
- 근거: 사용자 관찰로 "가톨릭은 보수적"이라는 인식이 강하며, 기존 방식(수기/엑셀/회의록)을 오래 유지하는 경향이 있다
- 유사 사례: 봉헌금 페이 서비스 - 현금 외 지불 방식의 편리함에도 앱 다운로드 전환율이 낮아 상용화 실패 (2026-01-25 피드백)
- 검증:
  - 위험: 도입 제안→승인 전환율, 보류/거절 기록, 기존 도구 병행 기간
  - 대응: 파일럿 전후 추천 전환율/재방문율 비교

### 리스크 2: 사제 동의 지연
- 가설: 최종 의사결정자인 사제의 동의가 지연되면 파일럿 이후 전환이 늦어진다
- 근거: 도입과 결제의 최종 승인이 사제에게 있음
- 검증:
  - 위험: 파일럿 종료 후 동의까지 리드타임, 보류/거절 코드
  - 대응: 보고 템플릿 제공 전후 동의 리드타임 비교

### 리스크 3: 새로운 도구에 대한 학습/적응 부담
- 가설: 대부분의 교리교사는 새로운 도구 학습에 익숙하지 않아 초기 적응에서 이탈이 발생할 수 있다
- 근거: 디지털 전환 경험이 낮고 기존 방식에 익숙한 경우가 많다는 가정
- 검증:
  - 위험: 첫 2주 온보딩 완료율, 기능 사용률, 초기 이탈률
  - 대응: 온보딩 체크리스트 도입 전후 활성률/이탈률 비교

### 리스크 4: 출석/통계 UX 복잡성으로 인한 사용 저하
- 가설: 출석 체크, 정렬, 통계 조회가 복잡하면 일상 사용이 감소한다
- 근거: 사용자 피드백에서 출석 UX 개선 요구가 반복됨(상세: `docs/business/0_feedback/feedback.md`)
- 검증:
  - 위험: 출석 처리 시간/클릭 수, 지연/누락률, 기능 사용률
  - 대응: 개선 후 처리 시간/클릭 수 감소, 지연/누락률 개선

### 리스크 5: B2B2C 구조로 인한 최종 사용자 정착 실패
- 가설: 도입 결정 주체와 실제 사용자가 분리되어 있으면 최종 사용자(교리교사) 정착이 어려울 수 있다
- 근거: B2B2C에서는 도입과 사용 동기가 다를 수 있다는 가정
- 검증:
  - 위험: 도입 경로별 4주 리텐션 및 활성도 비교
  - 대응: 경로별 핵심 기능 사용률/리텐션 비교

### 리스크 6: 학생/보호자 정보의 민감성
- 가설: 학생 정보(연락처/출석 등) 취급에 대한 불안으로 도입이 지연될 수 있다
- 근거: 주일학교 데이터는 미성년자 정보가 포함될 가능성이 높다
- 검증:
  - 위험: 동의서 제출 누락/지연, 권한 변경/접근 차단 요청 건수
  - 대응: 개인정보 안내 적용 전후 누락률 비교

### 리스크 7: 교리교사 교체로 인한 인수인계 공백
- 가설: 교리교사 교체 시 계정/데이터 인수인계가 어려우면 사용이 중단된다
- 근거: 교리교사 역할은 봉사/순환성이 높다는 가정
- 검증:
  - 위험: 본당 10곳 교리교사 교체 주기/인수인계 방식 조사
  - 대응: 교체 후 유지율/재설정 요청 건수

### 리스크 8: 개인 결제 중심 전환 실패
- 가설: 교리교사 개인 결제가 주로 형성되지 않으면 유료 전환이 지연되고 확산 속도가 낮아진다
- 근거: 본당 예산 결제는 승인/결재 절차로 리드타임이 길어질 수 있다
- 검증:
  - 위험: 실제 결제 주체 분포와 승인 리드타임
  - 대응: 결제 옵션별 전환율/해지율 비교

### 리스크 9: 광고로 인한 사용자 이탈
- 가설: 광고가 사용자 경험을 해쳐 이탈률이 증가할 수 있다
- 근거: 광고는 UX 흐름을 방해할 수 있으며, 무료 사용자 중심 서비스에서 이탈 요인이 된다
- 검증:
  - 위험: 광고 전후 주간/월간 이탈률 비교
  - 대응: 광고 위치별 이탈률/불만 신고 건수 비교

### 리스크 10: 부적절한 광고 노출
- 가설: 종교/교육 환경에 맞지 않는 광고가 노출되면 신뢰도가 하락한다
- 근거: 주일학교는 종교 교육 기관이며, 성인/도박 등 부적절한 광고는 심각한 반발을 유발할 수 있다
- 검증:
  - 위험: 광고 필터링 적용 후 부적절 광고 신고 건수
  - 대응: 필터링 적용 전후 신고 건수 비교

### 리스크 11: 광고 수익 불확실성
- 가설: 틈새 시장의 낮은 트래픽으로 광고 수익이 운영 비용을 충당하지 못할 수 있다
- 근거: 주일학교 시장은 규모가 작고, 사용 빈도가 주 1회 중심이라 광고 노출 기회가 제한적이다
- 검증:
  - 위험: 월간 광고 수익/운영 비용 비율, CTR, CPM 추적
  - 대응: 목표 미달 시 유료 전환 캠페인 효과 측정

### 리스크 12: 가치 체감 부족으로 인한 빠른 이탈(장위동 초등부 사례)
- 가설: 진입 장벽을 낮추고 사용성을 개선하더라도 핵심 가치가 체감되지 않으면 빠르게 이탈한다
- 근거: 장위동 초등부 파일럿 안착 실패와 출석 관련 피드백 집중(초등부 전체 의견 일반화 아님)
- 검증:
  - 위험: 초기 핵심 기능 사용률과 2주 이탈률 상관
  - 대응: 최소 입력 플로우/즉시 효용 제공 후 활성률/이탈률 비교

### 리스크 13: 개신교 ChMS의 가톨릭 시장 진입
- 가설: 개신교 ChMS 업체가 가톨릭 시장으로 확장하면 경쟁 우위가 약화될 수 있다
- 근거: 오직교적(9,500+ 교회), 지저스온 등 개신교 ChMS가 이미 대규모 사용자 기반을 보유하며, 용어/구조만 변경하면 가톨릭 진출이 가능
- 검증:
    - 위험: 개신교 ChMS의 가톨릭 대응 기능 출시 여부 모니터링
    - 대응: 가톨릭 특화 UX (의무축일, 학년 전환, 가톨릭 용어)로 전환 비용을 높이고, 먼저 사용자 기반 확보

### 리스크 14: 범용 출석 앱이 "충분히 좋은" 대안이 됨
- 가설: Attendable, 구글 시트 등 무료 범용 도구가 교리교사에게 "그냥 이거면 되지" 수준으로 인식될 수 있다
- 근거: 범용 출석 앱은 이미 무료이며, 교회 특화 기능 없이도 기본 출석 체크는 가능
- 검증:
    - 위험: 파일럿 이탈자 대상 "왜 그만뒀는지" 인터뷰에서 대체재 사용 여부 확인
    - 대응: 가톨릭 특화 기능 (의무축일, 학년 자동 전환, 통계/보고)이 범용 앱과의 차이를 만드는지 검증

### 리스크 15: 개인정보보호법 미준수
- 가설: 개인정보 수집·이용 동의 절차가 미흡하면 법적 리스크(과태료/시정명령)가 발생할 수 있다
- 근거: 학생 이름·성별·학년·출석 기록 등 개인정보를 수집하고 있으며, 현재 동의 절차 없이 운영 중. 개인정보보호법 위반 시 최대 5천만원 과태료
- 검증:
    - 위험: 개인정보 수집 항목/목적/보유기간 명시 누락, 기존 회원 소급 동의 미수집, 제3자 제공 동의 미확보
    - 대응: 법률 자문을 통한 개인정보 처리방침 검토 (2단계 내 진행), 회원가입 시 동의 절차 구현 (SDD 진행 중)

## 대응 전략
### 전략 1: 보수적 문화 대응
- 가설: 소규모 파일럿과 성공사례 공유가 도입 저항을 낮춘다
- 근거: 신뢰 기반 추천과 실제 성공 사례가 설득에 유리하다

### 전략 2: 사제 동의 지연 대응
- 가설: 사제 보고용 요약(효과/통계)과 짧은 설명이 동의 획득 시간을 줄인다
- 근거: 의사결정자는 요약 지표와 효과를 빠르게 확인하려는 경향이 있다

### 전략 3: 학습/적응 부담 대응
- 가설: 최소 단계 온보딩과 짧은 가이드가 초기 이탈을 줄인다
- 근거: 초기 1~2개의 핵심 작업(학생 등록/출석 기록)이 성공하면 지속 사용 가능성이 높다

### 전략 4: 출석/통계 UX 복잡성 대응
- 가설: 출석/통계 화면을 단순화하고 자동 저장을 제공하면 사용 저하를 줄일 수 있다
- 근거: 출석 체크 간편화, 자동 저장, 정렬 기준 등 개선 요구가 반복됨

### 전략 5: B2B2C 정착 리스크 대응
- 가설: 최종 사용자의 즉시 효용(출석/학생 관리)을 강조하면 정착률이 높아진다
- 근거: 사용자가 체감하는 가치가 도입 이후 유지에 결정적이다

### 전략 6: 개인정보 우려 대응
- 가설: 최소 수집 원칙과 권한 분리 안내로 우려를 완화할 수 있다
- 근거: 민감 데이터는 수집 범위/보관 기간/접근 권한의 명확성이 중요하다

### 전략 7: 인수인계 리스크 대응
- 가설: 계정 이전/권한 관리 프로세스를 제공하면 사용 중단이 줄어든다
- 근거: 교리교사 교체는 반복되는 이벤트이며, 명확한 절차가 필요하다

### 전략 8: 개인 결제 중심 전환 대응
- 가설: 개인 결제 플로우를 우선 제공하고 본당 결제는 보조 경로로 두면 전환 속도가 개선된다
- 근거: 개인 결제는 승인 리드타임이 짧고, 본당 결제는 예외 케이스로 관리하는 편이 효율적이다

### 전략 9: 광고 이탈 대응
- 가설: 광고 위치/빈도를 핵심 기능 흐름 외부에 배치하면 이탈을 최소화할 수 있다
- 근거: 출석 체크 등 핵심 작업 중 광고가 노출되면 불편이 커지지만, 대시보드/통계 조회 등 비핵심 화면은 허용 범위가 넓다

### 전략 10: 부적절한 광고 대응
- 가설: 광고 네트워크의 카테고리 필터링과 사용자 신고 기능으로 부적절한 광고를 차단할 수 있다
- 근거: 대부분의 광고 플랫폼은 카테고리 제외 옵션을 제공하며, 사용자 신고로 누락된 케이스를 보완할 수 있다

### 전략 11: 광고 수익 불확실성 대응
- 가설: 광고 수익이 목표에 미달할 경우 유료 전환 촉진 또는 비용 구조 조정으로 대응한다
- 근거: 광고는 보조 수익원이며, 핵심 수익은 유료 구독에서 확보해야 한다

### 전략 12: 가치 체감 강화 대응
- 가설: 입력 최소화와 즉시 효용 제공으로 핵심 가치를 초기에 체감하게 하면 이탈이 줄어든다
- 근거: 장위동 초등부 파일럿에서 입력 불편과 가치 체감 부족이 이탈로 연결됨(초등부 전체 의견 일반화 아님)

## 가정
### 가정 1
- 가설: 중고등부 교리교사가 1차 도입 주체가 될 가능성이 높다
- 근거: 중고등부가 1차 타겟이며 업무 복잡도가 높은 편이라는 가정
- 검증: 중고등부 계정 비중과 활성 사용률 측정

### 가정 2
- 가설: 교리교사 네트워크 추천이 초기 확산에 유효하다
- 근거: 주일학교 내 도구 선택이 신뢰 기반 추천에 영향을 받는다는 가정
- 검증: 추천 기반 유입 비중/전환율 측정

### 가정 3 ⚠️ 재검토 필요
- 가설: 사제는 도입 자체에 우호적이며, 교리교사의 파일럿 결과를 기반으로 최종 결정을 한다
- 근거: 사용자 합의(사제는 도입 자체에 찬성)와 현장 의사결정 구조 가정
- 반론 (2026-01-25 피드백):
  - 사제/사무실은 현행 방식(한글/엑셀 파일)에 불편함을 느끼지 않을 가능성
  - 프로세스 전환 자체가 불편 요인
  - "우호적"이라는 가정은 "반대하지 않음"과 다름 - 적극적 지지 vs 소극적 허용 구분 필요
- 검증:
  - 기존: 사제 동의 리드타임/동의율 측정
  - 추가: 사제 대상 현행 방식 만족도, 새 도구 도입 의향 인터뷰
